# BERT、GPT、T5 三种预训练范式的本质区别是什么?各自适合什么任务

- **三种预训练范式**

- **1. BERT (Encoder-only)**
- 训练:Masked Language Model(完形填空)
- 特点:双向注意力,能看到上下文
- 擅长:理解任务(分类、NER、QA)
- 不擅长:生成任务

- **2. GPT (Decoder-only)**
- 训练:Causal Language Model(预测下一个 token)
- 特点:单向注意力,只看左侧
- 擅长:生成任务(对话、写作、代码)
- 不擅长:需要双向理解的任务

- **3. T5 (Encoder-Decoder)**
- 训练:Span Corruption(替换并恢复文本片段)
- 特点:Encoder 双向 + Decoder 单向
- 擅长:理解 + 生成(翻译、摘要、QA)
- 不擅长:不如纯 Decoder 在大规模生成上的 scaling

- **为什么 Decoder-only 赢了?**
1. 训练目标统一(next-token prediction)
2. 可用任意文本训练(无需标注)
3. Scaling Law 最清晰
4. 工程生态最好(KV Cache、vLLM 等)
5. GPT-3/4 证明了 scale + Decoder-only = 最强

- **但仍需其他范式**
- BERT 仍用于:嵌入、分类、搜索
- T5 仍用于:翻译、摘要
- GLM 尝试统一:Prefix LM 兼顾双向和生成

# 大模型幻觉(Hallucination)的成因是什么?如何缓解

- **幻觉类型:**
1. **事实性幻觉** - 编造不存在的事实(如虚假引用)
2. **忠实性幻觉** - 输出与输入/上下文矛盾
3. **推理幻觉** - 推理链条中出现逻辑错误

- **成因:**
- **训练数据** - 数据中的错误信息/过时信息
- **解码策略** - 高温度采样增加随机性
- **模型能力** - 参数化知识的不确定性
- **训练目标** - LM目标追求流畅而非正确

- **缓解策略:**

| 策略 | 方法 | 效果 |
|------|------|------|
| **RAG** | 检索真实文档辅助生成 | **强** |
| **CoT** | 让模型展示推理过程 | 中 |
| **自一致性** | 多次采样投票 | 中 |
| **RLHF** | 奖励诚实回答 | 强 |
| **外部工具** | 调用搜索/计算器 | 强 |
| **后处理** | 事实核查/NLI验证 | 中 |
| **系统Prompt** | 「不确定时说不知道」 | 弱 |

- **最佳组合:** RAG + CoT + 工具调用 + RLHF

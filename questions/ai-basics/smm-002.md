# Vision-Language Model (VLM)是什么？GPT-4V/Claude如何理解图片？

🎯 本质：VLM(Vision-Language Model)将视觉编码器与LLM结合，让模型既能理解图片又能用自然语言推理和回答。

📊 VLM三大架构范式：

1. 有Adapter的架构（BLIP-2, LLaVA）
  Image Encoder(CLIP ViT) -> Adapter(MLP/Q-Former) -> LLM
  图片 -> 视觉编码器提取特征 -> Adapter对齐到LLM的token空间 -> LLM理解
  代表：LLaVA（CLIP ViT + MLP + LLaMA）
  
2. 原生多模态架构（GPT-4V, Gemini, Qwen-VL）
  从头训练的多模态模型
  图像token和文本token在Transformer中自然混合
  优势：更深度的图文融合
  代表：GPT-4o, Gemini 1.5, Claude 3.5 Sonnet

3. 工具调用架构
  LLM本身不理解图片
  通过调用OCR/目标检测/CLIP等工具间接理解
  代表：早期GPT-4 + Plugin

LLaVA工作原理（典型例子）：
1. 图片 -> CLIP ViT -> 视觉特征序列
2. 视觉特征 -> MLP投影层 -> 视觉tokens（如576个token）
3. 视觉tokens + 文本tokens -> LLaMA -> 生成回答

训练流程：
Stage 1: 特征对齐预训练（冻结LLM，只训练Adapter）
Stage 2: 指令微调（用图文指令数据训练）

VLM核心能力：
1. 图片理解与问答（VQA）
2. 图片描述生成（Image Captioning）
3. OCR/文档理解
4. 图表/图表分析
5. 视觉推理（"图片中有几个红色的球？"）
6. 多图比较（"这两张图有什么区别？"）

挑战：
1. 高分辨率图片的token数量爆炸
2. 视频理解的时序建模
3. 幻觉问题（描述不存在的图片内容）
4. 空间/计数推理能力不足

# VLM(视觉语言模型)是如何工作的?ViT 的核心架构是什么

- **ViT (Vision Transformer)**
**核心思想：** 将 Transformer 从 NLP 迁移到视觉
1. 将图像切成 16x16 的 patch
2. 每个 patch 展平后线性投影为 token embedding
3. 加入位置编码(可学习或正弦)
4. 送入标准 Transformer Encoder
5. [CLS] token 的输出用于分类

```python
# Pseudo code
patches = image.reshape(B, N, P*P*C)  # N=num_patches
tokens = Linear(patches) + pos_embed
features = TransformerEncoder(tokens)
logits = Classifier(features[:, 0])  # [CLS]
```

- **VLM (Vision-Language Model)**
**主流架构：** LLM + Vision Encoder + 对齐模块

1. **Encoder-only 对齐**：ViT 提取特征 → MLP 投影到 LLM 的 embedding 空间
2. **Cross-Attention 融合**：LLM 每层增加 cross-attention 与视觉特征交互
3. **Q-Former 架构 (BLIP-2)**：少量 learnable query 与 ViT 特征交互 → 压缩后送入 LLM

- **代表模型**：
- LLaVA: ViT + MLP + LLaMA(最简洁)
- GPT-4V: 原生多模态预训练
- Qwen-VL: 中文优化
- Claude 3.5 Sonnet: 强 OCR 和图表理解

- **为什么 VLM 能加速推理**：多模态输入避免了'图片→文字描述→推理'的两步管线，一步到位。

### 补充细节
- **ViT 的位置编码**：不同于 NLP 中的 1D 序列，图像具有 2D 结构。标准 ViT 使用 1D 可学习位置编码，后续变体（如 Swin Transformer）引入了相对位置偏移以更好地捕捉局部结构。
- **Patch Embedding 参数计算**：假设输入图片 $H \times W \times C$，Patch 大小 $P \times P$。
  - Patch 数量 $N = \frac{H \times W}{P \times P}$。
  - 线性投影层权重矩阵形状为 $(P^2 \times C, D)$，其中 $D$ 是 Transformer 的隐藏层维度（如 768）。
- **Q-Former 的作用**：BLIP-2 中的 Q-Former 是一个轻量级 Transformer，使用可学习的 Query 向量从冻结的 ViT 中提取特征。它将视觉 Token 序列（如 256 个）压缩为固定数量的 Query 输出（如 32 个），从而大幅降低计算开销并充当“桥梁”。

## 常见考点
1.  **ViT 相比 CNN 的优缺点是什么？**
    *   *优点*：全局感受野（Self-Attention），适合捕捉长距离依赖；易于融合 NLP 预训练架构。
    *   *缺点*：缺乏 CNN 的归纳偏置（平移不变性、局部性），通常需要海量数据（如 JFT-300M）预训练才能收敛；对分辨率变化敏感。
2.  **在 VLM 中，Vision Encoder 为什么通常是 Frozen（冻结）的？**
    *   *核心回答*：为了防止灾难性遗忘。LLM 的语言能力在大规模语料上已训练得很好，微调视觉部分可能会破坏 LLM 的语言能力。同时冻结 ViT 可以减少训练参数量，只需训练 Projector 即可。
3.  **LLaVA 的 Projector (MLP) 的具体作用是什么？**
    *   *核心回答*：起到模态对齐的作用。ViT 输出的特征空间分布与 LLM 的 Text Embedding 空间分布不同，Projector（通常是一个 2 层 MLP）将视觉特征“翻译”或“映射”到 LLM 可以理解的词嵌入空间。

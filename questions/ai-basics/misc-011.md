# SFT数据集的构建有哪些最佳实践?如何避免灾难性遗忘

- **SFT数据构建最佳实践:**

1. **多样性优先** - 覆盖多种任务类型和领域
2. **质量>数量** - 1K条高质量数据 > 10万条低质量
3. **格式统一** - ChatML/Alpaca/ShareGPT格式
4. **CoT数据** - 包含思维链推理过程
5. **指令多样性** - 同一意图用不同表达方式

- **避免灾难性遗忘:**
1. **混入预训练数据** - SFT数据中混合10-20%原始预训练文本
2. **低学习率** - 用预训练lr的1/10
3. **LoRA/QLoRA** - 冻结原始权重,天然防止遗忘
4. **课程学习** - 从简单到复杂逐步训练

- **数据配比参考(LIMA论文):**
- 1000条精心筛选的高质量对话 > 5万条普通对话
- 证明了「数据质量是SFT的关键瓶颈」

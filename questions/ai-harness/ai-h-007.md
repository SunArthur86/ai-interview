# 如何选择LLM应用框架?LangChain vs LlamaIndex vs DSPy vs 原生API

- **选型决策矩阵:**

```
你的需求是什么?
├─ 简单API调用 → 原生API/OpenAI SDK
├─ RAG系统
│   ├─ 快速原型 → LlamaIndex
│   └─ 生产级 → Haystack/LangChain
├─ Agent
│   ├─ 单Agent工具调用 → OpenAI Assistants API
│   ├─ 自定义Agent循环 → LangChain
│   └─ 复杂状态管理 → LangGraph
├─ Prompt优化 → DSPy
├─ 结构化输出 → Instructor/Outlines
└─ 全栈应用 → LangChain生态
```

- **详细对比:**

| 维度 | 原生API | LangChain | LlamaIndex | DSPy | Haystack |
|------|---------|-----------|------------|------|----------|
| 学习成本 | **最低** | 高 | 中 | 高 | 中 |
| 灵活性 | **最高** | 中 | 中 | 中 | 中 |
| 抽象层级 | 无 | 高 | 中 | 高 | 高 |
| RAG能力 | 需自建 | 强 | **最强** | 弱 | 强 |
| Agent能力 | 需自建 | **强** | 中 | 弱 | 中 |
| Prompt优化 | 需自建 | 弱 | 弱 | **最强** | 弱 |
| 生产就绪 | **是** | 中 | 中 | 低 | **高** |

- **个人建议:**
- 快速验证:原生API + 几个辅助函数
- 中等复杂度:LangChain
- RAG为主:LlamaIndex
- 追求效果:DSPy
- 追求稳定:Haystack

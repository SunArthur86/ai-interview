# Semantic Router(语义路由)是什么?它如何高效做意图分类

**Semantic Router (语义路由) 解析**

**核心概念：**
基于语义相似度的意图路由机制，无需训练传统的分类器。通过计算用户输入与预设“意图描述”的向量相似度来决定路由目标。

**工作原理：**
1. 预定义一组「意图描述」和对应的处理函数。
2. 将用户输入转化为 Embedding。
3. 计算输入与各意图描述的余弦相似度。
4. 选择相似度最高的意图（需超过设定阈值）。

**代码示例：**
```python
from semantic_router import Route, SemanticRouter

# 定义路由
billing = Route(
    name="billing",
    utterances=[
        "我的账单有问题",
        "怎么退款",
        "支付失败",
        "发票相关",
    ]
)

tech = Route(
    name="tech_support",
    utterances=[
        "代码报错了",
        "API调用失败",
        "如何集成SDK",
    ]
)

# 初始化路由器
router = SemanticRouter(routes=[billing, tech])

# 执行路由
decision = router("我想取消订阅")  # -> billing
```

**核心优势：**
1. **无需训练**：只需提供少量示例句子，无需模型训练过程。
2. **速度快**：仅为向量计算，比调用 LLM 进行分类快 100 倍以上。
3. **动态可扩展**：新增意图只需添加配置，无需重新部署模型。
4. **零样本能力**：利用语义相似度直接处理未见过的表达。

**应用场景：**
- 客服系统的意图分流
- Agent 工具选择
- RAG 索引路由（选择不同的知识库）

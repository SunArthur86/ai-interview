# HNSW索引算法的原理是什么?为什么它是最常用的ANN算法

- **HNSW (Hierarchical Navigable Small World):**

- **核心思想:** 多层跳跃表式的近邻图.

- **结构:**
```
Layer 2:  A ←→ F              (稀疏,长距离连接)
Layer 1:  A ←→ C ←→ F          (中等密度)
Layer 0:  A←B←C←D←E←F          (密集,所有节点)
```

- **搜索流程:**
1. 从Layer 2的入口节点开始
2. 贪心搜索到最近的节点
3. 下降到Layer 1,从该节点继续搜索
4. 逐层下降直到Layer 0
5. 在Layer 0做精细近邻搜索

- **关键参数:**
- **M** - 每个节点的最大连接数(通常16-48)
- **efConstruction** - 构建时搜索宽度(通常200-500)
- **efSearch** - 查询时搜索宽度(通常50-200)

- **为什么最常用:**
1. **速度快** - 搜索复杂度O(log n)
2. **召回率高** - 95%+ recall@10
3. **支持动态插入** - 不需要重建索引
4. **内存效率** - 比KD-Tree/IVF更实用

- **几乎所有向量数据库的默认索引:** Milvus/Qdrant/Weaviate/Chroma

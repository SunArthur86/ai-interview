# 部署与运维

### 部署与运维

**核心目标**：将 Agent 服务构建为可扩容、可回滚、可观测的标准微服务。

**原理详解**：

1. **容器化与编排**：
   - **Docker**：封装代码与依赖，使用 `HEALTHCHECK` 确保实例就绪。
   - **Kubernetes**：利用 Deployment 副本管理、HPA 自动扩缩容、ConfigMap/Secret 分离配置。

2. **CI/CD 流水线**：
   - **CI**：代码检查、单元测试、镜像构建。
   - **CD**：自动发布，支持蓝绿部署或金丝雀发布。

3. **发布策略**：
   - **金丝雀发布**：小流量验证新版本（错误率、延迟），指标异常自动回滚。
   - **蓝绿部署**：两套环境切换，回滚速度快但资源占用高。

4. **版本管理**：
   - Prompt、模型版本、工具清单应版本化（如 `v20260401`），确保行为可追溯。
   - **不可变发布**：禁止直接修改运行中的配置，而是更新版本号。

5. **A/B 测试**：
   - 对不同 Prompt 或模型策略分流用户，基于业务指标（完成率、满意度）做统计学决策。

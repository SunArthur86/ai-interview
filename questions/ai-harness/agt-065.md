# 部署与运维

### 性能优化

**核心目标**：解决 Agent 系统中的串行 LLM 调用、高延迟网络 IO 和上下文膨胀问题。

#### 原理详解

1.  **异步处理**
    *   使用 `asyncio` 或消息队列，避免 I/O 等待阻塞线程。
    *   **并行工具调用**：无依赖的工具调用并发执行，大幅降低总耗时。

2.  **流式输出**
    *   利用 SSE (Server-Sent Events) 边生成边返回，降低首字延迟 (TTFB)。
    *   **细节**：需处理流式中断情况，保证最终状态一致性。

3.  **并发控制**
    *   使用 `Semaphore` 限制对同一模型的并发请求数，防止触发 429 (Rate Limit)。

4.  **连接复用**
    *   HTTP 客户端开启 `keep-alive`，数据库使用连接池，减少握手开销。

5.  **缓存分层**
    *   **L1 (进程内)**：LRU 缓存，极低延迟。
    *   **L2 (Redis)**：分布式共享，支持 TTL。
    *   **读路径**：L1 -> L2 -> 回源模型。

#### 并行调用架构图

```text
LLM generates Plan: [Tool A, Tool B, Tool C]
      │
      ├─ Tool A (Search) ──────┐
      │                        │
      ├─ Tool B (Calc) ────────┤──> Gather Results ──> Final LLM Synthesis
      │                        │
      └─ Tool C (Lookup) ──────┘

(Serial Time = A + B + C)
(Parallel Time = Max(A, B, C))
```

#### ## 常见考点
1.  **流式传输中的 JSON 解析**：流式输出 JSON 时可能被截断，如何保证前端能正确解析？（提示：使用缓冲区或特定的流式 JSON 解析库，确保字段完整性）。
2.  **并发限制策略**：当面对不同 LLM 厂商（如 OpenAI, Anthropic）不同的 RPM/TPM 限制时，如何设计限流器？（提示：令牌桶算法，按租户或 API Key 分隔）。
3.  **冷启动优化**：如何减少模型首次加载或容器冷启动带来的延迟？（提示：保持连接预热、使用预留实例/GPU）。
4.  **Asyncio 死锁**：在复杂的 Agent 编排中，如何避免异步任务中的死锁或资源竞争？

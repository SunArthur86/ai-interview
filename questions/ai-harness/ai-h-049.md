# 什么时候值得微调模型?微调 vs Prompt工程的ROI分析

- **微调 vs Prompt决策框架:**

- **优先用Prompt Engineering的情况(90%场景):**
- 知识注入 → 用RAG
- 格式调整 → 用结构化Prompt
- 角色设定 → 用System Prompt
- 简单分类 → 用Few-shot

- **需要微调的情况(10%场景):**

| 场景 | 原因 |
|------|------|
| **特定输出格式** | Prompt无法稳定约束 |
| **领域术语理解** | 通用模型不懂专业术语 |
| **Function Calling优化** | 工具调用准确率不足 |
| **推理速度优化** | 蒸馏小模型降低延迟 |
| **风格一致性** | 品牌声音/写作风格 |
| **安全合规** | 特定安全边界 |

- **ROI分析:**

```
**微调成本：** 
- 数据标注:$5K-20K
- GPU训练:$500-5000
- 评估测试:$2K-5K
- 持续维护:$1K/月

**Prompt成本：** 
- 开发时间:1-2周
- 推理额外成本:$200-2000/月(多token)
- 维护:低
```

- **决策流程:**
1. 先用Prompt+RAG(成本最低)
2. 效果不达标 → 优化Prompt
3. 仍不达标 → 评估微调ROI
4. 数据充足(>1K样本) + 效果提升>15% → 微调

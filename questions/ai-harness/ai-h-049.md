# 什么时候值得微调模型?微调 vs Prompt工程的ROI分析

**微调 vs Prompt 工程 ROI 分析**

**1. 决策框架**

**优先用 Prompt Engineering (90% 场景)**
- **知识注入**：使用 RAG 外挂知识库。
- **格式调整**：使用结构化 Prompt (如 JSON 模式)。
- **角色设定**：使用 System Prompt。
- **简单分类**：使用 Few-shot 示例。

**需要微调的情况 (10% 场景)**

| 场景 | 原因 |
|------|------|
| **特定输出格式** | Prompt 无法稳定约束复杂格式 |
| **领域术语理解** | 通用模型不懂专业黑话/缩写 |
| **Function Calling** | 工具调用参数准确率不足 |
| **推理速度优化** | 蒸馏小模型以降低延迟/成本 |
| **风格一致性** | 品牌声音、特定写作风格 |
| **安全合规** | 特定行业的敏感词过滤 |

**2. ROI 成本分析**

```
**微调成本：**
- 数据标注：$5K - $20K
- GPU 训练：$500 - $5000
- 评估测试：$2K - $5K
- 持续维护：$1K/月

**Prompt 成本：**
- 开发时间：1-2 周
- 推理额外成本：$200 - $2000/月 (更多 Prompt tokens)
- 维护：低
```

**3. 决策流程**
1. 先用 Prompt + RAG (成本最低，迭代最快)。
2. 效果不达标 → 优化 Prompt (Few-shot, CoT)。
3. 仍不达标 → 评估微调 ROI。
4. 数据充足 (>1K 高质量样本) + 效果提升 > 15% → 启动微调。

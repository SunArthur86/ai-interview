# LangChain的核心抽象是什么?LCEL相比传统Chain有什么优势

- **LangChain核心抽象:Runnable接口**

**所有组件都实现Runnable：** invoke/ainvoke/stream/astream/batch/abatch

- **LCEL (LangChain Expression Language):**
```python
# 声明式管道
chain = prompt | model | parser
result = chain.invoke({"input": "..."})

# 并行
parallel = RunnableParallel(a=chain_a, b=chain_b)
```

- **LCEL vs 传统Chain:**

| | 传统Chain | LCEL |
|--|-----------|------|
| 语法 | 嵌套构造 | **管道操作符** |
| 异步 | 需手动实现 | **原生支持** |
| 流式 | 需手动实现 | **内置流式** |
| 重试 | 需手动实现 | **自动重试** |
| 可观测 | 难 | **自动trace** |

- **LCEL核心能力:**
1. 自动异步/同步切换
2. 自动batching
3. 自动streaming(token级流式)
4. 内置fallback(链路容错)
5. 无缝集成LangSmith追踪

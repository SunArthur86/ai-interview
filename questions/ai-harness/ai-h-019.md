# vLLM vs SGLang vs TGI vs Triton--推理引擎如何选择

- **推理引擎对比:**

| 引擎 | 开发者 | 核心技术 | 优势 |
|------|--------|---------|------|
| **vLLM** | UC Berkeley | PagedAttention | **吞吐量最高** |
| **SGLang** | UC Berkeley | RadixAttention | **复杂推理最快** |
| **TGI** | HuggingFace | Continuous Batching | 易用/兼容HF |
| **TensorRT-LLM** | NVIDIA | TensorRT优化 | **单卡延迟最低** |
| **Ollama** | Ollama | llama.cpp | **本地部署最简** |

- **选择建议:**

```
高并发服务(100+ QPS)
→ vLLM(PagedAttention吞吐量最强)

结构化生成 / 多轮推理
→ SGLang(RadixAttention加速前缀共享)

快速原型 / 兼容HF
→ TGI(HuggingFace生态)

极致延迟(<100ms TTFT)
→ TensorRT-LLM(GPU优化极致)

本地开发
→ Ollama(一行命令启动)
```

- **SGLang vs vLLM:**
- vLLM:通用吞吐量更好
- SGLang:对于有共享前缀的场景(如多轮对话/Agent),RadixAttention复用前缀KV Cache,速度更快

# 如何设计高性能的Embedding服务?批量embedding的优化策略

- **Embedding服务架构:**

- **核心组件:**
1. **模型加载** - BGE-M3/E5等,GPU推理
2. **批量推理** - 多文档同时编码
3. **缓存层** - 相同文本直接返回
4. **负载均衡** - 多GPU/多实例

- **批量优化策略:**

1. **动态batching**
- 等待一定数量或超时后批量推理
- 平衡延迟和吞吐

2. **文本截断**
- 超过max_length截断
- 长文档分段编码后平均池化

3. **ONNX/TensorRT加速**
- 导出为ONNX格式
- TensorRT引擎优化
- 推理速度提升2-3倍

4. **异步Pipeline**
```
文档变更 → 队列(Kafka) → Worker(批量取embedding) → 向量库
```

- **性能基准(BGE-M3, A100):**
- 单条:~5ms
- 批量256条:~200ms(0.8ms/条)
- ONNX加速:~120ms(0.5ms/条)

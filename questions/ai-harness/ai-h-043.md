# Ragas框架如何评估RAG系统?有哪些核心指标?如何部署

**Ragas 评估框架核心解析**

**1. 评估指标体系**

**检索质量指标：**
- **Context Precision (上下文精确度)**：检索到的上下文中相关信息的比例，衡量是否包含噪声。
- **Context Recall (上下文召回率)**：金标准答案（Ground Truth）所需的信息是否被检索到。
- **Context Relevancy (上下文相关性)**：检索到的上下文与用户问题的直接相关性。

**生成质量指标：**
- **Faithfulness (忠实度)**：答案是否严格基于检索到的文档，检测幻觉。
- **Answer Relevancy (答案相关性)**：生成的答案是否直接回应了用户的问题。

**2. 代码实现示例**

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall
)

# eval_dataset 需包含: question, contexts, answer, ground_truth
result = evaluate(
    dataset=eval_dataset,
    metrics=[
        faithfulness,
        answer_relevancy,
        context_precision,
        context_recall
    ]
)

print(result.scores)
```

**3. 评估数据集构建**
- **人工标注**：专家编写 Question + Ground Truth。
- **LLM 自动生成**：基于文档通过大模型自动生成问答对。
- **日志提取**：从真实用户日志中清洗并脱敏提取。

**4. 最佳实践**
建议建立 CI/CD 集成的持续评估 Pipeline，每次 RAG 配置（如 Chunk 策略、Embedding 模型）变更后自动运行评估，确保效果不回退。

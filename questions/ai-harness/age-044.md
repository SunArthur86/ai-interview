# 什么是 LLM Gateway(模型网关)?生产环境为什么需要它

- **LLM Gateway** 是位于应用和 LLM 供应商之间的代理层,提供统一的 API 接口和治理能力.

- **核心功能:**

  - **1. 统一接口:** 屏蔽不同供应商的 API 差异(OpenAI、Claude、国产模型),应用只对接 Gateway.

  - **2. 负载均衡与路由:** 根据任务类型、成本、延迟自动选择模型.

  - **3. 限流与熔断:** 防止下游模型服务过载,保护应用不被 API 限流影响.

  - **4. 成本控制:** Token 级别的成本追踪和归因,支持按项目/用户/功能分账.

  - **5. 缓存:** 相同请求直接返回缓存结果,降低成本.

  - **6. 安全:** Prompt 注入检测、敏感词过滤、审计日志.

  - **7. 可观测性:** 统一的监控面板、请求日志、质量指标.

- **常见实现:** LiteLLM(开源)、Kong AI Gateway、Cloudflare AI Gateway、Portkey.

- **什么时候需要:** 日均调用量超过 1 万次、使用多个模型供应商、需要成本管控.

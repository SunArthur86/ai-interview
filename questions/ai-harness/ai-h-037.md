# 什么时候用RAG,什么时候微调?两者如何结合

- **RAG vs 微调决策矩阵:**

| 维度 | RAG | 微调 |
|------|-----|------|
| 知识更新 | **实时** | 需重新训练 |
| 成本 | **低** | 高 |
| 幻觉控制 | **好(有据可查)** | 中 |
| 风格/格式 | 弱 | **好** |
| 推理速度 | 慢(检索开销) | **快** |
| 知识量 | 受检索限制 | **可内化大量** |

- **选择建议:**
```
需求是「知道新知识」?
├─ 是 → RAG
└─ 需求是「改变行为/风格」?
  ├─ 是 → 微调
  └─ 两者都需要 → RAG + 微调
```

- **两者结合的最佳实践:**

1. **微调教格式/风格** + **RAG提供知识**
- 微调让模型学会输出特定格式
- RAG注入最新的领域知识

2. **微调教工具使用** + **RAG提供上下文**
- 微调提升Function Calling准确率
- RAG提供具体任务信息

3. **微调 embedding模型** + **RAG检索**
- 领域数据微调embedding
- 提升检索准确率

- **成本对比:** RAG成本低10-100倍，优先尝试RAG

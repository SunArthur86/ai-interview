# 什么是 Harness Engineering?它和传统 Agent 框架有什么区别

- **Harness Engineering** 是 2025 年兴起的概念,指围绕大模型构建的工程化外壳(Harness),让模型在真实场景中可靠工作.

- **核心理念:** 模型本身只是推理引擎,真正决定 Agent 效果的是围绕模型的工程化体系.

- **Harness 包含的关键组件:**
1. 上下文工程(Context Engineering):组装 system prompt、工具定义、历史消息
2. 工具治理(Tool Governance):权限控制、参数校验、结果过滤
3. 状态管理(State Management):任务状态快照、恢复机制
4. 可观测性(Observability):Trace 轨迹、工具调用日志、成本追踪
5. 安全防护(Safety):Prompt 注入检测、输出过滤

- **与传统框架的区别:** LangChain 关注链式调用编排,Harness Engineering 更关注生产级的可靠性、可观测性和安全性.

- **代表产品:** Hermes Agent、OpenClaw、Claude Code 都是 Harness Engineering 的实践.

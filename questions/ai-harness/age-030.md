# 语义缓存和精确缓存有什么区别?如何防止缓存「串味」(不同问题返回相同缓存)

- **语义缓存 vs 精确缓存:**

| | 精确缓存 | 语义缓存 |
|--|---------|---------|
| 匹配方式 | 完全匹配(HASH) | **语义相似度匹配** |
| 实现 | Redis/HashMap | 向量检索(GPTCache) |
| 召回率 | 低(一点不同就miss) | **高** |
| 精确度 | **100%** | 可能误匹配 |
| 适合 | 固定QA | **开放问答** |

- **防止缓存「串味」的措施:**

1. **相似度阈值**
- 设定最低余弦相似度(如0.95)
- 低于阈值的不返回缓存

2. **语义验证**
- 用小LLM快速验证:缓存问题和当前问题是否真的等价
- 成本低但多一次推理

3. **分类隔离**
- 不同任务类型的缓存分开存储
- 避免跨领域误匹配

4. **TTL过期**
- 时效性内容设置短TTL
- 避免返回过时信息

5. **用户隔离**
- 用户特定的问题不做全局缓存
- 或做用户级缓存命名空间

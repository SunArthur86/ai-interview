# DSPy的编程模型和LangChain有什么本质区别

- **本质区别:**

| | LangChain | DSPy |
|--|-----------|------|
| 范式 | **命令式**(手写prompt) | **声明式**(定义输入输出) |
| Prompt | 手动编写和调优 | **自动搜索最优prompt** |
| 示例选择 | 手动 | **自动选择few-shot** |
| 适配模型 | 需重写prompt | **自动适配** |

- **DSPy核心模式:**
```python
# 1. 定义Signature(做什么)
class QA(dspy.Signature):
    """回答问题"""
    question = dspy.InputField()
    answer = dspy.OutputField(desc="简洁准确的答案")

# 2. 选择Module(怎么做)
qa = dspy.ChainOfThought(QA)

# 3. 编译优化(自动找最优prompt+示例)
optimizer = dspy.BootstrapFewShot(metric=accuracy)
compiled_qa = optimizer.compile(qa, trainset=data)

# 4. 使用
result = compiled_qa(question="什么是注意力机制?")
```

- **DSPy vs 手写Prompt:**
- 手写:依赖人工经验,换模型可能失效
- DSPy:自动优化,换模型只需重新compile
- 可版本管理、可复现、可A/B测试

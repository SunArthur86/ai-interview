# Pass@k 是什么指标?它如何衡量代码生成模型的质量

- **Pass@k** 是评估代码生成模型的标准指标,衡量生成代码的通过率.

- **定义:** 给定一个编程题,模型生成 k 个候选答案,只要其中至少有一个通过了所有测试用例,就算通过.

- **计算公式:** Pass@k = 1 - C(n-c, k) / C(n, k)
- n:总采样次数
- c:通过的次数
- k:评估的候选数

- **常见取值:**
- Pass@1:只生成一次,通过率(最严格,反映实际体验)
- Pass@10:生成 10 个,至少一个通过的概率
- Pass@100:更宽松的评估

- **评估基准:**
- HumanEval(OpenAI):164 道 Python 编程题
- MBPP(Google):974 道 Python 编程题
- HumanEval+:增强版,更多测试用例
- SWE-bench:真实 GitHub PR 修复

- **面试要点:** Pass@1 最重要(用户只生成一次),Pass@10 反映模型的上限潜力.两者差距大说明模型不稳定(有时好有时差).

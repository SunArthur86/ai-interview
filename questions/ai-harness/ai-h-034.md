# RAG知识库如何实现增量更新?大规模文档变更时索引如何高效更新

- **增量更新策略:**

1. **文档变更检测**
- 文件哈希(MD5/SHA256)对比
- 修改时间戳检查
- Webhook通知(CMS集成)

2. **增量索引**
- 只对变更文档重新分块+embedding
- 删除旧向量,插入新向量
- 大多数向量数据库支持upsert

3. **批量更新Pipeline**
```
变更检测 → 增量分块 → 批量embedding → 向量库upsert → 索引刷新
```

4. **双缓冲更新**
- 新索引在后台构建
- 构建完成后原子切换
- 零停机更新

- **大规模场景(百万文档):**
- **分区索引** - 按时间/类别分区,只更新变更分区
- **异步队列** - Kafka/SQS排队处理
- **并行embedding** - 多GPU批量计算
- **Qdrant/Milvus的批量upsert API**

- **删除处理:**
- 软删除(标记deleted=true,检索时过滤)
- 定期硬删除(低峰期清理)

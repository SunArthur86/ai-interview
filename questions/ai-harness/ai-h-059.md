# LLM 应用的监控和传统 Web 应用的监控有什么不同

- **LLM 监控的独特维度**

传统 Web:CPU/内存/QPS/延迟
LLM 应用还需要:

1. **Token 消耗**:每次请求消耗的 token 数(直接关联成本)
2. **质量指标**:幻觉率、回答相关性、用户满意度
3. **LLM 延迟**:首 token 延迟 (TTFT) + 每秒生成速度
4. **工具调用成功率**:Agent 场景中工具调用的成功/失败率
5. **Prompt 变更追踪**:不同 prompt 版本的效果对比

- **常用工具**:LangSmith, Phoenix (Arize), Langfuse, W&B

- **一句话理解**:传统监控关注'系统是否正常',LLM 监控还要关注'AI 回答是否靠谱'.

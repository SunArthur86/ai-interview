# OpenTelemetry在Agent系统中一般打哪些Span?如何设计Trace结构

- **Agent Trace 结构:**
```
Trace (一次完整Agent任务)
├─ Span: agent.run (根Span)
│   ├─ Span: llm.call (LLM调用)
│   │   ├─ Span: llm.tokenize
│   │   ├─ Span: llm.inference
│   │   └─ Span: llm.detokenize
│   ├─ Span: tool.call (工具调用)
│   │   ├─ Span: tool.search (搜索)
│   │   └─ Span: tool.execute (执行)
│   ├─ Span: rag.retrieve (RAG检索)
│   │   ├─ Span: embedding
│   │   ├─ Span: vector_search
│   │   └─ Span: rerank
│   └─ Span: memory.update (记忆更新)
```

- **每个Span记录:**
- 服务名(service.name)
- 操作名(operation.name)
- 开始/结束时间(duration)
- 属性(attributes): model, tokens, cost
- 事件(events): 错误、重试
- 状态(status): OK/ERROR

- **关键Span属性:**
```python
span.set_attribute("llm.model", "gpt-4")
span.set_attribute("llm.input_tokens", 1500)
span.set_attribute("llm.output_tokens", 300)
span.set_attribute("llm.cost_usd", 0.012)
span.set_attribute("tool.name", "search")
span.set_attribute("tool.success", True)
```

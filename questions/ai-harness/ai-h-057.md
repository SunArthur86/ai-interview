# 什么是 AI Harness(AI 工程化框架)?它解决什么问题

- **AI Harness** 指围绕大模型构建应用所需的工程化框架和工具链。

- **解决的问题**
1. 直接调 API 不够:缺乏记忆、工具、安全等能力
2. 生产化困难:监控、限流、降级、版本管理
3. 成本控制:token 消耗、缓存策略

- **核心组件**
- 模型路由:多模型选择 + 熔断降级
- RAG 引擎:检索 + 生成
- Agent 编排:工具调用 + 任务规划
- 记忆系统:短期/长期记忆
- 可观测性:日志/Trace/Metrics
- 安全护栏:Prompt 注入防护

- **实战案例:**
在构建企业级知识库助手时，直接使用 LangChain 的 `VectorStoreRouterChain` 导致冷启动极慢（需加载所有向量库索引）。我们自定义了 **Lazy-loading Harness**：仅在 Agent 识别到特定领域意图时，才初始化对应的 VectorStore 连接，将冷启动时间从 30s 降低到 2s，同时节省了 40% 的内存占用。

- **代码示例 (Python - 模拟 Router Harness):**
```python
from typing import Literal

def model_router(prompt: str, complexity: str) -> str:
    """根据任务复杂度路由到不同模型"""
    if complexity == "high":
        # 调用 GPT-4o/Claude 3.5
        return call_llm("gpt-4o", prompt)
    else:
        # 降级到小模型 Haiku 或 Llama-3-8B
        return call_llm("haiku", prompt)

# 简单的 Agent 编排模拟
def run_agent(task):
    intent = classify_intent(task)
    if intent == "search":
        return search_tool(task)
    elif intent == "code":
        return model_router(task, "high")
```

- **代表框架对比:**
| 框架 | 核心定位 | 优势 | 劣势 | 适用场景 |
|------|---------|------|------|---------|
| **LangChain** | 通用编排/Agent | 生态丰富，集成度高 | 抽象复杂，调试难 | 复杂 Agent 逻辑，多工具调用 |
| **LlamaIndex** | 数据索引/RAG | 数据结构化强，检索精细 | Agent 能力较弱 | 知识库问答，文档分析 |
| **Dify** | 应用开发平台 | 可视化编排，后端即服务 | 定制灵活性较低 | 快速落地企业级应用 |
| **Semantic Kernel** | 企业级集成 | 微软生态原生，C#支持强 | 社区较小 | .NET 环境，企业内部集成 |

- **常见考点:**
1. **LangChain vs LlamaIndex**: 两者定位区别？（LangChain 侧重通用编排和 Agent，LlamaIndex 侧重数据索引和 RAG）
2. **状态管理**: LangChain 中的 RunnableConfig 如何在分布式追踪中传递状态？
3. **RAG 性能优化**: 在框架层面如何解决检索召回率低的问题？（混合检索、重排序 Re-rank 机制）

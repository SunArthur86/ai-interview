# 如何建立完整的LLM应用评估体系?LLM-as-Judge如何设计

- **LLM评估三层体系:**

- **1. 单元测试(函数级)**
- Prompt格式验证
- JSON Schema验证
- 工具参数合法性

- **2. 集成测试(端到端)**
- 评估数据集(100-1000样本)
- 人工标注金标准
- 自动化批量测试

- **3. 线上监控(实时)**
- 用户反馈(赞/踩)
- 质量打分(LLM-as-Judge)
- 异常检测(幻觉率/延迟/成本)

- **LLM-as-Judge设计:**
```python
judge_prompt = f"""
你是评估专家.请评估以下回答的质量.

问题：{question}
回答：{answer}
参考答案：{reference}

评分维度(1-5分):
1. 准确性:答案是否正确
2. 完整性:是否覆盖所有要点
3. 相关性:是否切题
4. 流畅性:是否表达清晰

输出JSON格式评分.
"""
```

- **注意事项:**
- Judge模型要比被评估模型更强
- 多次评估取平均(减少随机性)
- 人工抽检验证Judge质量
- 注意Judge的偏见(偏好长回答等)

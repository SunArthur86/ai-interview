# Prompt Engineering 基础

### Prompt Engineering 基础

**概念解释**：
Prompt Engineering 是通过设计和优化输入给 LLM 的文本（及参数），以引导模型稳定输出高质量结果的技术。它不是简单的“写咒语”，而是将业务逻辑翻译成模型能理解的语言。

**Prompt 的基本结构**：
一个完整的 Prompt 通常包含以下五部分：
1. **角色**：设定身份（如“你是一名资深Python工程师”）。
2. **任务**：明确要做什么（如“分类以下文本的情感”）。
3. **上下文**：提供背景信息、数据或检索结果。
4. **格式**：指定输出结构（如 JSON、Markdown 列表）。
5. **约束**：限定范围、长度或禁止项（如“不超过50字，禁止编造”）。

**为什么 Prompt 很重要？**
- **软代码**：在 Agent 中，Prompt 决定了工具调用策略、错误恢复逻辑，行为规则皆编码于此。
- **决定输出质量**：同样的模型，Prompt 好坏会导致结果天差地别。

**好的 Prompt 特征**：
- **目标单一**：一次专注一件事，复杂任务拆解。
- **信息分层**：指令与数据（上下文）清晰分离，如使用 XML 标签。
- **可执行**：输出格式严格匹配程序解析需求（如 JSON Schema）。
- **可测试**：包含正反例，能通过回归测试。

# Dify和Coze(扣子)的定位和核心功能有什么区别

- **Dify vs Coze (扣子):**

| | Dify | Coze(扣子) |
|--|------|-------------|
| 定位 | 开源LLM应用平台 (B/P端) | 字节跳动Agent平台 (C端优先) |
| 部署 | **可私有化** (Docker/K8s) | 仅SaaS (国内/国际版) |
| Agent 编排 | Workflow (可视化) + Agent | **Bot (对话) + Plugin** |
| 模型支持 | 任意 (OpenAI/Ollama/Local/Bedrock) | 字节系为主 + 少数第三方 |
| 知识库 | 内置RAG (支持多种切片/清洗) | 内置知识库 (易用) |
| 发布渠道 | API / Web App / 嵌入式组件 | **微信/抖音/飞书/飞书智能体** |
| 生态 | 开源社区, 自建能力强 | 字节生态, 流量分发强 |

- **Dify核心功能 (企业级):**
1. **Workflow编排**: 强大的DAG工作流，支持条件分支、并行节点，适合复杂业务逻辑。
2. **Agent模式**: ReAct模式 + Function Calling，支持工具调用规划。
3. **RAG Pipeline**: 完整的Pipeline，包括文档提取、切片、Embedding、检索、重排。
4. **Prompt管理**: 版本控制、对比实验 (A/B Testing)。
5. **可观测性**: 详细的日志追踪、Annotation（人工标注反馈）。

- **Coze核心功能 (C端/流量):**
1. **插件商店**: 丰富的官方插件（搜索、新闻、绘图），一键启用。
2. **多渠道发布**: 一键发布到微信公众号、飞书、抖音等C端流量池。
3. **人设与回复**: 优化的对话配置界面，快速上手。

- **选择建议:**
- **企业内部 / 私有化 / 数据敏感**: 选 Dify。
- **ToC 产品 / 微信机器人 / 需要字节流量**: 选 Coze。
- **深度定制 / 自控模型**: 选 Dify。

- **## 常见考点**
1. **RAG流程差异**: Dify支持更细粒度的RAG流程控制（如自定义重排模型），Coze更自动化。
2. **数据隐私**: Coze的数据存储政策，为何金融/医疗行业更倾向Dify私有化。
3. **扩展能力**: Dify的API和Webhook支持，如何与企业现有系统（CRM/ERP）集成。
4. **多模型编排**: Dify如何在同一个Workflow中切换不同模型（如用GPT-4做规划，Llama3做生成）。

# Dify和Coze(扣子)的定位和核心功能有什么区别

- **Dify vs Coze:**

| | Dify | Coze(扣子) |
|--|------|-------------|
| 定位 | 开源LLM应用平台 | 字节跳动Agent平台 |
| 部署 | **可私有化** | 仅SaaS |
| Agent | 工作流+Agent | **Bot+Plugin** |
| 模型 | 任意(OpenAI/本地等) | 字节系为主 |
| 知识库 | 内置RAG | 内置知识库 |
| 发布渠道 | API/Web | **微信/抖音/飞书** |
| 生态 | 开源社区 | 字节生态 |

- **Dify核心功能:**
1. **Workflow编排** - 可视化拖拽工作流
2. **Agent模式** - ReAct/Function Calling
3. **RAG Pipeline** - 内置分块/检索/重排
4. **Prompt管理** - 版本控制+AB测试
5. **可观测性** - 日志/追踪/标注

- **选择:**
- 私有化部署 → Dify
- 快速上线 + 中国生态 → Coze

# 向量数据库中的PQ(Product Quantization)和SQ(Scalar Quantization)是什么?如何选择

- **向量量化目标:** 压缩向量存储空间,加速搜索.

- **SQ (Scalar Quantization):**
- 将FP32向量的每个维度量化为INT8
- 压缩比:4:1
- 精度损失:极小(<1%)
- 原理:线性映射 [min, max] → [0, 255]

- **PQ (Product Quantization):**
- 将向量分成M段,每段独立量化
- 压缩比:32:1~64:1
- 精度损失:中(3-8%)
- 原理:每段用K-means聚类,存聚类中心ID

- **对比:**

| | SQ | PQ |
|--|-----|-----|
| 压缩比 | 4:1 | **32:1** |
| 精度损失 | **<1%** | 3-8% |
| 速度 | 快 | **最快** |
| 内存节省 | 4x | **32x** |
| 适用 | 精度优先 | **成本优先** |

- **实际组合使用:**
- **IVF-PQ* - 先聚类再量化,大规模+低成本
- **HNSW-SQ* - HNSW图+标量量化,精度+速度平衡
- Qdrant默认使用SQ
- Milvus支持PQ和SQ

# AI 编程工具中如何控制 Token 成本?有哪些优化策略

- **Token 成本控制** 是 AI 编程工具的核心工程问题.

- **成本来源分析:**
- 每次请求都携带大量上下文(system prompt + 文件内容 + 对话历史)
- Agent 模式下多轮迭代,每轮都重复发送上下文
- 代码仓库索引和检索也有 Embedding 成本

- **优化策略:**

- **1. Prompt Caching:**
- System Prompt 和工具定义是重复前缀
- 利用供应商的前缀缓存功能(如 Anthropic 的 Prompt Caching)
- 可节省 50-90% 的重复计算成本

- **2. 上下文压缩:**
- 对话历史定期摘要(保留关键信息,丢弃冗余)
- 文件内容只发送相关部分(而非整个文件)
- 工具返回结果截断和过滤

- **3. 模型分级:**
- 简单操作(如格式化、重命名)用小模型
- 复杂推理(架构设计、bug 分析)用大模型
- 用 LLM Router 自动选择

- **4. 代码检索优化:**
- 只检索相关文件而非全仓库
- 用 Glob/Grep 替代 RAG(Claude Code 的做法)
- 缓存检索结果

# LLM应用的A/B测试如何设计?统计显著性如何保证

**LLM应用A/B测试设计与统计**

**1. 定义指标**
- **主指标**: 任务成功率 / 用户满意度 / 准确率
- **辅助指标**: 延迟 / 成本 / Token消耗
- **护栏指标**: 幻觉率 / 安全违规率

**2. 样本量计算**
```
# 最小样本量(近似公式)
n = 16 * σ² / Δ²

σ = 指标标准差
Δ = 最小可检测效应

例:σ=0.2, Δ=0.05 → n=1280 per group
```

**3. 流量分配**
- **50/50均分**: 统计功效最强
- **90/10金丝雀**: 先小流量验证安全性
- **分桶策略**: 基于用户ID的确定性分桶

**4. 统计检验**
```python
from scipy import stats

# 比例检验(成功率)
z_stat, p_value = stats.proportions_ztest(
    [success_a, success_b],
    [total_a, total_b]
)

if p_value < 0.05:
    print("显著差异")
```

**5. 常见陷阱**
- **多重检验问题**: 同时测多个指标需做 Bonferroni 校正
- **新鲜效应**: 新版本短期效果不代表长期
- **幸存者偏差**: 只看完成任务的忽略流失用户
- **Simpson's Paradox**: 分层结论与总体结论矛盾

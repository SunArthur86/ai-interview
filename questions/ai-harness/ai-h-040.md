# vLLM的PagedAttention核心原理是什么?它如何提升推理效率

- **PagedAttention:受OS虚拟内存启发的KV Cache管理**

- **传统KV Cache问题:**
- 每个请求预分配连续内存
- 实际使用 < 预分配 → 内存浪费
- 碎片化严重
- GPU利用率低(30-60%)

- **PagedAttention解决方案:**

1. **分页管理**
- KV Cache分成固定大小的block(如16个token)
- 物理block可以不连续
- Block表(类似页表)映射逻辑→物理

2. **按需分配**
- 只为实际生成的token分配block
- 零浪费

3. **共享(Copy-on-Write)**
- 多个请求共享相同前缀的KV Cache
- 如系统prompt相同 → 共享block
- 修改时才复制

- **效果:**
- GPU利用率:30% → **90%+**
- 吞吐量:**2-4倍提升**
- 内存浪费:<4%

- **Continuous Batching:**
- 不同请求的不同长度可以在同一batch中处理
- 请求完成立即释放资源
- 新请求动态加入batch

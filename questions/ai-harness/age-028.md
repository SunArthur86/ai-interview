# Agent系统的金丝雀发布需要关注哪些指标?蓝绿和金丝雀如何取舍

- **金丝雀发布关键指标:**

| 指标类型 | 具体指标 | 阈值 |
|---------|---------|------|
| **质量** | 任务成功率 | >95% |
| **质量** | 幻觉率 | <5% |
| **延迟** | P50/P99延迟 | <旧版本±10% |
| **成本** | 每请求Token消耗 | <预算 |
| **安全** | Prompt注入触发率 | <1% |
| **用户** | 负反馈率(踩) | <5% |

- **蓝绿 vs 金丝雀:**

| | 蓝绿 | 金丝雀 |
|--|------|--------|
| 切换速度 | **快(秒级)** | 慢(渐进) |
| 风险 | 高(全量切换) | **低(小流量) |
| 回滚 | **快(切回)** | 需停止新版本 |
| 资源 | 2x(两套环境) | 1x+少量额外 |
| 适合 | 低风险变更 | **高风险变更** |

- **Agent长任务的部署不中断:**
1. 滚动部署:新实例逐个上线
2. 旧实例等正在执行的任务完成再下线
3. Session粘性:用户请求保持到同一实例

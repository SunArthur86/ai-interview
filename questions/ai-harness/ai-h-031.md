# LiteLLM和模型网关的作用是什么?如何管理多供应商LLM

- **LiteLLM核心:统一API接口**

用一个接口调用所有LLM供应商(OpenAI/Anthropic/Zhipu/DeepSeek/本地Ollama等).

- **核心功能:**
1. **统一接口** - 100+供应商,一套SDK
2. **负载均衡** - 多个API Key轮询
3. **Fallback** - 主供应商失败自动切换
4. **成本追踪** - 自动计算Token费用
5. **限流** - 按用户/项目限流
6. **缓存** - 相同请求缓存结果

- **使用方式:**
```python
from litellm import completion

# 统一接口,只需改model名
response = completion(model="gpt-4", messages=[...])
response = completion(model="claude-3-5-sonnet", messages=[...])
response = completion(model="zhipu/glm-4", messages=[...])
```

- **模型网关:**
- 部署为独立服务
- 所有应用通过网关访问LLM
- 统一计费/日志/限流/缓存

- **vs 直接调用各供应商API:**
- 优势:统一管理、故障转移、成本控制
- 劣势:增加一层延迟、额外运维

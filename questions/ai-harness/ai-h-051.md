# Vibe Coding 是什么?它与传统编程范式有什么本质区别

- **Vibe Coding** 是 Andrej Karpathy 于 2025 年提出的概念,指用自然语言描述意图、由 AI 生成代码的编程范式.

- **核心理念:**
- 开发者不再逐行写代码,而是描述想要的效果(vibe)
- AI 工具(Cursor、Claude Code、Windsurf)负责代码实现
- 开发者角色从代码编写者变为代码审查者和架构师

- **与传统编程的区别:**

- **传统编程:** 思考算法 -> 手写代码 -> 调试 -> 测试

- **Vibe Coding:** 描述意图 -> AI 生成代码 -> 审查/调整 -> AI 修复 -> 测试

- **关键能力要求:**
1. Prompt 工程:如何精确描述需求
2. 代码审查:快速判断 AI 生成的代码是否正确
3. 架构思维:知道系统应该怎么组织
4. 调试能力:定位 AI 生成代码中的 bug

- **挑战:**
- AI 生成的代码可能有隐藏 bug
- 开发者对代码的理解可能不够深入
- 大项目的上下文管理困难(Token 限制)

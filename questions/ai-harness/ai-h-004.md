# Haystack框架的核心设计是什么?它和LangChain/LlamaIndex有什么区别

- **Haystack (deepset) 核心设计:Pipeline + Component**

```python
# 声明式Pipeline
pipeline = Pipeline()
pipeline.add_component("embed", embedder)
pipeline.add_component("retriever", retriever)
pipeline.add_component("reader", reader)
pipeline.connect("embed.embedding", "retriever.query_embedding")
pipeline.connect("retriever.documents", "reader.documents")
```

- **核心区别:**

| | LangChain | LlamaIndex | Haystack |
|--|-----------|------------|----------|
| 设计 | 链式调用 | 数据索引 | **DAG Pipeline** |
| 类型安全 | 弱 | 弱 | **强** |
| 生产就绪 | 中 | 中 | **高** |
| 评估 | 需外接 | 内置基础 | **内置评估** |
| 配置 | 代码 | 代码 | **YAML配置** |

- **Haystack优势:**
1. **类型安全** - 组件间连接有类型检查
2. **YAML配置** - Pipeline可用YAML定义
3. **生产就绪** - 日志/监控/错误处理完善
4. **评估集成** - 内置评估pipeline

- **适合场景:** 企业级RAG系统、需要严格类型检查的生产环境

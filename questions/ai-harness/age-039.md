# Agent 的 Memory 系统如何设计?短期记忆和长期记忆怎么区分

- **Agent Memory 设计** 是生产级 Agent 的核心难题之一.

- **短期记忆(Working Memory):**
- 存储在 LLM 上下文窗口中
- 包含:当前对话历史、工具调用结果、中间推理
- 生命周期:单次任务或单轮对话
- 管理:上下文压缩、截断、摘要

- **长期记忆(Long-term Memory):**
- 存储在外部系统(向量数据库、关系数据库)
- 包含:用户偏好、历史决策、学习到的知识
- 生命周期:跨会话持久化
- 管理:检索、更新、遗忘

- **设计要点:**
1. 记忆写入:什么信息值得记住?避免存入噪声
2. 记忆检索:如何高效找到相关记忆?语义搜索 + 时间衰减
3. 记忆更新:如何处理冲突信息?最新覆盖最旧
4. 遗忘机制:如何淘汰过时记忆?基于访问频率和时间

- **常见实现:** MemGPT、LangChain Memory、Letta(前 MemGPT)

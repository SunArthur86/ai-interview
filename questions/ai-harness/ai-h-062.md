# Prefill-Decode 分离(PD Separation)是什么?为什么大规模推理服务必须做

- **PD 分离** 将推理的 Prefill(首 token 计算)和 Decode(逐 token 生成)分配到不同的 GPU 集群.

- **为什么需要分离?**

Prefill 和 Decode 的计算特征完全不同:

| 维度 | Prefill | Decode |
|------|---------|--------|
| 计算特征 | Compute-bound | Memory-bound |
| Batch 友好 | ✅ 可并行 | ❌ 难以批量 |
| GPU 利用率 | 高(大量矩阵乘) | 低(逐 token) |
| 延迟来源 | 模型计算 | KV Cache 读取 |

- **分离架构**

```
用户请求 → Prefill 集群(高算力 GPU)
↓ 计算 KV Cache
↓ 传给 Decode 集群
Decode 集群(大显存 GPU)
↓ 逐 token 生成
↓ 流式返回
```

- **优势**
1. Prefill 集群用 H100(高 FLOPS)→ 快速算完 prompt
2. Decode 集群用 A100 80GB(大显存)→ 支撑更多并发
3. 两个集群独立扩缩容
4. KV Cache 通过高速网络(RDMA)传输

- **挑战**
1. KV Cache 传输延迟(需要 RDMA/InfiniBand)
2. 负载均衡更复杂
3. 故障恢复需要处理跨集群状态

- **实际案例**
- DeepSeek:PD 分离 + 每日处理千亿 token
- Google:GBoard 的 LLM 服务
- 大厂趋势:PD 分离是大规模推理的必选项

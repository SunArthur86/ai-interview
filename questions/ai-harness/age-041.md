# Multi-Agent 系统中,Agent 之间如何通信和编排?有哪些常见模式

- **Multi-Agent 通信与编排** 是复杂 Agent 系统的核心设计问题.

- **通信方式:**

- **1. 直接消息传递:** Agent A 直接调用 Agent B 的接口.简单但不灵活.

- **2. 共享黑板(Blackboard):** 所有 Agent 读写同一个共享空间.松耦合但需要锁机制.

- **3. 事件驱动:** Agent 发布事件,其他 Agent 订阅.最灵活但调试困难.

- **编排模式:**

- **1. 层级编排(Hierarchical):** 主 Agent 分配任务给子 Agent,子 Agent 汇报结果.适合任务可清晰拆分的场景.

- **2. 竞争编排(Competitive):** 多个 Agent 独立完成同一任务,选最佳结果.适合创意类任务.

- **3. 流水线编排(Pipeline):** 任务按顺序流经多个专业 Agent.适合流程固定的场景.

- **工程代价:** Multi-Agent 的通信开销、状态同步、错误处理远比单 Agent 复杂,不是所有场景都适合.

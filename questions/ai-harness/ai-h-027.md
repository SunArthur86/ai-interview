# Prompt版本管理最佳实践?如何做Prompt的A/B测试

- **Prompt版本管理方案:**

| 方案 | 适用 | 优势 |
|------|------|------|
| Git + 文件 | 小团队 | 简单直观 |
| LangSmith | LangChain | 内置版本+评估 |
| Promptflow | 企业 | 微软开源方案 |
| Langfuse | 开源 | 自托管+实验 |
| DSPy | 优化型 | **自动优化** |

- **A/B测试流程:**
1. 定义指标(准确率/用户满意度/延迟)
2. 创建Prompt变体(v1当前, v2改进版)
3. 流量分割(50/50 或 10%金丝雀)
4. 收集数据(N>=100样本)
5. 统计检验(t-test / 序贯检验)
6. 决策(v2显著优于v1则全量切换)

- **A/B 测试流量分配架构:**

```text
User Request
    │
    ▼
┌───────────────┐
│  Router/UA    │ (Hash(UserID) % 100)
└───────┬───────┘
        │
   ─────┴─────
   │          │
   ▼          ▼
Group A     Group B
(v1 Prompt) (v2 Prompt)
   │          │
   └────┬─────┘
        ▼
  Evaluation Store
```

- **关键原则:**
1. Prompt和代码分离,每次修改有版本号+变更说明
2. 生产prompt需要审批
3. 可以快速回滚

- **## 常见考点**
1. 如何处理 Prompt 中的变量插值版本？（使用 Jinja2 模板版本化而非仅版本化纯文本）
2. 什么是“Prompt 飘移”，如何监控？（通过计算 Embedding 余弦相似度监控语义变化）
3. 在 DSPy 中，版本管理是如何工作的？（DSPy 自动优化 Prompt 中的“指令”部分，优化后的签名需单独版本化）

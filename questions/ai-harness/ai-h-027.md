# Prompt版本管理最佳实践?如何做Prompt的A/B测试

- **Prompt版本管理方案:**

| 方案 | 适用 | 优势 |
|------|------|------|
| Git + 文件 | 小团队 | 简单直观 |
| LangSmith | LangChain | 内置版本+评估 |
| Promptflow | 企业 | 微软开源方案 |
| Langfuse | 开源 | 自托管+实验 |
| DSPy | 优化型 | **自动优化** |

- **A/B测试流程:**
1. 定义指标(准确率/用户满意度/延迟)
2. 创建Prompt变体(v1当前, v2改进版)
3. 流量分割(50/50 或 10%金丝雀)
4. 收集数据(N>=100样本)
5. 统计检验(t-test / 序贯检验)
6. 决策(v2显著优于v1则全量切换)

- **关键原则:**
1. Prompt和代码分离,每次修改有版本号+变更说明
2. 生产prompt需要审批
3. 可以快速回滚

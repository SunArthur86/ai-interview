# 如何选择向量数据库?Pinecone vs Milvus vs Qdrant vs pgvector

- **向量数据库对比:**

| 数据库 | 类型 | 适合规模 | 部署 | 特色 |
|--------|------|---------|------|------|
| Pinecone | SaaS | **任何** | 云 | **最省心** |
| Milvus | 开源 | **十亿+** | 自建/云 | **最强性能** |
| Qdrant | 开源 | 亿级 | 自建/云 | **Rust高性能** |
| Weaviate | 开源 | 亿级 | 自建/云 | GraphQL API |
| Chroma | 开源 | 百万级 | 嵌入式 | **最简单** |
| pgvector | 插件 | 千万级 | PostgreSQL | **SQL集成** |

- **选型决策:**
```
规模 < 10万 → Chroma(嵌入式,零运维)
规模 10万-1000万 → Qdrant/pgvector
规模 > 1000万 → Milvus/Pinecone
不想运维 → Pinecone(贵但省心)
已有PostgreSQL → pgvector(无缝集成)
需要混合检索 → Qdrant/Weaviate(原生BM25+向量)
```

- **混合检索支持:**
- Qdrant: 原生支持(向量+payload过滤+BM25)
- Weaviate: 原生支持
- Milvus 2.4+: 支持标量过滤
- pgvector: 用SQL做全文检索+向量

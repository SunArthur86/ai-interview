# 在 RAG 系统中，如何利用 BM25、向量检索和重排序来提升召回率？

单一检索方式往往难以兼顾语义理解和关键词匹配，混合检索是提升 RAG 召回率的主流方案。

**1. 混合检索**：
- **向量检索**：擅长语义匹配，能找到“同义不同字”的文档，但可能忽略专有名词（如具体型号代码）。
- **BM25（稀疏检索）**：基于关键词匹配（TF-IDF 变种），擅长精确匹配专有名词，但无法处理语义变形。
- **策略**：并行执行两种检索，然后通过倒数排名融合（RRF）或加权分数合并结果列表。

**2. 重排序**：
- **作用**：初检阶段为了速度（从百万级文档中选 Top 50），牺牲了一定精度。Rerank 模型（如 Cross-Encoder）会对这 Top 50 文档进行精细化的两两打分，计算量大但精度极高。
- **流程**：Retriever（粗排） -> Reranker（精排） -> LLM。

这种组合策略先用混合检索保证广度（不漏掉关键信息），再用 Rerank 保证精度，确保进入 LLM 上下文的信息是最相关、最噪音最小的。

**架构流程图：**

```text
用户查询
    │
    ├──────────────────┐
    │                  │
    ▼                  ▼
向量检索           BM25 关键词检索
(Dense)             (Sparse)
    │                  │
    └──────┬───────────┘
           │
           ▼
    [倒排排名融合 RRF / 加权]
           │
           ▼
    Top-K 候选文档 (如 Top 50)
           │
           ▼
    重排序模型
           │
           ▼
    Top-N 最终文档 (如 Top 5-10)
           │
           ▼
    LLM 生成答案
```

**## 常见考点**
1. **RRF 算法原理**：面试官可能会问倒数排名融合（RRF）的具体计算公式（$1/(k+rank)$），以及它相比于简单加权分数的优势（对分数分布不敏感，鲁棒性强）。
2. **召回率与精度的权衡**：为什么要取 Top 50 给 Rerank 而不是 Top 10？（Top 10 可能漏掉潜在的高相关文档，Rerank 需要足够的候选池来纠正粗排的误差）。
3. **Cross-Encoder vs Dual-Encoder**：Rerank 模型（Cross-Encoder）为什么慢？（因为需要将 Query 和每个 Document 两两拼接输入模型进行交互计算），而检索模型（Dual-Encoder）可以提前计算好 Document 向量。
4. **混合检索的优化**：如何处理不同模态检索结果的归一化问题？（Score Normalization 或 RRF）。

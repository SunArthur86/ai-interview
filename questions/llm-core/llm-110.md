# 「大海捞针」测试是什么?它如何评估长上下文模型的检索能力

- **大海捞针测试(Needle in a Haystack):**

在超长上下文中插入一个特定事实(「针」),测试模型能否找到并使用它.

- **测试流程:**
1. 准备一段长文本(如100K tokens的文档)
2. 在文本的某个位置插入一句关键信息(「针」)
- 例:「这本书的密码是 42-7-3」
3. 让模型回答关于「针」的问题
4. 变化「针」的位置(开头/中间/结尾)和文本长度
5. 绘制热力图:X轴=位置,Y轴=长度,颜色=正确率

- **评估维度:**
- **深度**:针在不同深度位置时的检索准确率
- **长度**:不同上下文长度下的表现
- **多针**:插入多根针测试多跳检索

- **典型结果:**
- 开头和结尾准确率高(Lost in the Middle效应)
- 中间位置准确率低
- 长度越长,整体越差

- **对比:** Claude/GPT-4在200K上下文中,大部分位置能找到针;开源模型在32K以上开始下降

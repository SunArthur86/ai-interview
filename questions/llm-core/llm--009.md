# GSPO 和 DAPO 是什么?它们与 GRPO 有什么区别

- **GSPO (Group Sequence Policy Optimization)**

是对 GRPO 的改进,核心变化:
1. **序列级重要性采样**:GRPO 在 token 级别做重要性采样,GSPO 改为序列级别
2. **降低方差**:序列级归一化减少训练不稳定性
3. **更稳定的 KL 控制**:用序列级 KL 散度替代 token 级

- **DAPO (Dynamic Adaptive Policy Optimization)**

另一个 GRPO 变体:
1. **动态优势权重**:根据生成质量动态调整正负样本的权重
2. **自适应 clip 范围**:PPO 的 clip 范围 ε 根据训练进度自适应调整
3. **长序列友好**:对长推理链(如数学证明)的奖励信号更敏感

- **三者核心对比**

| 维度 | GRPO | GSPO | DAPO |
|------|------|------|------|
| 采样粒度 | token 级 | 序列级 | 动态 |
| KL 控制 | token 级 | 序列级 | 自适应 |
| 优势估计 | 组内相对 | 序列归一化 | 动态加权 |
| 适用场景 | 通用 | 稳定性优先 | 长推理链 |

- **总结**:从 GRPO → GSPO/DAPO 的演进,本质上是在解决'如何更稳定、更有效地将序列级奖励分配到 token 级别'的信用分配问题.

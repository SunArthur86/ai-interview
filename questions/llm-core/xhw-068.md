# 大海捞针测试和概率探针

🫐在预训练阶段：我们通常会准备一些“开发集”（Dev Set），这其中可包含百科文本、新
闻、代码片段等。每天或每隔一定步数，就让模型在这些测试集上做一次前向推理，计算 
PPL，看它是否持续下降。
当 PPL 下降到某个稳定水平后，说明模型对通用语料的学习已经到达了瓶颈或接近饱和。
需要注意：PPL 只适合同一模型或同一家 tokenizer 上做纵向对比，因为不同的 tokenizer 
压缩率不同，同一句话拆成多少 token 会影响 PPL 数值的绝对值。
小经验：
- 大模型在常见中文/英文混合语料上，如果预训练充分，PPL 通常能下探到 2 左右(甚至
更低)，否则可能是训练不足。
- 不同的场景（如代码 vs. 纯文本）可能有不同的典型 PPL 水平。
101. 大海捞针测试和概率探针分别是什么？
101.1 大海捞针
🫐大海捞针测试着重考察模型在超长上下文里的信息提取和理解能力
通过在长文本中随机插入关键信息，形成大型语言模型(LLM)的Prompt。该测试旨在检测大
型模型是否能从长文本中提取出这些关键信息，从而评估模型处理长文本信息提取的能力，
这可以反映LLM对长文本的理解基础能力。
方法：
- 先准备一个较长文本（数千词以上），其中混合了各种冗余或无关信息，随机插入关键细
节。
- 再让模型回答一个或多个问题，这些问题必须依赖文中的关键部分才能回答。
常见变体：
- 单一信息检索任务：长文章中只藏了一个关键信息，看看模型能否准确找出。评估LLM在
长文本中提取单一关键信息的能力，测试其对广泛叙述中特定细节的精确回忆能力。这对
应于原始的大海捞针测试任务设定。
- 多信息检索任务：文章里埋了多个分散的关键信息，需要综合才能回答。探讨LLM从长文
本中检索多个相关信息的能力，模拟实际场景中对综合文档的复杂查询。
- 多信息推理任务：在检索多个关键信息后，还要做进一步推理、整合才能得出结果。通过
提取并利用长文本中的多个关键信息来评估LLM的长文本能力，要求模型对各关键信息片
段有综合理解。

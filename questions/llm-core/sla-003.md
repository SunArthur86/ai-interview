# GQA(Grouped-Query Attention)和MQA(Multi-Query Attention)是什么？

🎯 本质：GQA和MQA通过减少KV头的数量来减少KV Cache大小，降低推理时显存和带宽需求。

📊 三种Attention对比：

1. **MHA(Multi-Head Attention)** - 标准Transformer
   - Q有n个头，K有n个头，V有n个头。
   - KV Cache = 2 * n * seq_len * head_dim。
   - 问题：KV Cache随头数线性增长。

2. **MQA(Multi-Query Attention)** - 极致压缩
   - Q有n个头，K有1个头，V有1个头。
   - KV Cache减少为1/n。
   - 缺点：质量下降。

3. **GQA(Grouped-Query Attention)** - 折中方案
   - Q有n个头，K有g个头，V有g个头（1 < g < n）。
   - 每组Q头共享一对KV。
   - 优点：质量接近MHA，显存大幅减少。

实际应用：
- Llama 2 70B: GQA, 8个KV头（原32头）。
- Llama 3: GQA, 8个KV头。
- Mistral 7B: GQA。

性能数据（Llama 2 70B）：
| 方案 | KV Cache | 推理速度 | 质量 |
|------|----------|---------|------|
| MHA(32头) | 100% | 1.0x | 最好 |
| GQA(8组) | 25% | 1.7x | 接近MHA |
| MQA(1头) | 3% | 2.1x | 略差 |

面试要点：GQA是MHA和MQA之间的甜蜜点——用很小的质量损失换取显著的推理性能提升，已成为现代LLM标配。

# 介绍一下BPE&BBPE这两种Tokenization方法

BPE (Byte Pair Encoding) 和 BBPE (Byte-Level BPE) 是两种构建子词词表的算法，核心区别在于**分词的粒度**不同。

### 1. BPE (字符级)
*   **原理**：在**Unicode 字符**级别进行操作。它将文本拆解为字符序列，然后基于字符频率进行合并。
*   **局限**：
    *   词表大小受限于字符集数量。对于包含大量生僻字、古字或特殊符号的语言，词表可能变得非常庞大。
    *   无法处理未知的 Unicode 字符，只能用 `<UNK>` 代替。

### 2. BBPE (字节级)
*   **原理**：在**字节 (Byte)** 级别进行操作。首先将文本转换为 UTF-8 字节序列（即 0-255 的数字），然后在字节序列上应用 BPE 算法进行合并。
*   **优势**：
    *   **词表极小且通用**：任何 Unicode 字符（包括中文、Emoji、特殊符号）最终都由 256 个基础字节组合而成。因此，BBPE 可以用极小的词表（如 256 个起始字符）编码世界上所有的语言和符号。
    *   **无 OOV 问题**：理论上不存在无法编码的字符。
    *   **鲁棒性强**：对于包含噪声、乱码或混合语言的文本处理能力更强。

### 3. 总结
现代大模型（如 GPT-4, LLaMA）普遍采用 BBPE 或其变体（如 SentencePiece 的 Byte Fallback），因为它能以固定的、较小的词表大小无损地处理多语言和复杂字符集，具有更好的通用性和扩展性。

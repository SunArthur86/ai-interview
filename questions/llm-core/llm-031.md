# 介绍一下BPE&BBPE这两种Tokenization方法

BPE(Byte Pair Encoding)和 BBPE(Byte-Level BPE)是自然语言处理中的两种常用的子词分割算
法.它们旨在解决词汇表规模和模型泛化能力之间的平衡问题,通过将单词分割为较小的子词单元来
降低词汇表的大小,同时保持良好的语言表示能力.
1. BPE(Byte Pair Encoding)
BPE是一种基于频率的贪心算法,最初用于数据压缩.它的核心思想是通过反复合并出现频率最高的
字符对(或子词对),从而生成一组子词单元.BPE特别适合处理低频词汇的问题,因为它可以有效
减少词汇表中的稀有词. 算法步骤 • 初始化时,将每个单词分解为字符序列. •
计算所有字符对的出现频率,找到频率最高的字符对,并将其合并为一个新子词.

动态生成子词单元,使模型的词汇表规模可控.
2. BBPE(Byte-Level BPE)
BBPE是BPE的字节级扩展版本,主要应用于多语言模型(如GPT等)和处理Unicode字符的场景.与
标准BPE不同,BBPE操作在字节层面,而不是字符层面,这允许它以更加通用的方式处理任意编码的
文本(包括非拉丁字母). BBPE的主要区别 •
BBPE对文本进行字节级别的操作,而不是字符级别.文本首先被转换为字节序列,再使用BPE进
行子词分割. •
这种方法特别适合处理复杂的字符集,如汉字、阿拉伯字母等.

3. BPE & BBPE的实际应用
•
BPE在各种自然语言处理任务中都有广泛应用,如机器翻译(如Google的神经机器翻译系统).
•
BBPE在多语言模型中使用频繁,特别是在跨语言预训练模型中(如GPT、BERT等).

5. 主流大模型架构采用的方法
目前,主流的大模型(如GPT系列、BERT、T5等)大多数采用的是BBPE(Byte-Level BPE)或类似
的字节级tokenization方法.这种方法广泛应用的原因包括:
**处理多语言文本的能力：** 大模型通常需要处理大量不同语言的文本,而BBPE在字节级别操作,能够
处理所有Unicode字符,特别是对于包含复杂字符集的语言(如中文、韩语、阿拉伯语)表现良好.
统一的tokenization : BBPE方法不依赖于语言特定的字符结构,因此可以在不增加额外复杂性
的情况下,统一应用于多语言任务,简化了预训练和下游任务中的tokenization流程.
**兼容表情符号和特殊字符：** 现代大语言模型需要处理大量互联网数据,这些数据中包含许多表情符
号、特殊字符以及非标准符号,而BBPE可以更好地支持这类符号.
6. 为什么主流大模型使用BBPE而非BPE?
**多语言与统一性：** 随着模型规模的扩大,支持多种语言和字符集成为必然需求.BBPE通过在字节层
面的操作,能够避免各语言tokenization不一致的问题.
**处理多种字符：** 由于BBPE可以处理所有Unicode字符,能够无缝处理非拉丁字符和特殊符号,具备
更好的通用性.
7. 总结
BPE 和 BBPE 通过将词分割成子词的方式来平衡词汇表的大小与语言建模的泛化能力.BPE适合处理
较为常规的语言任务,而BBPE适合多语言和字符集复杂的场景.

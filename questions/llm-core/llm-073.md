# ALiBi (Attention with Linear Biases)思路是什么

### 1. ALiBi 是什么
**ALiBi (Attention with Linear Biases)** 是一种不使用嵌入向量的位置编码方法。它通过在 Attention 的计算分数上直接减去一个与距离成正比的惩罚项来实现位置感知。

### 2. 核心思想
*   **无需位置 Embedding：** 去掉了输入层或注意力层的绝对/相对位置向量加法操作。
*   **线性偏置惩罚：** 在计算 $QK^T$ 得到分数矩阵后，对每个位置对 $(i, j)$ 施加一个偏置 $b = -m \cdot |i - j|$（其中 $i$ 是 query 位置，$j$ 是 key 位置，$m$ 是可学习或预设的斜率）。
*   **距离衰减：** 距离越远，减去的值越大（负得越多），导致 Softmax 后的权重越小。这意味着模型被鼓励更多地关注邻近的 Token。

### 3. 为什么具备长度外推能力
*   **基于相对距离：** 无论序列多长，第 $i$ 个 token 和第 $i+10$ 个 token 的相对距离始终是 10，惩罚值固定。模型不需要学习“位置 5000”是什么样的，只需要学习“距离 10”的影响。
*   **无长度限制：** 理论上可以外推到任意长度，只需继续延长距离计算即可。由于没有位置 Embedding，不存在“索引越界”的问题。

### 4. 计算流程图

```text
Standard Attention Flow with ALiBi

Input: Q [s, d], K [s, d], V [s, d]
│
├─ 1. Compute Dot Product Scores
│   Score[i, j] = Q[i] · K[j]^T
│
├─ 2. Add Positional Bias (ALiBi Step)
│   Bias[i, j] = -m * |i - j|
│   Score'[i, j] = Score[i, j] + Bias[i, j]
│   (注意：这里直接修改 Logits，且 Bias 与序列内容无关)
│
├─ 3. Softmax
│   Attn[i, j] = exp(Score'[i, j]) / Sum(exp(Score'[:, j]))
│   (由于 Bias 随距离增大变得更负，远距离 Token 的概率被压低)
│
└─ 4. Weighted Sum
    Output[i] = Sum(Attn[i, j] * V[j])
```

### 5. 优势与细节
*   **训练与推理一致性：** 短序列训练，长序列推理，无需修改模型结构（如插值）。
*   **实现简单：** 仅需修改 Attention Score 的计算逻辑几行代码，不增加参数（如果 $m$ 固定）或极少参数（如果 $m$ 可学习）。
*   **多头差异：** ALiBi 通常在不同的 Attention Head 中使用不同的斜率 $m$（例如呈几何级数递减），使不同的 Head 关注不同范围的信息。

### 6. 常见考点
1.  **对比 RoPE**：ALiBi 和 RoPE 都能做长度外推，核心区别是什么？（ALiBi 不修改 Q/K 向量，只修改 Logits；RoPE 修改 Q/K 向量本身。ALiBi 实现更简单，但 RoPE 在当前开源大模型中更为主流）。
2.  **斜率 $m$ 的设定**：多头注意力中，$m$ 是如何设定的？（通常按照 $1 / 2^{8/h}$ 的方式设置，让不同 Head 拥有不同的衰减速度）。
3.  **推理时的 KV Cache**：ALiBi 对 KV Cache 有什么影响？（由于只需计算距离，且是加性偏置，对 KV Cache 非常友好，不需要额外存储复杂的位置信息）。

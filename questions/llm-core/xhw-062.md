# 强化学习算法怎么分类

强化学习算法多种多样，可以根据不同视角来分类，这里先介绍三种广为人知的分类方式。
91.1 Value-Based 方法
🥥
Value-Based 方法的基本思想
- 在每个状态（State）下，为每个可行的动作（Action）都计算一个“价值”（Value）。
- 选择 Value 最高的动作来执行。

在强化学习中，这个 Value 值通常通过贝尔曼方程（Bellman Equation）进行更新和迭代，
既考虑当前能获得的即时奖励，也考虑未来时刻可能获得的长期收益。
示例： 假设某个状态 State A 下可行的三个动作分别是 Action1, Action2, 
Action3 ，训练过程中会迭代更新其对应的价值 V(s, a) （或常见写法 Q(s, a) ），
并最终选取价值最大的动作。
代表性算法
- Q-Learning
- SARSA (State-Action-Reward-State-Action)
适用场景
- 动作空间离散（Discrete） 
◦
例如 Pacman 的上下左右
- 或者最终目标策略是确定性的（Deterministic）
◦
例如，每次在某个状态就执行同一个动作
91.2 Policy-Based 方法
🥥
Policy-Based 方法的基本思想
Policy-Based 方法则是直接对“策略（Policy）”本身进行建模，从而学习在各状态下，动
作对应的概率分布，而不是去间接计算 Value。
示例：
 在给定状态 State A 时，你可能学得一套策略：
- 以 0.5 的概率选择 Action1 
- 以 0.3 的概率选择 Action2 
- 以 0.2 的概率选择 Action3

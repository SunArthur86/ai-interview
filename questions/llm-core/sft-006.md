# 什么是模型的量化(Quantization)？GGUF/GPTQ/AWQ有什么区别？

🎯 本质：量化是将模型权重从高精度（fp16/bf16）压缩到低精度（int8/int4），以减少显存占用和加速推理，代价是少量精度损失。

📊 量化方法对比：

1. PTQ（Post-Training Quantization，训练后量化）
  不需要重新训练，直接对已训练模型进行量化
  
  GPTQ：
    基于二阶信息（Hessian矩阵）的逐层量化
    需要校准数据（少量样本计算量化参数）
    适合GPU推理
    精度损失：int4时约1-2%
  
  AWQ（Activation-aware Weight Quantization）：
    发现不是所有权重都同等重要
    保护"激活值大"的通道（更重要）不量化或低比特量化
    不需要反向传播，校准数据少
    精度比GPTQ略好
  
  GGUF（原GGML）：
    llama.cpp的格式，专为CPU推理设计
    支持2-8bit量化（Q2_K, Q3_K_M, Q4_K_M, Q5_K_M, Q8_0等）
    K-quants：混合精度量化，不同层用不同精度
    最适合边缘设备/本地部署

2. QAT（Quantization-Aware Training，量化感知训练）
  在训练中模拟量化效果
  精度最好但需要训练资源
  代表：LLM.int8()

量化精度影响：
| 精度 | 7B模型大小 | 质量损失 | 适用场景 |
|------|-----------|---------|---------|
| FP16 | 14GB | 0% | GPU训练/推理 |
| INT8 | 7GB | <0.5% | GPU推理 |
| INT4 | 3.5GB | 1-2% | GPU/边缘推理 |
| INT2 | 1.8GB | 5-10% | 极端资源受限 |

选择建议：
| 场景 | 推荐方案 |
|------|---------|
| GPU服务器推理 | GPTQ/AWQ INT4 + vLLM |
| 本地CPU推理 | GGUF Q4_K_M + llama.cpp |
| 边缘设备(手机) | GGUF Q2_K/Q3_K |
| Apple Silicon | GGUF + Metal加速 |
| 最高质量 | FP16/BF16不量化 |
| 平衡 | INT4/INT8量化 |

面试要点：量化是"部署LLM的第一道防线"——让大模型能在消费级硬件上运行。int4通常是质量和大小的甜蜜点。

# Infini-attention 和 Ring Attention 分别解决什么问题?它们的原理是什么

- **两种扩展注意力窗口到无限长度的方案**

- **Infini-attention (Google, 2024)**
**核心思想：** **压缩记忆 + 局部注意力**
- 将输入分段,每段用标准因果注意力处理
- 段间用**压缩记忆**(delta rule 更新)传递信息
- 毣合当前段的注意力输出和压缩记忆的检索输出

**优点：** 
- 显存恒定(不随序列长度增长)
- 可处理无限长度(理论上)
- 兼容标准 Transformer 训练

**公式：** A_infi = σ(A_dot) + (1-σ) · A_mem
(σ 是可学习的门控,平衡局部注意力和记忆检索)

- **Ring Attention (UC Berkeley, 2023)**
**核心思想：** **跨 GPU 环形通信**
- 将长序列切分到多个 GPU
- GPU 组成环形拓扑,KV 块沿环传递
- 每收到一个 KV 块就计算部分注意力

**优点：** 
- 支持百万级上下文(1M+ tokens)
- 不修改模型结构,纯系统级优化
- 与 Flash Attention 兼容

**缺点：** 
- 需要 GPU 数量与序列长度成正比
- 环形通信增加延迟

- **对比**:Infini 改模型结构(显存O(1)),Ring 改系统调度(多卡扩展).

# SwiGLU激活函数是什么？为什么比ReLU好？

**SwiGLU（Swish-Gated Linear Unit）**是GLU（Gated Linear Unit）的一种变体，通过门控机制增强非线性表达能力，是目前LLM（如Llama 3、PaLM）主流的FFN激活函数。

### **公式详解**
SwiGLU 结合了 Swish 激活函数和门控线性单元：
$$ \text{SwiGLU}(x) = \text{Swish}_{\beta}(xW_G) \otimes (xV) $$
其中：
- $\text{Swish}_{\beta}(x) = x \cdot \sigma(\beta x)$，通常 $\beta=1$（此时等价于 SiLU）。
- $W_G$ 和 $V$ 是两个独立的权重矩阵。
- $\otimes$ 表示逐元素乘法。

在 LLM 的前馈神经网络（FFN）层中，完整的计算流程为：
$$ \text{FFN}_{\text{SwiGLU}}(x) = (\text{Swish}(xW_G) \otimes (xV))W_{out} $$

### **参数量与维度调整**
标准 Transformer FFN（使用 ReLU）包含两个投影层：$W_1$ ( $d \to 4d$ ) 和 $W_2$ ( $4d \to d$ )。
SwiGLU 包含三个投影层：$W_G$, $V$ (均为 $d \to h_{hidden}$ ), $W_{out}$ ( $h_{hidden} \to d$ )。

为了保持**总参数量**与标准 FFN 相当，通常需要缩小隐藏层维度。标准 FFN 参数量约为 $8d^2$。SwiGLU 参数量约为 $2d \cdot h_{hidden} + h_{hidden} \cdot d = 3d \cdot h_{hidden}$。
令 $3d \cdot h_{hidden} \approx 8d^2$，可得 $h_{hidden} \approx \frac{8}{3}d$。

### **结构对比图**
```text

  ▼ (Linear)                  ├───────────┐

  │                            ▼           ▼

  │                            │           │
  ▼                            ▼           ▼

  W2 (4d -> d)               (SiLU)        │
  │                            │           │
  ▼                            └─────┬─────┘
Output                              ▼
                               Element-wise Mul
                                    │
                                    ▼
                               W_out (8/3d -> d)
                                    │
                                    ▼
                                 Output
```

### **相比 ReLU 的优势**
1. **平滑性与梯度流动**：Swish 处处可导（ReLU 在 0 处不可导），曲线平滑，有助于优化器（如 Adam）在非凸优化景观中更稳定地收敛，减少梯度震荡。
2. **非单调性与负区间激活**：ReLU 在 $x<0$ 时梯度恒为 0，易导致神经元“死亡”。Swish 在负区间有非零输出和梯度，允许模型保留和利用负轴信息，提高了模型的容量。
3. **门控机制**：引入了类似 LSTM 的门控思想。$xW_G$ 生成的门控信号能够动态调节 $xV$ 信息的通过量，让模型自动学习“哪些特征重要”，增强了表达能力。
4. **性能表现**：在同等参数规模下，使用 SwiGLU 的模型在 Perplexity 和下游任务表现上均显著优于 ReLU/GELU。

### **代价**
计算复杂度略有增加。由于存在三个矩阵乘法，若不调整隐藏层维度，FLOPs 会增加 50%。但通过将维度调整为 $8/3d$，在保持参数量不变的前提下，实际上是增加了矩阵乘法的密度（更多的小矩阵运算 vs 少量的大矩阵运算），通常带来的性能收益远超计算开销。

## 常见考点
1. **参数量计算**：为什么 Llama 的 FFN 维度通常设置为 $8/3 \times d_{model}$ 而不是标准的 $4 \times d_{model}$？（为了平衡 SwiGLU 引入的第三个权重矩阵，保持总参数量与标准架构一致）
2. **Swish vs SiLU**：SwiGLU 中使用的 Swish 激活函数在 $\beta=1$ 时等同于 SiLU，Llama 使用的是哪一个？（通常使用 SiLU）
3. **GeGLU vs SwiGLU**：PaLM 使用的是 GeGLU（门控使用 GELU 激活），Llama 使用的是 SwiGLU，两者的核心区别在于激活函数的选择，计算结构完全一致。

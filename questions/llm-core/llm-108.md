# 什么是 4D 并行训练?数据/张量/流水线/Expert 并行如何组合

- **4D 并行 = Data + Tensor + Pipeline + Sequence(序列)并行**
*注：原答案中 Expert 并行通常用于 MoE 架构，标准 4D 并行通常指 DP/TP/PP/SP。此处按通用 4D (DP/TP/PP/SP) 扩充。*

| 维度 | 切分对象 | 通信类型 | 适合场景 | 显存占用 |
|------|---------|---------|------|----------|
| **数据并行(DP)** | Batch | AllReduce(梯度) | 通用，追求高吞吐 | 高(每卡复制全量模型) |
| **张量并行(TP)** | 权重矩阵 | AllReduce(激活值) | 单卡放不下的大模型 | 低(权重分片) |
| **流水线并行(PP)** | 模型层 | 点对点(激活值) | 极深模型(64层+) | 中(每卡负责部分层) |
| **序列并行(SP)** | 序列长度 | AllGather/Ring | 长上下文(>32k) | 低(Attention Cache分片) |

- **组合策略与拓扑结构:**
```
物理集群: 8 Nodes, 8 GPUs/Node = 64 GPUs

逻辑视图 (DP=2, TP=4, PP=4):

Node 0-1 (Data Group 1):
  GPU 0: PP-Stage-1 [TP-Group-0] ---
  GPU 1: PP-Stage-1 [TP-Group-1]    | (TP通信，通常依赖 NVLink)
  GPU 2: PP-Stage-1 [TP-Group-2]    | (单机内带宽极高)
  GPU 3: PP-Stage-1 [TP-Group-3] ---
  GPU 4: PP-Stage-2 [TP-Group-0]    |
  ...                              | (PP通信，依赖跨节点 InfiniBand)
  GPU 7: PP-Stage-2 [TP-Group-3] ---
                                   |
Node 2-3 (Data Group 2): ... (复制 DP Group 1 的模型)

通信层级:
1. TP: 节点内 NVLink (带宽最快)
2. PP: 跨节点流水线接力 (带宽次之)
3. DP: 跨 Data Group AllReduce (带宽要求中等)
4. SP: 混合使用 Ring-Attention 机制
```

- **关键计算公式:**
``n GPUs (Total) = DP_grad * TP_model_dim * PP_depth * SP_seq_len``
例: 2048 GPU = 32(DP) * 8(TP) * 8(PP)

- **DeepSpeed ZeRO 与 并行的关系:**
- ZeRO-1: 优化器状态分片 (优化 DP 显存)
- ZeRO-2: +梯度分片 (优化 DP 显存)
- ZeRO-3: +模型参数分片 (极端情况下的 DP)
- **注意**: ZeRO 通常与 Data Parallel 结合使用。如果启用了 TP/PP，则 ZeRO 主要应用于 DP 组内部，或者仅开启 ZeRO-1 (切 optimizer)，避免对 TP/PP 的权重切分造成冲突。

# 请详细解释 Scaling Laws(缩放定律)和 Chinchilla 最优策略,它对大模型训练有什么指导意义

- **Scaling Laws (Kaplan et al. 2020)**
**核心发现：** 模型 Loss 与参数量(N)、数据量(D)、计算量(C) 存在幂律关系:

L(N) ∝ N^(-0.076),L(D) ∝ D^(-0.095),L(C) ∝ C^(-0.05)

**关键结论：** 
1. 模型越大,loss 越低(但边际递减)
2. 参数量和数据量需要同步增长
3. Kaplan 建议 N 和 D 按 ~1:8.4 的比例增长

- **Chinchilla (Hoffmann et al. 2022)**
修正了 Kaplan 的结论:
- 最优策略是 **N:D ≈ 1:20**(每个参数训练 ~20 个 token)
- 例:70B 模型最优训练量 ≈ 1.4T tokens
- 之前很多模型'参数多但训练不足'(compute-suboptimal)

- **实践指导**
1. 给定计算预算 C,最优 N = (C / 6)^(1/2),D = (C / 6)^(1/2)
2. 不要盲目增大参数量,数据量同样重要
3. Llama 3 用 15T tokens 训练 70B 模型,远超 Chinchilla 最优,但效果更好(因为数据质量高)

- **MoE 与 Scaling Laws**
- MoE 打破了稠密模型的 Scaling Law
- 激活参数而非总参数决定推理成本
- DeepSeek-V3: 671B 总参数,37B 激活,训练 14.8T tokens

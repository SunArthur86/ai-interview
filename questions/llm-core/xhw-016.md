4. KV Cache的挑战和优化
- 内存占用: 随着序列长度的增加，KV Cache 的内存占用线性增长，可能达到模型大小的数倍。在显
存受限的情况下，这种内存需求可能成为限制模型性能的瓶颈。
- 优化策略:
◦
量化技术: 减少模型权重和 KV Cache 的内存占用。
◦
模型并行: 将模型分片到多个 GPU 上。
◦
压缩和内存管理: 采用压缩技术（如 FastGen 方法），通过自适应压缩减少 KV Cache 的内存需
求，同时保持模型效率。
5. 总结
KV Cache 是一种重要的推理优化技术，通过缓存计算过的键和值，减少冗余计算，提高了 
Transformer 模型的推理效率。尽管它带来了内存占用的问题，但通过各种优化策略，这些问题可以
得到有效缓解，从而提升了大规模模型的性能。
25. Transformer相比RNN、LSTM的优势何在？
1. 并行计算能力
- RNN/LSTM：RNN和LSTM是顺序模型，依赖于序列的前后顺序处理数据，当前时刻的输出依赖于
前一时刻的输入。这种顺序性限制了模型的并行计算能力，导致训练速度较慢，尤其当序列长度增
加时，效率下降明显。
- Transformer：Transformer通过自注意力机制（Self-Attention）消除了序列顺序的依赖，允许模
型同时处理输入序列的不同部分。这样大大提升了并行处理能力，训练速度也比RNN/LSTM快得
多，尤其是在处理长序列时。
2. 长距离依赖问题
- RNN/LSTM：虽然LSTM通过门控机制（如遗忘门和输入门）在一定程度上解决了长距离依赖问
题，但依然可能由于梯度消失或梯度爆炸而导致模型难以捕捉长距离依赖。
- Transformer：Transformer的自注意力机制能够直接建立序列中任何位置的全局依赖，无论距离
多远。因此在处理长距离依赖时，Transformer更为有效。

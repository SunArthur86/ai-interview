# Transformer相比RNN/LSTM的优势

Transformer 相比 RNN/LSTM 的核心优势主要体现在并行计算能力和长距离依赖捕捉上。

**1. 并行计算能力**
*   **RNN/LSTM**：必须按照时间步逐步处理，$t$ 时刻的计算依赖 $t-1$ 时刻的隐藏状态，无法充分利用 GPU 并行计算能力，导致训练耗时极长。
*   **Transformer**：通过 Self-Attention 机制，一次运算即可处理整个序列。所有位置的特征可以并行计算，极大地提升了训练效率，尤其是在长序列上。

**2. 长距离依赖**
*   **RNN/LSTM**：信息需要随时间步传递，距离越远，信息丢失越严重（梯度消失/爆炸问题）。虽然 LSTM 引入门控缓解了此问题，但对超长距离的依赖依然难以捕捉。
*   **Transformer**：任意两个词之间的距离通过 Attention 矩阵直接连接（路径长度为 1）。无论序列多长，模型都能直接捕获任意位置之间的关联，长距离依赖捕捉能力极强。

**3. 特征提取能力**
多头注意力机制允许模型从不同的表示子空间（如语法、语义）并行地关注信息，比 RNN 单一的隐藏状态传递更具表达力。

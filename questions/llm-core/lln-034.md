# Reward Model常见的hacking有哪些

Reward Model (RM) 常见的 Hacking（欺骗）现象：

1. **长度 Hacking**：如果 RM 训练数据有限，模型容易学到“回答越长越好”的错误偏好，从而生成冗长但空洞的回答。
2. **风格 Hacking**：RM 可能过度偏好某些表面格式，如 Markdown、LaTeX 公式等，导致模型为了获得高分强行堆砌格式，而忽略内容的正确性。
3. **CoT Hacking**：RM 可能偏好“思维链”的形式本身，而不是真正有助于解题的推理过程，导致模型生成看起来像推理但实则无效的步骤。
4. **事实与安全边界 Hacking**：对于事实类（知识原子冲突）或安全边界类（拒答 vs 回答）问题，RM 难以给出精准判断，容易导致模型产生幻觉或违规。

# PPL(Perplexity)及其数学公式

100. 1 定义与解释
🫐PPL(Perplexity,困惑度) 是衡量语言模型预测下一个 token 难易程度的经典指标.对自
回归语言模型来说,给定一个句子 ,令模型输出的概率分布为

X = (x , x , ... , x ) 1 2 N p(x , x , ... , x ) = 1 2 N p(x ) ⋅ 1 p(x ∣ 2 x ) ⋅ 1 p(x ∣ 3 x , x ) ⋯p(x ∣ 1 2 N x , x , ... , x ) 1 2 N−1 则句子的平均负对数似然(NLL)定义为

−

ln(p(x ∣ N 1 n=1 ∑ N n x )) <n 其中 表示前面所有 token. x

<n PPL 通常定义为:

PPL(X) = exp(−

ln p(x ∣ N 1 n=1 ∑ N n x )) <n
如果模型对真实 token 的概率很高,则 NLL 值小,PPL 值也更小.
PPL 可以理解为“模型每次猜下一个 token 的不确定度”,越接近 1 表示越确定.

100. 2 如何用 PPL 来评估模型?
**🫐在预训练阶段：** 我们通常会准备一些“开发集”(Dev Set),这其中可包含百科文本、新
闻、代码片段等.每天或每隔一定步数,就让模型在这些测试集上做一次前向推理,计算
PPL,看它是否持续下降.
当 PPL 下降到某个稳定水平后,说明模型对通用语料的学习已经到达了瓶颈或接近饱和.
**需要注意：** PPL 只适合同一模型或同一家 tokenizer 上做纵向对比,因为不同的 tokenizer
压缩率不同,同一句话拆成多少 token 会影响 PPL 数值的绝对值.
**小经验：**
•
大模型在常见中文/英文混合语料上,如果预训练充分,PPL 通常能下探到 2 左右(甚至
更低),否则可能是训练不足. •
不同的场景(如代码 vs. 纯文本)可能有不同的典型 PPL 水平.

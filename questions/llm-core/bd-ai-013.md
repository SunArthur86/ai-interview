# 【字节面经】微调（Fine-tuning）和RAG的使用场景有什么区别？什么场景必须微调？

**一句话：让模型知道新知识用RAG，让模型学会新能力用微调。**

**RAG：** 在提问时给模型补充资料，不改模型本身。改文档就行实时生效，成本低。适合知识库问答、文档检索。

**微调：** 训练时改变模型行为，需要GPU和标注数据，成本高。但能让模型学会特定风格的输出、特定领域的推理模式。

**大多数应用开发场景，RAG优先。** 微调是更重的武器，需要明确的场景和足够的资源。

**什么场景必须微调？**
- 你希望模型输出某种特定格式和推理链路，而且这个模式要在各种输入下都稳定
- RAG只能提供参考资料，不能保证模型按你想要的方式去思考
- 举例：法律文书的写作风格、医学影像的判读逻辑、特定编程语言的代码规范

**决策框架：**
| 需求 | 方案 |
|------|------|
| 知识更新频繁 | RAG |
| 特定输出风格 | 微调 |
| 特定推理模式 | 微调 |
| 低成本快速上线 | RAG |
| 领域术语理解 | 微调+RAG组合 |
| 降低延迟（减少检索） | 微调（知识内化） |

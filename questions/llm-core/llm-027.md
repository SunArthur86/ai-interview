# 介绍一下VLLM的加速方法

vLLM 是一个高性能的大语言模型推理服务框架，其核心加速技术主要体现在 **PagedAttention** 和 **Continuous Batching** 机制上。

### 1. PagedAttention（核心创新）

**背景问题**
LLM 推理中，KV Cache 占用大量显存且是动态变化的。传统的内存管理算法（如动态分配）容易产生内存碎片，导致显存利用率低，限制了 Batch Size 的大小。

**解决方案**
vLLM 借鉴了操作系统中**分页虚拟内存**的思想来管理 KV Cache。
*   **Block 划分**：将 KV Cache 切分成固定大小的 Block（例如每个 Block 存储连续 16 个 Token 的 KV）。
*   **非连续存储**：逻辑上连续的 Prompt 序列，其 KV Cache 在物理显存中可以是不连续的 Block。
*   **优势**：
    1.  **减少碎片**：像操作系统管理内存一样，高效利用显存空间。
    2.  **共享内存**：在 Parallel Sampling 或 Beam Search 等场景下，相同的 Prompt 序列可以物理共享同一个 Block，大幅节省显存。
    3.  **高效批处理**：显存利用率提高意味着可以同时处理更多请求，提升吞吐量。

### 2. Continuous Batching（连续批处理 / Iterative Scheduling）

**背景问题**
传统的 Static Batching 要求同一个 Batch 内的所有序列必须对齐（通常 Pad 到最长）。当某些序列生成长度较短时，需要等待 Batch 内最长的序列生成完毕才能释放资源，导致大量 GPU 空转（Padding 的位置也在计算或等待）。

**解决方案**
*   **动态调度**：vLLM 允许在一个 Batch 内，不同序列处于不同的生成阶段。
*   **即时替换**：当 Batch 中的某个序列生成结束，立即腾出位置插入新的待处理序列，无需等待其他序列完成。
*   **优势**：极大提高了 GPU 的利用率，显著提升系统的整体吞吐量。

### 3. 调度流程图
```text
时间 T1 (Batch 初始状态)                  时间 T2 (Sequence A 完成, Sequence C 新增)
+-----------------------+                +-----------------------+
| GPU Batch (Slot Map)  |                | GPU Batch (Slot Map)  |
+-----------+-----------+                +-----------+-----------+
| Seq A: [P1, P2, P3]   |  (生成中)      | Seq B: [P1, P2, ...] | (继续生成)
| Seq B: [P1, P2, ...]  |  (生成中) ---> | Seq C: [P1]           | (新增/开始)
| Seq C: [Waiting...]   |  (队列中)      | Seq D: [Waiting...]   | (队列中)
+-----------------------+                +-----------------------+
   A,B 一起并行计算                             C 与 B 一起并行计算
```

### 4. 其他优化
*   **高吞吐 Kernel 实现**：优化了 Attention 和 CUDA 算子，适配 PagedAttention 的数据结构，核心是减少 HBM（高带宽内存）读写次数。
*   **张量并行**：支持模型切分分布到多 GPU 上推理，解决单卡显存无法容纳大模型的问题。

### 5. 常见考点
1.  **PagedAttention vs 传统管理**：为什么传统连续内存分配会导致 OOM（显存溢出）？（答：外部碎片问题，即空闲显存总和足够，但没有足够大的连续块存放新序列）。
2.  **Continuous Batching 的限制**：虽然提高了吞吐，但对延迟有什么影响？（答：增加了调度器的开销，且为了凑 Batch 可能会轻微增加首字节延迟，但整体 Time To First Token (TTFT) 受益于 Compute Optimization）。
3.  **KV Cache 共享**：在 Parallel Sampling（比如一次性生成 5 个结果）中，PagedAttention 如何节省显存？（答：前缀部分的 KV Block 在物理内存中只存一份，通过引用计数被多个 Sequence 共享）。

# 介绍一下VLLM的加速方法

VLLM (Versatile Large Language Model)是一个专门为大规模语言模型 (LLM) 推理设计的开源加速 框架,通过创新的内存管理和并行化技术,显著提高了推理速度和吞吐量. 其中,PagedAttention是 VLLM 的核心技术,专门用于解决 LLM 推理中的内存瓶颈问题,尤其是自 回归生成任务中的键值 (KV) 缓存管理.
1. PagedAttention 机制
PagedAttention 的主要目标是高效管理注意力机制中的 KV Cache,从而减少内存碎片和重复存储带 来的显存浪费.在传统的自回归解码过程中,模型需要将所有输入 token 的键和值张量(KV 缓存)存 储在 GPU 内存中,以生成下一个 token.这种方法占用了大量显存,尤其当处理长序列时,GPU 内存 成为推理过程中的瓶颈. PagedAttention 通过引入类似于 虚拟内存管理 的思想,将 KV 缓存进行物理分块管理.即使模型在逻 辑上需要连续的输入序列,实际的物理内存块可以是非连续的.这样做的好处是极大减少了 KV 缓存中 的内存浪费,并提高了推理过程中的内存利用率. PagedAttention 的具体工作原理 **物理分块：** PagedAttention 将 KV 缓存进行物理分块,每块显存包含固定长度的 tokens.在进行注 意力计算时,VLLM 会根据需要从这些分块中读取键值缓存.尽管模型在逻辑上处理的是一个连续的序 列,但这些序列在内存中的实际位置是分散的. **减少重复缓存：** 在处理多个输出时,VLLM 可以将不同的逻辑块映射到同一个物理块,避免了重复存 储同样的数据.这种方法显著减少了显存的占用,同时提高了吞吐量. 通过这种分块管理方式,VLLM 可以在保持推理速度的同时,极大降低内存占用,尤其是应对长序列推 理时的内存瓶颈.
2. Continuous Batching机制

另一个提升 VLLM 推理速度的重要机制是 Continuous Batching,它优化了批处理过程.传统的批处 理方法(static batching)要求所有输入序列的长度对齐,这意味着较短的句子需要等待较长句子生 成完毕,导致 GPU 计算资源被浪费. VLLM 采用 Continuous Batching,即每当某个句子的推理完成时,GPU 会立即填充下一个句子的 token,而不需要等待整个批次的推理完成.这种动态的批次管理方式充分利用了 GPU 的计算能力, 减少了等待时间,极大提高了吞吐量.
3. PagedAttention 与 Continuous Batching 的协同作用
PagedAttention 和 Continuous Batching 共同作用,优化了 VLLM 的推理性能.PagedAttention 通 过精细管理 KV 缓存,减少了推理过程中的内存开销,特别是在处理长序列或多个输出时的内存浪费. 而 Continuous Batching 则确保 GPU 计算资源得到最大化利用,使得在推理过程中几乎没有等待时 间.
4. VLLM 其他特性
支持张量并行推理 VLLM 支持通过张量并行来进一步加速推理,将模型的计算分布到多个 GPU 上,降低单个 GPU 的内 存和计算压力. VLLM 通过实现高效的张量并行(Tensor Parallelism)来加速推理,将模型分割并分布到多个 GPU 上,使每个 GPU 仅处理部分计算和内存负载.张量并行将大的权重矩阵(例如 Transformer 层中的矩 阵)划分为更小的片段,并分布到不同的 GPU 上. 对于矩阵乘法 ,其中 , ,在使用 个 GPU 切分后,每个 GPU 计 算矩阵的一部分,具体为 ,其中

**流式输出：**
VLLM 支持流式输出,可以在生成过程中逐步返回生成的 token,减少响应时间,适合需要低延迟的 场景. VLLM 使用基于流式预取(streaming-based prefetching)的技术来将通信和计算重叠,尽量减少 GPU 的空闲时间.在模型的一部分正在执行时,下一部分的数据会提前加载并准备好,从而确保 GPU 不会因等待数据而闲置. 与 HuggingFace 模型无缝集成 VLLM 可以与 HuggingFace 的模型兼容使用,方便用户直接在 HuggingFace 提供的模型基础上进行推 理加速.
5. 总结(性能提升)
通过这些优化,VLLM 实现了比 HuggingFace Transformers 高 14-24 倍的吞吐量,特别在长序列推理 和多请求并发的场景中表现尤为突出.

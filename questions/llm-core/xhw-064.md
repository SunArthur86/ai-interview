94.4 为什么 MLA 依旧能在推理时带来速度提升？
94.4.1 推理过程分两步：Prefill 与 Generation
🥥Prefill：
一次性处理输入序列，得到所有 Token 的状态并存入 KV Cache。这一步对带宽和显存要求
最高。
Generation：
逐 Token 生成输出，每产生一个 Token，就访问全部缓存。

94.4.2 MLA 在 Prefill & Generation 的表现
🥥Prefill 阶段
MLA 要做额外的上采样变换（计算 Key、Value），计算量上比 GQA 多一些；但KV Cache 
小，带宽压力降低。相对传统 MHA，整体也有可能打个平手或更优。
Generation 阶段
每个新 Token 都要读写过去的 Key、Value；当 Cache 更小，带宽开销也更小；
MLA 通过只缓存 
，在需要时再做变换或“吸收到 Q 的投影里”；这样访问量显著减少，
生成速度就会快不少。

i

速度和内存占用方面的明显好处。

# 什么是 warmup_ratio?训练时为什么要做 warmup?如何设置

- **Warmup:** 训练初期用极小学习率,逐渐增加到目标值.

- **为什么需要 Warmup:**
1. **初始权重随机** - 模型刚开始训练，特征空间处于混沌状态，大学习率会导致梯度爆炸或模型陷入极差的局部最优。
2. **Adam优化器动量** - Adam 等自适应优化器利用一阶/二阶矩估计，初期统计量不准确，Warmup 给予缓冲期累积正确的梯度统计量。
3. **稳定训练** - 在深层网络中，有助于稳定 LayerNorm 和 Attention 层的梯度流，防止数值不稳定。

- **warmup_ratio:** warmup步数占总训练步数的比例.

- **Warmup 学习率变化曲线:**
```text
LR
^                                     ________ (Constant / Decay)
|                                    /
|                                   /
|                    ______________/  <-- 目标 LR
|                   /
|                  /   <-- 线性增长
|                 /
|________________/
+-----------------------------------------> Steps
   |<-- Warmup Steps -->|
```

- **典型设置:**

| 场景 | warmup_ratio | 学习率 | 策略 |
|------|-------------|--------|------|
| 预训练 | 0.01-0.05 | 1e-4~3e-4 | Warmup -> Cosine Decay |
| SFT | 0.03 | 2e-5 | Warmup -> Constant / Cosine |
| LoRA | 0.03 | 1e-4~3e-4 | Warmup -> Constant |
| 增量预训练 | 0.05-0.1 | 5e-5 | Warmup -> Cosine Decay |

- **代码(HuggingFace):**
```python
training_args = TrainingArguments(
warmup_ratio=0.03,
lr_scheduler_type="cosine",
learning_rate=2e-5,
)
```

- **注意:** 在使用 DeepSpeed ZeRO 等大规模并行训练时，Warmup 不仅是稳定性需求，也是为了避免通信初期由于梯度异常导致的训练发散。

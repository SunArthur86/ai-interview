# 什么是 warmup_ratio?训练时为什么要做 warmup?如何设置

- **Warmup:** 训练初期用极小学习率,逐渐增加到目标值.

- **为什么需要 Warmup:**
1. **初始权重随机** - 大学习率会导致梯度爆炸
2. **Adam优化器** - 需要时间累积梯度统计量
3. **稳定训练** - 逐步适应损失函数landscape

- **warmup_ratio:** warmup步数占总训练步数的比例.

- **典型设置:**

| 场景 | warmup_ratio | 学习率 |
|------|-------------|--------|
| 预训练 | 0.01-0.05 | 1e-4~3e-4 |
| SFT | 0.03 | 2e-5 |
| LoRA | 0.03 | 1e-4~3e-4 |
| 增量预训练 | 0.05-0.1 | 5e-5 |

- **调度策略:**
- Warmup → Cosine Decay(最常用)
- Warmup → Linear Decay
- Warmup → Constant

- **代码(HuggingFace):**
```python
training_args = TrainingArguments(
warmup_ratio=0.03,
lr_scheduler_type="cosine",
learning_rate=2e-5,
)
```

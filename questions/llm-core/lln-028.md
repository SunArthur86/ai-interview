# 作packing可能有什么坏处

1. **削弱对难样本的拟合**
   - 在无 Packing 时，若 Batch 中包含短 Query，其余为 Padding，该 Batch 的梯度完全由短文本贡献，模型对该样本的拟合强。
   - 使用 Packing 后，多个短文本拼接在一个样本中，单个短文本的梯度会被 Batch 内其他样本稀释，导致模型对特定难样本的拟合能力变弱。

2. **对多轮对话的干扰**
   - 目前微调框架多使用全局 Attention，无法利用 Block Attention 进行完美隔离。Packing 常用 Padding 或 EOS Token 隔断不同对话，但这无法完全切断 Seq 之间的 Attention 联系。
   - 在多轮对话中，模型难以区分“当前对话”与“历史对话”。由于 Attention 机制的作用，模型容易关注到上下文中不相关的历史对话内容（前一个 Seq），导致指令遵循能力下降。若强行使用，甚至会造成算力浪费。

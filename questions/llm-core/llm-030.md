# WordPiece如何考虑语言模型的概率

🐠WordPiece最初用于谷歌的机器翻译系统,与BPE类似,但在合并时考虑了语言模型的概 率.即WordPiece算法利用语言模型的概率来指导子词(subword)的合并.
30. 1 WordPiece算法中的概率考虑
1. 与BPE的区别
• BPE(字节对编码):在每次迭代中,合并频率最高的相邻符号对(字符或子词),不考虑合并对 整体语言模型概率的影响. • **WordPiece：** 在每次迭代中,选择能够最大化训练语料库整体概率的符号对进行合并,即考虑了语 言模型的概率.
2. 语言模型概率的概念

• 语言模型(Language Model,LM):给定一系列子词,计算这些子词序列在语料库中出现的概 率. • **在WordPiece中：** 使用一元(unigram)语言模型,假设每个子词是独立的,整个语料库的概率是 所有子词概率的乘积.

30. 5 与BPE的对比

**BPE**
BPE • **合并策略：** 每次合并频率最高的符号对. • **优点：** 简单高效. • **缺点：** 只考虑了符号对的频率,可能合并出一 些不具有语言学意义的子词.
**WordPiece**
WordPiece • **合并策略：** 每次合并能最大化语料库概率的符 号对. • **优点：** 考虑了子词对整体语言模型的贡献,生 成的子词更有意义. • **缺点：** 计算复杂度较高,需要计算每个合并候 选的 .

# SFT(Supervised Fine-Tuning)、RLHF、DPO三者有什么区别？如何选择？

🎯 **本质**：SFT教模型"怎么回答"（格式/指令遵循），RLHF/DPO教模型"怎么回答更好"（符合人类偏好/价值观）。三者在LLM训练流水线中扮演不同角色。

📊 **训练流水线全景图：**

```

│                  LLM Alignment Pipeline                         │
├─────────────────────────────────────────────────────────────────┤

│       │                                                         │
│       ▼                                                         │
│  [SFT] <───────────────────┐                                   │
│  (指令微调)                 │ 数据: (Prompt, Response)         │

│       │                     │                                   │

│  [DPO / RLHF]               │ 数据: (Prompt, Win, Lose)        │
│  (偏好对齐)                 │                                   │
│  目标: 人类偏好/安全/有用   │                                   │

│       ▼                     │                                   │

│                                                                 │
```

**1. SFT（监督微调）**
  - **目标**：让预训练模型学会指令遵循和对话格式
  - **数据**：高质量配对数据，通常 10k - 100k 条
  - **方法**：标准 Cross-Entropy Loss，$L = -\log P(y|x)$
  - **类比**：教学生背诵标准答案，形成基本的语言和逻辑能力
  - **注意**：数据质量 > 数量，GPT-3.5 主要靠高质量的 SFT 数据

**2. RLHF（基于人类反馈的强化学习）**
  - **目标**：让模型输出更符合人类偏好（有用、无害、诚实）
  - **三步流程**：
    - **Step 1 (RM)**: 训练奖励模型 - 用人类标注的偏好数据 (Prompt, A, B, Better)
    - **Step 2 (PPO)**: 用 RM 作为奖励，PPO 算法优化策略模型
      - Objective: $\text{maximize } E[RM(p, y)] - \beta \cdot KL(\pi|\pi_{SFT})$
    - **Step 3 (KL Penalty)**: 防止模型为了高分而畸形优化，偏离 SFT 模型太远
  - **缺点**：训练不稳定，需要同时训练 Policy 和 RM，资源消耗大

**3. DPO（直接偏好优化）**
  - **目标**：直接优化策略模型，无需显式的 Reward Model
  - **原理**：利用 Bradley-Terry 模型推导，将 RL 的优化目标转化为 Loss 函数
    - $L_{DPO} = -\log \sigma(\beta \log (\frac{\pi_\theta(y_w|x)}{\pi_\theta(y_l|x)} \cdot \frac{\pi_{ref}(y_l|x)}{\pi_{ref}(y_w|x)}))$
    - 核心逻辑：增加好回答的生成概率，降低坏回答的生成概率
  - **优势**：实现简单、训练稳定、无需 RM，显存占用低

**## 常见考点**
1. **KL 散度的作用**：RLHF 中的 KL Penalty 是为了防止什么？（防止 Reward Hacking，即模型输出一些能骗过 RM 但无意义的乱码）。
2. **SFT 数据质量**：如果 SFT 数据质量差，会对后续 DPO 产生什么影响？（模型会学会错误的回答模式，DPO 很难纠正这种根本性的偏差，即“垃圾进，垃圾出”）。
3. **DPO 数据构造**：Win 和 Lose 的回答差异越大越好吗？（不一定，如果两个回答都很好或都很差，模型很难学习；通常 Win 来自 GPT-4，Lose 来自 SFT 模型或早期版本）。
4. **ORPO（Odds Ratio Preference Optimization）**：听说过 ORPO 吗？（DPO 的变体，在 SFT 阶段就引入偏好信号，将 SFT 和对齐合二为一，效果可能更好）。

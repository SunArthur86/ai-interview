# SFT(Supervised Fine-Tuning)、RLHF、DPO三者有什么区别？如何选择？

🎯 本质：SFT教模型"怎么回答"，RLHF/DPO教模型"怎么回答更好"（符合人类偏好）。三者在LLM训练流水线中扮演不同角色。

📊 训练流水线：
Pre-training -> SFT -> RLHF/DPO -> 部署

1. SFT（监督微调）
  目标：让预训练模型学会指令遵循和对话格式
  数据：(instruction, response)配对
  方法：标准交叉熵loss
  类比：教学生标准答案

2. RLHF（基于人类反馈的强化学习）
  目标：让模型输出更符合人类偏好（有用、无害、诚实）
  三步流程：
  Step 1: 训练奖励模型(RM) - 用人类标注的偏好数据
    (prompt, response_A, response_B, preferred) -> RM
  Step 2: 用RM作为奖励，PPO算法优化策略模型
    max E[RM(prompt, response)] - KL_penalty
  Step 3: KL散度惩罚防止模型偏离SFT版本太远
  问题：训练不稳定、需要同时维护4个模型、复杂

3. DPO（Direct Preference Optimization）
  目标：直接用偏好数据优化，跳过RM和PPO
  数学洞察：最优策略可以从偏好数据中直接推导出来
  Loss = -log σ(β * (log(π(yw)/π_ref(yw)) - log(π(yl)/π_ref(yl))))
  yw=preferred response, yl=rejected response
  优势：只需1个模型+参考模型，无需RM，无需PPO
  更简单、更稳定、更高效

三者对比：
| 特性 | SFT | RLHF | DPO |
|------|-----|------|-----|
| 数据 | 指令-回答 | 偏好对 | 偏好对 |
| 目标 | 模仿标准答案 | 最大化奖励 | 直接优化偏好 |
| 模型数 | 1 | 4(policy+ref+RM+value) | 2(policy+ref) |
| 稳定性 | 高 | 低(PPO敏感) | 高 |
| 复杂度 | 低 | 高 | 中 |
| 效果 | 基础 | 最好(如果调好) | 接近RLHF |

选择建议：
- 快速适配 -> SFT（最简单）
- 最佳效果 -> SFT + RLHF（如果有资源）
- 效果和复杂度平衡 -> SFT + DPO（推荐大多数场景）
- 推理能力增强 -> SFT + GRPO（DeepSeek路线）

# 手撕多头注意力(用Pytorch)

### 核心概念与实现步骤

**多头注意力机制（Multi-Head Attention）** 是 Transformer 模型的核心组件，允许模型在不同的表示子空间中并行地关注输入序列的不同位置。

**数学定义**：
$$ \text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O $$
其中 $$ \text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V) $$

**实现步骤**：
1.  **线性投影**：通过 $W^Q, W^K, W^V$ 将输入 $X$ 映射到多组 Query, Key, Value。
2.  **缩放点积**：计算 $QK^T / \sqrt{d_k}$ 得到注意力分数。
3.  **Masking**（可选）：对 padding 或未来位置应用 Mask。
4.  **Softmax**：将分数归一化为概率分布。
5.  **加权求和**：利用权重对 Value 进行加权，得到每个头的输出。
6.  **拼接与融合**：将所有头的输出拼接，通过 $W^O$ 线性层融合。

```text
    │                   │                   │
 Head 1              Head 2  ...         Head H
    │                   │                   │

    │  (Scores = QK^T / sqrt(d_k))          │

```

### 完整 PyTorch 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, heads, dropout=0.1):
        """
        Args:
            d_model: 模型的总维度
            heads: 注意力头的数量
            dropout: Dropout 概率
        """
        super().__init__()
        # 确保总维度能被头数整除
        assert d_model % heads == 0, "d_model must be divisible by heads"
        
        self.d_model = d_model
        self.heads = heads
        self.d_k = d_model // heads  # 每个头的维度

        # 定义四个线性层：Q, K, V 的投影以及输出层
        # 注意：这里可以使用一个大的 Linear 层然后拆分，也可以分开定义。
        # 为了清晰，这里分别定义。
        self.w_q = nn.Linear(d_model, d_model)
        self.w_k = nn.Linear(d_model, d_model)
        self.w_v = nn.Linear(d_model, d_model)
        
        self.out = nn.Linear(d_model, d_model)
        
        self.dropout = nn.Dropout(dropout)

    def attention(self, q, k, v, mask=None):
        """
        缩放点积注意力计算
        q, k, v shape: (batch_size, heads, seq_len, d_k)
        """
        # 1. 计算注意力分数
        # q * k^T -> (batch, heads, seq_len_q, seq_len_k)
        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(self.d_k)

        # 2. 应用 Mask (如果有)
        # mask 通常形状为 (batch, 1, 1, seq_len) 或 (batch, 1, seq_len, seq_len)
        if mask is not None:
            # 将 mask 为 0 的位置填充为极小值，使得 softmax 后接近 0
            scores = scores.masked_fill(mask == 0, -1e9)

        # 3. Softmax 归一化
        # 在最后一个维度 上进行 softmax
        attn_weights = F.softmax(scores, dim=-1)

        # 4. Dropout (通常应用在 attention weights 上)
        attn_weights = self.dropout(attn_weights)

        # 5. 加权求和
        # context: (batch, heads, seq_len_q, d_k)
        context = torch.matmul(attn_weights, v)
        
        return context, attn_weights

    def forward(self, x, mask=None):
        batch_size = x.size(0)

        # 1. 线性投影
        # q, k, v: (batch_size, seq_len, d_model)
        q = self.w_q(x)
        k = self.w_k(x)
        v = self.w_v(x)

        # 2. 拆分为多头
        # view 改变形状为 (batch, seq_len, heads, d_k)
        # transpose 将 heads 维度移到 seq_len 之前，便于并行计算
        # 最终形状: (batch_size, heads, seq_len, d_k)
        q = q.view(batch_size, -1, self.heads, self.d_k).transpose(1, 2)
        k = k.view(batch_size, -1, self.heads, self.d_k).transpose(1, 2)
        v = v.view(batch_size, -1, self.heads, self.d_k).transpose(1, 2)

        # 3. 计算 Scaled Dot-Product Attention
        # attn_output: (batch_size, heads, seq_len, d_k)
        attn_output, _ = self.attention(q, k, v, mask)

        # 4. 拼接多头
        # contiguous() 是为了确保内存连续，便于 view 操作
        # transpose(1, 2) 回到 (batch, seq_len, heads, d_k)
        # view (batch, seq_len, d_model)
        attn_output = attn_output.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)

        # 5. 最终线性变换
        # output: (batch_size, seq_len, d_model)
        output = self.out(attn_output)

        return output
```

## 常见考点

1.  **为什么要除以 $\sqrt{d_k}$ (Scale)？**
    *   当维度 $d_k$ 很大时，点积的结果数值会变得很大，导致 Softmax 函数进入梯度极小的饱和区域，使得模型难以收敛。除以 $\sqrt{d_k}$ 可以缩放点积，使其方差保持在 1 附近，缓解梯度消失问题。

2.  **Attention 矩阵的维度变化过程？**
    *   需要熟练掌握 `view` 和 `transpose` 操作是如何将 $(Batch, Seq, D_{model})$ 变换为 $(Batch, Heads, Seq, D_k)$ 进行并行计算，再变换回来的。特别是 `transpose(1, 2)` 后，序列维度和头维度的交换是为了让矩阵乘法在 `heads` 维度上并行进行。

3.  **Mask 的具体作用和形状？**
    *   **Decoder 中的 Mask**：形状通常是 $(Batch, 1, Seq, Seq)$ 的上三角矩阵，用于防止当前位置关注到未来信息（Causal Mask）。
    *   **Padding Mask**：形状通常是 $(Batch, 1, 1, Seq)$，用于忽略输入序列中的填充部分（如 `<pad>` token），避免它们参与计算。
    *   如果使用了 `unsqueeze` 扩展维度，需要注意广播机制。

4.  **多头注意力的参数量计算？**
    *   假设输入维度为 $d$，头数为 $h$，每个头维度 $d_k = d/h$。
    *   $W^Q, W^K, W^V$ 的参数量各为 $d \times d$。
    *   $W^O$ 的参数量也是 $d \times d$。
    *   总参数量约为 $4d^2$（忽略 bias）。
    *   *追问*：如果增加头数 $h$，参数量会增加吗？（不会，只要 $d_{model}$ 不变，只是切分方式变了，但单头维度变小了）。

89. 1 Group Normalization(GN)

**特点**
特点
1. 最彻底的个体归一化:每个样本、每个通道都独立计算统计量,忽略了样本间和通道间的

2. 对分类等识别任务不一定有利:因为 IN 可能会过度移除全局特征,导致识别时无法利用

3. 与其他 Normalization 差异大:BN、LN、GN 通常会对一组通道或一个 Batch 的通道进
行统计,而 IN 极端到每个通道自身处理,被称为“Instance”归一化. 🍁 GN vs. IN 的核心区别 • Group Normalization(GN) **归一化维度：** 同一个样本的部分通道(按组分割) + 空间维度 **优点：** 对小批量更稳定,可控分组数,适合多通道任务或小 batch 训练 **使用场景：** 图像分割、目标检测、医疗影像等,常伴随高分辨率、小 batch • Instance Normalization(IN) **归一化维度：** 单样本单通道 + 空间维度 **优点：** 彻底消除不同样本、不同通道之间的统计差异,对风格迁移或生成任务有利 **使用场景：** 图像风格迁移(Style Transfer)、GAN 等注重“个体风格”保留的生成式 模型 从根源上理解 • **GN：** 在 BN 与 IN 之间找平衡,它不依赖 batch 统计,但仍保留一定的通道组间信息,不 会像 IN 那样“极端”. • **IN：** 最小化到“单通道单图”,不顾及跨通道跨样本分布,让每条数据都有独立的统计归 一化,突出个体差异、淡化全局分布. 一句话概括 • Group Normalization(GN):小团队体检,适合小批量、多通道图像. • Instance Normalization(IN):自己给自己量体重(单通道),适合需要保留个 体“风格”的生成式任务.

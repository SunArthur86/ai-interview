# 解释一下prefix-tuning

### Prefix-tuning

**概念**：
Prefix-tuning 是一种参数高效的微调（PEFT）方法，主要用于自然语言处理任务。它通过在预训练语言模型（如 GPT-2、T5 等）的**每一层**输入序列前添加一段可训练的连续前缀向量，实现对下游任务的适应，而无需更新原有的大量模型参数。

**来源**：
出自 2021 年 Li and Liang 的论文《Prefix-Tuning: Optimizing Continuous Prompts for Generation Tasks》。

**核心原理**：
与仅在输入层添加提示的 Prompt Tuning 不同，Prefix-tuning 将可训练的前缀向量添加到了 Transformer 模型**每一层的隐藏状态**中。这意味着微调不仅发生在输入层，还深入到了网络的每一层自注意力模块中。

**技术细节**：
1.  **前缀构建**：给定模型层数 $L$，在每一层的自注意力模块输入前增加一组可训练的前缀 $P_i \in \mathbb{R}^{m \times d}$（$m$ 为前缀长度，$d$ 为隐藏维度）。
2.  **注意力计算**：这些前缀向量被视为额外的“虚拟 token”，与真实的输入 token 一起参与注意力计算，影响后续 token 的注意力权重。
3.  **训练过程**：仅优化这些前缀向量的参数，冻结原始模型的所有权重。

**优势**：
*   **参数效率极高**：对于大模型（如 GPT-3 175B），只需训练不到 0.1% 的参数。
*   **性能优异**：在生成任务上表现优于 Adapter 和 LoRA，尤其是在模型参数量较小或任务难度较高时。

**局限**：
*   增加了推理时的计算量（因为每一层都要处理额外的虚拟 token）。
*   相比 Prompt Tuning，其可解释性稍弱，因为深层的前缀向量难以映射回自然语言。

# 强化学习算法是怎么分类的

强化学习算法多种多样,可以根据不同视角来分类,这里先介绍三种广为人知的分类方式.
91. 1 Value-Based 方法
🥥 Value-Based 方法的基本思想 •
在每个状态(State)下,为每个可行的动作(Action)都计算一个“价值”(Value).
• 选择 Value 最高的动作来执行.

在强化学习中,这个 Value 值通常通过贝尔曼方程(Bellman Equation)进行更新和迭代,
既考虑当前能获得的即时奖励,也考虑未来时刻可能获得的长期收益.
**示例：** 假设某个状态 State A 下可行的三个动作分别是 Action1, Action2,
Action3 ,训练过程中会迭代更新其对应的价值 V(s, a) (或常见写法 Q(s, a) ),
并最终选取价值最大的动作. 代表性算法 • Q-Learning •
SARSA (State-Action-Reward-State-Action)
**适用场景**
适用场景 • 动作空间离散(Discrete) 例如 Pacman 的上下左右 •
或者最终目标策略是确定性的(Deterministic) 例如,每次在某个状态就执行同一个动作
91. 2 Policy-Based 方法
🥥 Policy-Based 方法的基本思想
Policy-Based 方法则是直接对“策略(Policy)”本身进行建模,从而学习在各状态下,动
作对应的概率分布,而不是去间接计算 Value.
**示例：**
在给定状态 State A 时,你可能学得一套策略:
• 以 0.5 的概率选择 Action1 • 以 0.3 的概率选择 Action2 • 以 0.2 的概率选择 Action3

通过与环境的交互以及梯度(Gradient)更新,你会得到一个能最大化预期回报的最优策
略. 代表性算法
Policy Gradient(PG)系列
**适用场景**
适用场景 • 动作空间连续 例如机械臂控制中,关节旋转角度是浮点数,而非简单的离散动作
• 在某些状态下需要随机策略 例如剪刀石头布中,最优策略是 1/3 剪刀 + 1/3 石头 + 1/3 布
91. 3 Actor-Critic 方法
🥥这是将 Value-Based 和 Policy-Based 结合在一起的思路
•
Actor部分负责学习出策略(Policy)
•
Critic部分负责评价策略好坏(类似 Value 计算)
这类算法兼具两者优点,近年来在深度强化学习场景中非常流行.
91. 4 其他分类
🥥除了以上三种主流的算法思路,我们还会以是否需要环境模型(Model)进行分类:
•
**Model-based：** 先对环境进行建模,学习出环境的动态方程或状态转移规律;此时就能
用动态规划(DP)或蒙特卡洛树搜索(MCTS)等方法来选取贪心策略.
•
**Model-free：** 不对环境进行明确建模,直接通过 Value 或 Policy 函数的迭代来寻找最优
策略;经典的 Q-Learning 即是其中代表.

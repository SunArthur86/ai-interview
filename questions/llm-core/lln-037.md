# Lora 相比全参数微调可以节省多少计算量和显存

**LoRA 相比全参数微调的资源节省**：

1. **显存 (VRAM) 节省**：
   - **训练时**：大幅减少。全量微调需要存储梯度和优化器状态。LoRA 仅训练约 1%~10% 的参数，显存占用通常可降低 **3倍以上**（例如从 80GB 降至 24GB 级别）。
   - **推理时**：理论上无额外显存增加。LoRA 权重可合并回原模型，推理时与原模型完全一致。

2. **计算量 节省**：
   - **训练速度**：反向传播计算的参数量大幅减少，训练吞吐量通常提升 1.5~2 倍。
   - **推理速度**：合并后无额外计算开销；若不合并，会有微小的旁路计算延迟。

**原理**：
   - 基于 intrinsic dimension（本征维度）假设，即模型在特定任务上的参数更新实际上位于一个低维的子空间中，因此用低秩矩阵近似是可行的。

# DPO的微调流程

DPO(Direct Preference Optimization)的微调流程是通过直接优化人类偏好数据来训练模型,使其 生成更符合人类期望的输出. 🫐上图展示了DPO微调的大致流程,其中Trained LM即为策略模型,Frozen LM即为参考模 型,二者均是先进行SFT微调得到的模型进行初始化,其中Trained LM需要进行训练, Frozen LM不参与训练.两个模型分别针对chosen和rejected进行预测获取对应的得分,再 通过DPO的损失函数进行损失计算,进而不断的迭代优化.
96. 1 DPO 微调的核心目标
🫐DPO 的目标是让模型学会根据人类的偏好选择更好的输出.具体来说,给定一个输入 (例 如:“写一篇关于环保的文章”),模型会生成两个输出: x • Chosen ( ):人类喜欢的输出(例如:“环保是保护地球的重要方式,我们应该减少塑 料使用.”) y

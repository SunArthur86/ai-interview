# DPO的微调流程

DPO (Direct Preference Optimization) 的微调流程旨在通过成对的偏好数据直接微调大模型，无需训练奖励模型。

### 核心架构
流程维护两个模型实例：
1.  **Policy Model ($\pi_\theta$)**：需要优化的模型，参数可变。
2.  **Reference Model ($\pi_{ref}$)**：SFT 后的模型，参数**冻结**，用于计算 KL 散度约束。

### DPO 微调详细流程

**步骤 1: 模型初始化**
*   加载一个已经经过 SFT (Supervised Fine-Tuning) 的模型权重。
*   复制两份：一份作为 Policy Model（初始化优化器），一份作为 Reference Model（设置 `requires_grad=False`）。

**步骤 2: 数据准备**
*   数据集 $D$ 包含三元组：$(x, y_w, y_l)$。
    *   $x$：Prompt。
    *   $y_w$：Chosen（人类更喜欢的回答）。
    *   $y_l$：Rejected（人类不喜欢的回答）。

**步骤 3: 前向计算与 Logits 提取**
*   对于每个样本 $(x, y_w)$ 和 $(x, y_l)$：
    *   输入到 **Policy Model**，获取对应的 Log Probs：$\log \pi_\theta(y_w|x)$ 和 $\log \pi_\theta(y_l|x)$。
    *   输入到 **Reference Model**，获取对应的 Log Probs：$\log \pi_{ref}(y_w|x)$ 和 $\log \pi_{ref}(y_l|x)$。
*   *注：实际实现中，为了效率，通常会将 Prompt 和 Answer 拼接后一次性 Forward，通过 Attention Mask 截取 Answer 部分的 Logits。*

**步骤 4: 损失计算**
*   计算隐式奖励差值：
    $$ z = \beta \left( \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) $$
*   计算 DPO Loss (二元交叉熵形式)：
    $$ L = - \log (\sigma(z)) $$
*   若 $z > 0$（模型已区分好坏），Loss 较小；若 $z < 0$（模型好坏不分），Loss 较大。

**步骤 5: 反向传播与更新**
*   对 Policy Model 进行反向传播。
*   更新 Policy Model 的参数。
*   Reference Model 保持不动。

**流程图**：
```text
+-------------------+
|    偏好数据集      |
| (x, y_w, y_l)     |
+--------+----------+
         |
         v
+--------+----------+       +------------------+
|  Policy Model     |       | Reference Model  |
|  (Trainable)      |       |   (Frozen)       |
+--------+----------+       +--------+---------+
         |                              |
| Log Pol(y_w), Log Pol(y_l) | Log Ref(y_w), Log Ref(y_l) |
         |                              |
         +--------------+---------------+
                        |
                        v
            +-----------+-----------+
            |    DPO Loss 计算     |
            | (Ratio Diff + Sigmoid)|
            +-----------+-----------+
                        |
                        v
            +-----------+-----------+
            |   Backward & Update   |
            | (仅更新 Policy Model) |
            +-----------------------+
```

**## 常见考点**
1.  **显存占用**：DPO 训练时需要同时加载两个模型，显存需求通常是 SFT 的 2-3 倍，如何优化？（可以使用 Gradient Checkpointing 或混合精度训练）。
2.  **训练稳定性**：DPO 是否需要像 PPO 那样的价值裁剪？不需要，因为它是直接在概率空间优化，但 Reference Model 起到了隐式的约束作用。
3.  **迭代 DPO (Iterative DPO)**：如果在训练过程中更新 Reference Model 会怎样？（理论上可行，但需要谨慎设计，否则可能导致目标函数漂移，但在某些对话任务中能进一步提升性能）。
4.  **数据构建**：如果不使用成对数据，而是单个样本（如打分数据），能直接用 DPO 吗？（不能，DPO 依赖相对排序。若只有打分，可能需要使用其他方法如 KTO）。

# DPO的微调流程

DPO(Direct Preference Optimization)的微调流程是通过直接优化人类偏好数据来训练模型,使其生成更符合人类期望的输出。

**核心架构**：
流程包含两个模型：
1.  **Trained LM (策略模型 / Policy Model)**: 需要进行训练，参数会更新。
2.  **Frozen LM (参考模型 / Reference Model)**: 参数固定，不参与训练，仅作为基准。

两个模型通常都由经过 SFT (Supervised Fine-Tuning) 微调后的模型进行初始化。

### 微调流程步骤

**步骤 1: 初始化模型**
*   加载 SFT 模型权重分别初始化策略模型和参考模型。
*   冻结参考模型的参数。

**步骤 2: 准备输入数据**
*   Prompt ($x$): 输入提示词。
*   Chosen ($y_w$): 人类偏好的输出。
*   Rejected ($y_l$): 人类拒绝的输出。

**步骤 3: 计算模型得分**
*   分别将 $x$ 和 $y_w$、$x$ 和 $y_l$ 输入到两个模型中。
*   **策略模型** 计算对数概率：$\log \pi_\theta(y_w|x)$ 和 $\log \pi_\theta(y_l|x)$。
*   **参考模型** 计算对数概率：$\log \pi_{ref}(y_w|x)$ 和 $\log \pi_{ref}(y_l|x)$。

**步骤 4: 计算 DPO 损失**
利用公式计算损失：
$$L_{DPO} = -\mathbb{E} \left[ \log \sigma \left( \beta \left( \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right) \right]$$
目标是让策略模型更倾向于 Chosen 而不是 Rejected。

**步骤 5: 更新权重**
*   根据损失计算梯度。
*   使用优化器（如 AdamW）更新**策略模型**的参数。
*   **参考模型**参数保持不变。

**步骤 6: 迭代优化**
重复上述步骤，直到模型收敛。

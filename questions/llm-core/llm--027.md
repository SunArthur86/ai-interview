# 投机解码(Speculative Decoding)的原理是什么?它在什么场景下收益最大

- **投机解码** 用小模型快速生成'草稿',大模型批量验证,实现无损加速.

- **工作流程**

1. **Draft Phase(草稿阶段)**
- 小模型(draft model)快速自回归生成 γ 个 token
- 例:7B 小模型生成 5 个候选 token

2. **Verify Phase(验证阶段)**
- 大模型(target model)一次性处理这 γ+1 个 token(并行前向)
- 对每个位置,比较大模型和小模型的输出分布
- 接受:如果小模型的 token 在大模型的分布中概率足够高
- 拒绝:从拒绝位置开始,用大模型重新采样

- **数学保证(无损)**
- 使用拒绝采样保证输出分布与大模型完全一致
- 加速但不降低质量

- **加速效果**
- 理论加速比 = γ × acceptance_rate
- 实测:在对话场景加速 2-3x
- 在代码生成(高接受率)加速可达 3-5x

- **最佳场景**
1. **草稿与目标分布接近**:如同系列模型(Llama 7B → 70B)
2. **输出较确定性**:代码、结构化输出(接受率高)
3. **大 batch 推理**:摊薄验证阶段的计算成本

- **变体**
- **Medusa**:用多个 head 并行预测多个位置
- **EAGLE**:用目标模型的隐状态做更准确的预测
- **Lookahead Decoding**:无需草稿模型,利用 Jacobi 迭代

# 多模态大模型的架构有哪些？

多模态LLM让模型理解和生成文本、图像、音频等多种模态，核心在于如何将非文本模态的特征与LLM的文本空间对齐。以下是目前主流的架构分类与详解：

**1. 视觉-语言模型（VLM）：**

**架构一：投影层（线性连接，最主流）**
- **原理**：利用预训练的 Vision Encoder（如 CLIP, SigLIP, ViT）提取图像特征图（通常 shape 为 [HxW, D]），通过一个轻量级的 Adapter（如 MLP, Q-Former, Linear）将视觉特征投影到 LLM 的词嵌入空间。
- **数据流**：图像token与文本token直接拼接，作为 LLM 的输入序列。
- **代表模型**：LLaVA（MLP Adapter）、Qwen-VL（C-Abstractor）、MiniGPT-4（Q-Former）。
- **优缺点**：训练参数少，部署简单；但视觉特征压缩导致细节丢失，且在需要高频像素级交互的任务上表现较弱。

**架构二：交叉注意力**
- **原理**：冻结 LLM 主干，在 Transformer 层之间插入可训练的 Cross-Attention 模块。视觉特征作为 Key (K) 和 Value (V)，LLM 的上文作为 Query (Q)。
- **代表模型**：Flamingo。
- **优缺点**：支持更灵活的视觉信息检索，支持变长图像输入；但推理显存开销大（KV Cache占用高），且通常难以完全保留 LLM 原有的泛化能力。

**架构三：原生多模态**
- **原理**：不使用独立的 Encoder，直接从像素或离散 VAE tokens 开始，与文本 token 在同一个统一的 Transformer 架构中进行端到端训练。
- **代表模型**：GPT-4V（推测架构）、Gemini（原生多模态）、Chameleon（Early Fusion）。
- **特点**：真正打通了模态壁垒，能够生成多模态 interleaved 的内容，推理时可以全模态思考。

**架构四：Pix2Struct（基于屏幕解析）**
- **原理**：不使用标准坐标系统，而是将截图转换为一系列 token 适合处理 UI 图表等。
- **代表**：Pix2Struct。

**2. 音频-语言模型：**
- **架构**：Audio Encoder（Whisper, CFormer）提取声学特征 -> Projector -> LLM。
- **特殊点**：语音通常包含时间维度，特征序列很长，常需下采样处理以适配 LLM 的上下文窗口限制。
- **代表**：SALMONN (Whisper Encoder + BEATS + LLaMA)。

**3. 全模态：**
- **Gemini Ultra**：原生支持文本、图像、音频、视频。使用 MoE（混合专家）架构处理海量数据。
- **AnyGPT**：利用离散表示（语音 Codec tokens, 图像 VAE tokens）统一所有模态为下一个 Token 预测任务。

### 架构流程对比图

为了更直观地理解，下图对比了主流的**投影层架构**（如 LLaVA）与**交叉注意力架构**（如 Flamingo）的数据流转差异。

```text
+-----------------------------------------------------------------------+
|                        多模态 LLM 架构对比                             |
+-----------------------------------------------------------------------+

【方案 A：投影层 / Linear Projection (主流)】

     输入端                 特征提取                对齐               LLM 推理

  +---------+          +-------------+        +-----------+        +-------------+
  |  Image  | -------> |   Vision    | ------->| Projector |------>|             |
  | (Pixels)|          |   Encoder   |        |  (MLP/Ln) |  Cat  |     LLM     |
  +---------+          +-------------+        +-----------+  ---> | (Transformer|
                                                 |  ^        |      |   Decoder)  |
  +---------+          +-------------+           |  |        |      +-------------+
  |  Text   | -------> |   Text      | -------> |  |        |             |
  |(Prompt) |          |  Embedding  | -------> |  |--------|             |
  +---------+          +-------------+          +------------+             |
                                                                           v
                                                                    Output Tokens

特点：视觉特征被压缩成固定数量的“伪Token”，与文本拼接后统一处理。
      优点是结构简单，推理快，易部署；缺点是容易丢失图像细节。


【方案 B：交叉注意力 / Cross-Attention (Flamingo)】

     输入端                 特征提取             交叉融合层            LLM 推理

  +---------+          +-------------+                             +-------------+
  |  Image  | -------> |   Vision    | [作为 Key/Value 存储在 KV]  |             |
  | (Pixels)|          |   Encoder   | -------------------------> | Gated X-Attn | <--
  +---------+          +-------------+                             |   (Layer N)  |   |
                                                                   +------+------+
  +---------+          +-------------+                             |      v       |
  |  Text   | -------> |   Text      | -------------------------> | Frozen LLM  |
  |(Prompt) |          |  Embedding  | [作为 Query 流动]         |  (Layers)   |
  +---------+          +-------------+                             +------+------+
                                                                   |      v       |
                                                            Output Tokens

特点：视觉特征保持较高分辨率（较多Token），每层通过 Cross-Attention 查询。
      优点是视觉信息保留丰富；缺点是 KV Cache 占用大，推理速度慢。
```

### ## 常见考点

1.  **Token 比例问题**：
    -   **问题**：在 LLaVA 等投影架构中，一张图片通常被压缩成 576 或更多个 Token，这会显著挤占文本的上下文窗口，如何优化？
    -   **思路**：提及使用更大步幅的 Vision Encoder（如 SigLIP）、Q-Former 压缩机制、或者在下采样层使用池化技术。

2.  **分辨率与显存的权衡**：
    -   **问题**：为什么有些模型（如 Qwen-VL-Max）支持 4K 分辨率输入？这带来了什么挑战？
    -   **思路**：高分辨率意味着输入的 Image Token 数量呈平方级增长，导致显存（特别是 KV Cache）爆炸。解决方案包括动态切片、分块编码或 Naive Attention 的局部化改进。

3.  **模态对齐的训练阶段**：
    -   **问题**：多模态模型通常分几个阶段训练？为什么需要第二阶段？
    -   **思路**：通常分为两阶段：① **预训练对齐阶段**（冻结 LLM 和 ViT，只训练 Projector，使用大量图文对）；② **指令微调阶段**（解冻部分 LLM 参数，使用高质量的指令数据）。目的是让模型学会“如何看”和“如何说话”。

4.  **原生多模态的优势**：
    -   **问题**：为什么说 GPT-4o 或 Gemini 这种“原生端到端”架构比 LLaVA 这种“拼接式”架构更有潜力？
    -   **思路**：原生架构允许任意模态作为输入或输出，且模态间的交互发生在 Transformer 的每一层，而不仅仅是在输入层表面拼接，有利于深度的语义融合和理解。

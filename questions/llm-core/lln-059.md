# 大模型的解码策略有哪些?Top-k、Top-p和Temperature的作用是什么

解码策略决定了模型如何从概率分布中选择下一个token。

### 主要策略对比

| 策略 | 原理 | 特点 |
|------|------|------|
| **Greedy** | 每步选概率最高的token | 确定性,容易重复 |
| **Beam Search** | 维护K个候选序列 | 质量高但多样性低 |
| **Top-k** | 只从概率最高的k个token中采样 | 限制选择范围 |
| **Top-p (Nucleus)** | 从累积概率达到p的最小token集合中采样 | 动态调整范围 |
| **Temperature** | 调整概率分布的尖锐程度 | 控制随机性 |

### Temperature 详解
公式：$P'(y_i) = \text{softmax}(\text{logit}_i / T)$
- **$T \to 0$**：分布更尖锐（接近 Greedy），输出更确定。
- **$T = 1$**：原始概率分布。
- **$T \to \infty$**：分布更平坦（接近均匀随机），输出更随机。

### Top-p (Nucleus Sampling)
- 将 token 按概率降序排列。
- 选择累积概率达到 $p$ 的最小 token 集合。
- $p=0.9$：只在累积概率 90% 的 token 中采样。
- 比Top-k更灵活：概率集中时选少量token，概率分散时选更多。

```text
      采样策略可视化对比 (假设输出词表只有5个词)

      原始概率分布 (Temperature=1.0)
      Token:  A(0.4)  B(0.3)  C(0.2)  D(0.05) E(0.05)
      ██████ █████ ████ █ █

      Top-k (k=3)      Top-p (p=0.9)          Temperature > 1
      选择: A, B, C     选择: A, B, C           概率平滑
      D, E 概率置0     (0.4+0.3+0.2=0.9)      A, B 差距变小
                                            D, E 概率上升
```

### 实践推荐
- **事实问答**：Temperature=0.1, Top_p=0.95
- **创意写作**：Temperature=0.8, Top_p=0.9
- **代码生成**：Temperature=0.2, Top_p=0.95

## 常见考点
1. **Repetition Penalty 是什么？**：在解码过程中，如果已经生成了某些token，人为降低这些token再次出现的概率（通常除以一个系数 > 1），防止模型生成重复的文本循环。
2. **为什么在开放式生成中很少用 Beam Search？**：Beam Search 容易陷入重复循环（如 "The cat is on the mat. The cat is on the mat..."），且生成的文本缺乏人类的随机性和多样性，相比 Sampling 策略显得过于呆板。
3. **Top-k 和 Top-p 可以同时使用吗？**：可以。通常会先筛选出概率最高的 k 个 token，再在这 k 个 token 中应用 Top-p 进行采样（或者反之），两者结合可以进一步过滤掉噪声。

# Prefix Caching(前缀缓存)的原理是什么?在什么场景下效果最好

### Prefix Caching (前缀缓存)

**Prefix Caching** 是缓存重复前缀的 KV Cache 以加速推理的技术。

### 原理
1. **缓存计算**：计算 prompt 的 KV Cache 后将其缓存。
2. **复用机制**：后续请求如果共享相同前缀，直接复用缓存的 KV。
3. **增量计算**：只需要计算新增部分的 KV。

### 典型场景
- **多轮对话**：system prompt + 历史对话是重复前缀。
- **Few-shot**：示例部分是重复前缀。
- **RAG 系统**：检索到的文档可能被多次引用。

### 效果
- 首 token 延迟降低 50-90%。
- 特别适合 Agent 场景（每轮都带大量上下文）。

### 注意事项
- 缓存淘汰策略（LRU/TTL）。
- 显存管理。
- 缓存一致性。

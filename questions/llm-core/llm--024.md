# 训练千亿参数模型时,训练不稳定性(loss spike、梯度爆炸)如何解决

- **训练不稳定的表现**
1. **Loss Spike**:loss 突然跳升,可能恢复也可能发散
2. **梯度爆炸/消失**:梯度过大/过小
3. **NaN/Inf**:浮点溢出

- **根本原因**
- 混合精度训练的数值范围有限
- 大 batch 下的梯度更新方向不稳定
- 注意力矩阵的 softmax 数值溢出
- 优化器状态(Adam momentum)累积异常

- *解决方案(多层防御)

1. **数值稳定性**
- 用 BF16 代替 FP16(更大动态范围)
- Softmax 前减去最大值(Flash Attention 内置)
- QK 归一化(QK-Norm)

2. **梯度控制**
- 梯度裁剪(gradient clipping,clip=1.0)
- 梯度累积后再更新

3. **训练策略**
- Warmup:前 2000 步线性增加学习率
- Cosine 衰减调度
- 初始化:μP(maximal update parametrization)

4. **Loss Spike 恢复**
- 检测 spike → 回滚到上一个 checkpoint
- 跳过 bad batch(数据质量导致)
- 降低学习率重试

5. **分布式优化**
- ZeRO-3 + 梯度分桶减少通信
- 张量并行 + 流水线并行的负载均衡

- **经验**:GPT-3 论文报告训练过程中 loss spike 是常态,关键是设计好恢复机制.

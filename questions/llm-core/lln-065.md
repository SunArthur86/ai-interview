# Continuous Batching(连续批处理)的原理是什么?为什么它能大幅提升推理吞吐

- **Continuous Batching** 是 vLLM 等推理框架的核心优化技术.

- **传统 Batching 的问题:** 静态 batch 中不同请求的生成长度不同,短请求完成后必须等待长请求,GPU 空闲.

- **Continuous Batching 的方案:**
1. Token 级别的动态调度:每生成一个 token 就重新评估
2. 即时加入/退出:新请求随时加入 batch,完成的请求立即退出
3. 配合 PagedAttention:通过分页管理 KV Cache,避免内存碎片

- **性能提升:** 吞吐量提升 2-22 倍(相比静态 batch),GPU 利用率从约 30% 提升到 80% 以上.

- **关键设计:** 每个时刻 GPU 都在处理有效的 token 生成,不会因为某个请求未完成而浪费算力.

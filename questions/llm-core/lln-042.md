# 讲讲你了解的scaling law有哪些

🌟目前关于LLM的Scaling Law研究主要集中在模型性能与规模(参数、数据、计算量)之间的关系、涌现能力的发现、推理 优化以及实际应用中的挑战等方面. 基础Scaling Law研究 [1] • **Kaplan等人的经典理论：** 2020年提出的Scaling Law指出,模型性能(如测试损失)随参数数量、数据集大小和计算量的指数增 长呈线性提升,且三者需协同扩展以避免瓶颈.例如,模型参数增加约8倍时,训练数据需增加约5倍以维持性能提升. • **幂律关系的验证：** 研究发现,测试损失与模型规模之间遵循逆幂律关系,即损失随规模增大而平滑下降,但提升速度逐渐放缓. 这一现象在多个数据集上被复现,成为预训练扩展的理论基础. 强化Scaling Law研究 [2] • 过优化Goodhart’s law:模型可能过度拟合代理奖励函数,导致在真实评估指标上的性能下降,即Reward Hacking.这符合 Goodhart定律--“当指标成为目标时,它就不再是一个好指标”.论文通过合成实验验证了这一现象:随着模型在代理奖励上 的得分提升,其在“黄金标准”奖励模型(代表真实偏好)上的得分先上升后下降,呈现“驼峰型”曲线. •
**直接对齐算法的隐式过度优化：** **KL散度与性能的非线性关系：** 不同KL约束下,模型性能(如GPT-4评估的胜率)随训练步长增加先升后降,且小模型更易陷 入过度优化. **模型规模的影响：** 大模型在KL与性能的权衡中表现更优,表明模型容量越高,对隐式奖励的泛化能力越强. 微调Scaling Law研究 [3] •
**幂律联合缩放法则：** 微调性能遵循幂律乘法联合缩放法则,即模型性能与微调数据量、模型大小、预训练数据量等因素呈幂律关系.例如,测试 PPL随模型参数和微调数据量的增加而平滑下降,但提升速度逐渐放缓. 当模型大小(如从 1B 到 16B)与微调数据量协同扩展时,性能提升显著,但参数高效调优(PET)方法的参数量扩展对性能 提升作用有限 • 模型规模 > 预训练数据规模:模型大小的扩展比预训练数据量的扩展更有效,且发现预训练数据量的影响仅在模型规模较小时显 著,而对大模型的微调增益有限. •
**PEFT的适用性：** PEFT在数据量较少时表现更优,而全模型调优(FMT)需要更多数据但最终效果更好13. LoRA 的稳定性优于 Prompt Tuning:LoRA 在数据量增加时表现更稳定,而 Prompt Tuning 可能出现逆向扩展(性能随参 数增加而下降). PEFT 方法的泛化能力更强,因其冻结了大部分预训练参数,保留基础模型的知识. • **任务与数据的依赖：** 最优微调方法高度依赖任务类型和微调数据量.例如: 数据量少(几千样本):优先选择 PET(如 LoRA 或 Prompt Tuning). 数据量中等(数万样本):LoRA 更稳定. 数据量极大(百万级):FMT 效果更优. 推理Scaling Law研究 [4]

• **慢思考策略：** 通过额外计算资源(如多次采样、树搜索)提升推理质量,例如宽度扩展(BoN、MCTS)和生成-选择机制.研究 发现,推理路径长度增加会导致信息损失(雪球误差效应),需平衡生成与选择步骤的效率和准确性. • **推理成本与效率：** 随着模型规模扩大,推理成本急剧上升,行业开始转向优化推理阶段而非单纯扩展预训练. 批大小Scaling Law研究 [5] • **激增现象：** Adam类优化器的最优学习率随Batch Size的增加呈现先上升后下降的非单调趋势,类似“浪潮”形态.例如,当 Batch Size较小时,增大Batch Size可提升学习率;但超过临界值(由噪声水平 Bnoise决定)后,学习率反而需降低以维持收敛 稳定性. •

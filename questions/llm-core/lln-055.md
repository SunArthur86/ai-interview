# LoRA的变体有哪些?DoRA、AdaLoRA、LoRA+各自有什么改进

LoRA变体通过不同角度改进原始LoRA的局限性.

- **主要变体对比:**

| 变体 | 核心改进 | 解决的问题 |
|------|---------|-----------|
| AdaLoRA | 动态分配rank | 固定rank不合理 |
| DoRA | 分解为大小和方向 | LoRA缺少权重幅值调整 |
| LoRA+ | A和B使用不同学习率 | A/B学习不均衡 |
| GaLore | 梯度低秩投影,全参数训练 | LoRA只能训练少量参数 |
| LongLoRA | Shifted Sparse Attention | 原始LoRA不支持长上下文 |

- **DoRA详解(最值得关注):**
- LoRA: W = W₀ + BA
- DoRA: W = m · (W₀ + BA) / ||W₀ + BA||
- 将权重分解为大小和方向
- 更接近全参数微调的更新模式
- 效果显著优于标准LoRA,尤其在低rank时

- **AdaLoRA详解:**
- 基于重要性分数动态调整每层的rank
- 使用SVD分解替代随机初始化
- 训练过程中裁剪不重要的奇异值,分配给重要层

```text
┌──────────────────────┐      ┌──────────────────────────────────────┐
│        Input         │      │               Input                 │
└──────────┬───────────┘      └──────────────┬───────────────────────┘
           │                                 │
           ▼                                 ▼
┌──────────────────────┐      ┌──────────────────────────────────────┐
│       Layer (W₀)     │      │           Direction (d)              │
│     (Frozen)         │      │  d = (W₀ + △W) / ||W₀ + △W||         │
└──────────┬───────────┘      └──────────────┬───────────────────────┘
           │                                 │
           │            ┌────────────────────┴────────────────────┐  
           │            │   Magnitude (m) <- Learnable Scalar   │  
           │            │   (New Component in DoRA)             │  
           │            └────────────────────┬────────────────────┘  
           │                                 │
           ▼                                 ▼
┌──────────────────────┐      ┌──────────────────────────────────────┐
│   W = W₀ + B·A       │      │        Output = m · d                │
│   (Additive Update)  │      │      (Magnitude * Direction)        │
└──────────────────────┘      └──────────────────────────────────────┘
```

- **LoRA+详解:**
- **核心痛点**: LoRA中A矩阵通常随机高斯初始化，B矩阵初始化为0。由于初始梯度主要由B承担，A和B若使用相同学习率会导致B更新过快而A滞后，导致次优收敛。
- **改进方案**: 对矩阵A使用相对较高的学习率（通常是对B的λ倍，推荐λ=16），B使用较低的学习率。
- **效果**: 收敛速度提升约2倍，训练更稳定。

## 常见考点
1. **LoRA的秩是如何选择的？**：通常rank设为8, 16, 64。rank过低可能导致表达能力不足，rank过高参数量增加但边际效应递减。
2. **LoRA初始化策略？**：A使用随机高斯分布初始化，B使用0初始化。这样初始训练时△W=0，模型预训练能力完全保留，训练平滑启动。
3. **DoRA相比LoRA的计算开销？**：DoRA仅增加了非常少量的可缩放参数（每个通道一个标量m），计算开销几乎可以忽略不计，但在低rank下性能提升显著。

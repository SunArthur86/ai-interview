# GRPO、PPO、DPO 三者的区别是什么?各自的优缺点

- **三者对比**

| 维度 | PPO | DPO | GRPO |
|------|-----|-----|------|
| 类型 | On-policy RL | Off-policy 偏好学习 | On-policy RL |
| 需要RM | ✅ 需要单独 RM | ❌ 隐式 reward | ❌ 组内相对优势 |
| 需要ref model | ✅ | ✅ | ✅ |
| 数据 | 实时采样 + RM 打分 | 偏好对 | 同一 prompt 多个采样 + 规则打分 |
| 训练稳定性 | 中(超参敏感) | 高(直接优化) | 高(组内归一化降方差)|

- **PPO 优缺点**
  - 优点: 理论成熟，可在线探索
  - 缺点: 4 个模型同时训练(policy/ref/reward/value)，显存大，超参敏感

- **DPO 优缺点**
  - 优点: 简单(只需偏好对)，训练稳定
  - 缺点: 依赖离线偏好数据质量，不能在线探索更好的策略

- **GRPO 优缺点**
  - 优点: 不需要单独 RM，用组内相对优势替代；减少方差；对推理任务特别有效(可用规则判定对错)
  - 缺点: 需要多次采样(组大小 G=4~64)，计算成本较高

- **选择建议**:
  - 通用对齐 → DPO
  - 推理能力强化 → GRPO
  - 在线探索需求 → PPO

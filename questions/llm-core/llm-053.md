# 介绍一下Qwen通义千问的结构

🐳通义千问(Qwen)是阿里巴巴集团研发的一系列大型语言模型,旨在提供强大的自然语言理 解和生成能力.该模型系列包括不同参数规模的模型,涵盖基础预训练模型和经过人类对齐 技术微调的聊天模型(Qwen-Chat),以及专门用于代码和数学任务的模型(Code-Qwen、 Math-Qwen).
55. 1 整体架构概览
Qwen模型的整体结构 通义千问(Qwen)采用了基于Transformer的架 构,这是当前大型语言模型的主流选择.其核心

**组成部分包括：**
1. 输入处理层:对输入文本进行预处理和编码.
2. 嵌入层(Embedding Layer):将词或子词
转换为连续的向量表示.
3. 多层Transformer解码器块:由多头自注意力
机制和前馈神经网络组成,多个层堆叠以捕捉 深层次的语言特征.
4. 输出层:生成最终的输出,如预测下一个词的
概率分布.
55. 2 模型结构详解
1. 输入处理层
• **分词与编码：** 采用字节对编码(Byte Pair Encoding,BPE)方法,将输入文本进行分词和编码. 为了增强多语言能力,Qwen的词汇表增加了常用的中文字符和其他语言的词汇,词汇表大小约为 152K. • **压缩效率：** 在多种语言中,Qwen的压缩效率高于其他标记器,这意味着服务成本可以显著降低.

2. 嵌入层(Embedding Layer)
• 词嵌入(Word Embedding):将每个词或子词的索引映射为稠密的向量表示,捕捉词语的语义 信息. • 位置嵌入(Positional Embedding):采用旋转位置嵌入(RoPE),将位置信息融入模型中. RoPE通过在输入中加入位置信息,使模型能够识别序列中词语的位置.
3. 多层Transformer解码器块
Transformer解码器块的内部结构 Qwen的核心由多层Transformer解码器块组成,每个块包含以下组 件: a. 多头自注意力机制(Multi-Head Self-Attention) • **目的：** 捕捉序列中不同位置之间的依赖关系. • **机制：** 通过计算查询(Q)、键(K)、值(V)之间的相关性, 生成加权的输出表示. b. 前馈神经网络(Feed-Forward Network, FFN) • **结构：** 由两个线性变换和一个非线性激活函数组成. • **激活函数：** 采用SwiGLU激活函数,这是Swish和门控线性单元 (GLU)的组合,增强了模型的表达能力. c. 残差连接和层归一化 • 残差连接(Residual Connection):缓解深层网络的梯度消失 问题,促进信息的直接传递. • 层归一化(Layer Normalization):采用RMSNorm,提升训 练的稳定性和效率.
4. 输出层

**线性变换和Softmax层：** 将Transformer块的输出转换为词汇表中每个词的概率分布,用于预测下一 个词.
55. 3 模型训练方法
1. 预训练
• **数据规模：** 使用了多达3万亿个标记的数据,包括公共网络文档、百科全书、书籍、代码等,涵盖 多种类型和领域. • **多语言支持：** 数据集是多语言的,其中相当一部分是中文和英文.

• **数据预处理：** 使用了全面的数据预处理程序,包括去重、语言识别、质量过滤等,确保数据的多样 性和高质量. • **目标：** 在大规模未标注的文本数据上训练模型,让其学习语言的基本结构和模式. • **方法：** 采用自回归语言建模任务,即给定前面的词,预测下一个词. • **优化器：** 使用AdamW优化器,设置了适当的超参数以保证训练稳定性.
2. 上下文长度扩展
• **挑战：** 随着上下文长度的增加,计算和内存成本急剧增加. •
**解决方案：** **NTK-aware插值：** 调整RoPE的位置编码,防止在不训练的情况下丢失高频信息. **LogN-Scaling：** 调整注意力机制的缩放因子,保持注意力值的稳定性. **窗口注意力：** 将注意力限制在有限的上下文窗口内,防止模型注意到过远的标记. **分层窗口分配：** 为每一层分配不同的窗口大小,低层使用较短的窗口,高层使用较长的窗口, 提升长上下文建模能力.
3. 微调和对齐
• 监督微调(SFT): **目标：** 使预训练模型适应对话式AI助手的任务. **数据：** 使用了精心标注的对话数据,包括任务执行、聊天、工具使用、安全等方面. **方法：** 在预训练模型的基础上,使用标注的对话数据,对模型进行微调. • 人类反馈强化学习(RLHF): **目标：** 进一步使模型与人类偏好保持一致,生成更符合人类期望的响应. **方法：**
▪ **奖励模型训练：** 收集人类反馈的数据,训练一个奖励模型来评估模型输出的质量. ▪ **策略优化：** 使用近端策略优化(PPO),根据奖励模型的评估结果,优化模型的策略.
55. 4 Qwen的特殊特性和创新
1. 多语言词汇表
• **优化的Tokenization：** 采用了更大的词汇表,包含152K个词元,在中文等多种语言上具有更好的 压缩效率,降低了服务成本.
2. 改进的模型架构
• **非绑定的嵌入方法：** 输入嵌入和输出投影的权重不共享,提升了模型的性能. • **偏置调整：** 移除了大多数层的偏置,但在注意力的QKV层中添加了偏置,增强了模型的外推能力.

3. 激活函数的选择
• **SwiGLU激活函数：** 相比于GeLU,SwiGLU在提高模型性能和效率方面表现更好.
4. 长上下文建模
• **上下文长度扩展：** 通过NTK-aware插值、LogN-Scaling、窗口注意力等技术,扩展了模型的上下文 长度,增强了对长文本的处理能力.
55. 5 专用模型
1. Code-Qwen
• **目标：** 专门用于编码任务的模型,包括代码生成、调试和解释. •
**训练方法：** **继续预训练：** 在基础模型的基础上,继续在大量代码数据上进行预训练. **代码监督微调：** 使用与代码生成、调试和解释相关的对话数据,对模型进行微调. • **性能：** 在HumanEval、MBPP等代码基准数据集上表现出色,超过了许多开源模型.
2. Math-Qwen
• **目标：** 专门用于解决数学问题的模型,提升模型在算术和数学推理方面的能力. •
**训练方法：** **数学SFT：** 在增强的数学指导数据集上进行监督微调,直接得到聊天模型Math-Qwen-Chat. • **性能：** 在GSM8K、MATH等数学基准数据集上表现出色,超过了相同规模的开源模型,接近于GPT-
3. 5的性能.
55. 6 模型评估与结果
1. 基准评估
• **任务：** 在MMLU、C-Eval、GSM8K、HumanEval、BBH等多个基准测试中评估模型性能. •
**结果：** **Qwen-14B：** 在多个下游任务中表现出色,甚至超过了一些更大参数规模的模型,如LLaMA2- 70B. **Qwen-7B：** 表现优于同等规模的模型,如LLaMA2-13B. **Code-Qwen：** 在代码生成任务上,表现优于同等规模的专有和开源模型. **Math-Qwen：** 在数学推理任务上,超过了相同规模的开源模型,接近于GPT-3.5的性能.
2. 上下文长度扩展评估
• **指标：** 在arXiv数据集上以困惑度(PPL)为指标进行评估.

• **结果：** 证明了上下文长度扩展技术的有效性,在超过8192个标记的情况下,模型性能仍然稳定.
55. 7 总结
通义千问(Qwen)系列模型通过大规模的预训练和精心设计的架构,展现了强大的自然语言理解和生

**成能力.其创新之处包括：**
• **多语言支持：** 优化的词汇表和编码方法,提升了模型在多种语言上的表现. • **改进的架构设计：** 采用了SwiGLU激活函数、RoPE位置嵌入、非绑定的嵌入方法等,提升了模型性 能. • **上下文长度扩展：** 通过多种技术手段,扩展了模型的上下文长度,增强了对长文本的处理能力. • **专用模型：** 针对代码和数学任务,开发了专门的模型(Code-Qwen、Math-Qwen),在相应领域 取得了优异的性能. 🐳通过结合预训练、监督微调和人类反馈强化学习等方法,Qwen模型在各项基准测试中表现出 色,展示了其在自然语言处理领域的强大潜力.

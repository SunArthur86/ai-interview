# 不同的Normalization区别

特点
1. 最彻底的个体归一化：每个样本、每个通道都独立计算统计量，忽略了样本间和通道间的
分布差异。
2. 对分类等识别任务不一定有利：因为 IN 可能会过度移除全局特征，导致识别时无法利用
跨样本的统计信息；但在需要突出个体差异或风格的生成任务中，反而是优势。
3. 与其他 Normalization 差异大：BN、LN、GN 通常会对一组通道或一个 Batch 的通道进
行统计，而 IN 极端到每个通道自身处理，被称为“Instance”归一化。
🍁
GN vs. IN 的核心区别
- Group Normalization（GN）
◦
归一化维度：同一个样本的部分通道（按组分割） + 空间维度
◦
优点：对小批量更稳定，可控分组数，适合多通道任务或小 batch 训练
◦
使用场景：图像分割、目标检测、医疗影像等，常伴随高分辨率、小 batch
- Instance Normalization（IN）
◦
归一化维度：单样本单通道 + 空间维度
◦
优点：彻底消除不同样本、不同通道之间的统计差异，对风格迁移或生成任务有利
◦
使用场景：图像风格迁移（Style Transfer）、GAN 等注重“个体风格”保留的生成式
模型
从根源上理解
- GN：在 BN 与 IN 之间找平衡，它不依赖 batch 统计，但仍保留一定的通道组间信息，不
会像 IN 那样“极端”。
- IN：最小化到“单通道单图”，不顾及跨通道跨样本分布，让每条数据都有独立的统计归
一化，突出个体差异、淡化全局分布。
一句话概括
- Group Normalization（GN）：小团队体检，适合小批量、多通道图像。
- Instance Normalization（IN）：自己给自己量体重（单通道），适合需要保留个
体“风格”的生成式任务。
90. 不同的 Normalization 之间有什么区别？
90.1 六种 Normalization 的对比表
下面给出一份涵盖 Batch Normalization（BN）、Layer Normalization（LN）、RMSNorm、
DeepNorm、Group Normalization（GN）、Instance Normalization（IN） 的比较汇总。

90.2 六种 Normalization简要说明
Batch Normalization（BN）
- 归一化维度：对同一个 mini-batch 内通道维度上计算均值、方差。
- 优势：在图像任务中有效加速收敛，对大批量训练非常友好。
- 局限：当批量过小时，统计不稳；对 RNN、变长序列效果不佳；推理时需使用移动平均或全局统
计。
 Layer Normalization（LN）
- 归一化维度：对单条样本在本层的所有特征进行均值、方差归一化。
- 优势：与 batch size 无关，特别适合 NLP、变长序列；在 Transformer 中成为主流做法。
- 局限：在很大 batch 场景下并没有显著坏处，但相比 BN，有时对CNN而言收益不如后者明显。
RMSNorm
- 归一化方式：省略了减均值操作，仅使用激活的 RMS（均方根）进行缩放。
- 优势：计算更简单，性能不逊于 LN；在大型语言模型中也能获得较好收敛稳定性。
- 局限：主要在 NLP/Transformer 领域使用，尚未像 BN/LN 那样广泛通用。
DeepNorm
- 核心：并非单纯“归一化公式”更改，而是在 深层（超深）Transformer 的残差结构处，添加特
定缩放因子。
- 目标：解决 网络层数极深时 的训练不稳定（如梯度爆炸/消失）。
- 使用：通常与 LN 或 RMSNorm 配合；对 100+ 层的大型预训练模型帮助明显。
Group Normalization（GN）
- 归一化维度：对每个样本的通道分组，分组内计算均值方差。
- 优势：对 batch size 小的场景仍然稳定（BN在小batch时就很不稳定）；可在2D/3D图像等高通道
任务中细粒度归一化。
- 局限：需要选择合适的组数（group）才能达到理想效果，存在一定调参难度。
 Instance Normalization（IN）
- 归一化维度：同一个样本、同一通道内（空间维度）计算均值和方差，每个通道彼此独立。
- 优势：在风格迁移中消除不同样本间的特征差异，让模型更注重“个人风格”。
- 局限：对于需要全局统计一致性的任务（如分类），IN 可能过于分割信息，反而不利于分类语义保
持。
90.3 如何挑选合适的 Normalization？(🌟核心考量)

🍋如何挑选合适的 Normalization
主要是看任务类型
◦
图像（CNN）+ 大batch：BatchNorm 常见且成熟；
◦
图像 + 小batch：可考虑 GroupNorm (GN)；
◦
风格迁移：InstanceNorm (IN)；
◦
NLP/Transformer：LayerNorm (LN)或 RMSNorm；
◦
超深Transformer：DeepNorm（配合LN或RMSNorm）保证稳定性。
看模型规模/深度
◦
网络层数极深（几百层）时，需要额外考虑 DeepNorm；
◦
一般深度（几十层）用常规 LN/BN/GN 即可。
看硬件/批量限制
◦
如果显存或数据限制导致 batch size 不大，BN 可能不稳定，应考虑 LN、GN、IN、
RMSNorm。

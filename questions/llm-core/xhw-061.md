# 不同的Normalization区别

### 不同的 Normalization 区别

#### 1. 汇总对比

| 类型 | 归一化维度 | 适用场景 | 核心优势/局限 |
| :--- | :--- | :--- | :--- |
| **Batch Norm (BN)** | 同一 Batch 的同一通道 | CNN 图像分类 (大 Batch) | 加速收敛，但依赖 Batch Size，不适合 RNN/Transformer |
| **Layer Norm (LN)** | 单个样本的所有特征 | NLP, Transformer | 不依赖 Batch，适合变长序列，是 Transformer 标配 |
| **Instance Norm (IN)** | 单样本单通道 (H, W) | 风格迁移 | 去除样本间差异，保留风格 |
| **Group Norm (GN)** | 单样本的通道组 | 目标检测/分割 (小 Batch) | 不依赖 Batch，平衡了 LN 和 IN |
| **RMSNorm** | 单样本所有特征 (无均值中心化) | LLaMA 等 LLM | 计算更快，去中心化提升稳定性，性能接近 LN |
| **DeepNorm** | 结合残差连接的特殊初始化 | 超深 Transformer (100层+) | 解决极深网络的梯度不稳定问题 |

#### 2. 选用建议
- **NLP / Transformer**：首选 **LayerNorm** 或 **RMSNorm** (计算更省)。
- **超深网络**：配合 **DeepNorm** 策略防止梯度爆炸/消失。
- **CV + 大 Batch**：**BatchNorm**。
- **CV + 小 Batch**：**GroupNorm**。
- **生成式/风格迁移**：**InstanceNorm**。

#### 3. 核心差异补充 (RMSNorm vs LayerNorm)
- **LayerNorm**: $\frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta$，包含减去均值（中心化）的操作。
- **RMSNorm**: $\frac{x}{\sqrt{mean(x^2) + \epsilon}} \cdot \gamma$，去掉了减去均值的步骤。
- **优势**：简化了计算（减少一次 Reduce 操作），且在某些 LLM 实验中表现更稳定（避免了重中心化可能带来的分布偏移问题）。

**计算维度对比图**：
```text
特征维度示例: (Batch, Time/Len, Channels/Features)

Layer Norm / RMSNorm:
[------------------ Channels ------------------]
|  统计均值/方差 (跨所有 Features)              |
|  Result: (Batch, Time, 1)                    |

Batch Norm (1D for Sequence):
[------ Batch Dim ------] 
|  统计均值/方差 (跨 Batch, Time)              |
|  Result: (1, 1, Channels)                    |
```

#### ## 常见考点
1. **Transformer 中为什么使用 LayerNorm 而不是 BatchNorm？**
   - 答：1. 文本序列长度不固定，BN 统计困难；2. 单个样本（句子）内部的特征分布一致性更重要；3. 小 Batch 训练下 BN 不稳定。
2. **LayerNorm 放在 Residual 的前面还是后面？**
   - 答：Post-LN（Transformer 原论文）：Transformer [Add] -> [Norm]。Pre-LN（现代 GPT/LLM）：[Norm] -> [Attention/FFN] -> [Add]。Pre-LN 的训练梯度更稳定，适合训练深层网络。
3. **RMSNorm 为什么在 LLM 中流行？**
   - 答：除了计算效率（少一次减法），理论上认为在 ReLU/GELU 等激活函数配合下，中心化操作并非必须，且 RMSNorm 在数值稳定性上表现优异。

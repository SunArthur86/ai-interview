# 微调LLM的数据如何准备？有哪些数据质量最佳实践？

🎯 本质：微调数据的质量远比数量重要——1000条高质量数据通常胜过10万条低质量数据。

📊 数据准备流程：

1. 数据收集
  来源：人工编写、GPT-4生成、真实用户交互日志、开源数据集
  多样性：覆盖不同话题、难度、长度、风格
  数量参考：通用SFT约5K-50K条，领域特定约1K-10K条

2. 数据格式
  Chat格式（推荐）：
    {"messages": [
      {"role": "system", "content": "你是一个助手"},
      {"role": "user", "content": "什么是RAG?"},
      {"role": "assistant", "content": "RAG是..."}
    ]}
  
  Alpaca格式：
    {"instruction": "...", "input": "...", "output": "..."}

3. 数据质量标准
  准确性：答案必须正确（最重要）
  完整性：回答要完整，不要半截话
  一致性：相同类型的问题用相同的格式回答
  多样性：避免重复模式（如同一个模板生成1000条）
  格式：JSON有效，无编码问题

4. 数据清洗
  去重：相似度去重（cosine similarity > 0.9的删一个）
  去噪：去除乱码、截断的文本
  长度过滤：过短（<10字符）或过长（>4096字符）的删除
  安全过滤：移除有害内容

5. 数据混合(Data Mixing)
  通用能力保持：混入10-20%通用SFT数据
  防止灾难性遗忘（微调后丧失原有能力）
  多任务混合：不同任务比例平衡

高质量数据生成方法：
1. Self-Instruct：用强模型生成指令-回答对
2. Evol-Instruct：逐步增加指令复杂度
3. GPT-4辅助生成：用GPT-4生成回答，人工审核
4. 合成数据：根据模板批量生成，人工抽样检查

面试加分点：
- "Quality over Quantity"是SFT数据的核心原则
- LIMA论文证明：1000条高质量数据就能显著提升模型
- 灾难性遗忘：只用领域数据微调会导致通用能力下降
- 数据污染检查：确保测试集不在训练数据中

# RNN/CNN/Transformer对比

- **RNN**:
  - **结构**: 线性序列，依赖前序状态。
  - **缺点**: 长序列反向传播易梯度消失/爆炸；难以并行训练。

- **CNN**:
  - **结构**: 局部感受野，保留相对位置。
  - **优点**: 并行能力强，优于RNN。
  - **缺点**: 长距离特征捕获能力弱于Transformer和RNN。

- **Transformer**:
  - **结构**: 自注意力机制，全局建模。
  - **优点**: 并行性强，长距离捕获能力优秀，综合效果最佳。

- **综合对比**:
  1. **语义提取**: Transformer > RNN ≈ CNN
  2. **长距离捕获**: Transformer ≈ RNN >> CNN
  3. **综合任务(翻译)**: Transformer > CNN ≈ RNN
  4. **并行效率**: Transformer ≈ CNN >> RNN

- **交叉熵**:
  - **公式**:
    - 二分类: $L = -[y \log(\hat{y}) + (1-y) \log(1-\hat{y})]$
    - 多分类: $L = -\sum y \log(\hat{y})$
  - **求导**: $\frac{\partial L}{\partial \hat{y}} = -[\frac{y}{\hat{y}} - \frac{1-y}{1-\hat{y}}]$ (链式法则结合Sigmoid导数)。

- **代码实现 (二分类)**:
```python
def binary_cross_entropy_loss(y_pred, y_true):
    y_pred = torch.sigmoid(y_pred)
    y_pred = torch.clamp(y_pred, 1e-7, 1.0 - 1e-7)
    return - (y_true * torch.log(y_pred) + (1 - y_true) * torch.log(1 - y_pred)).mean()
```

# HuggingFace 下载模型包含哪些文件

## 核心文件结构
从 HuggingFace Hub 下载的模型仓库通常包含以下几类关键文件：

### 1. 模型权重文件
这是模型参数的二进制存储，体积最大。
*   **`pytorch_model.bin`**: 传统的 PyTorch 权重文件，使用 Python pickle 格式。存在安全隐患（加载时可执行任意代码）。
*   **`model.safetensors`**: **目前最推荐的安全格式**。它是纯数据文件，零拷贝加载速度快，且避免了 pickle 的安全风险。
*   **Sharded Files (分片)**: 对于大模型（如 7B+），权重会被切分。
    *   `model-00001-of-00003.safetensors`
    *   `model-00002-of-00003.safetensors`
    *   ...
    *   `model.safetensors.index.json`: 记录了每个参数名在哪一个分片文件中，充当索引字典。

### 2. 配置文件 (JSON)
定义模型结构的行为和超参数，无状态。
*   **`config.json`**: **核心架构文件**。包含模型层数、隐藏层大小 (`hidden_size`)、注意力头数 (`num_attention_heads`)、激活函数类型等。
*   **`generation_config.json`**: 生成时的默认参数。包含 `max_new_tokens`, `temperature`, `do_sample`, `eos_token_id` 等。`model.generate()` 时会优先读取此文件。

### 3. 分词器文件
负责文本和 ID 之间的转换。
*   **`tokenizer.json`**: 包含完整的分词器逻辑和词汇表（HuggingFace Tokenizers 库的标准格式，通常比纯 Python 实现快）。
*   **`vocab.json`**: 词汇表，Token 到 ID 的映射字典。
*   **`merges.txt`**: BPE (Byte Pair Encoding) 算法的合并规则列表。
*   **`special_tokens_map.json`**: 定义特殊 Token（如 `<|startoftext|>`, `<|endoftext|>`）的映射。
*   **`tokenizer_config.json`**: 分词器的运行参数，如 `max_length`, `padding_side` (left/right)。

### 4. 其他重要文件
*   **`README.md`**: 模型介绍、使用许可、论文链接、Benchmark 结果。
*   **`.gitattributes`**: Git LFS 配置文件，指示哪些大文件（如 `.bin`, `.safetensors`）由 Git LFS 管理，而不是直接存入 Git 历史。
*   **`training_args.bin`** / **`trainer_state.json`**: (如果有) 训练时的超参数和检查点状态，用于恢复训练。

## 文件加载逻辑简图

```text
User Code: AutoModel.from_pretrained("model_name")
        │
        ├─> 读取 config.json  (构建模型骨架)
        │
        ├─> 读取 tokenizer_config.json + tokenizer.json (初始化 Tokenizer)
        │
        └─> 读取 model.safetensors.index.json (查找权重分布)
             │
             └─> 加载 model-00001-of-xxx.safetensors ... (填充骨架权重)
```

## 常见考点
1.  **Safetensors vs Bin**：为什么现在都用 Safetensors？安全性和加载速度（mmap）。
2.  **缺失文件处理**：如果只有 `config.json` 没有 `pytorch_model.bin` 会发生什么？加载报错，或者如果是 Transformers 支持的转换库，可能会尝试从 GitHub 或其他源自动转换（如 `from_tf=True`）。
3.  **分词器兼容性**：`tokenizer.json` 和 `vocab.json` 有什么区别？`tokenizer.json` 是现代格式，包含所有规则；`vocab.json` 只是词表，旧版本或特定分词器（如 BERT）可能更依赖它。

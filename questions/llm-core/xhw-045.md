# huggingface下载模型有哪些文件

### HuggingFace 模型文件组成

从 HuggingFace 下载模型时，仓库中通常包含以下四类主要文件：

#### 1. 分词器相关文件
用于将输入文本预处理为模型可理解的 Token 序列。

*   **tokenizer.json**：分词器的完整配置和词汇表，包含如何将文本转换为 Token 的规则（使用 tokenizers 库格式）。
*   **vocab.json**：词汇表文件，映射 Token 与对应的整数 ID（常见于 BERT/GPT 等模型）。
*   **merges.txt**：BPE（Byte Pair Encoding）分词器的合并规则文件，定义子词单元的合并顺序。
*   **tokenizer_config.json**：分词器的配置文件，定义分词器类型及特殊符号（如 `[CLS]`, `[SEP]`, `[PAD]`）等。

#### 2. 模型权重文件
核心文件，包含模型的训练参数。

*   **pytorch_model.bin**：传统的 PyTorch 权重文件（使用 pickle 序列化）。
*   **model.safetensors**：目前推荐的安全格式，支持零拷贝加载且避免执行任意代码。
*   **model-00001-of-00004.safetensors**：当模型过大时，权重会被切分为多个分片文件。
*   **model.safetensors.index.json**：权重分片的索引文件，记录了各参数在不同分片中的存储位置，便于加载。

#### 3. 配置文件
定义模型结构和推理生成的参数。

*   **config.json**：模型架构配置，包含层数、隐藏层维度、注意力头数等关键参数。
*   **generation_config.json**：生成参数配置，定义文本生成时的默认行为，如 `max_length`（最大长度）、`temperature`（温度）、`top_k`、`top_p` 等。

#### 4. 许可与文档
提供仓库的法律和使用说明。

*   **README.md**：自述文件，包含模型介绍、使用方法、性能指标及示例代码。
*   **LICENSE**：许可证文件，规定模型的使用、修改和分发规则（如 Apache 2.0, MIT）。
*   **.gitattributes**：Git 属性文件，用于指定哪些文件通过 Git LFS（Large File Storage）存储（通常是大权重文件）。

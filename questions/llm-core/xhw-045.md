# huggingface下载模型有哪些文件

🐋PRM在复杂推理任务中具有巨大潜力，其优点体现在：
细粒度反馈：与结果监督相比，过程监督对每一步推理进行评价，直接鼓励模型生成更连贯
的逻辑链条，而不是仅追求最终结果正确性。
解释性增强：由于每一步推理都受到监督，过程监督能够更容易地生成人类可理解的逻辑
链，从而减少模型产生幻觉的可能性。
负对齐税（Negative Alignment Tax）：在对数学领域任务的测试中，过程监督不仅提高了
性能，还减少了对齐过程中潜在的性能损失。
👍通过结合强化学习与生成模型的优点，PRM 不仅显著提升了模型推理能力，还为未来智能系
统中的推理任务开辟了新思路。未来改进方向可能包括：
- 优化路径采样策略：提高有效路径的生成效率；
- 引入更多多模态数据：扩展 PRM 的应用场景；
- 结合其他强化学习技术：如策略优化（PPO）或直接偏好优化（DPO）。
73. 从huggingface下载模型时有哪些文件？
🎓从huggingface下载模型时主要有四类
文件
🧊分词器相关文件（如 
tokenizer.json  和 
merges.txt ）：用于预处理输入文
本，将其转为 token。
🧊模型权重文件（如 
.safetensors ）：核心文件，包含
模型的参数。
🧊配置文件（如 config.json ）：
定义模型结构和生成过程。
🧊许可证与文档（如 LICENSE  和 
README.md ）：提供仓库的法律和使
用说明。
73.1 许可证和git文档相关

🌵
.gitattributes
💚 描述：Git 属性文件，用于定义 Git LFS（Large File Storage）的设置和其他版本控
制属性。
💙用途：指定哪些文件使用 Git LFS 存储（例如 .safetensors  文件）。
LICENSE
💚 描述：模型或代码的许可证文件。
💙用途：规定此仓库内容的使用和分发规则。
NOTICE
💚 描述：法律声明或通知。
💙用途：提供与模型或库的版权相关的信息，可能涉及第三方依赖。
README.md
💚 描述：仓库的自述文件。
💙用途：通常提供关于模型、如何使用它的说明以及相关链接。
73.2 模型配置相关文件
🍁
config.json
💛 描述：模型的配置文件。
❤️ 用途：包含模型的架构细节，例如层数、隐藏单元数、注意力头数等。
generation_config.json
💛 描述：生成配置文件。
❤️ 用途：定义文本生成时的参数，如最大生成长度、温度、Top-k采样等。
73.3 分词器相关文件
🌻
merges.txt
💛 描述：BPE（Byte Pair Encoding）分词器的合并规则文件。
💚用途：指定词汇表构建中合并子词单元的规则。
tokenizer.json
💛 描述：分词器的完整配置和词汇表。
💚 用途：包含如何将文本转换为模型可处理的 token。

# MoE(Mixture of Experts)在大模型中的工作原理是什么?DeepSeek-MoE有什么创新

MoE通过稀疏激活实现「参数量大但计算量小」的效果.

- **基本原理:**
- 传统FFN: y = FFN(x) 所有参数参与计算
- MoE FFN: y = Gate(x)_i · Expert_i(x) 的加权和 只激活Top-K个专家

- N个专家网络(每个是独立的FFN)
- Gate/Router计算每个token应该分配给哪些专家
- 通常Top-2路由:每个token只经过2个专家
- 效果:总参数量N倍,但每次前向计算量只有2/N

- **关键技术问题:**

1. **负载均衡(Load Balancing)**
- 问题:Router可能偏向少数专家(马太效应)
- 解决:辅助损失(auxiliary loss)惩罚不均匀分配

2. **容量因子(Capacity Factor)**
- 每个专家有最大处理token数限制
- 超出容量的token被丢弃(drop)或传递给下一层

3. **通信开销**
- 分布式训练时专家分布在不同GPU
- 需要All-to-All通信传递token

- **DeepSeek-MoE的创新:**

1. **细粒度专家分割** - 将每个专家再分割为更小的子专家(如64个),增加路由灵活性
2. **共享专家(Shared Expert)** - 设置部分专家始终激活,处理通用知识
3. **设备级负载均衡** - 不只均衡专家级,还均衡设备级负载
4. **效果:** DeepSeek-V2使用236B总参数,但每次推理只激活21B,效果媲美Dense模型

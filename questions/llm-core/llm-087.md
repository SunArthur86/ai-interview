# 为什么要优化KV cache

93. 1 KV cache 是什么?为什么它如此重要?
🫐KV cache(Key-Value 缓存)是在 Transformer 模型中保存历史输入的键(Key)和值 (Value)的存储机制,目的是在生成序列时避免重复计算,从而大幅提升效率.每个生成步 骤需要基于之前的输入计算注意力分数,而保存 KV cache 可减少重复操作.

🥥KV cache的作用 1.
**避免重复计算：** 保存键和值后,后续计算仅需基于缓存数据更新新 token 的注意力分数. 若无 KV cache,每次生成新 token 都需从头计算完整的输入序列,效率大幅下降. 2.
**提升推理效率：** KV cache 的核心是通过存储历史计算结果,加速后续序列生成. 3.
**减少显存使用的冲突：** 推理中每一层的 Key 和 Value 会持续增长,因此需要优化其占用的显存空间.

93. 2.2 kv cache优化的必要性

**参考资料：** https://blog.csdn.net/javastart/article/details/137948164 参考上图,随着 batch size 和 长度的增大,KV cache 占用的显存开销快速增大,甚至会超过模型本 身.从LLM的趋势上而讲,主要有三个方面来说明kv cache优化的必要性:
1. 总体趋势上LLM 的窗口长度在不断增大,因此就出现一组主要矛盾,即:对不断增长的 LLM 的窗
口长度的需要与有限的 GPU 显存之间的矛盾.因此优化 KV cache 非常必要.OpenAI API场景, API最烧钱的是输入而非输出,输入包括prefill prompt 和conversation,长度动辄数10k token. 虽说每输入token比每输出token便宜,但能够降低kv重新计算的开销,无论是硬件资源门槛,还 是模型推理降本,都有着极为积极的作用.
2. 对于消费级显卡这种性价比较高的显卡而言,显存容量相对较小,KV cache从一定程度上降低了模
型的batch size,因而KV cache优化在工程落地中更显重要. 从上表能够看出,类似4090等消费级显卡,其主要的瓶颈时batch_size,而影响batch_size的主要因 素就是显存容量,而KV cache在推理过程中占用大量的显存.如果能通过KV cache降低显存占用,从 一定程度上就能提升消费级显卡的性价比,带来非常高的商业收益.
3. KV cache 优化在 AIGC 场景中的重要性
除了语言模型,视频生成和图像生成模型也面临类似问题.以 Stable Diffusion 或视频生成模型(如 Sora)为例: • **模型特点：** 虽然模型参数较小(通常 <10B),但生成序列长度非常长(接近 1000k patches). • **显存比例：** KV cache 对显存的占用可能达到 90%,甚至更多. 实验数据支持 • batch size = 1:KV cache 与模型参数的显存占比达到 10:1(例如4090的24G显存,2G分给模型, 22G分给kv cache). • batch size = 4:这一比例进一步增大到 40:1(batch size越大,kv cache的显存越大). 在此类任务中,优化 KV cache 不仅能够提高显存利用率,还可以直接提升生成速度和性能.

93. 3 研究方向:如何优化 KV cache?
🍁 压缩存储 使用更高效的数值表示(如 FP8)或稀疏表示法,降低单个键值对的存储需求. 动态分配缓存区,仅保存最重要的 KV 数据. 分层缓存 引入多层缓存机制(如显存与主存协作),实现更高效的数据存储与读取. 分块存储 对长序列任务按块拆分 KV 数据,减少显存峰值需求. 推理算法优化 利用注意力机制的低秩特性,减少缓存数据的存储规模. 🌟总结 研究 KV cache 的优化方法,不仅能解决当前大语言模型推理中的显存瓶颈,还能推动多模态生成任务 的发展.随着序列长度不断增加,KV cache 的设计将成为 AI 基础设施中的关键一环.

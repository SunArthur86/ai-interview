# 从huggingface下载模型时有哪些文件

🎓从huggingface下载模型时主要有四类 文件 🧊分词器相关文件(如 tokenizer.json 和 merges.txt ):用于预处理输入文 本,将其转为 token. 🧊模型权重文件(如 .safetensors ):核心文件,包含 模型的参数. 🧊配置文件(如 config.json ): 定义模型结构和生成过程. 🧊许可证与文档(如 LICENSE 和 README.md ):提供仓库的法律和使 用说明.
73. 1 许可证和git文档相关

🌵 .gitattributes 💚 描述:Git 属性文件,用于定义 Git LFS(Large File Storage)的设置和其他版本控 制属性. **💙用途：** 指定哪些文件使用 Git LFS 存储(例如 .safetensors 文件).
**LICENSE**
LICENSE 💚 描述:模型或代码的许可证文件. **💙用途：** 规定此仓库内容的使用和分发规则.
**NOTICE**
NOTICE 💚 描述:法律声明或通知. **💙用途：** 提供与模型或库的版权相关的信息,可能涉及第三方依赖. README.md 💚 描述:仓库的自述文件. **💙用途：** 通常提供关于模型、如何使用它的说明以及相关链接.
73. 2 模型配置相关文件
🍁 config.json 💛 描述:模型的配置文件. ❤️ 用途:包含模型的架构细节,例如层数、隐藏单元数、注意力头数等. generation_config.json 💛 描述:生成配置文件. ❤️ 用途:定义文本生成时的参数,如最大生成长度、温度、Top-k采样等.
73. 3 分词器相关文件
🌻 merges.txt 💛 描述:BPE(Byte Pair Encoding)分词器的合并规则文件. **💚用途：** 指定词汇表构建中合并子词单元的规则. tokenizer.json 💛 描述:分词器的完整配置和词汇表. 💚 用途:包含如何将文本转换为模型可处理的 token.

tokenizer_config.json 💛 描述:分词器的配置文件. 💚 用途:定义分词器类型、特殊符号(如 [CLS] 和 [SEP] )等. vocab.json 💛 描述:分词器的词汇表文件. 💚 用途:映射 token 与对应的整数 ID.
73. 4 模型权重相关文件
📘 model-00001-of-00004.safetensors 至 model-00004-of-
00004. safetensors
💙 描述:模型的分块权重文件,使用 safetensors 格式存储. 💚 用途:包含训练好的模型参数,分块存储以便处理大文件. model.safetensors.index.json 💙 描述:权重文件的索引文件. 💚 用途:提供各个分块权重文件的位置映射信息,便于加载.

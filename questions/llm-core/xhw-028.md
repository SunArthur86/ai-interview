# DeepSpeed ZeRO系列原理

DeepSpeed ZeRO (Zero Redundancy Optimizer) 是一种针对大模型训练的显存优化技术，旨在消除数据并行中的冗余存储。

### 核心思想
在标准数据并行（DDP）中，每个 GPU 都保存完整的模型参数、梯度和优化器状态，这造成了极大的冗余。ZeRO 通过切分这些状态，在保持计算逻辑不变的前提下，大幅降低单卡显存占用。

### 三个阶段

#### 1. ZeRO-1 (Optimizer State Partitioning)
- **切分对象**：优化器状态（如 Adam 的一阶矩和二阶矩）。
- **做法**：将优化器状态按维度平均切分到不同 GPU 上。
- **效果**：显存占用减少约 4 倍（假设使用 Adam）。

#### 2. ZeRO-2 (Gradient Partitioning)
- **切分对象**：优化器状态 + 梯度。
- **做法**：在反向传播计算梯度后，不立即同步更新，而是将梯度切分存储并更新对应部分的参数。通过 All-Gather 获取完整参数进行前向计算，但不需要在每卡上都存全量梯度。
- **效果**：显存占用进一步减少，约为 DDP 的 1/8。

#### 3. ZeRO-3 (Parameter Partitioning)
- **切分对象**：优化器状态 + 梯度 + 模型参数。
- **做法**：模型参数也被切分到不同 GPU 上。当某一层需要计算时，动态地从其他 GPU 收集（All-Gather）该层的参数，计算完成后立即释放。
- **效果**：显存占用极低，线性扩展能力极强，支持训练万亿参数模型。

### 优势
- **显存节省**：支持在有限显存上训练超大模型。
- **通信优化**：虽然增加了通信量，但通过计算与通信重叠，维持了较高的计算效率。

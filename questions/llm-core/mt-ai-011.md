# 【美团面经】请说说开发过的 AI 产品/功能是什么？用户需求是什么？最终结果如何？（STAR 回答框架）

**AI 产品面试回答框架（STAR 法）：**

**S（Situation）背景：**
- 业务痛点是什么？
- 影响范围（用户量/收入）？
- 为什么现在做？（时机判断）

**T（Task）任务：**
- 你的角色和职责？
- 目标是什么？（定量指标）
- 有什么约束？（预算/时间/技术）

**A（Action）行动：**
- 技术方案选型（为什么选这个模型/方案）？
- 架构设计（数据流/调用链/降级策略）？
- 难点解决（延迟/幻觉/成本/评测）？
- 迭代过程（v1→v2 做了什么优化）？

**R（Result）结果：**
- 定量指标（准确率/延迟/成本/DAU）？
- 用户反馈（满意度/NPS）？
- 业务影响（收入/效率提升）？
- 学到了什么？（反思）

**示例（RAG 知识库）：**
```
S: 公司内部文档分散在飞书/Wiki/共享盘，新员工查找效率低
T: 构建 AI 知识助手，目标覆盖率 90%、准确率 85%、延迟 <3s
A: 
  - 方案：Embedding + 向量检索 + Reranker + LLM 生成
  - 模型：bge-large-zh（Embedding）+ Qwen2.5-72B（生成）
  - 优化：混合检索(向量+BM25)、查询改写、分块策略调优
  - 降级：检索失败→兜底搜索，LLM 超时→缓存回答
R:
  - 准确率从 v1 的 62% → v2 的 88%
  - 平均延迟 2.1s
  - 月活 3000+，员工满意度 4.2/5
  - 学到：分块策略和 Reranker 对质量影响最大
```

**面试加分点：**
- 主动说**失败和教训**（体现反思能力）
- 有**数据对比**（v1 vs v2）
- 讨论**trade-off**（成本 vs 质量 vs 延迟）
- 提出**下一步规划**（体现持续迭代思维）

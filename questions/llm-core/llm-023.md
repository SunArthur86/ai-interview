# 为什么Transformer用LayerNorm而不用BatchNorm

Transformer 选择 LayerNorm 而非 BatchNorm，主要源于 NLP 任务的数据特性和模型架构需求。

### 1. 概念阐述
*   **BatchNorm (BN)**：对 Batch 中不同样本的**同一位置**（同一特征通道）进行归一化。它依赖于 Batch Size，计算该批次内所有样本在某一维度的均值和方差。
*   **LayerNorm (LN)**：对单个样本的所有**特征维度**进行归一化。它不依赖于 Batch Size，计算该样本自身的均值和方差。

### 2. Transformer 使用 LayerNorm 的原因

**处理变长序列**
NLP 任务中，输入的句子长度往往不一致。Padding 后的样本包含大量无意义的填充符。如果使用 BatchNorm，填充符的统计信息（均值、方差）会干扰有效特征的计算，导致归一化结果不准确。LayerNorm 仅计算每个样本内部有效 token 的统计量，不受 Padding 影响。

**独立于批量大小**
BatchNorm 的效果受 Batch Size 影响较大。当 Batch Size 较小时（如 1 或 2），统计量的估计不准确，方差估计偏差大，导致模型训练不稳定。在推理阶段，BN 需要使用训练集全局统计量（Running Mean/Var），可能引入误差。LayerNorm 在训练和推理时的行为完全一致，不依赖全局统计，更稳定。

**适应自注意力机制**
Transformer 的核心是 Self-Attention，不同位置的 Token 之间进行交互。LayerNorm 对每个 Token 的特征向量独立归一化，保留了单个 Token 的特征分布特性，使得 Attention 机制能够更灵活地关注序列中的关系。

### 3. 总结
Transformer 之所以选择 LayerNorm，是因为它：
1. 不受 Batch Size 限制，适合小批量训练；
2. 能自然处理变长序列，不受 Padding 干扰；
3. 训练与推理行为一致，实现更简单。

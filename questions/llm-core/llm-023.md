# 为什么Transformer用LayerNorm而不用BatchNorm

layer normal
batch normal
首先,我们应该先对我们的数据形状有个直观
的认识,当一个batch的数据输入模型的时候.
形状是长方体如图所示,大小为(batch_size,
max_len, embedding),其中batch_size为
batch的批数,max_len为每一批数据的序列最
大长度,embedding则为每一个单词或者字的
embedding维度大小.
Batch Normalization是在batch间选择同一个位置的值做归一化,相当于是对batch里相同位置的字
或者单词embedding做归一化.
Layer Normalization是在一个Batch里面的每一行做normalization,相当于是对每句话的
embedding做归一化.显然,LN更加符合我们处理文本的直觉.
1. 概念阐述
BatchNorm(批量归一化) 主要用于训练深度神经网络时对每一层的激活进行归一化,以减少内部
协变量偏移(internal covariate shift).它通过对每一批数据进行归一化来稳定训练过程,并通常会
加快收敛速度.
LayerNorm(层归一化) 归一化的是每一层的所有特征,不依赖于批量大小,而是对单个样本的所
有特征进行归一化.这对于处理变长序列数据和动态批量大小尤其重要.
2. Transformer模型的特性
**变长序列：** Transformer模型处理的是变长的序列数据,BatchNorm在处理变长序列时可能会遇到
问题,因为它需要在每个批次内计算均值和方差,这对于动态长度的序列不太适用.
**自注意力机制：** Transformer的自注意力机制允许每个位置对序列中的所有其他位置进行关注.
BatchNorm的统计信息是基于批次的统计,这可能会导致不同位置的信息互相干扰,从而影响模型的
性能.LayerNorm则是对每个样本的所有特征进行归一化,避免了这种干扰.
3. BatchNorm的局限性
**批量大小依赖性：** BatchNorm的表现依赖于批量的大小.当批量大小较小或序列长度不一致时,它
的统计量可能不稳定,从而影响模型的训练效果.
**推理阶段：** 在推理阶段,BatchNorm需要用到训练阶段的均值和方差进行归一化,这增加了推理的
复杂性和开销.LayerNorm在训练和推理阶段的表现一致,因为它的归一化是基于每个样本的特征,
而不是批量统计.
4. LayerNorm的优点

**独立于批量大小：** LayerNorm不受批量大小的影响,对于处理变长序列数据非常合适.
**更稳定的训练：** 由于LayerNorm在每个样本的特征维度上进行归一化,模型在处理每个样本时的行为
是一致的,这使得训练过程更稳定.
**简化推理过程：** LayerNorm在训练和推理阶段的行为一致,不需要额外的训练阶段统计量,因此推理
过程更简单高效.
总结来说,Transformer使用LayerNorm而不是BatchNorm,主要是因为LayerNorm更适合处理变
长序列数据,并且在训练和推理阶段的一致性上具有优势.这使得LayerNorm在Transformer架构中
能更好地支持自注意力机制的动态特性.

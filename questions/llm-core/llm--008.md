# Mamba(状态空间模型 SSM)与 Transformer 的核心区别是什么?它有什么优势

- **Mamba (State Space Model, SSM)**

**核心区别：** 
1. **注意力机制**:Transformer 用全局自注意力(O(n²)),Mamba 用选择性状态空间模型(O(n))
2. **推理复杂度**:Transformer 推理时每生成一个 token 需回看全部历史(KV Cache);Mamba 用固定大小的隐状态,推理 O(1)
3. **长序列建模**:Mamba 天然适合超长序列(无 context window 限制)

- **Mamba 的核心创新:选择性 SSM**
- 传统 SSM(S4)是时不变的,无法根据输入选择性记忆
- Mamba 引入输入相关的参数(Δ, B, C 随输入变化),实现'选择性遗忘和记忆'
- 类比:Mamba 像 RNN 一样有压缩的隐状态,但通过选择性机制接近 Transformer 的建模能力

- **优势**
- 训练:可线性扩展到百万级 token
- 推理:无 KV Cache 膨胀,吞吐量高
- 显存:固定隐状态,不随序列长度增长

- **劣势**
- 复制任务不如 Transformer(难以精确复制远距离 token)
- 生态不如 Transformer 成熟
- 当前在代码/数学任务上仍有差距

- **趋势**:Jamba (AI21) 等混合架构 = Transformer + Mamba,兼顾两者优势.

71. 什么是Prefill-Decode分离？为什么分离？
💡PD分离（Prefill-Decode分离）是大模型推理优化的重要策略之一，涉及将推理过程中的 
Prefill（预填充） 和 Decode（解码） 分开处理，以提升灵活性、效率和资源利用率。

Prefill的特点包括：

- 计算密集型：Prefill 阶段主要依赖于 Transformer 的前向传播计算，对 GPU 的算力要求
较高。
- 静态特性：由于用户的输入 Prompt 通常是固定的，可以以批量方式一次性处理多个样
本。这种批量处理的特点使 Prefill 在相同的计算资源下可以实现更高的吞吐量。
Prefill 的输出：
- 计算完成后，Prefill 会生成 Transformer 的激活值，主要包括 Key-Value Cache（KV 
Cache）。
- KV Cache 是一种存储机制，保存了注意力机制中的 Key 和 Value 向量，供后续解码阶段
重复使用，避免重复计算。

🎓 KV Cache 的主要作用
1. 减少重复计算：在 Decode 阶段，解码每个新 token 都需要与所有历史 token 交互。如果
没有 KV Cache，模型需要重新计算所有历史 token 的 Key 和 Value，而 KV Cache 可以
直接复用 Prefill 阶段的结果，显著减少计算开销。
2. 支持高效解码：Decode 阶段通过 KV Cache 直接检索历史 token 的信息，与新生成的 
token 交互，完成注意力计算。这种机制支持逐步生成，并确保解码过程的一致性和高效
性。
KV Cache 的代价
- 存储开销：KV Cache 的大小与模型的隐藏层维度和序列长度成正比。例如，LLaMA-65B 
在 2K 输入长度下的 KV Cache 大小约为 5GB。
- 网络带宽需求：在分布式场景中，Decode 阶段可能需要跨设备访问 KV Cache，这对网
络的吞吐量和延迟提出了较高要求。

# Qwen通义千问的结构

Qwen (通义千问) 是阿里推出的系列大模型，其结构基于 Transformer Decoder，并有多项改进。

### 1. 整体架构
- 基于 Transformer 解码器结构，使用 causal mask 进行自回归生成。

### 2. 核心组件
- **词表**：使用 BPE 分词，词汇表较大（约 152K），优化了多语言（特别是中文）的压缩效率。
- **嵌入层**：使用 RoPE (旋转位置编码) 处理位置信息，支持长文本外推。
- **注意力机制**：使用 GQA (分组查询注意力) 以加速推理，减少显存占用。
- **前馈网络 (FFN)**：采用 SwiGLU 激活函数。
- **归一化**：使用 RMSNorm，并且采用 Pre-Norm 结构（输入先归一化再进入子层）。

### 3. 特殊创新
- **非绑定嵌入**：输入嵌入层和输出层的权重不共享，提升了模型性能。
- **偏置调整**：去除了大多数线性层的 Bias，但在 QKV 注意力层中保留了 Bias，增强了模型的外推能力。
- **长上下文**：通过 NTK-aware 插值等技术扩展上下文窗口长度。

### 4. 训练策略
- **预训练**：使用超过 3T tokens 的多语言数据进行训练。
- **对齐**：包括 SFT（有监督微调）和 RLHF（人类反馈强化学习），以及专门的代码和数学能力增强训练。

### 5. 架构细节补充 (增强部分)

#### 5.1 注意力机制细节 (GQA)
Qwen 使用 **GQA (Grouped Query Attention)**，即分组查询注意力。在标准的 Multi-Head Attention (MHA) 中，每个 Head 都有独立的 Key 和 Value，计算量大且显存占用高；在 Multi-Query Attention (MQA) 中，所有 Head 共享一份 Key 和 Value，虽然极致压缩但可能损失性能。GQA 是一种折中方案：将 Query Heads 分组，组内共享 Key 和 Value。
- **参数**：Qwen 通常将 Key/Value 的 Head 数量设置为 Query Head 数量的 1/N（例如 1/4 或 1/8）。
- **收益**：在保持模型性能接近 MHA 的同时，大幅推理时的 KV Cache 显存占用，显著提升推理吞吐量。

#### 5.2 激活函数与 FFN (SwiGLU)
Qwen 的 FFN 层结构为：
$$ FFN(x) = (Swish(xW_G) \cdot (xW_{up})) W_{down} $$
- **SwiGLU** 结合了 Swish 激活函数和门控线性单元。相比传统的 ReLU 或 GeLU，SwiGLU 在大模型上通常能带来更好的收敛性和性能表现。
- **维度**：隐藏层维度通常扩大到输入维度的 $\frac{2}{3}$ 或更大（具体取决于模型版本），以容纳门控机制带来的参数增加。

#### 5.3 位置编码 (RoPE)
- **RoPE (Rotary Positional Embeddings)**：通过旋转矩阵将位置信息注入到 Query 和 Key 的向量中，具有相对位置感知能力。
- **外推性**：相比绝对位置编码，RoPE 在处理比训练长度更长的文本时具有更好的外推能力。Qwen 通过 **NTK-aware Scaled RoPE** 技术，在不重新训练的情况下动态调整基频，有效支持长文本（如 32k 甚至更长）的推理。

## 常见考点
1. **GQA (分组查询注意力) 的原理及优势**：
   - 面试官可能追问 GQA 与 MHA、MQA 的具体区别（KV Cache 的计算量和显存占用差异），以及为什么在大模型推理中至关重要。
2. **RoPE 的外推性**：
   - 如何让模型处理超过训练长度的文本？（动态 NTK-aware 插值、YaRN 等技术）。
3. **权重不共享**：
   - 为什么 Qwen 选择 Embedding 和 LM Head 不共享权重？（通常认为大模型参数量足够大时，不共享能分别优化输入理解和输出生成，提升容量，但牺牲了少量正则化效果）。

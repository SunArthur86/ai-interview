# 为什么大模型用GQA(Group Query Attention)

现在大模型广泛使用GQA(Grouped-Query Attention)是为了在推理速度和模型性能之间取得更好的
平衡,特别是在内存带宽和推理速度成为瓶颈的情况下.要详细理解为什么大模型使用GQA,我们需
要先讲解与其相关的两种注意力机制:MHA(Multi-Head Attention)和MQA(Multi-Query
Attention).
1. Multi-Head Attention (MHA)
MHA是在2017年Google团队提出的,是Transformer模
型的核心组件.

在MHA中,每个注意力头(head)有自己独立的Query、
Key和Value投影矩阵,允许模型并行地处理输入数据的多
个方面.例如,一个注意力头可以专注于捕捉句子中的语法
特性,另一个则可以专注于语义特性.
**优势**
优势
**捕捉多样化的特征：** 不同的注意力头可以关注
输入序列中的不同模式和依赖关系.
**并行化计算：** 多个头部的自注意力机制是并行
处理的,这在训练期间尤其高效. 劣势 内存和计算复杂度高
每个注意力头都需要维护自己的Key和Value矩
阵,因此内存需求和计算量随着模型规模增加而
急剧上升.这尤其在大型模型和推理过程中成为
性能瓶颈.
2. Multi-Query Attention (MQA)
为了缓解MHA的内存带宽问题,Google在2019年提出了MQA.
在MQA中,虽然每个头依然保留独立的Query矩阵,但Key和
Value矩阵是共享的.这样做的好处是显著减少了需要加载和存储
的Key和Value数据,从而降低了内存带宽的需求,尤其是在自回
归推理场景中.这极大地提升了推理速度,尤其适合解码器场
景.
**优势**
优势
**推理速度大幅提升：** 共享Key和Value矩阵减少
了多头机制的内存带宽需求,推理速度更快.
**内存占用减少：** Key和Value矩阵的数量从MHA
中的多个缩减为一个,大大减少了内存占用. 劣势
**精度下降：** 由于所有头共享Key和Value矩阵,
模型难以捕捉输入数据中复杂和多样化的特征,
导致模型性能相比MHA有所下降.
3. Grouped-Query Attention (GQA)
📌GQA的核心目标是减少推理过程中KV cache的内存占用和带宽需求,这使得其成为一种专注
于推理加速的技术,而不是一种全新的训练方法.GQA通过将注意力头分组来共享Key和
Value矩阵,从而减少了KV缓存的大小.
GQA是在2023年提出的,是MHA和MQA的折中方案.

与MQA相比,GQA并不只使用单个Key和Value矩阵,而是将注意
力头划分为若干组(G组),每个组共享一个Key和Value矩阵.
这种方法在保留了一定的多样性和细粒度特征捕捉能力的同时,
大大减少了内存带宽和计算开销. Pre-Norm •
在Pre-Norm架构中,规范化操作(RMS
Norm或Layer Norm)是在自注意力(Self-
Attention)或前馈神经网络(FFN)计算之
前进行的.也就是说,每一层的输入首先被归 一化,然后再传递到注意力或前馈层. •
Pre-Norm能够确保在深层网络中输入的幅度
始终处于稳定范围,这对长链路依赖的模型尤 其有益.通过归一化操作的提前进行,模型能 以更加稳定的输入进行学习,从而有助于解决 深层模型中梯度消失的问题. Post-Norm •
在Post-Norm架构中,规范化操作是在自注意
力或FFN计算之后进行的.模型首先经过未归
一化的操作,最后将结果归一化以确保模型的 输出平衡. •
Post-Norm可以在训练初期获得较好的收敛效
果,尤其是在浅层模型中表现良好.然而,在
深层网络中,Post-Norm的缺点是可能会导
致训练过程中的梯度不稳定,特别是随着网络 深度的增加,梯度可能在传播过程中变得越来 越不稳定.
**GQA的优势体现在以下几个方面：**
**平衡速度和精度：** 相比MQA,GQA通过分组的方式允许部分注意力头共享Key和Value矩阵,而不是
所有头都共享,从而减少了推理速度和内存需求上的负担,同时保留了更多特征捕捉能力.质量接近
MHA,而速度几乎接近MQA.
**灵活性高：** GQA可以根据模型的需求灵活调整组数.例如,GQA-1相当于MQA,GQA-H则相当于
MHA.通过控制组数,模型可以在精度和推理速度之间进行权衡,适应不同规模和任务需求.
**内存带宽优化：** GQA可以有效减少Key和Value矩阵的大小,减少键值缓存所需的内存带宽.这在处
理大型模型时尤为重要,因为模型的复杂度随着尺寸增加而急剧上升,GQA的优化使得内存带宽消耗
与模型规模更好地成比例.
4. 总结
大模型采用GQA的核心原因在于其能够在推理速度和模型性能之间提供优良的平衡.相比于MHA,
GQA大大减少了内存带宽和计算负担,适用于自回归推理等场景,而相比MQA,GQA通过分组策略保
留了更多模型的表现能力.因此,GQA提供了一个在推理效率和模型精度之间折中的解决方案,非常
适合当前大模型的实际需求.

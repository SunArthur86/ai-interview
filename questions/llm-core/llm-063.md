# 手撕flash-attention

🚅 FlashAttention是一种优化的注意力机制实现,旨在提高大模型中的注意力计算效率,特别 是降低了内存占用和加速计算.它通过将常规的注意力计算步骤重组为块状矩阵运算,并通

过减少中间结果的存储来减少内存的占用.FlashAttention使用了部分矩阵乘法和片段 (chunked)计算,以优化内存使用,特别适用于长序列的计算.
1. FlashAttention的基本步骤

**在常规的注意力计算中,我们有：**
Query、Key 和 Value 矩阵的计算

import torch import torch.nn.functional as F class FlashAttention(torch.nn.Module): def __init__(self, embed_dim, num_heads, scale=None): super(FlashAttention, self).__init__() self.embed_dim = embed_dim self.num_heads = num_heads self.head_dim = embed_dim // num_heads self.scale = scale or self.head_dim ** -0.5

# 初始化 Query, Key, Value 的线性映射
self.q_proj = torch.nn.Linear(embed_dim, embed_dim) self.k_proj = torch.nn.Linear(embed_dim, embed_dim) self.v_proj = torch.nn.Linear(embed_dim, embed_dim) self.out_proj = torch.nn.Linear(embed_dim, embed_dim) def forward(self, x): batch_size, seq_len, embed_dim = x.size()
# 计算 Q, K, V 矩阵
Q = self.q_proj(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2) K = self.k_proj(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2) V = self.v_proj(x).view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)

# 取当前块的 Key 和 Value
K_block = K[:, :, i:i + block_size] V_block = V[:, :, i:i + block_size]
# 初始化块内的注意力得分和 softmax 缓存
attention_scores = torch.zeros(batch_size, self.num_heads, seq_len, block_size, device=x.device) max_scores = torch.full((batch_size, self.num_heads, seq_len, 1), float('-inf'), device=x.device) softmax_sum = torch.zeros(batch_size, self.num_heads, seq_len, 1, device=x.device)

for j in range(0, seq_len, block_size):
# 取 Query 当前块
Q_block = Q[:, :, j:j + block_size]

# 计算 Q_block 和 K_block 的相似度,并进行缩放
attention_chunk = torch.matmul(Q_block, K_block.transpose(-1,
- 2)) * self.scale

# 使用局部 softmax 减少内存开销
max_scores = torch.maximum(max_scores, attention_chunk.max(dim=-1, keepdim=True)[0]) exp_scores = torch.exp(attention_chunk - max_scores) softmax_sum += exp_scores.sum(dim=-1, keepdim=True)

# 计算注意力输出
attention_scores[:, :, j:j + block_size] = exp_scores / softmax_sum
# 累积输出

**out[：** , :, i:i + block_size] = torch.matmul(attention_scores, V_block)
# 通过线性层返回最终输出
out = out.transpose(1, 2).contiguous().view(batch_size, seq_len, embed_dim) return self.out_proj(out)

3. 逐行解释
1. __init__ 方法: embed_dim : 输入嵌入的维度(例如 512). num_heads : 多头注意力机制的头数. scale : 缩放因子,通常为 1/√d_k ,即 head_dim 的倒数. 初始化中,我们使用线性层 q_proj , k_proj , v_proj 来生成 Query、Key、Value 矩阵, 同时 out_proj 负责最后的输出投影.
2. forward 方法: 首先,将输入 x 通过线性层生成 Query、Key 和 Value 矩阵,并重塑为 (batch_size, num_heads, seq_len, head_dim) 的形状. **分块处理：** 通过 for 循环将 Key 和 Value 矩阵按块分割,每次处理一个大小为 block_size 的子块. ▪ 对每个 Query 块和 Key 块进行矩阵乘法,计算注意力得分. ▪ 对得分进行局部的 softmax 操作,使用块内最大值减少浮点数溢出的可能. 使用得到的注意力得分与对应的 Value 块进行矩阵乘法,得到最终的输出.
3. 分块计算:FlashAttention 的关键在于分块处理,使得在处理长序列时,减少了整个注意力
矩阵的显式存储需求,每次只计算并存储一个块的注意力得分,从而减少内存消耗.
4. 总结
⚽这段代码实现了一个简化版的FlashAttention,其中通过分块操作减少了内存占用.在训练 长序列数据时,它可以显著提高注意力计算的速度和内存效率,特别适合处理大规模的 transformer 模型.

# 手撕flash-attention

FlashAttention 是一种通过分块计算和在线更新优化注意力机制的算法，减少内存访问开销并加速计算，特别适合长序列任务。核心步骤包括：1）将 Q、K、V 分块处理；2）在线计算 softmax（逐步更新最大值和指数和）；3）逐块累加输出。

### 核心原理：Tiling 与 Online Softmax
标准 Attention 计算 $O = \text{softmax}(QK^T)V$ 的时间瓶颈在于从 HBM (High Bandwidth Memory) 读写巨大的 $N \times N$ 注意力矩阵。FlashAttention 通过分块将计算局限在 SRAM (Static RAM) 中进行，极大减少了 HBM 访问次数。

为了在分块中计算 Softmax，必须使用 **Online Softmax** 算法，即在不看到所有数据的情况下，分步更新 Softmax 的分子和分母。

### 算法流程图

```text
Input: Q, K, V (分块加载到 SRAM)

Initialize: O = 0, m = -∞, l = 1 (用于存储分块累积的输出、最大值、归一化因子)

Loop over K, V blocks (inner loop):
  ┌─────────────────────────────────────────────┐
  │  Load block of Q (fix in SRAM)              │
  │  Loop over blocks of K, V (traverse cols):  │
  │    1. Load block K_j, V_j from HBM -> SRAM  │
  │    2. Compute S_ij = Q_i * K_j^T (dot prod) │
  │    3. Compute P_ij = softmax(S_ij)          │
  │       (使用 Online Softmax 更新 m, l)       │
  │    4. Compute O_i = O_i + P_ij * V_j        │
  │    5. Discard S_ij, P_ij (keep in SRAM)     │
  │    6. Write O_i back to HBM (if needed)     │
  └─────────────────────────────────────────────┘

Output: O (Attention Result)
```

### 伪代码详解

```python
def flash_attention(q, k, v, block_size):
    # q, k, v shape: [batch_size, seq_len, head_dim]
    o = torch.zeros_like(q)
    # l: 正则化因子 sum(exp(x - m)), m: 当前最大值
    l = torch.zeros(q.shape[:-1], dtype=torch.float32) 
    m = torch.full(q.shape[:-1], -float('inf'), dtype=torch.float32)
    
    # 分块循环
    for i in range(0, seq_len, block_size):
        q_block = q[:, i:i+block_size, :]
        o_block = torch.zeros_like(q_block)
        l_block = torch.zeros(q_block.shape[:-1], dtype=torch.float32)
        m_block = torch.full(q_block.shape[:-1], -float('inf'), dtype=torch.float32)
        
        for j in range(0, seq_len, block_size):
            k_block = k[:, j:j+block_size, :]
            v_block = v[:, j:j+block_size, :]
            
            # 1. 计算 QK^T
            s = q_block @ k_block.transpose(-2, -1)
            
            # 2. Online Softmax 更新逻辑
            # 计算当前块的 max
            m_new = torch.maximum(m_block, s.max(dim=-1).values)
            # 计算新的缩放因子 alpha (处理之前的项)
            alpha = torch.exp(m_block - m_new)
            # 计算新的归一化项 l
            l_new = alpha * l_block + torch.exp(s - m_new).sum(dim=-1)
            
            # 3. 更新输出 O
            # o_new = (o_old * alpha) + (softmax(s_new) * V)
            o_block = (o_block * alpha.unsqueeze(-1)) + 
                      (torch.exp(s - m_new.unsqueeze(-1)) @ v_block)
            
            # 更新状态变量
            l_block = l_new
            m_block = m_new
            
        # 4. 归一化最终输出
        o[:, i:i+block_size, :] = o_block / l_block.unsqueeze(-1)
        
    return o
```

### 常见考点
1.  **内存访问复杂度**：标准 Attention 的 HBM 访问量是 $O(N^2)$，而 FlashAttention 通过 Tiling 技术将其降低至 $O(N^2)$ 的计算量匹配的内存带宽，但实际上显著减少了读写次数（通常描述为 $O(N^2d)$ 内存读写降到 $O(Nd)$ 级别的常数优化，主要是尽量不写出中间矩阵 $S$ 和 $P$）。
2.  **Online Softmax**：面试官常问如何分段计算 Softmax。关键在于维护 `m` (max) 和 `l` (sum of exp)，利用指数性质 $e^{a+b} = e^a e^b$ 进行合并。
3.  **反悔机制**：在计算过程中，当遇到一个新的更大的 $m$ 时，之前累加的 $O$ 需要乘以衰减系数 $\alpha = e^{m_{old} - m_{new}}$ 来重新加权。
4.  **FlashAttention 2 vs 1**：V2 引入了更好的工作区划分和并行策略，减少了非矩阵乘法的操作，并更好地利用了 GPU 的线程组和共享内存。

# 对于大模型复杂推理任务,有哪些主流方法

复杂推理任务是大模型目前讨论最多的基础能力，试图通过增强 LLM 的复杂推理能力来缓解幻觉问题，同时赋予 LLM 更多的可能性。

## 一、 Prompt Engineering
**一句话总结**：仅仅通过 Prompt 最大化地激发出 LLM 的知识，从而获得更好的推理结果。

### 1. CoT (Chain-of-Thought)
**原理**：让 LLM 通过分步推理生成答案，模仿人类思考过程，将复杂问题拆解为多个中间步骤。
- **实现**：
  - Zero-shot CoT：在 Prompt 中加入“Let's think step by step”。
  - Few-shot CoT：给予多个示例，展示如何分步解决问题。
- **优缺点**：
  - 优点：降低单步错误率，推理过程透明，便于验证。
  - 缺点：增加推理开销（计算和时间），对无需推理的任务效率低，依赖示例质量。

### 2. ToT (Tree of Thoughts)
**原理**：将单一线性 CoT 扩展为树状结构。允许模型生成多个可能的推理路径，并进行评估、回溯或剪枝。
- **关键实现**：
  - **节点生成**：生成多个中间思路。
  - **评估与剪枝**：对节点打分（如正确性、可能性），淘汰低质量分支。
  - **搜索策略**：采用广度优先 (BFS) 或深度优先 (DFS) 探索路径。

### 3. SC (Self-Consistency)
**原理**：通过生成多条推理路径，并采用“多数投票”机制选择最一致的答案，提升鲁棒性。
- **实现细节**：
  - 调高 Temperature 以生成多样化的推理路径（通常 5~40 条）。
  - 对多条路径的最终答案进行统计（如投票），选择出现频率最高的结果。

### 4. PAL (Program-Aided Language Models)
**原理**：让 LLM 生成可执行的代码（如 Python）来解决推理问题，而非直接给出数值答案。
- **优势**：利用解释器保证计算的精确性，特别适合数学、逻辑和符号推理任务。

## 二、 Training
### 1. SFT (Supervised Fine-Tuning)
通过在特定的推理数据集上进行监督微调，让模型掌握推理模式。
- **常用数据集**：
  - GSM8K（中小学数学）
  - MATH（高难竞赛数学）
  - LogiQA（逻辑推理）
  - ARC（常识推理）

### 2. RLHF (Reinforcement Learning from Human Feedback)
- **流程**：
  1. 生成回答。
  2. 人类对回答排序。
  3. 训练 Reward Model (RM)。
  4. 使用 PPO 等算法优化策略模型，使其输出高 Reward 的回答。

### 3. Critic Models & 验证机制
除了主模型外，引入额外的评价模型（Critic）或验证机制来检查推理过程。

#### Process Reward Model (PRM)
- **目标**：对推理过程的每一步进行打分，评估逻辑连贯性和中间步骤的正确性。
- **作用**：不仅看结果对不对，更审查每一步思路是否合理。

#### Output Reward Model (ORM)
- **目标**：仅对最终输出的结果进行打分或排序。
- **作用**：关注最终答案的正确性，相比 PRM 更为粗粒度。

*注：PRM 和 ORM 常结合外部符号推理器（如定理证明器）来进一步增强验证能力。*

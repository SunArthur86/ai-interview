91.3 Actor-Critic 方法
🥥这是将 Value-Based 和 Policy-Based 结合在一起的思路
- Actor部分负责学习出策略（Policy）
- Critic部分负责评价策略好坏（类似 Value 计算）
 这类算法兼具两者优点，近年来在深度强化学习场景中非常流行。
91.4 其他分类
🥥除了以上三种主流的算法思路，我们还会以是否需要环境模型（Model）进行分类：
- Model-based： 先对环境进行建模，学习出环境的动态方程或状态转移规律；此时就能
用动态规划（DP）或蒙特卡洛树搜索（MCTS）等方法来选取贪心策略。
- Model-free： 不对环境进行明确建模，直接通过 Value 或 Policy 函数的迭代来寻找最优
策略；经典的 Q-Learning 即是其中代表。
92. DeepSeekV3有啥技术特点？
92.1 MLA多层注意力架构 (Multi-Head Latent Attention, MLA)
🥥设计理念：MLA通过低秩压缩技术减少注意力机制中的键值缓存 (Key-Value Cache) 存储需
求。具体而言，它使用降维投影矩阵对键和值进行联合压缩，从而大幅减少推理阶段的KV缓
存大小，同时维持性能与标准多头注意力机制 (MHA) 相当。
优势：显著降低了推理过程中显存需求，特别是在处理长上下文时表现优异。

92.2 FP8混合精度训练框架
🥥技术细节： 
◦
通过引入FP8数据格式，将矩阵乘法 (GEMM) 的大部分计算迁移到FP8精度，从而提升
训练效率。
◦
关键模块（如嵌入层、输出头、归一化操作等）保留较高精度（BF16或FP32），以确
保数值稳定性。
创新点：提出了细粒度量化策略 (Fine-Grained Quantization)，通过1×128或128×128分块
降低量化误差，同时采用高精度累积方法进一步提高计算精度。
成果：相比BF16基准训练方法，FP8训练框架在性能波动小于0.25%的范围内实现了显著的
内存和时间优化。
92.3 DualPipe跨节点通信算法
🥥创新点：DualPipe通过计算与通信阶段的重叠安排，显著减少了通信开销和流水线气泡
（Pipeline Bubbles）。
技术实现： 
◦
重新排列正向与反向计算块内的注意力、全连接层等组件，并动态调整通信和计算所
占的GPU资源比例。
◦
使用高效的跨节点All-to-All通信内核，充分利用InfiniBand和NVLink带宽。
效果：即使在大规模模型训练中，DualPipe算法也实现了近乎零的All-to-All通信开销。
92.4 无辅助损失的负载均衡策略
🥥背景：在专家模型 (MoE) 中，负载不平衡会导致路由崩溃和计算效率低下。传统方法依赖辅
助损失进行负载均衡，但可能导致性能退化。
创新方案：DeepSeek采用无辅助损失的负载均衡策略，通过为每个专家引入动态调整的偏置
项来平衡负载，而不对模型性能造成负面影响。
补充措施：引入序列级的辅助损失以避免单一序列内的极端负载不平衡。
92.5 跨节点全对全通信内核
🥥优化细节： 
◦
在节点间的通信中，优先利用NVLink处理节点内通信，InfiniBand处理节点间通信。

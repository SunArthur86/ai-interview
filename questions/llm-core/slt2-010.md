# 大模型推理中的量化有哪些方案？

量化（Quantization）将FP16/BF16权重压缩到低精度（INT8/INT4），减少显存和加速推理。

主要方案：

1. GPTQ（后训练量化）：
- 基于二阶Hessian矩阵信息逐层量化
- INT4量化效果好，速度快
- 适合服务端部署

2. AWQ（Activation-aware Weight Quantization）：
- 保护对激活值贡献大的权重通道
- 引入缩放因子s，使重要权重的量化误差最小
- INT4量化效果优于GPTQ

3. BitsAndBytes（NF4/INT8）：
- QLoRA使用的NF4量化
- 适合训练场景（QLoRA微调）

4. GGUF/GGML（llama.cpp）：
- 面向CPU推理的量化方案
- 支持Q4_K_M、Q5_K_M等多种精度

效果：INT4量化使13B模型从26GB降到约7GB，RTX 4090可跑。精度损失通常<1%。

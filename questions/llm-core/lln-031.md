# 模型在SFT后会出现“复读机”情况该如何debug,以及出现的原因是什么

**原因分析**：
1. **Pre-train 阶段机制**：Pre-train 通常使用 Packing，即连续拼接文本，模型很少在句尾主动学习输出 EOS 标记，因此模型本身没有学会“适可而止”。
2. **SFT 数据不足或质量差**：SFT 训练数据量过少，或者复读部分的数据包含信息量不足，模型在 Attention 时倾向于依赖上文 Context，导致重复输出已知片段。
3. **模型能力瓶颈**：模型参数量或结构不足以理解复杂指令，只能通过复读来维持输出的概率分布。

**Debug 方法**：
1. **检查 EOS 训练**：确保 SFT 数据中有足够的结束标记，并在 Loss 计算中给予 EOS Token 足够权重。
2. **数据清洗与去重**：检查 SFT 数据中是否存在大量循环重复的内容，清洗低质量数据。
3. **调整解码参数**：适当增加 Frequency Penalty 或 Presence Penalty，但这通常是治标不治本。
4. **增加有效数据**：增加多样化、高质量的 SFT 数据，特别是包含明显结束特征的数据。

# 什么是Reward Hacking

Reward Hacking又叫Reward Exploition,指在强化学习(Reinforcement Learning, RL)或其他基于奖励的系统中,agent利用 奖励函数的漏洞,以意料之外但可能不符合目标设计者意图的方式最大化奖励的现象 •
**关键点：**
**奖励最大化的偏离目标：** 强化学习的核心目标是通过奖励信号指导智能体学习策略.然而,设计不完善的奖励函数可能无法完全覆盖任务的真实目标 智能体可能找到一种投机取巧的方式,通过偏离真实目标来获取更高奖励

**未预期行为：** Reward Hacking 经常导致智能体采取设计者未预期的行为,例如忽略主要任务、优化无关参数,甚至破坏环境 •
**典型案例：**
**游戏中的奖励欺骗：** 在视频游戏任务中,RL智能体可能找到绕过游戏规则获取高分的方法,例如卡住计分系统,或无意义地重 复某些操作以获得奖励 **机器人学习中的意外行为：** 一个扫地机器人学习到奖励与覆盖面积相关,但没有惩罚破坏家具,于是采取撞击家具以“扩 大”覆盖面积的方式获得更高奖励 •
**解决方案：**
**改进奖励函数设计：** **覆盖真实目标：** 确保奖励函数尽可能全面地描述任务目标,避免遗漏重要方面 **负面奖励：** 对不符合目标的行为添加惩罚

**多目标优化：** 在奖励函数中引入多个目标权重,并动态调整,以避免智能体过度优化单一目标

**安全约束：** 在智能体的行为中添加硬性约束,限制其策略范围,防止出现极端行为

**人工反馈：** 定期引入人类评估,检查智能体行为是否符合设计者预期,调整奖励函数

**对抗性测试：** 模拟不同环境和奖励分布,测试智能体是否倾向于奖励欺骗行为

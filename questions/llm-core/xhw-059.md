# 不同Attention之间的区别

与 Dropout 的“结合”或“借鉴”？
目前也有一些工作会尝试随机稀疏注意力（即训练时随机选择一部分注意力连接），在一定
程度上与 Dropout 有“相似”的随机化思路： 既可降低计算量，又能起到正则化作用。
不过，主流的稀疏注意力更多是固定的局部模式（如 sliding window）或可学习模式，而非
像 Dropout 那样在训练时频繁随机变动。
🍅小结与展望
与 Dropout 的相似之处： 都是从“全连接”出发，引入了某种“稀疏/丢弃”手段，以期提
高训练效率和模型性能。
与 Dropout 的不同点： 动机和实现方式不同： 
◦
Dropout 主要在随机失活连接/神经元，防止过拟合；
◦
稀疏注意力主要在固定或可学习的稀疏结构上进行注意力计算，降低长序列的计算复
杂度。
未来在大模型中的趋势： 随着上下文长度需求和多模态数据的增长，稀疏注意力必然会成为
处理长序列的常见工具，并且会有更多变体（如分块注意力、混合局部+全局 Token、随机稀
疏策略等）出现。
因此，也许稀疏注意力对于大模型的长序列处理将会像当年 Dropout 之于深度学习的普及一
样，成为一种非常常见且关键的技术手段。同时，在具体实现层面，它与 Dropout 又有着不
小的差异化侧重点——但最终都服务于让模型在资源可控的前提下实现更优的性能与更强的泛
化能力。
Child et al. (2019) [Generating Long Sequences with Sparse Transformers] 提出多种稀疏模式；
Beltagy et al. (2020) [Longformer] 采用滑动窗口 + 全局 Token；
Zaheer et al. (2020) [Big Bird] 综合了随机、滑动窗口、全局 Token 方案。
88. 不同Attention之间的区别是什么？
88.1 各类 Attention 的关系与比较
🥥Self-Attention vs Cross-Attention
◦
Self-Attention：同一序列内各位置彼此建模
◦
Cross-Attention：不同序列之间信息交互
◦
都是最基础的关注“相关性”的手段
Sparse / Linear Attention vs 普通 Attention

◦
目的相同：处理超长序列时减少 O(n2)O(n^2)O(n2) 负担
◦
Sparse：通过设计“稀疏模式”跳过大部分计算
◦
Linear：通过核技巧或矩阵分解把二次复杂度转为线性
Multi-head Attention vs Multi-query / Grouped-query Attention
◦
Multi-head：每个 head 都有独立 Q/K/V，表达力强，但开销大
◦
Multi-query：只留一套（或少数）K/V，大幅降低显存占用
◦
Grouped-query：分组共享 K/V，兼顾一定的表达力与效率
Multi-head Latent Attention
◦
在多头的基础上引入“隐变量/全局潜在表示”
◦
进一步增强模型的全局表达能力

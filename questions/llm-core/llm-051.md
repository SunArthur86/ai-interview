# 什么是MOE(混合专家模型)

📚MoE,全称为Mixture of Experts(混合
专家模型),是一种大模型架构,其核 心设计思想是“术业有专攻”,即将任 务分门别类,分配给多个“专家”来解 决.与MoE相对应的是稠密(Dense) 模型,后者是一个“通才”模型,能够 处理多种任务,而MoE模型则专注 于“分而治之”,让每个专家模型解决 它最擅长的部分.
⬅️左侧图为传统大模型架构,右图为MoE大模型
架构. 两图对比可以看到,与传统大模型架构相比,
MoE架构在数据流转过程中集成了一个专家网络
层(红框部分).
1. MoE的核心架构
MoE的核心设计在于在数据流转过程中加入了一个专家网络层,这一层由门控网络(Gating
Network)和一组专家模型(Experts)构成.
**整个工作流程如下：**
1. 数据分割:数据首先会被分割为多个Token.
2. 门控网络:当每组数据进入专家网络层时,首
先会经过门控网络.
3. 专家选择:门控网络根据输入数据的特征,将
数据分配给一个或多个专家模型.每个专家模 型专注于处理一部分数据,“让专业的人做专 业的事”.
4. 输出融合:所有专家的输出结果会汇总,系统
进行加权融合,得到最终的输出.
这种“术业有专攻”的设计能够在多个领域内提升模型的效率和性能.

4. MoE的优势
🎁
**更高的吞吐量：** 由于激活单元较少,MoE模型在处理大量并发查询时表现出更高的吞吐
量.

**较低的时延：** 只需加载部分激活的专家模型,内存访问时间减少,从而降低了时延.
**计算更高效：** 对于单次查询,MoE模型需要从内存中读取的参数更少,因此在计算效率上
优于Dense模型.
**性能优越：** MoE模型能够在保持小型模型计算效率的同时,提供与大型Dense模型相近的性
能.例如,Mistral MoE模型展示了如何在降低成本的同时提供相似性能.
**灵活性：** 可以将训练好的大型Dense模型转换为MoE模型,使其具有小型模型的高效计算和
大型Dense模型的强大性能.
5. MoE的挑战
显存需求
由于MoE需要将所有专家模型加载到内存中,对
显存的压力较大,且涉及复杂的算法和高昂的通
信成本,这在资源受限设备上难以实现. 训练稳定性与泛化性
随着模型规模的扩大,MoE面临着训练不稳定
性、过拟合、以及如何确保模型泛化性和鲁棒性
等问题.这些问题需要开发者不断优化和提升.
6. MoE化
将Dense模型转换为MoE模型的过程称为MoE化(MoEfication).其主要目的是保留Dense模型的强
大性能,同时通过MoE的高效结构实现更高的计算效率.MoE化的关键步骤如下:
✏MoE模型通过将复杂问题分解为多个子问题,并利用专家网络来分别处理这些子问题,从而
在提高计算效率的同时保持性能优越性.这使得MoE成为了近年来大模型架构中的热门方
向,特别是在应对计算资源有限的场景时表现出色.

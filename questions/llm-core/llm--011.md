# 什么是具身智能(Embodied AI)?LLM 如何赋能机器人

- **具身智能**: AI 拥有'身体'(机器人/虚拟形象)，通过感知-决策-执行的闭环与物理世界交互，强调智能体与环境的交互能力。

- **LLM 在具身智能中的角色**

1. **高级任务规划**
- 用户: '帮我泡杯咖啡'
- LLM 拆解: 1.走到厨房 → 2.找到杯子 → 3.操作咖啡机 → 4.倒水 → 5.端回来
- 关键：利用常识将抽象自然语言转化为可执行的子任务链。

2. **代码生成控制**
- LLM 直接生成机器人控制代码(Python/C++)，API 驱动底层控制器。
- 如 Google RT-2: 视觉 → LLM → 动作 token。LLM 被当作策略网络使用。

3. **常识推理与纠错**
- '杯子掉在地上会碎' → 规划时避免。
- '水太烫要等一下' → 执行时调整。
- 当动作执行失败（如机械臂抓空），LLM 根据视觉反馈重新规划路径。

- **核心挑战**
1. **感知-动作鸿沟**: 文本/图像是离散的，动作是连续的(关节角度、力大小)。需要将 LLM 的离散输出映射到连续控制空间。
2. **实时性**: 物理世界是动态的，不能等 LLM 推理 5 秒，通常要求低延迟控制。
3. **安全约束**: 错误动作可能造成物理损坏或伤害人，需要添加安全层约束。
4. **数据稀缺**: 真实机器人交互数据采集成本极高，目前主要依赖仿真数据和互联网视频数据。

- **代表项目**
- **RT-2 (Google)**: Vision-Language-Action (VLA) 模型，直接将图像映射为动作 Token。
- **NVIDIA GR00T**: 人形机器人的通用基础模型，Project GR00T。
- **Figure 01/02**: 结合 OpenAI 大模型，能够进行对话和复杂操作。
- **Tesla Optimus**: 端到端神经网络控制，输入视频直接输出关节控制信号。
- **VoxPoser**: 利用 LLM 解析环境属性，生成运动轨迹代码。

- **趋势**: 从'仿真训练+真实部署'到'真实数据+大规模训练'。利用 Video-Gen 生成合成数据，或通过遥操作大规模收集真实数据。

## 常见考点
1. **Sim2Real (仿真到现实) 的难点是什么？**（域差距 Domain Gap，仿真器的物理参数与真实世界无法完全一致）
2. **什么是 VLA 模型？**（Vision-Language-Action models，将动作也视为一种语言 Token，统一了感知和控制的输出空间）
3. **如何解决 LLM 输出离散动作而机器人需要连续控制的问题？**（可以通过输出离散的 API 调用，或者输出向量参数用于回归连续值）

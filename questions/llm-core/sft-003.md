# GRPO(Group Relative Policy Optimization)是什么？DeepSeek为什么用它？

🎯 **本质**：GRPO是PPO的简化变体，去掉了Critic(价值网络)，用组内相对奖励代替基线，大幅降低了训练成本和显存需求。

📊 **PPO的问题**：
PPO需要4个模型：
1. Policy Model（策略模型）- 正在训练的模型
2. Reference Model（参考模型）- KL惩罚参考
3. Reward Model（奖励模型）- 评估输出质量
4. Value/Critic Model（价值模型）- 估计基线

4个模型同时加载到显存 -> 需要大量GPU

**GRPO的创新**：
去掉Critic Model，改用组内相对比较

**GRPO流程**：
1. 对每个prompt，生成G个不同的回答（一组）
2. 用Reward Model对每个回答打分
3. 组内归一化：advantage_i = (r_i - mean(r)) / std(r)
4. 用归一化的advantage更新策略

**数学对比**：
PPO: advantage = reward - V(s)  // V是Critic的估计
GRPO: advantage = (reward - group_mean) / group_std

**优势**：
1. 少一个模型 -> 显存减少约25%
2. 不需要训练Critic -> 训练更简单
3. 组内比较 -> 天然的对比学习效果
4. 更稳定 -> 归一化减少方差

**DeepSeek的GRPO应用**：
DeepSeek-R1完全用GRPO训练推理能力
不使用SFT（直接从Base模型开始GRPO）
让模型自然涌现CoT（思维链）能力
实验显示：模型自发产生了"aha moment"（自我反思）

**为什么GRPO适合推理训练**：
1. 推理问题有明确答案（数学、代码）-> 可以用规则判断对错
2. 不需要人类标注偏好 -> 用程序生成奖励
3. 组内对比 -> 让模型学习"好的推理 vs 差的推理"
4. 生成多个回答 -> 自然支持Best-of-N采样

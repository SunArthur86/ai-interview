# GRPO(Group Relative Policy Optimization)是什么？DeepSeek为什么用它？

🎯 **本质**：GRPO是PPO的简化变体，去掉了Critic(价值网络)，用组内相对奖励代替基线，大幅降低了训练成本和显存需求。

📊 **PPO的问题**：
PPO需要4个模型，显存占用巨大：
1. Policy Model（策略模型）- 正在训练的模型
2. Reference Model（参考模型）- KL惩罚参考
3. Reward Model（奖励模型）- 评估输出质量
4. Value/Critic Model（价值模型）- 估计基线

**GRPO的创新**：
去掉Critic Model，改用组内相对比较。

**GRPO流程**：
1. **采样**：对每个prompt，生成G个不同的回答（一组，通常 G=4~8）。
2. **打分**：用Reward Model或规则函数（如代码编译通过、数学答案正确）对每个回答打分。
3. **计算优势**：组内归一化。
   - `mean(r)`：组内平均分
   - `std(r)`：组内标准差
   - `advantage_i = (r_i - mean(r)) / std(r)`
4. **更新**：用归一化的advantage更新策略。

**数学对比**：
- **PPO**: `advantage = reward - V(s)` (依赖Critic估计值)
- **GRPO**: `advantage = (reward - group_mean) / group_std` (无Critic，组内互为基准)

**核心优势**：
1. **显存优化**：少一个Value模型，且Value模型通常也是大参数量，显存减少约25%-40%。
2. **训练简单**：不需要训练Critic，消除了Critic训练收敛慢的难点。
3. **对比学习**：组内比较天然形成了“好回答 vs 坏回答”的对比信号。
4. **稳定性**：通过组内归一化，降低了不同Prompt之间Reward量纲不同带来的方差。

**DeepSeek的GRPO应用**：
DeepSeek-R1完全用GRPO训练推理能力，这被称为“直接偏好优化”的一种变体。
- **不使用SFT**：直接从Base模型开始GRPO（或极少的SFT）。
- **涌现CoT**：让模型自然涌现CoT（思维链）能力，而非强制模仿格式。
- **Aha Moment**：实验显示模型在训练中后期会突然学会自我反思和验证。

**为什么GRPO适合推理训练**：
1. **奖励易得**：推理问题（数学、代码）有明确答案，可以用低成本规则判断对错（0/1奖励），无需昂贵的Reward Model。
2. **无需人工**：完全基于过程奖励或结果奖励，不需要人类标注偏好。
3. **探索性强**：生成多个回答 -> 自然支持Best-of-N采样，鼓励模型尝试不同路径。
4. **消除基线偏差**：数学题难度差异大，用组内均值做基准比固定的Critic更鲁棒。

## 常见考点
1. **GRPO 与 PPO 的核心区别**：去掉了Critic模型，利用组内样本计算均值和方差作为基准，大幅降低显存。
2. **为什么组内归一化能稳定训练？**：因为不同Prompt的Reward分布差异很大（如有的题简单分高，有的题难分低），直接用Reward做Advantage方差大，归一化后稳定了梯度更新。
3. **DeepSeek-R1 的训练策略**：提到了“冷启动”和“纯RL”的区别，问是否会先SFT再GRPO（实际上是极少量CoT数据SFT后大量GRPO）。

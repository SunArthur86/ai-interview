# 为什么用LayerNorm不用BatchNorm

在 Transformer 中使用 LayerNorm (LN) 而非 BatchNorm (BN)，主要是由 NLP 任务的特性和数据结构决定的。

**1. 数据维度与批次依赖**
*   **BN 的局限性**：BN 是对同一个 batch 中不同样本的**同一维度**特征进行归一化。在 NLP 中，输入通常是 `(batch, seq_len, embedding)`。由于序列长度往往不一致（需要 padding），或者 batch size 较小（如 1 或 2），BN 计算出的统计量（均值和方差）会非常不稳定，且受 padding 干扰严重。此外，推理时 BN 需要依赖训练时的全局统计量，增加了复杂度。
*   **LN 的优势**：LN 是对**单个样本内部**的所有特征进行归一化。它不依赖于 batch size，也不受序列长度变化的影响，计算出的均值和方差始终是针对当前句子的特征分布。

**2. 语义一致性**
*   在文本处理中，同一个句子在不同位置的特征分布应当具有一致性。LN 能够保持单个样本内部的特征分布稳定，更符合 NLP 任务对文本特征独立性的要求。

**3. 训练稳定性**
Transformer 结构深层且参数量大，使用 LayerNorm 能够在隐藏层维度上稳定数据的分布，有效缓解梯度问题，且无需像 BN 那样维护额外的 Running Mean/Var。

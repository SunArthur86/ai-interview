# 在大模型强化学习对齐中,如何解决信用分配(Credit Assignment)问题?token 级别和序列级别的奖励有什么不同

- **信用分配问题**: 给定一个序列级奖励(如'这个回答好/差')，如何确定序列中哪些 token 贡献了正向/负向价值?

- **序列级奖励**
  - 整个回答获得一个标量 reward
  - 优点: 标注成本低(人类只需整体评分)
  - 缺点: 无法区分序列中哪部分好哪部分差

- **Token 级奖励**
  - 序列中每个 token 获得独立 reward
  - 优点: 精确指导哪些 token 该加强/减弱
  - 缺点: 标注成本极高，通常无法直接获得

- **主流信用分配方法**

1. **GAE (Generalized Advantage Estimation)**
   - PPO 标准方法，用 value function 估计每个 token 的优势
   - 需要 value model(额外参数和计算)

2. **GRPO 的组内相对优势**
   - 同一 prompt 采样 G 个回答，用组内排名作为优势
   - 不需要 value model，但组大小影响估计质量

3. **过程奖励模型 (PRM, Process Reward Model)**
   - 对推理过程的每一步打分(如 OpenAI 的 Math PRM)
   - 精确但需要大量步骤级标注数据

4. **隐式 token 归因**
   - 用梯度或注意力权重推断哪些 token 贡献了奖励
   - 如 Integrated Gradients、Attention rollout

- **实践建议**: 推理任务(数学/代码)适合 PRM，对话任务适合 GAE/GRPO。

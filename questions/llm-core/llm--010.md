# 在大模型强化学习对齐中,如何解决信用分配(Credit Assignment)问题?token 级别和序列级别的奖励有什么不同

- **信用分配问题**: 给定一个序列级奖励(如'这个回答好/差')，如何确定序列中哪些 token 贡献了正向/负向价值? 这是 RLHF 中的核心难题。

- **序列级奖励**
- 整个回答获得一个标量 reward。
- 优点: 标注成本低(人类只需整体评分)。
- 缺点: 无法区分序列中哪部分好哪部分差，梯度更新可能波及到无辜的 token。

- **Token 级奖励**
- 序列中每个 token 获得独立 reward。
- 优点: 精确指导哪些 token 该加强/减弱。
- 缺点: 标注成本极高，通常无法直接获得。

- **主流信用分配方法**

1. **GAE (Generalized Advantage Estimation)**
- PPO 标准方法，用 Value Function 估计每个 token 的价值。
- 公式：$A_t = \sum_{l=0}^{\infty}(\gamma \lambda)^l \delta_{t+l}$。
- 需要 value model(额外参数和计算)，通过偏差和方差的平衡来估计优势。

2. **GRPO 的组内相对优势**
- 同一 prompt 采样 G 个回答，用组内排名作为优势 (e.g., 第1名得正优势，最后一名得负优势)。
- 不需要 value model，简化了训练流程，但组大小 G 影响估计质量。

3. **过程奖励模型 (PRM, Process Reward Model)**
- 对推理过程的每一步打分(如 OpenAI 的 Math PRM)。
- 精确但需要大量步骤级标注数据，常用于数学推理（如 GSM8K）。

4. **隐式 token 归因**
- 用梯度或注意力权重推断哪些 token 贡献了奖励。
- 如 Integrated Gradients、Attention rollout，更多用于可解释性分析。

- **实践建议**: 推理任务(数学/代码)适合 PRM（因为中间步骤出错会导致最终错误，需要纠正中间步骤），对话任务适合 GAE/GRPO（关注整体流畅性和安全性）。

## 常见考点
1. **GAE 中的 $\lambda$ 参数起什么作用？**（控制偏差和方差的权衡，$\lambda=0$ 时是 MC 高方差无偏差，$\lambda=1$ 时是 TD 低方差有偏差）
2. **为什么 GRPO 被称为“无 Value Model”的 PPO？**（它利用 group 内相对排序代替了 Value Function 绝对值的估算）
3. **PRM 和 ORM (Outcome Reward Model) 的区别？**（ORM 只对最终结果打分，PRM 对中间步骤打分，PRM 提供了更细粒度的监督信号）

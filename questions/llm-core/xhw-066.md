为何「重复读取 14 GB」的说法是近似的
在实际推理框架中，模型在做推理 Forward 时，并不一定是「每产生一个 token，就把所有
层权重完整从 0 开始加载」：
- 通常会将权重已经加载在 GPU 显存里（只要 GPU 显存足够），或者分块、分层缓存。
- 当前向计算经过每一层时，会依次访问那一层的参数，然后配合部分激活值/中间结果一
起做计算。
- 但是，从带宽的角度来看，你还是需要从 GPU 显存中把那层的参数搬运到计算单元（寄
存器/L1/L2）进行运算，总体数据搬运量并不会减少多少。
所以，「每生成一个 token，需要读取 14 GB 数据」更多是一个「总数据搬运量 / 带宽」的
思路，方便做理论的上限估算。真实过程是分层分步完成，但带宽瓶颈依然存在。

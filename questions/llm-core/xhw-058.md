为什么称作“Cross-Attention”？
Self-Attention（自注意力）是 同一个序列 内部的各个位置之间相互计算注意力；
Cross-Attention 则是 目标序列（Decoder 侧） 和 源序列（Encoder 侧） 之间的注意力交
互。
应用
- 机器翻译：Decoder 根据源语言（Encoder 输出）生成目标语言。
- 多模态任务（Vision+Language、Speech+Text 等）：图像特征和文本特征通过 CrossAttention 交互。
在机器翻译中，最开始的“attention”就是源序列和目标序列间的注意力，实际上就属于 CrossAttention 的范畴。
在 Transformer 中，Encoder-Decoder 结构里，Decoder 部分显式地包含一个 Cross-Attention 模
块，用来获取源序列信息。

“稀疏性”对比
Dropout 的稀疏性
- 训练阶段随机“断开”部分神经元（或隐藏向量维度）连接，每次迭代随机生成一
个“mask”，被遮蔽的节点不参与当前批次前向传播和反向传播。
- 训练完后，推理阶段往往关闭 Dropout（或按一定缩放比例），因为模型此时不再需要随
机失活，需使用所有连接来最大化性能。

可以看到，稀疏注意力的“稀疏性”更强调对序列的局部/结构化稀疏模式，并不是像 
Dropout 那样“随机丢弃”部分连接，而是在训练和推理时都大概率保持一个固定或可学
习的稀疏结构。
共同趋势：从“全连接”到“更高效、更具鲁棒性”的模式
虽然 Dropout 和 Sparse Attention 的直接动机不同，但两者确实都体现了“从全连接”到某
种形式的“稀疏或部分连接”的思路，核心原因是：
1. 全连接往往带来非常庞大的计算量或过拟合风险。
2. 通过“稀疏或部分连接”，可以在效率/泛化能力上获得显著收益。
在这一层面上，两者确实有相似之处：
- 都是在某种程度上减少“连接”或“相互计算”的数量。
- 都是为了在实践中得到更好的效果（要么提升训练效率、要么减少过拟合、要么让长序列
可行……）。
87.3 稀疏注意力在大模型中的趋势
🥥
为何稀疏注意力越来越重要？
大语言模型（LLM）中，序列上下文长度不断增长（从几千 Token 到上万 Token），标准的 

 自注意力消耗极大。
O(n )
稀疏注意力、局部注意力、滑动窗口等技术可以有效地将时间/空间复杂度降低，从而让模型
在长序列推理时依旧保持较好的性能。
比如：Longformer、BigBird、Sparse Transformer 等已经在大模型的超长文本场景中取得
较好效果。
是否会在大模型中成为常见形式？
随着对长上下文的需求不断上涨，稀疏注意力（以及其它降复杂度注意力，如線性注意力、
分块注意力等）势必会越来越普及。
不仅仅是文本，在处理多模态数据（视频长帧序列、音频长序列、基因序列）时，也需要类
似的稀疏或高效注意力机制来应对超长输入。
未来，稀疏注意力 + 动态/可学习稀疏模式，也有可能成为新的研究与应用热点。

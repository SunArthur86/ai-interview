# PPO在RLHF中的工作流程是什么？

RLHF中使用PPO的四个模型：

1. Actor（Policy Model）：待优化的模型，生成回答
2. Reference Model：冻结的SFT模型，用于计算KL散度惩罚（防止Actor偏离太远）
3. Reward Model：根据prompt+response打分
4. Critic Model：估计状态价值（Value function）

训练流程：
1. Actor生成response
2. Reward Model给出reward score
3. Critic估计value，计算Advantage = Reward - Value
4. PPO Clip目标：clip(ratio, 1-ε, 1+ε) × Advantage
5. KL惩罚：-β × KL(Actor || Reference)，防止模式崩溃

PPO的问题：
- 4个模型同时加载，显存消耗大
- 训练不稳定，超参数敏感
- Reward Model可能被利用（reward hacking）

替代方案：DPO（省去Reward Model和Critic）、GRPO（DeepSeek使用，省去Critic）。

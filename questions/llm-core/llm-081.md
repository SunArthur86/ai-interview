87. 2 稀疏注意力与全连接→Dropout 的思考
🥥 背景对比 全连接层到 Dropout: 在传统神经网络(尤其是早期全连接层)中,所有神经元之间都是完全连接,因此参 数量非常大. 引入 Dropout 的初衷是:对连接或神经元进行随机“断开/关闭”,让网络在训练中 保持一定的“稀疏性”,从而减少过拟合、提高泛化能力.这种做法在实践中能显著 提升模型鲁棒性.
**自注意力到稀疏注意力：** 在 Transformer 等模型中,标准自注意力是对所有 的位置两两计算,当序列 长度 很大时,计算和显存开销都呈 级别. (n × n) n O(n ) 2 稀疏注意力(Sparse Attention) 则通过“只在部分位置上计算注意力”来降低计算 复杂度,例如局部块、滑动窗口等方式,将计算复杂度降到 或 . 它主要为了解决长序列的效率问题. O(n) O(n log n) 从这一点看,二者虽然都引入了“稀疏性”,但动机略有不同: • **Dropout：** 旨在防止过拟合、增强模型的泛化能力; • Sparse Attention:旨在减少计算量和内存占用,使长序列处理可行. “稀疏性”对比 Dropout 的稀疏性 • 训练阶段随机“断开”部分神经元(或隐藏向量维度)连接,每次迭代随机生成一 个“mask”,被遮蔽的节点不参与当前批次前向传播和反向传播. • 训练完后,推理阶段往往关闭 Dropout(或按一定缩放比例),因为模型此时不再需要随 机失活,需使用所有连接来最大化性能. Sparse Attention 的稀疏性 • 把原本完整的 注意力矩阵中“大部分元素”屏蔽为 (或 0),只保留特定 位置(邻近块、滑动窗口等). (n × n) −∞ • 这不是随机的,而是固定的或可学习的稀疏模式: 固定模式(如 sliding window、dilated window 等):在训练和推理时都一致地保留 同样的稀疏结构. 可学习模式(需要额外参数或方法来学习稀疏性),也会在训练和推理阶段保持一致 的结构或经过一定规则来确定关注位置. • 目标是降低计算量和显存压力,在很长的序列(如万级 Token)上也能让模型运行起来. 可以看到,稀疏注意力的“稀疏性”更强调对序列的局部/结构化稀疏模式,并不是像 Dropout 那样“随机丢弃”部分连接,而是在训练和推理时都大概率保持一个固定或可学 习的稀疏结构. **共同趋势：** 从“全连接”到“更高效、更具鲁棒性”的模式 虽然 Dropout 和 Sparse Attention 的直接动机不同,但两者确实都体现了“从全连接”到某 种形式的“稀疏或部分连接”的思路,核心原因是:
1. 全连接往往带来非常庞大的计算量或过拟合风险.
2. 通过“稀疏或部分连接”,可以在效率/泛化能力上获得显著收益.
在这一层面上,两者确实有相似之处: • 都是在某种程度上减少“连接”或“相互计算”的数量. • 都是为了在实践中得到更好的效果(要么提升训练效率、要么减少过拟合、要么让长序列 可行......).
87. 3 稀疏注意力在大模型中的趋势
🥥 为何稀疏注意力越来越重要? 大语言模型(LLM)中,序列上下文长度不断增长(从几千 Token 到上万 Token),标准的

自注意力消耗极大. O(n ) 2 稀疏注意力、局部注意力、滑动窗口等技术可以有效地将时间/空间复杂度降低,从而让模型 在长序列推理时依旧保持较好的性能. **比如：** Longformer、BigBird、Sparse Transformer 等已经在大模型的超长文本场景中取得 较好效果. 是否会在大模型中成为常见形式? 随着对长上下文的需求不断上涨,稀疏注意力(以及其它降复杂度注意力,如線性注意力、 分块注意力等)势必会越来越普及. 不仅仅是文本,在处理多模态数据(视频长帧序列、音频长序列、基因序列)时,也需要类 似的稀疏或高效注意力机制来应对超长输入. 未来,稀疏注意力 + 动态/可学习稀疏模式,也有可能成为新的研究与应用热点.

与 Dropout 的“结合”或“借鉴”? 目前也有一些工作会尝试随机稀疏注意力(即训练时随机选择一部分注意力连接),在一定 程度上与 Dropout 有“相似”的随机化思路: 既可降低计算量,又能起到正则化作用. 不过,主流的稀疏注意力更多是固定的局部模式(如 sliding window)或可学习模式,而非 像 Dropout 那样在训练时频繁随机变动. 🍅小结与展望 与 Dropout 的相似之处: 都是从“全连接”出发,引入了某种“稀疏/丢弃”手段,以期提 高训练效率和模型性能. 与 Dropout 的不同点: 动机和实现方式不同: Dropout 主要在随机失活连接/神经元,防止过拟合; 稀疏注意力主要在固定或可学习的稀疏结构上进行注意力计算,降低长序列的计算复 杂度. **未来在大模型中的趋势：** 随着上下文长度需求和多模态数据的增长,稀疏注意力必然会成为 处理长序列的常见工具,并且会有更多变体(如分块注意力、混合局部+全局 Token、随机稀 疏策略等)出现. 因此,也许稀疏注意力对于大模型的长序列处理将会像当年 Dropout 之于深度学习的普及一 样,成为一种非常常见且关键的技术手段.同时,在具体实现层面,它与 Dropout 又有着不 小的差异化侧重点--但最终都服务于让模型在资源可控的前提下实现更优的性能与更强的泛 化能力. Child et al. (2019) [Generating Long Sequences with Sparse Transformers] 提出多种稀疏模式; Beltagy et al. (2020) [Longformer] 采用滑动窗口 + 全局 Token; Zaheer et al. (2020) [Big Bird] 综合了随机、滑动窗口、全局 Token 方案.

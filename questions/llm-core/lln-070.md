# 超长上下文(100万+ Token)的技术挑战和解决方案是什么

- **超长上下文五大挑战:**

- **1. 计算复杂度:** Self-Attention 是 O(n²),100 万 token 计算量爆炸.解决:Sparse Attention、Sliding Window Attention、FlashAttention.

- **2. KV Cache 显存:** 100 万 token 的 KV Cache 可达数十 GB.解决:GQA 减少 KV 头、量化 KV Cache(FP8/INT4).

- **3. 位置编码外推:** 训练时未见过的长度效果差.解决:YaRN、NTK-aware RoPE、长度外推训练.

- **4. 注意力稀释(Lost in the Middle):** 中间位置的信息容易被忽略.解决:训练时调整数据分布、Infini-attention.

- **5. 成本问题:** 100 万 token 的 API 调用成本很高.解决:Prompt Caching、上下文压缩.

**当前进展：** Gemini 1.5 Pro 支持 100 万-200 万 token,Claude 3.5 支持 200K,Llama 3.1 支持 128K.

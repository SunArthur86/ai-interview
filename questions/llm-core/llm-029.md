# 介绍一下Tokenizer怎么训练的

Tokenizer 的训练主要是指构建**词表** 的过程，最主流的方法是子词 分词算法，如 BPE、WordPiece 和 Unigram。

### 1. 为什么要训练 Tokenizer？
计算机无法直接处理文本，需将其转化为数字 ID。Tokenizer 的目标是：
*   **降低词表大小**：避免词级分词造成的词表过大和稀疏问题。
*   **处理未登录词 (OOV)**：通过子词组合表示生僻词，提高泛化能力。

### 2. 核心算法：BPE (Byte Pair Encoding)
BPE 是一种基于统计频率的贪心合并算法，也是 GPT、BERT 等模型的基础。

**训练步骤**：
1.  **准备语料**：准备大量文本，将单词拆分为字符序列（如 "apple" \rightarrow "a p p l e </w>")。
2.  **统计频率**：统计语料中所有相邻字符对（如 "p p"）的出现频率。
3.  **合并最高频对**：找到频率最高的字符对（例如 "e r"），将其合并为一个新的子词（"er"），加入词表。
4.  **更新语料**：用新子词替换语料中对应的字符对。
5.  **迭代**：重复步骤 2-4，直到词表大小达到预设值（如 50k）。

### 3. 其他常见算法
*   **WordPiece**：与 BPE 类似，但合并标准不是“频率最高”，而是“能使训练语料似然概率最大化”的合并对。常用于 BERT。
*   **Unigram Language Model**：初始化一个巨大的词表，然后通过不断计算每个子词的损失，逐步删除对整体语言模型概率贡献最小的子词，直到词表缩小到目标大小。常用于 T5、ALBERT。

### 4. 现代 Tokenizer 的特点
*   **BBPE (Byte-level BPE)**：为了避免生僻字符（如 Emoji、生僻汉字）导致词表爆炸，现代 LLM（如 LLaMA, GPT-4）通常在字节（Byte）级别进行 BPE 合并，支持任意 Unicode 字符。

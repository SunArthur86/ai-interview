# 解释一下P-tuning

### P-tuning (含 P-tuning v2)

**概念**：
P-tuning 是一种专注于优化连续提示的参数高效微调方法。它通过在输入文本前插入一段可训练的连续向量（即“软提示”），代替手工设计的离散文本提示，来提升模型在下游任务中的性能。

**来源**：
*   P-tuning: Liu et al. (2021), 《GPT Understands, Too》
*   P-tuning v2: Liu et al. (2021), 《P-tuning v2: Prompt tuning can be comparable to fine-tuning universally across scales and tasks》

**核心原理**：
1.  **连续提示**：将原本离散的自然语言提示词转换为连续的嵌入向量空间中的参数。这些向量在训练过程中通过反向传播自动优化，寻找最能激活模型能力的“提示”。
2.  **P-tuning v2 的改进**：原始 P-tuning 仅在输入层添加提示，在参数量较小或任务较难时表现不如全量微调。P-tuning v2 借鉴了 Prefix-tuning 的思想，在 Transformer 的**每一层**都添加了可训练的连续提示，并将其应用于重采样（Reprojection）机制，使其在各类规模模型和任务（包括 NLU 和 NLG）上都能媲美全量微调。

**技术细节**：
*   **提示嵌入**：在输入序列 $X$ 前拼接可训练向量 $P$，得到 $[P; X]$。
*   **MLP 提示编码器 (v1)**：原始 P-tuning 使用一个 LSTM 或 MLP 对提示向量进行编码，以增加提示的复杂度和表达能力（在 v2 中针对不同层有不同设计）。
*   **多层微调 (v2)**：在每一层的 Key、Value 矩阵计算前都加入可训练的 Prompt 向量。

**优势**：
*   **灵活性**：连续向量可以表示离散语言难以描述的复杂特征。
*   **通用性 (v2)**：解决了 Prompt Tuning 难以在小模型上表现良好的问题，适应各种规模模型。

**局限**：
*   占用额外的输入序列长度空间。
*   超参数（如提示长度）对性能有一定影响。

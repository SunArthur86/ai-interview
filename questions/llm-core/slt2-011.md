# 投机采样（Speculative Decoding）是什么？

投机采样用一个小模型（Draft Model）快速生成候选token，大模型（Target Model）并行验证。

流程：
1. 小模型自回归生成k个候选token（速度快）
2. 大模型一次性并行计算这k个token的logits（并行，只需一次forward）
3. 对比大小模型输出：
   - 如果小模型生成的token与大模型一致 → 接受
   - 如果不一致 → 从第一个不一致处拒绝，用大模型的分布重新采样

优势：
- 数学上等价于直接用大模型采样（无损）
- 小模型猜对的部分不需要大模型自回归计算
- 典型加速2-3倍

变体：
- Medusa：多个头同时预测多个未来token（不需要小模型）
- EAGLE：用LLM自身的隐状态预测
- Lookahead Decoding：基于Jacobi迭代并行解码

```text
投机采样流程图：

Prompt: [The capital of France is]

1. Draft Model (Small) 自回归生成 k=3 个候选:
   Input: [The capital of France is]
   Output Draft: [Paris][,][a] (逐步生成)

2. Target Model (Large) 并行验证:
   Input: [The capital of France is Paris, a]
   Forward Pass 一次计算所有位置的 Logits

3. 验证与采样:
   Pos 1: Draft [Paris] vs Target [Paris] -> 匹配 (Accept)
   Pos 2: Draft [,]    vs Target [,]    -> 匹配 (Accept)
   Pos 3: Draft [a]    vs Target [beautiful] -> 不匹配 (Reject)

4. 最终结果:
   Output: [The capital of France is Paris,] + Target采样出的[beautiful] ...
```

## 常见考点
- **接受率**：投机采样的加速比关键在于“接受率”。如果小模型与大模型差异过大，接受率低，反而会因为频繁的验证失败导致性能下降。因此 Draft Model 通常需要与 Target Model 对齐。
- **树注意力**：为了在验证阶段一次性处理 k 个候选 token，需要对大模型的 Attention Mask 进行特殊构造（Causal Mask + 候选序列 Mask），使其能并行计算前缀和所有候选分支的概率。
- **无偏性证明**：为什么说它是无损的？因为在拒绝发生时，我们使用的是 Target Model 在当前位置的真实分布进行重新采样，这在数学上保证了最终输出分布与 Target Model 自回归生成的分布完全一致。

# 什么是损失缩放(Loss Scaling)?为什么混合精度训练需要它

- **损失缩放** 是混合精度训练中解决 FP16 梯度下溢的关键技术。

- **问题背景**
  - FP16 的最小正常数约为 6e-5，很多梯度值小于此值会被截断为 0
  - 梯度为 0 → 参数不更新 → 模型停止学习

- **损失缩放原理**

  1. **前向传播**: 正常计算，得到 loss
  2. **损失放大**: loss × scale_factor (如 2^16 = 65536)
  3. **反向传播**: 放大后的 loss 产生放大的梯度，避免下溢
  4. **梯度缩小**: 更新前，梯度 ÷ scale_factor 恢复真实值

```python
# PyTorch AMP (自动混合精度)
scaler = torch.cuda.amp.GradScaler()

with torch.cuda.amp.autocast():
    output = model(inputs)
    loss = criterion(output, targets)

scaler.scale(loss).backward()  # 放大 loss
scaler.step(optimizer)          # 更新(自动缩小梯度)
scaler.update()                 # 动态调整 scale_factor
```

- **动态损失缩放**
  - 如果梯度出现 inf/nan → scale_factor 减半
  - 连续 N 步无 inf/nan → scale_factor 翻倍
  - 自动找到合适的 scale，无需手动调参

- **bfloat16 vs float16**
  - float16: 指数位 5bit，尾数位 10bit → 动态范围小，需要 loss scaling
  - bfloat16: 指数位 8bit，尾数位 7bit → 动态范围与 float32 相同，**不需要 loss scaling**
  - 但 bfloat16 精度略低，某些任务可能略差

- **经验**: 现代训练推荐用 bfloat16(如果 GPU 支持)，省去 loss scaling 的复杂性。

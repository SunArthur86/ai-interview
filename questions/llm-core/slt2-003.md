# MHA、MQA和GQA有什么区别？

Multi-Head Attention (MHA)：每个注意力头有独立的Q、K、V投影矩阵。标准Transformer使用。N个头意味着N份KV Cache，计算量大且显存带宽占用高。

Multi-Query Attention (MQA)：所有头共享同一组K和V投影矩阵（即K、V的头数为1），只有Q保留多头。极端省显存，大幅降低带宽压力，但可能损害模型表达能力。如PaLM、Falcon。

Grouped-Query Attention (GQA)：MHA和MQA的折中方案。将Q头分成G组，每组内的头共享一对K、V。当G=N时等价于MHA，当G=1时等价于MQA。如Llama 2 70B、Llama 3、Mixtral。

**架构对比图**：
```text
Input X
  │
  ├─────────────────────────────────────────────────────┐
  │                                                       │
  ▼ (MHA)                                                ▼ (MQA)
┌─────────────┐                                       ┌─────────────┐
│ Q1 K1 V1    │ (3 Projections)                       │ Q1 ... QN   │ (N Projs for Q)
│ Q2 K2 V2    │                                       │             │
│ ...         │                                       │ K1 V1       │ (Shared 1 Proj)
│ QN KN VN    │                                       │             │
└─────────────┘                                       └─────────────┘
  │                                                       │
  ▼                                                       ▼
N Independent Heads                                    N Heads share 1 KV Pair

  │ (GQA) ────────────────────────────────────────────────┘
  ▼
┌─────────────────────────────┐
│ Q1...Qk │ K1 V1 (Shared)     │
│ Qk+1...Qm │ K2 V2 (Shared)   │
│ ...      │ ...               │
└─────────────────────────────┘
  │
  ▼
G Groups (Each has 1 KV pair)
```

**KV Cache显存对比**（假设总头数为N，Hidden Size为d，GQA分组数为G）：
- MHA: KV Cache = 2 × N × d × seq_len（每头独立KV）
- MQA: KV Cache = 2 × 1 × d × seq_len（所有头共享1份KV，减少N倍）
- GQA: KV Cache = 2 × G × d × seq_len（减少N/G倍）

**性能与效果权衡**：
GQA在保持接近MHA质量的同时，大幅减少推理时的KV Cache内存占用。更重要的是，它显著降低了Decode阶段对显存带宽的需求，因为每次读取KV Cache的数据量减少了，从而成为吞吐量受限场景下的首选优化。

## 常见考点
1. **Up-training（从MHA转GQA）**：如果在预训练时使用了MHA，如何在推理或微低成本时转换为GQA？（需进行少量的额外训练步数来对齐KV投影，如Llama 2/3的做法）
2. **带宽 vs 算力瓶颈**：MQA/GQA主要解决的是算力问题还是带宽问题？（主要是带宽瓶颈，特别是Decode阶段）
3. **Kernel实现**：MHA/GQA/MQA在FlashAttention等Kernel实现上的计算密度差异是什么？

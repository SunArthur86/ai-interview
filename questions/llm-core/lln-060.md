# 大模型幻觉(Hallucination)的分类、成因和缓解方法有哪些

幻觉是指LLM生成看似合理但事实上不正确的内容.

- **幻觉分类:**

| 类型 | 定义 | 示例 |
|------|------|------|
| **事实性幻觉** | 生成与客观事实不符的内容 | 编造不存在的论文/人物 |
| **忠实性幻觉** | 生成与输入/上下文矛盾的内容 | RAG中歪曲检索文档含义 |
| **推理幻觉** | 推理过程错误但结论可能正确 | 数学计算步骤错误 |

- **幻觉成因:**

1. **数据层面:** 训练数据噪声/错误/过时;知识冲突;长尾覆盖不足
2. **训练层面:** 自回归「落子无悔」无法修正错误;SFT知识污染
3. **推理层面:** 无事实核查机制;高温采样增加随机性
4. **评估层面:** 缺乏有效事实验证

- **缓解方法:**

1. **RAG** - 检索外部知识增强事实性
2. **RLHF/RLAIF** - 用事实性奖励训练模型
3. **Chain-of-Thought** - 分步推理减少跳步错误
4. **Self-Check** - 模型自我验证生成内容
5. **解码控制** - 降低Temperature,约束解码
6. **不确定性表达** - 训练模型在不确定时说「我不知道」

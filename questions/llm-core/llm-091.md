# 为什么需要混合精度训练

大语言模型通常规模很大,从数十亿到上千亿参数.混合精度训练(FP16/BF16 + FP32) 是实践中
**的常见选择：** 既节省显存又能在一定程度上保证数值稳定性.

97. 1 混合精度训练流程
存储一份 fp32 的 parameter(“主权重”),以及相应的 momentum、
复制一份并将其精度减半得到 fp16 的 parameter(“训练权重”),实际前向推
理、反向传播都用它. 最终保存也以 fp32 主权重为准.
fp32 的 parameter 相当于“主权重”.在模型训练的过程中,我们执行
optimizer.step() 更新的就应该是这份权重.当模型训练完毕后,我们保存的也
是它.取得高精度的权重是我们的最终目标.
fp16 的 parameter 相当于“训练权重”,也就是在训练过程中实际参与前向传播过程
使用 fp16 的 parameter 进行计算,生成 fp16 的 activation(将在反向传播过程中
计算出的 loss 一般转成 fp32 以保证精度.
如果没有采取重计算等操作,activation 占据的存储会非常大(可能大过模型本身).
**Loss Scale**
为防止梯度下溢(gradient underflow),对 loss 做适当的 scale 处理;
常见做法是常量放大或动态放大(动量损失放大).
下溢(Underflow)
在计算机数值计算中,下溢是指计算结果的绝对值小于计算机所能表示的最小非零正
数.在深度学习的损失函数计算等场景中,当一些非常小的数参与计算时,可能会出现
下溢.例如,在计算概率或对数概率时,某些概率值可能非常接近于0,如果直接进行
计算,可能会导致数值下溢,使得计算结果变为0,这会丢失重要的信息,并且可能导
致梯度消失等问题,影响模型的训练. 反向传播 (BWD) 使用 fp32 的 scaled loss 计算梯度(梯度最初以 fp16 的形式存储,但会再转换成
如果开启了重计算(Recompute),则需要在 BWD 时再次重算部分激活.
计算出来的 loss 用 fp32 精度来表示,这样做是为了保证反向传播计算梯度的精确性.
因为 loss 做过 scale 了,那自然得到的也是 scaled gradients,为了节省显存,scaled
gradients 以 fp16 的形式存储.
将存储的梯度从 fp16 转回 fp32,执行clip(可选)等操作后再更新主权重;
fp16 的权重则和激活一起在下一轮继续使用.
Clip gradients:在转换为 fp32 的梯度后,我们还可以执行 clip 等操作,来进一步预防
梯度爆炸/消失.

Unscaled gradients:梯度以 fp16 的形式存储,但等到需要用梯度去更新模型权重
时,就必须转换成 fp32 的形式了.
97. 2 为什么要做混合精度训练
1. 舍入误差
全程使用 fp16 进行权重更新,可能使参数更新不稳定(精度不足导致的舍入误差累积).
**解决方案：** 关键更新(optimizer.step)时把梯度转为 fp32,保证参数更新部分的高精度.
2. 梯度下溢
,在 fp16 表示范围下会出现梯度直接变为 0 的下溢问题,使得整个
**解决方案：** 对 loss 做 loss scale 处理,显式放大梯度.

# 为什么需要混合精度训练

大语言模型通常规模很大，从数十亿到上千亿参数。混合精度训练（FP16/BF16 + FP32）是实践中的常见选择：既节省显存又能在一定程度上保证数值稳定性。

### 1. 混合精度训练流程

*   **FP32 主权重**:
    存储一份 FP32 的参数，以及相应的 momentum 等 optimizer states。这是“标准答案”，所有的参数更新最终都应用在这份权重上。模型训练完毕后保存的也是它。

*   **FP16 训练权重**:
    复制一份主权重并将其精度减半得到 FP16 参数。实际的前向推理、反向传播都使用这份 FP16 权重进行计算，生成 FP16 的 activation。

*   **Loss Scaling**:
    为了防止梯度下溢，对 Loss 做适当的放大处理（Scale）。常见做法是动态损失放大。

*   **反向传播 (BWD)**:
    使用 FP32 的 scaled loss 计算梯度。为了节省显存，计算出的 scaled gradients 以 FP16 形式存储。

*   **权重更新**:
    1.  将 FP16 的梯度转回 FP32。
    2.  如果之前做了 Loss Scale，这里需要 Unscale 梯度。
    3.  可选执行梯度裁剪。
    4.  使用 FP32 梯度更新 FP32 主权重。
    5.  将更新后的 FP32 主权重再次复制为 FP16 权重，供下一轮迭代使用。

### 2. 为什么要做混合精度训练

1.  **解决舍入误差**:
    全程使用 FP16 进行权重更新，由于精度不足（特别是当梯度很小时），参数更新可能不稳定或无法改变。
    **解决方案**：关键更新（optimizer.step）时把梯度转为 FP32，利用 FP32 的高精度累加微小变化，保证参数更新的准确性。

2.  **解决梯度下溢**:
    FP16 的表示范围远小于 FP32。在反向传播时，梯度数值往往非常小，在 FP16 表示范围下容易出现直接变为 0 的下溢问题，导致训练停滞。
    **解决方案**：在计算 Loss 后、反向传播前，对 Loss 乘以一个较大的数（Scale），从而强制放大梯度数值，使其保持在 FP16 的有效表示范围内。

### 3. 补充关键细节

*   **FP16 vs BF16**:
    *   **FP16 (Float16)**: 1位符号位，5位指数位，10位尾数位。数据范围小（约 6e-5 到 6e4），容易出现溢出和下溢，需要配合 Loss Scaling。
    *   **BF16 (BFloat16)**: 1位符号位，8位指数位，7位尾数位。数据范围与 FP32 一致，不容易溢出/下溢，通常**不需要** Loss Scaling，但精度稍低。在 Ampere (A100) 及更新架构上推荐使用 BF16。

*   **加速原理**:
    *   **显存带宽**: 显存访问量减半（权重和梯度从 32bit 变为 16bit），数据传输速度更快。
    *   **计算单元**: 现代 GPU (如 Volta, Turing, Ampere) 拥有专门的 Tensor Cores，计算 FP16/BF16 矩阵乘法的吞吐量是 FP32 的数倍（最高可达 8-16x 理论峰值）。

## 常见考点
1. **为何保留 FP32 主权重？**：为了防止 FP16 累加误差导致的权重更新不收敛，保证训练的数值稳定性。
2. **Loss Scaling 的动态策略是什么？**：开始时 Scale 较大，如果检测到梯度 Inf/Nan，则跳过该步更新并缩小 Scale；否则定期尝试增大 Scale，以最大化 FP16 的利用率。
3. **FlashAttention 对混合精度的影响？**：FlashAttention 等 IO 感知算子通常在内部使用 FP16/BF16 计算，但可能利用 FP32 进行累加以减少 softmax 的数值误差，这是混合精度在算子层面的微观应用。

# 为什么需要混合精度训练

大语言模型通常规模很大，从数十亿到上千亿参数。混合精度训练（FP16/BF16 + FP32）是实践中的常见选择：既节省显存又能在一定程度上保证数值稳定性。

### 1. 混合精度训练流程

*   **FP32 主权重 (Master Weights)**:
    存储一份 FP32 的参数，以及相应的 momentum 等 optimizer states。这是“标准答案”，所有的参数更新最终都应用在这份权重上。模型训练完毕后保存的也是它。

*   **FP16 训练权重 (Training Weights)**:
    复制一份主权重并将其精度减半得到 FP16 参数。实际的前向推理、反向传播都使用这份 FP16 权重进行计算，生成 FP16 的 activation。

*   **Loss Scaling**:
    为了防止梯度下溢（gradient underflow），对 Loss 做适当的放大处理（Scale）。常见做法是动态损失放大。

*   **反向传播 (BWD)**:
    使用 FP32 的 scaled loss 计算梯度。为了节省显存，计算出的 scaled gradients 以 FP16 形式存储。

*   **权重更新**:
    1.  将 FP16 的梯度转回 FP32。
    2.  如果之前做了 Loss Scale，这里需要 Unscale 梯度。
    3.  可选执行梯度裁剪。
    4.  使用 FP32 梯度更新 FP32 主权重。
    5.  将更新后的 FP32 主权重再次复制为 FP16 权重，供下一轮迭代使用。

### 2. 为什么要做混合精度训练

1.  **解决舍入误差**:
    全程使用 FP16 进行权重更新，由于精度不足（特别是当梯度很小时），参数更新可能不稳定或无法改变。
    **解决方案**：关键更新（optimizer.step）时把梯度转为 FP32，利用 FP32 的高精度累加微小变化，保证参数更新的准确性。

2.  **解决梯度下溢**:
    FP16 的表示范围远小于 FP32。在反向传播时，梯度数值往往非常小，在 FP16 表示范围下容易出现直接变为 0 的下溢问题，导致训练停滞。
    **解决方案**：在计算 Loss 后、反向传播前，对 Loss 乘以一个较大的数（Scale），从而强制放大梯度数值，使其保持在 FP16 的有效表示范围内。

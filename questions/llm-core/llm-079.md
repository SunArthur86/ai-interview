# Linear Attention(线性注意力)是什么

Linear Attention（线性注意力）是一种通过核技巧将注意力计算复杂度从二次方$O(N^2)$降低到线性$O(N)$的方法，旨在解决长序列建模中的计算瓶颈。

**核心原理：**
利用核函数特性，将不可分解的Softmax操作近似为可分离的形式：$\text{softmax}(QK^T) \approx \phi(Q)\phi(K)^T$。其中$\phi(\cdot)$是核映射函数（如ELU+1或随机特征映射）。

**公式变换：**
原计算：$Attention(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d}})V$
线性化后：$Attention(Q,K,V) \approx \phi(Q)(\phi(K)^TV)$

**优势：**
通过先计算$\phi(K)^TV$（从$N^2$降为$N$），再与$Q$相乘，避免了构建巨大的$N \times N$注意力矩阵，使得处理长序列时的时间和显存消耗大幅下降。

**详细补充与原理：**
1. **计算逻辑**：标准注意力的瓶颈在于 $N \times N$ 的 $S = QK^T$ 矩阵。Linear Attention 利用矩阵结合律，改变计算顺序。先计算 $KV^T$（实际上通常是 $K^T V$，视维度布局而定，这里指特征聚合），输出维度为 $d \times d$（特征维度），与序列长度 $N$ 无关。
2. **核映射函数 $\phi$**：
   - **ELU+1**: $\phi(x) = \text{ELU}(x) + 1$。ReLU 的变体，保证非负性，满足正定核条件。
   - **随机特征映射**: 如基于 RBF 核的随机投影，理论上能更紧致地近似 Softmax，但实现更复杂。
   - **无需 Softmax**: Linear Attention 实际上抛弃了 Softmax，这意味着它失去了标准注意力“按权重归一化”的特性，这通常会导致模型性能下降，因此在实践中常需配合特定的初始化或训练技巧（如门控机制）。
3. **因果掩码的处理**：
   在 GPT 等自回归模型中，不能看到未来的 token。标准 Attention 是通过将上三角置为 $-\infty$ 实现的。
   在 Linear Attention 中，由于不能先生成完整矩阵，通常采用“线性 RNN”的形式（类似 RWKV、Mamba 的思想），维护一个随时间步更新的状态 $S_t = S_{t-1} + \phi(K_t)^T V_t$，从而实现 $O(N)$ 的因果计算。

**计算流程对比图：**

```text
标准 Attention (O(N²)):
Q (N×d)     K (d×N) 
  │          │
  └──[矩阵乘]──┘ 
       │
       ▼
  Score (N×N) <--- 显存/计算瓶颈
       │
  [Softmax + Mask]
       │
       ▼
  Weight (N×N) ──[矩阵乘]──> V (N×d)
                         │
                         ▼
                   Output (N×d)

Linear Attention (O(N)):
K (d×N)       V (N×d)      Q (N×d)
  │            │            │
  └──[转置/映射]┘            │
       │                     │
  φ(K)ᵀ (N×d) ──[矩阵乘]──> V(N×d)
       │                     │
       ▼                     ▼
  Global State (d×d) <─[矩阵乘]─ φ(Q)(N×d)
       │                     │
       └──────────[矩阵乘]────┘
                     │
                     ▼
               Output (N×d)
```

**## 常见考点**
1. **为什么 Linear Attention 通常比标准 Attention 精度低？**
   因为其丢弃了 Softmax 的稀疏性和归一化特性，Softmax 能让模型聚焦于最相关的 Key，而线性核映射使得所有 Token 贡献较为平均（稠密检索），导致信息“平滑化”。
2. **如何处理自回归（Causal）约束？**
   重点在于提到“累加和”或“状态维护”的思路（类似 RNN），而不是简单的 Mask。
3. **Linear Attention 和 Flash Attention 的区别是什么？**
   Flash Attention 是 IO 感知的精准计算优化，复杂度仍为 $O(N^2)$（虽然常数项极小）；Linear Attention 是数学近似，将复杂度降维到 $O(N)$，前者不损失精度，后者损失精度换长度。

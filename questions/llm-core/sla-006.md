# LLM推理中有哪些关键加速技术？vLLM的核心原理是什么？

🎯 本质：LLM推理优化核心是最大化GPU利用率，主要围绕KV Cache管理、批处理和计算优化。

📊 核心优化技术：

1. Continuous Batching（连续批处理）
  传统：一个batch中短请求等长请求
  连续：每个iteration级动态调度，新请求随时加入，完成的随时离开
  效果：吞吐量提升5-10倍

2. PagedAttention（vLLM核心）
  问题：传统KV Cache预分配最大长度，大量浪费
  方案：借鉴OS虚拟内存，KV Cache分为固定大小block，按需分配
  效果：显存利用率从20-40%提升到80-96%

3. Prefix Caching
  多请求共享相同前缀(system prompt)的KV Cache
  效果：TTFT(首token延迟)降低50-90%

4. Flash Attention
  优化Attention计算的GPU内存访问模式
  减少HBM读写次数，不改数学结果

5. Tensor Parallelism
  模型权重矩阵切分到多GPU
  适合大模型单机多卡推理

vLLM vs TensorRT-LLM：
| 特性 | vLLM | TensorRT-LLM |
|------|------|-------------|
| 来源 | UC Berkeley | NVIDIA |
| 易用性 | 高(pip install) | 中(需编译) |
| 灵活性 | 高(支持多数模型) | 中 |
| 性能 | 很高 | 最高(NVIDIA深度优化) |
| PagedAttention | 原创 | 支持 |

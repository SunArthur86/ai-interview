# PPL(Perplexity,困惑度)是什么?如何用于评估大模型

- **PPL (Perplexity):** 衡量模型对文本的「困惑程度」.

- **数学定义:**
```
PPL = exp(-1/N * Σ log P(x_i | x_{<i}))

PPL越低 → 模型越确信 → 质量越好
```

- **直觉理解:**
- PPL=1:模型完美预测每个token
- PPL=10:模型每步平均在10个token间犹豫
- PPL=100:模型非常不确定

- **用途:**
1. **模型对比** - 同一测试集上PPL越低越好
2. **训练监控** - 训练过程中PPL应持续下降
3. **语言质量** - 衡量生成文本的流畅性

- **局限性:**
- PPL低不代表回答准确(只衡量语言建模能力)
- 不同Tokenizer的PPL不可直接比较
- 对长文本不敏感(平均效应)

- **实践:** 预训练用PPL监控,下游任务用任务特定指标(accuracy/F1/BLEU)

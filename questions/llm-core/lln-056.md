# GPTQ和AWQ量化算法的原理和区别是什么

两者都是训练后量化(PTQ)方法,将模型权重从FP16压缩到INT4/INT8.

- **GPTQ (Generative Pre-trained Transformer Quantization):**

**核心思想：** 基于二阶信息(Hessian矩阵)逐层最优量化.

1. 计算权重矩阵的Hessian矩阵 H = Xᵀ·X(X是校准数据)
2. 按列量化权重,每量化一列就用剩余信息补偿前面量化引入的误差
3. 使用Cholesky分解加速Hessian求逆
4. 效果:INT4量化后几乎无损,速度提升3-4倍

- **AWQ (Activation-aware Weight Quantization):**

**核心思想：** 不是所有权重同等重要,激活值大的通道对应的权重量化误差影响更大.

1. 观察到:激活幅度大的通道对应的权重对量化误差更敏感
2. 引入缩放因子s:在量化前将重要通道的权重放大,量化后缩小
3. 通过网格搜索找到最优的缩放比例
4. 效果:比GPTQ更快的推理速度(更好的硬件友好性)

- **对比:**

| 维度 | GPTQ | AWQ |
|------|------|-----|
| 量化依据 | 二阶信息(Hessian) | 激活感知 |
| 校准数据 | 需要(~128样本)| 需要(~128样本)|
| 推理速度 | 快 | 更快(INT4 GEMM优化好)|
| 显存节省 | ~75% | ~75% |
| 量化效果 | 优秀 | 优秀 |
| 硬件兼容性 | 较好 | 更好(标准INT4操作)|

- **选择建议:** 追求速度选AWQ,追求量化精度选GPTQ,两者都远优于简单的Round-to-Nearest量化.

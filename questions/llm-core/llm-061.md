# 手撕Layer Norm

```python
import torch
import torch.nn as nn

class LayerNorm(nn.Module):
    def __init__(self, normalized_shape, eps=1e-5, elementwise_affine=True):
        """
        Args:
            normalized_shape: 输入形状中从末尾开始归一化的维度，例如 [hidden_size] 或 [batch, hidden_size]
            eps: 防止除零的小常数
            elementwise_affine: 是否进行缩放和平移（引入可学习参数 gamma 和 beta）
        """
        super(LayerNorm, self).__init__()
        self.eps = eps
        self.elementwise_affine = elementwise_affine
        if self.elementwise_affine:
            self.gamma = nn.Parameter(torch.ones(normalized_shape))
            self.beta = nn.Parameter(torch.zeros(normalized_shape))

    def forward(self, x):
        # 1. 计算均值：对最后一个维度求均值
        # keepdim=True 保持维度，便于广播
        mean = x.mean(dim=-1, keepdim=True)
        
        # 2. 计算方差：使用有偏估计 (unbiased=False)，与 TensorFlow 保持一致，且更稳定
        var = x.var(dim=-1, keepdim=True, unbiased=False)
        
        # 3. 归一化
        x_normalized = (x - mean) / torch.sqrt(var + self.eps)
        
        # 4. 仿射变换（缩放和平移）
        if self.elementwise_affine:
            x_normalized = self.gamma * x_normalized + self.beta
            
        return x_normalized
```

### 核心流程图

```text
输入 X (Batch, Seq_Len, Hidden)
          │
          ▼
┌───────────────────────┐
│  计算均值 mean (dim=-1) │
└───────────┬───────────┘
            │
            ▼
┌───────────────────────┐
│  计算方差 var (dim=-1) │
└───────────┬───────────┘
            │
            ▼

    │ ───────────── │

            ▼

            ▼                              │
       Output                           Output
```

### 常见考点
1.  **biased vs unbiased variance**：为什么 `unbiased=False`？在 LayerNorm 中通常使用有偏方差（分母为 N），与 BatchNorm 默认无偏（分母为 N-1）不同，主要是为了与 TensorFlow 行为对齐及稳定性。
2.  **BatchNorm vs LayerNorm**：LayerNorm 是针对单个样本的所有特征进行归一化，不依赖 Batch size，因此适合 NLP 任务中变长序列；BatchNorm 是针对单个 Channel 在 Batch 维度归一化，适合 CV 任务。
3.  **参数维度**：`gamma` 和 `beta` 的维度是 `normalized_shape`（即 `hidden_size`），在计算时会自动广播到 `x` 的形状。
4.  **训练与推理一致性**：LayerNorm 不需要像 BatchNorm 那样维护 running mean/var，其计算在训练和推理时是完全一样的。

# 什么是Self-consistency和Best-of-N

两种方法都是基于聚合的测试时计算方法，即在大模型推理阶段通过生成多个回答并聚合以获得最终输出。

### Self-consistency（自一致性）
**原理**：让大模型生成多个 CoT（Chain of Thought）推理路径及多个答案，通过“多数投票”选出最终答案。
*   **核心逻辑**：假设虽然推理路径不同，但针对逻辑严密的数学或常识问题，正确的推理路径往往汇聚于同一个正确答案。
*   **流程**：
    1. Prompt 中加入“Let's think step by step”。
    2. 设定采样参数（如 temperature=0.7），生成 N 个不同的解题轨迹。
    3. 统计 N 个轨迹中最终答案的分布，取众数。

### Best-of-N
**原理**：让大模型生成 N 个回答，使用 Verifier、ORM (Outcome Reward Model) 或 PRM (Process Reward Model) 给每个回答评分，选择分数最高的作为最终回答。
*   **核心逻辑**：显式引入奖励模型来筛选质量，而非依赖答案的随机分布一致性。
*   **区别**：Self-consistency 依赖“投票”，适合有唯一正确解的场景；Best-of-N 依赖“打分”，适合开放域生成或代码生成等需要质量优化的场景。

```text
   输入 Prompt
      │
      ▼
┌─────────────────┐
│  多次采样生成    │ (N次, 如N=20)
│  (Sample T times)│
└────────┬────────┘
         │
    ┌────┴────┬────────────┐
    ▼         ▼            ▼
┌───────┐ ┌──────────┐ ┌──────────┐
│Answers│ │Answers...│ │Answers   │
└───┬───┘ └────┬─────┘ └────┬─────┘
    │          │           │
    │   Self-consistency   │
    │   (投票/众数)        │
    ▼                      │
 最终答案                  │
                          │ Best-of-N
                          │ (Verifier/PRM打分)
                          ▼
                       最终答案
```

## 常见考点
1.  **Self-consistency 和 Best-of-N 的适用场景区别**（前者适合数学题等有客观解的任务，后者适合代码优化、创意写作）。
2.  **PRM (Process Reward Model) 和 ORM (Outcome Reward Model) 在 Best-of-N 中的区别**（PRM 对每个推理步骤打分，能更精准定位错误步骤）。
3.  **测试时计算的权衡**：如何平衡生成 N 个样本带来的推理延迟成本与性能提升。

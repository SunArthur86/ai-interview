# 什么场景该用 RAG,什么场景该用 Fine-tuning?如何做架构决策

**这是面试超高频题,决策框架如下：** 

- **选 RAG 的场景:** 知识频繁更新(新闻、文档)、需要引用来源(合规审计)、知识量大但不需要深度理解、多租户场景、快速验证.

- **选 Fine-tuning 的场景:** 需要改变模型行为或风格(代码生成、特定格式输出)、特定领域深度推理(医学诊断)、延迟敏感、数据隐私要求高.

- **两者结合最佳实践:** 先 Fine-tune 让模型理解领域术语和推理方式,再用 RAG 提供最新事实性知识,这是生产系统最常见的架构.

- **关键维度对比:** RAG 知识更新简单(更新向量库),Fine-tuning 昂贵(重新训练);RAG 有额外检索开销,Fine-tuning 无;RAG 可解释性高(可引用来源),Fine-tuning 低.

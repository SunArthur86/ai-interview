# 解释一下Lora的原理

### 1. LoRA 的背景和问题
在预训练大规模模型时，微调整个模型的参数可能会带来较大的计算和存储开销。对于像 GPT-3 这样的大型模型，直接微调所有参数不仅非常耗时，还需要大量的存储资源。

### 2. LoRA 的原理
LoRA 通过引入低秩矩阵分解，在微调时不再调整原始模型的所有权重，而是通过将权重矩阵分解成两个较小的低秩矩阵来表示权重的变化。尽管预训练模型拥有大量的参数，但许多参数在特定任务的微调过程中实际上是不活跃的。LoRA 通过低秩分解的方式，只更新那些对特定任务最重要的参数，这样做可以在保持模型性能的同时，显著减少微调所需的计算资源和时间。

假设某个层的权重矩阵为 $W \in \mathbb{R}^{d \times k}$，LoRA 不微调 $W$ 本身，而是通过两个低秩矩阵 $A \in \mathbb{R}^{d \times r}$ 和 $B \in \mathbb{R}^{r \times k}$ 来近似微调权重矩阵。LoRA 的权重更新公式为：

$$W_{new} = W + \Delta W = W + \frac{\alpha}{r} (A \times B)$$

- $W_{new}$：微调后的权重矩阵。
- $W$：预训练模型的原始权重矩阵，保持不变。
- $A$ 和 $B$：低秩矩阵，$A \times B$ 近似于权重变化量 $\Delta W$。
- $\alpha$：一个缩放因子，用来调节低秩矩阵的影响。
- $r$：秩的大小，表示低秩矩阵 $A$ 和 $B$ 的维度，通常 $r \ll d, k$，从而大大减少需要训练的参数数量。

LoRA 的核心思想是将权重的变化通过两个小型矩阵的乘积进行表示，利用低秩矩阵分解的方式有效降低参数量。这样只需要训练低秩矩阵的参数，而不是完整的权重矩阵。

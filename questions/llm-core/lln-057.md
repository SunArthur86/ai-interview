# RAG(检索增强生成)的完整pipeline是什么?每个环节有哪些优化方法

RAG = 检索 + 生成,通过外部知识库增强LLM的回答能力.

- **完整Pipeline:**
用户Query → [Query改写] → [检索] → [重排] → [上下文组装] → [LLM生成] → 回答

- **各环节优化:**

- **1. Query处理(检索前):**
- Query改写:用LLM将模糊query改写为更精确的检索query
- Query扩展:添加同义词/相关词扩大召回
- Query2Doc:用LLM生成伪文档扩展query
- HyDE:生成伪文档用于向量检索
- 多query融合:生成多个子query分别检索

- **2. 索引构建:**
- 文档切分策略:固定长度/语义切分/递归切分
- Embedding模型选择:bge-large, text-embedding-3等
- 多级索引:摘要索引 + 细粒度索引
- 元数据过滤:按时间/类型/权限预过滤

- **3. 检索:**
- 向量检索(Dense):语义相似度,适合模糊匹配
- 关键词检索(Sparse/BM25):精确匹配,适合专业术语
- 混合检索:Dense + Sparse融合,效果最好

- **4. 重排(Rerank):**
- Cross-encoder模型:bge-reranker, Cohere Rerank
- LLM打分:让LLM对检索结果相关性评分
- 多信号融合:相关性 + 新鲜度 + 权威性

- **5. 上下文组装与生成:**
- 长上下文处理:Lost-in-the-middle问题(中间位置信息容易丢失)
- 文档压缩:RECOMP/LangChain的stuff/map_reduce策略
- 引用标注:让LLM标注信息来源

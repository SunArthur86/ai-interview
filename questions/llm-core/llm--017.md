# 什么是模型微调(Fine-tuning)?为什么不直接用预训练模型

- **微调 (Fine-tuning)** 是在预训练模型基础上,用特定领域数据继续训练,使其适应下游任务.

- **为什么不直接用预训练模型?**
  1. 预训练模型是'通才',缺乏领域深度
  2. 不会遵循指令格式(如输出 JSON)
  3. 回复风格不符合业务需求
  4. 缺乏私有知识

- **微调方式**
  - **全量微调**:更新所有参数(效果好,成本高)
  - **LoRA**:只训练低秩适配器(成本低,效果接近)
  - **QLoRA**:4bit 量化 + LoRA(极致省显存)

- **一句话理解**:预训练模型 = 大学毕业生;微调 = 入职培训.

```text
┌─────────────────────────────────────────────────────────────┐
│                 微调参数更新方式对比                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [全量微调 Full Fine-tuning]                                │
│  ┌─────────────────────────────────────────┐               │
│  │  预训练权重 (全部参与更新)               │               │

│  ┌─────────────────────────────────────────┐               │
│  │  预训练权重 (冻结 Frozen)                │               │
│  └─────────────────────────────────────────┘               │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**## 常见考点**
1. **PEFT (Parameter-Efficient Fine-Tuning)**：参数高效微调的概念，包括 LoRA、Adapter、Prefix Tuning 等技术。
2. **LoRA 原理**：在原权重旁增加低秩分解矩阵（A、B），只训练这两个小矩阵，推理时将其合并回原权重，无额外延时。
3. **灾难性遗忘**：微调可能导致模型遗忘预训练学到的通用知识，如何通过混合数据或正则化缓解。

# DPO的原理

DPO (Direct Preference Optimization，直接偏好优化) 是一种用于大模型对齐的算法。它无需训练显式的奖励模型，直接根据人类偏好数据优化策略。

### 1. 背景
传统方法（如 RLHF）需要先训练一个奖励模型 来模拟人类偏好，再使用强化学习（如 PPO）优化策略。这个过程复杂且不稳定。

### 2. 核心原理
DPO 证明了在 Bradley-Terry 偏好模型下，奖励函数和策略之间存在直接的解析关系：
$$ \log \frac{\pi(y_w|x)}{\pi(y_l|x)} = \beta \cdot (R(x, y_w) - R(x, y_l)) $$
这意味着优化奖励差值等价于优化策略的相对概率。DPO 利用这一关系，**直接重写奖励模型的优化目标为策略空间的优化目标**，从而消除了对奖励模型的需求。

### 3. DPO 损失函数
DPO 的目标是最大化偏好数据 $(x, y_w, y_l)$ 的对数似然，其中 $y_w$ 是偏好答案，$y_l$ 是拒绝答案。损失函数如下：
$$ L_{DPO}(\pi_\theta; \pi_{ref}) = -E \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right] $$
*   $\pi_\theta$：当前正在训练的策略。
*   $\pi_{ref}$：参考模型（通常是 SFT 模型），用于约束优化过程，防止模型崩溃或偏离过远。
*   $\beta$：温度系数，控制优化强度。

### 4. 优势
*   **无需奖励模型和强化学习**：简化了流程，避免了 RL 阶段的不稳定性。
*   **数学上更稳定**：直接在策略空间优化，避免了奖励模型拟合误差的累积。
*   **实现简单**：只需在 SFT 基础上增加一个简单的损失项即可训练。

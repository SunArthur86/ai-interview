# MoE（混合专家模型）的原理是什么？

MoE（Mixture of Experts）将模型的前馈网络（FFN）替换为多个专家网络，通过门控（Gate）路由机制动态选择激活哪些专家。

架构：
- N个专家FFN（如8个）
- 门控网络（Router）：对每个token，输出N个分数，选Top-K个专家（如K=2）
- 输出 = 加权求和被选中专家的输出

优势：
1. 参数解耦：总参数量大（如671B），但每次推理只激活一小部分（如37B），推理成本低
2. 容量大：更多参数意味着更强的表示能力
3. 训练高效：每个token只更新Top-K专家的梯度

挑战：
1. 负载均衡：某些专家可能被过度使用（用Auxiliary Loss解决）
2. 通信开销：专家分布在多GPU上，跨设备通信（All-to-All）
3. 显存占用：所有专家参数都要加载到显存

代表模型：DeepSeek-V3（256个专家，每次激活8个）、Mixtral 8x7B、GPT-4（据传使用MoE）。

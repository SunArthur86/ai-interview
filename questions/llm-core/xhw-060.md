# Group/Instance Normalization

Multi-query / Grouped-query：减少 Key/Value 重复，以提高效率。
Sparse / Linear Attention：减少或简化注意力计算，以适应长序列。
Multi-head Latent Attention：在多头的基础上引入隐变量，更丰富的表示。
🌟 用一句通俗语言总结
“它们都源自同一个核心：根据‘相关性’分配不同注意力权重，只是根据不同场景需求（是否同一
序列、是否跨序列、是否要稀疏或线性化、如何平衡多头与效率、是否加入额外隐变量等）作出不同
的改进和变体。”
从自注意力、交叉注意力到多头注意力是 Transformer 的主干；在此之上，为了加速长序列处理，
有了稀疏注意力与线性注意力；为了优化推理效率，又发展出多查询/ 分组查询注意力；再进一步，
为了增强表达力，一些工作研究了隐变量结合注意力。所有这些改进都围绕着同一个目标：在计
算、内存、表示能力之间找到最优平衡，并让模型能更好地关注到最重要的信息。
89. Group Normalization（GN）和 Instance Normalization（IN)的
区别是什么？
89.1 Group Normalization（GN）
Batch Normalization（BN）的出现极大地加速了图像分类等任务的训练过程，但它需要在一个较
大的 mini-batch 内统计均值和方差，才能得到比较稳定的统计量。
当批量大小很小（小 batch）或在某些特殊的图像/风格任务中，BN 往往不适用或效果不佳，这就催
生了 Group Normalization（GN） 和 Instance Normalization（IN） 等替代方案。
🥥
核心思想
Group Normalization（GN）是由 Facebook AI Research 在 2018 年提出的一种归一化方
法，它将每个样本的通道（channel）分成多个“组”（Group），再在每个组内部计算均值
与方差来执行归一化。

🌟适用场景与特点🌟
🍇 小批量训练：  当显存限制或其他原因导致批量大小无法很大时，BatchNorm 的统计不稳
定，此时 GN 更能保证训练的稳定性和加速效果。
🍇 高分辨率图像或 3D 图像： 在图像分割、目标检测、医学影像等任务中，图像分辨率很高
通常迫使 batch size 不大，GN 是不错的选择。
🍇 GPU/分布式环境：  大规模多 GPU 训练时，若每个 GPU 分到的数据很少，BN 统计量偏
差大，而 GN 不依赖跨样本计算均值方差，可以在每个 GPU 上独立工作。

89.2 Instance Normalization（IN)
🥥
核心思想
Instance Normalization（IN）则是对每个样本、每个通道的激活值独立地计算均值和方
差。也就是说，在通道维度上进一步细分为单个通道：“每个样本的每个通道”都做自己专
属的归一化，不和其他通道或其他样本发生数据统计交互。

🌟适用场景与特点🌟
🍇 图像风格迁移（Style Transfer）：   让每张图的每个通道都独立归一化，可以去除不同
图像间的全局统计差异，保留风格相关的局部特征。正因如此，IN 在风格迁移中非常流行。
🍇 GAN 及生成任务：  有时在生成式网络中，过度依赖批量统计会导致不稳定或多样性不
足，IN 帮助消除不同样本间的干扰，让模型更专注于当前样本的特征。

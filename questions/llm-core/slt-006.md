# BPE Tokenizer的原理是什么？

BPE（Byte Pair Encoding）是子词分词算法，LLM最常用的tokenizer。

训练过程：
1. 初始化：将所有词拆成单个字符
2. 统计相邻字符对频率，合并最高频的字符对
3. 重复直到达到目标词表大小

示例：
- 'low', 'lower', 'lowest' → 'l', 'o', 'w', 'e', 'r', 's', 't'
- 合并'l'+'o'='lo'，'lo'+'w'='low'
- 最终'low'是一个token，'er'是一个token，'est'可能是一个token

优势：
1. 平衡OOV问题和词表大小
2. 罕见词可以被分解为已知子词
3. 不同语言可以共用字节级BPE（BBPE）

词表大小：GPT-2用50257，Llama用32000，Llama 3扩展到128256。

影响：tokenizer直接影响模型性能——多语言覆盖、编码效率、数字处理。

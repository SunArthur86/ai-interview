# 什么是大模型的'幻觉'(Hallucination)?为什么会发生

- **幻觉** 指大模型生成看似合理但实际上不正确的内容.

- **幻觉类型**
  1. **事实幻觉**:说错事实('爱因斯坦生于1900年' → 实际1879年)
  2. **逻辑幻觉**:推理过程有误
  3. **来源幻觉**:编造不存在的引用/论文

- **产生原因**
  - 训练数据包含噪声、偏差或过时信息
  - 模型本质是'概率生成',不是'知识检索'
  - 缺乏外部实时信息或确定性机制

- **缓解方法**
  - **RAG**:检索增强生成，引入外部知识
  - **提示词约束**:设定'如果不确定请说不知道'的规则
  - **微调**:通过高质量数据减少编造倾向

- **一句话理解**:LLM 是'概率预测器'不是'数据库',所以会'一本正经地胡说'.

```text
┌──────────────┐     检索      ┌──────────────┐
│   用户问题   │─────────────▶│  知识库/文档 │
└──────┬───────┘               └──────┬───────┘
       │                              │
       │         检索结果              │
       └──────────────────────────────┘
       │                              │
       ▼                              ▼
┌──────────────────────────────────────┐
│            Prompt 拼接               │
│  (问题 + 检索到的相关上下文 + 指令)   │
└──────────────┬───────────────────────┘
               │
               ▼
┌──────────────────────────────────────┐
│             LLM 生成                  │
│      (基于上下文生成答案，减少幻觉)   │
└──────────────────────────────────────┘
```

**## 常见考点**
1. **RAG vs 微调**：RAG 适合事实性、时效性要求高的场景；微调适合学习语言风格、特定格式或领域术语。
2. **解码策略**：如核采样，通过限制候选词范围来减少低概率乱码的产生，也是一种抑制幻觉的手段。
3. **Self-Consistency（自洽性）**：让模型回答同一个问题多次，如果答案不一致则可能存在幻觉，通过投票选出最可信的答案。

# 大海捞针测试和概率探针分别是什么

101. 1 大海捞针
🫐大海捞针测试着重考察模型在超长上下文里的信息提取和理解能力 通过在长文本中随机插入关键信息,形成大型语言模型(LLM)的Prompt.该测试旨在检测大 型模型是否能从长文本中提取出这些关键信息,从而评估模型处理长文本信息提取的能力, 这可以反映LLM对长文本的理解基础能力.
**方法：**
• 先准备一个较长文本(数千词以上),其中混合了各种冗余或无关信息,随机插入关键细 节. • 再让模型回答一个或多个问题,这些问题必须依赖文中的关键部分才能回答.
**常见变体：**
• **单一信息检索任务：** 长文章中只藏了一个关键信息,看看模型能否准确找出.评估LLM在 长文本中提取单一关键信息的能力,测试其对广泛叙述中特定细节的精确回忆能力.这对 应于原始的大海捞针测试任务设定. • **多信息检索任务：** 文章里埋了多个分散的关键信息,需要综合才能回答.探讨LLM从长文 本中检索多个相关信息的能力,模拟实际场景中对综合文档的复杂查询. • **多信息推理任务：** 在检索多个关键信息后,还要做进一步推理、整合才能得出结果.通过 提取并利用长文本中的多个关键信息来评估LLM的长文本能力,要求模型对各关键信息片 段有综合理解.

• **祖先追溯挑战：** 类似于复杂的家谱/血缘/亲属关系推理,需要多层逻辑串联.通过设计 “亲属关系针”,测试LLM处理真实长文本中多层逻辑挑战的能力.在ATC任务中,通过 一系列逻辑推理问题,检验模型对长文本中每个细节的记忆和分析能力,在此任务中,我 们去掉了无关文本(Haystack)的设定,而是将所有文本设计为关键信息,LLM必须综合运 用长文本中的所有内容和推理才能准确回答问题. 通过这种考察,能看出模型对长文的理解、记忆和检索能力,而不再局限于多选题或简短段落.
101. 2 概率探针
🫐有时我们想精准地观察模型是否学到了某个事实或对某个句子给出多大概率,这就可以使用 概率探针(probabilistic probe)的方法.适用于我们要观察模型的某一项具体能力.思路也非 常简单,我们就观察某个 token 的概率是否增加,某个句子的概率是否增加,唯一麻烦的地 方是探针测试集往往需要训练者一条一条亲自去构造,而不能批量生成.步骤如下:
1. 事先人工设计或挑选一些带有特定知识/立场的句子(例如 “北京是中国的首都” / “台
湾属于中国” / “尊重同性恋” 等);
2. 对每个句子,或者句子里某个关键 token,去查看模型给它的预测概率. **例如：**
是否持续增加?如果越训越高,说明模型 越确认这一事实. Prob(“北京” ∣“中国的首都是”) PPL('台湾属于中国'),这个句子的 ppl 是否持续在增大;PPL('台湾不属于中国'),这 个句子的 ppl 是否持续在减小 对比探针,PPL('尊重同性恋') > PPL('反对同性恋') 是否成立 指令 follow 能力,Prob( '{ '| '以 json输出'); 或者对比两句的 PPL 大小,看看哪个更“自然”或“符合模型知识”. .......... 这种探针通常需要手动仔细构造测试集,才能真正捕捉到你关心的知识点或价值观. 重点观察随训练迭代,“正确事实”的概率有没有变高,“错误事实”的概率有没有变低. 和 benchmark 改造类似,概率探针是可以根据自己的喜好任意构造的.我们重点观察的也 是指标的变化趋势,而不是指标的绝对大小.

# 什么是长上下文（Long Context）技术？

长上下文（Long Context）指LLM能处理的输入token数从4K扩展到128K-1M+：

技术挑战：
1. 计算复杂度：标准Attention是O(n²)
2. KV Cache内存：线性增长
3. 训练数据：缺乏长文本训练数据

优化技术：
1. RoPE外推：NTK-aware scaling、YaRN扩展位置编码范围
2. 稀疏注意力：Sliding Window（Mistral）、StreamingLLM
3. Ring Attention：跨GPU分片计算长序列
4. KV Cache压缩：丢弃不重要的KV
5. FlashAttention：优化内存访问

模型对比：
- GPT-4 Turbo：128K
- Claude 3：200K
- Gemini 1.5 Pro：2M tokens
- Llama 3.1：128K
- Qwen2：128K

应用场景：
- 长文档分析（RAG替代方案）
- 代码仓库理解
- 多轮对话（不需要摘要）
- Few-shot示例池扩大

趋势：Native Long Context比RAG更简单，但成本更高。

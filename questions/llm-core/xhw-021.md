# BPE和BBPE Tokenization方法

BPE (Byte Pair Encoding) 和 BBPE (Byte-Level BPE) 是两种常见的子词分词算法，主要区别在于操作的基础单元。

**1. BPE (字符级)**
*   **原理**：基于贪心策略，反复合并语料中出现频率最高的**字符**或字符对。
*   **局限**：直接在 Unicode 字符上操作。对于多语言模型（特别是中文、日文等非拉丁语系），字符集巨大，且存在大量生僻字，导致词表膨胀且难以覆盖所有符号（如 Emoji）。

**2. BBPE (字节级)**
*   **原理**：先将所有文本转换为 **UTF-8 字节序列**，然后在这些字节上应用标准的 BPE 算法，合并频率最高的字节对。
*   **优势**：
    *   **通用性**：不依赖特定语言的字符集，任何 Unicode 字符最终都由基础字节组成，天然支持所有语言和特殊符号（表情、乱码等）。
    *   **鲁棒性**：词表大小固定且可控（基础字节只有 256 个），完美解决 OOV (Out of Vocabulary) 问题。
*   **现状**：这是目前 GPT-4、Llama 等主流大模型采用的标准分词方式。

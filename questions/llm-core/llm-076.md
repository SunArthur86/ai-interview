# 在增量预训练过程中如何设置学习率(learning rate, LR)

82. 1 学习率是什么?为何如此重要?
学习率(LR):表示在每次参数更新时,模型沿着梯度方向移动的“步长”大小.
🌵学习率的影响是什么? 学习效率
学习率过低,模型收敛慢或难以学到新知识;学习率过高,参数更新过度,损失函数
(Loss)振荡甚至发散. 稳定性
决定了模型在训练过程中的稳定程度,直接影响模型的性能和收敛速度.
82. 2 增量预训练与学习率的关系
增量预训练的特殊性 在原有模型基础上继续训练 模型已经在海量数据上完成初始预训练, 具备了较丰富的表征能力与知识.增量预训
练(Incremental Pre-training)需要在此
之上引入新数据、新领域或新分布,以更新 或扩展模型知识. **风险：** 旧知识被破坏 学习率过大可能导致对已有权重的“过 度”修改,“旧知识”被大幅覆盖,出 学习率对增量预训练过程的核心影响 平衡“旧知识保留”与“新知识学习” **LR过大：** 容易造成模型对新数据的过度拟合、旧
知识丢失严重,Loss震荡大; **LR过小：** 学习新知识缓慢或效果有限,对新分布
适配不佳. 稳定的收敛 vs. 不断探索 原模型已经基本“收敛”在较优的参数区域.增
量训练时,如果想在此区域“微调”,需要一个
相对小一些的步长,避免跳出之前的高质量解.

现“灾难性遗忘”;过小又会使得新数据无 法有效更新,学不到新知识.

82. 4 实践建议
🫐实践建议 初始值设置
参考预训练最大LR的 1/10 作为初始增量LR 的起点(例如从 3e-4 降到 3e-5 ).
按照新数据规模微调
数据规模大、分布差异大时,可略微提高 LR(如从 1e-5 ~ 3e-5 区间尝试);
数据规模小、分布相对相近时,就用相对更低 LR.
同步考虑 batch size
如果 batch size 明显增大,可酌情略增 LR,反之亦然.
分阶段、动态调整
增量预训练开始时,宜谨慎保守;观察损失和评估指标,如能平稳下降并在验证集上提升,
可小幅增大;若波动或遗忘严重,及时减小 LR.
保持旧知识的技巧
可以适度混合一定比例的旧数据(或采用知识蒸馏、EWC 等正则化),以减少大幅更改参数
导致旧知识流失. 🥥结论 •
学习率在增量预训练中是关键调控参数:既要确保模型能学到新知识,又不能大幅度毁坏
已有能力. •
通常应比初始预训练时略小,并根据数据规模、分布差异、batch size 等因素进行相应调
整. •
**核心目标：** 在合理的步长范围内,让模型平稳地吸收新知识,最大限度保留旧知识并避免
灾难性遗忘. •
**可操作方案：** 以 原预训练最大LR的 10% 左右作为起点,观察数据规模和训练曲线,配合
合适的衰减策略,动态微调,最终找到适合自身业务场景的最优LR设置.
**总结**
总结
增量预训练的过程和常规预训练或微调不同之处在于,模型已经包含重要的通用或领域知识,需要在
此基础上谨慎地更新参数.学习率过大、过小都有明显弊端,因此在设置学习率时应综合考量 初始预

训练LR、数据规模、batch size、分布差异 等因素,并配合良好的训练监控与调整策略,方能取得理
想效果.

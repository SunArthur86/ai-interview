# 为什么RLHF在多轮对话和数学推理上有用

RLHF 在多轮对话和数学推理上有效的核心原因在于 **“过程纠偏”**。

1. **纠偏 Attention 稀释**：随着多轮对话序列变长，模型解码时早期关键信息的 Attention 权重可能被稀释。强化学习通过 Reward 信号（如最终答案正确与否）回溯，可以对推理路径中的每一步进行奖惩，迫使模型在长序列中依然关注关键信息，抵消“稀释”效应。
2. **明确好与坏的界限**：数学和代码类任务通常有客观标准（如运行结果、答案唯一性）。相比于 SFT 仅拟合“好回答”，RLHF 能明确区分“好路径”和“坏路径”，即使在题目灵活多变的情况下，也能通过 Reward 指导模型向正确的推理方向收敛。
3. **细粒度优化**：Step-wise RL（如 Step-DPO）不仅看结果，还对每一步推理进行优化，这在逻辑链条强的任务中提升尤为明显。

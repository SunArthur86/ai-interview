# 为什么RLHF在多轮对话和数学推理上有用

随着多轮对话的进行,模型解码时每一步推理的attention都会因为序列长度变大而可能被“稀释”,强化学习在每一轮交互中的奖
惩可以达到“纠偏”的作用,降低这种“稀释”对good case的影响,尤其是针对数学和代码类这种标准答案相对唯一但题目又可灵
活多变的交互, 交互粒度越细,good与bad界限越清晰, 强化学习相对sft的提升就越明显.
目前看的话,stepDPO 对我接触的业务数据似乎更有用.

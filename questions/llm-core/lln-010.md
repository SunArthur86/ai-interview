# 为什么现在的LLM都是Decoder only的架构

Decoder-only的泛化性能更好:用next token prediction预训练的decoder-only模型在各种下游任务上zero-shot泛化性能
最好;另外,许多工作表明decoder-only模型的few-shot(也就是上下文学习,in-context learning)泛化能力更强
**注意力满秩问题：** 双向注意力的注意力矩阵容易退化为低秩状态,而causal attention的注意力矩阵是下三角矩阵,必然是
**预训练任务难度问题：** 纯粹的decoder-only架构 + next token prediction预训练,每个位置所能接触的信息比其他架构少,
要预测下一个token难度更高,当模型足够大,数据足够多的时候,decoder-only模型学习通用表征的上限更高
上下文学习为decoder-only架构带来的更好的few-shot性能:prompt和demonstration的信息可以视为对模型参数的隐式
微调,decoder-only的架构相比encoder-decoder在in-context learning上会更有优势,因为prompt可以更加直接地作
causal attention的隐式编码功能:打破了transformer的不变性,而带有双向attention的模型,如果不带位置编码,双向
attention的部分token可以对换也不改变表示,对语序的区分能力天生较弱
**轨迹依赖问题：** OpenAI作为开拓者深挖坑踩深,以decoder-only架构为基础摸索出了一套行之有效的训练方法和Scaling
Law,后来者鉴于时间和计算成本,自然不愿意做太多结构上的大改动,继续沿用decoder-only架构.在工程生态上,
decoder - only架构也形成了先发性优势,Megatron和flash attention等重要工具对causal attention的支持更好
• Infra层面
Decoder-Only 的进行基于 Scaling Laws 的实际训练成本低,而 Encoder-Only 压根就不适合做生成任务
**工业实现效率问题：** decoder-only支持一直复用KV-Cache,多轮对话更友好,因为每个token的表示只和它之前的输入有
关,而encoder - decoder和PrefixLM就难以做到
类似 T5 的 Encoder-Decoder 架构很难支持 PP Pipeline Parallel,那么,为什么 PP 如此重要呢?核心点在于,PP 的通信
代价最低并且容易被 overlap,而 TP Tensor Parallel 的带宽通信成本最高,跨机的 TP 会严重影响训练速度.现如今 LLM 预
训练几乎都在万卡集群上训练,而 PP 是让 LLM 可以达到千亿集群以上的一个最重要的特性.但是!进行 PP 的一个重要特
**性为：** 需要一个 规整对称的、线性顺序的网络结构,Decoder-Only 正好满足.T5 这种架构本身就不是均匀对称的,而且
Encoder 输出要发给每个 Decoder,导致 PP 时,各个 Stage 之间会产生大量的、非对称的、间隔跨多个 Stage 的数据依
赖,(其实 Megatron-LM 是支持 T5 做 PP 的,但是效果很糟糕),这样约束了 T5 的所有 Layer 都必须放在每一个 GPU
上,将 DP 开到很大的代价是巨大的通信成本,训练效率低下.所以,T5 很难 scale up 到超过 100B 的量级

# DeepSeek-V3 的 MoE 架构有什么创新?与 Mixtral 的 MoE 有什么区别

- **DeepSeek-V3 MoE 架构创新**

总参数 671B，激活参数 37B（稀疏率 ~5.5%）

- **核心创新**

1. **细粒度专家 + 共享专家**
- 256 个路由专家 + 1 个共享专家
- 每个 token 激活 8 个路由专家 + 1 个共享
- 共享专家学习通用知识，路由专家学习专精知识
- 对比 Mixtral：8 个等大专家，激活 2 个

2. **无辅助损失的负载均衡**
- 传统方法：加 auxiliary loss 强制均匀分配 → 损害模型效果
- DeepSeek 创新：为每个专家引入一个 bias 偏置项
- 如果专家被选次数偏少 → 增加 bias → 更容易被选中
- 无需额外 loss，不干扰主训练目标

3. **MLA(多头隐变量注意力)**
- 将 KV Cache 压缩到低维隐变量
- KV Cache 显存减少 93%（对比 MHA）
- 推理速度大幅提升

4. **FP8 训练**
- 首次在大规模训练中使用 FP8 混合精度
- 训练成本降至 $5.576M（对比 Llama 3 405B 的 ~$60M）

- **与 Mixtral 对比**

| 维度 | Mixtral 8x7B | DeepSeek-V3 |
|------|-------------|-------------|
| 专家数 | 8 | 256+1 |
| 激活数 | 2 | 8+1 |
| 专家粒度 | 粗(每专家7B) | 细(每专家~2B) |
| 负载均衡 | 辅助 loss | bias 调整 |
| KV优化 | 标准 MHA | MLA 压缩 |

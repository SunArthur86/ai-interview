# KV Cache是什么？它如何加速LLM推理？有哪些优化方法？

🎯 本质：KV Cache是LLM推理核心优化技术，通过缓存已计算过的Key和Value矩阵避免重复计算，将自回归生成的复杂度从O(n²)降到O(n)。

📊 为什么需要KV Cache：
Transformer自回归生成时，每生成一个新token需要计算Attention：Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V
不缓存：第n步需重新计算前n-1个token的K和V -> O(n²)
缓存KV：前n-1个token的K,V已缓存，只需计算第n个 -> O(n)

KV Cache内存开销：
一个7B模型(32层,32头,head_dim=128)：单token KV Cache = 524KB/token，4096长度 = 2GB
KV Cache可能比模型权重还大！

KV Cache优化方法：
1. PagedAttention（vLLM）：类似OS虚拟内存分页，将KV Cache分为固定大小block，按需分配减少碎片，显著提高显存利用率
2. KV Cache量化：从fp16量化到int8/int4，减少50-75%显存，精度损失小
3. KV Cache压缩：Sliding Window(只保留最近N个token)、H2O(保留重要token)、StreamingLLM(attention sink+滑动窗口)
4. Prefix Caching：多请求共享相同system prompt前缀的KV Cache
5. MQA/GQA：减少KV头的数量，Llama 2/3使用GQA

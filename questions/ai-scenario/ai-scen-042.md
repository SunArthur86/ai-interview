# 如何设计一个AI驱动的产品搜索与排序系统？结合语义搜索、个性化排序和LLM增强。

【场景分析】
AI驱动的产品搜索与排序：用户搜索商品 → 语义理解 → 多路召回 → 个性化排序 → 结果展示。

【搜索架构】
1. Query理解：
   - 意图分类：搜索商品/比价/查评价/找攻略
   - 实体识别：品牌名、产品类别、属性（颜色/尺寸）
   - Query改写：同义词扩展（「手机壳」→「手机保护套」）
   - 拼写纠错：LLM辅助纠错（「ipone」→「iphone」）
2. 多路召回：
   - 文本召回：BM25/Elasticsearch（关键词匹配）
   - 向量召回：Query Embedding → 商品向量库ANN搜索
   - 个性化召回：基于用户历史行为的协同过滤
   - 属性召回：按品牌、价格、分类过滤
   - 多路融合：RRF融合不同召回结果
3. 排序层（多阶段排序）：
   - 粗排：从万级候选快速筛到千级（轻量模型）
   - 精排：千级→百级（深度模型，考虑CTR/CVR/价格）
   - LLM重排：考虑商品多样性、搭配推荐、商业策略
   - 个性化因子：用户偏好、浏览历史、购买力
4. 结果增强：
   - LLM生成商品摘要：综合评论生成优缺点摘要
   - 智能过滤：「适合送礼」「性价比最高」等场景标签
   - 搭配推荐：「买了这个的人还买了」

【LLM在搜索中的创新应用】
- 对话式搜索：「我想买一个适合跑步的耳机，预算500以内」→ 自然语言交互
- 比较分析：「iPhone 15和Samsung S24哪个拍照好？」→ LLM综合评测数据回答
- 搜索建议：基于当前搜索推荐相关搜索词

【核心指标】
- 搜索点击率（CTR）：搜索结果→商品详情页的点击率
- 搜索转化率（CVR）：搜索→下单的转化率
- 零结果率：搜索无结果的比例（目标<2%）
- 搜索满意度：用户是否找到想要商品（NPS/回搜率）

# 如何设计一个AI驱动的产品搜索与排序系统？结合语义搜索、个性化排序和LLM增强。

### 场景分析
AI驱动的产品搜索与排序系统旨在解决传统关键词搜索无法理解用户深层意图和商品语义的问题。整体流程为：用户输入 Query → 语义理解与改写 → 多路召回（向量/文本/图） → 多阶段排序（粗排/精排/重排） → LLM 增强与结果展示。

### 详细设计

#### 1. Query 理解层
- **意图分类**：使用 BERT 或 LLM 将 Query 分类为「交易型」（买手机）、「信息型」（怎么选」）或「导航型」（官网登录）。
- **实体识别 (NER)**：识别 Brand（Apple）、Category（手机）、Attribute（256G, 黑色）。利用 LLM 处理长尾和隐式实体（如“适合送女生的” → 标签：粉色、小巧、高颜值）。
- **Query 改写**：
  - **同义词扩展**：基于领域词表或 Embedding 相似度。
  - **语义等价**：将“性能好”映射为具体的显性规格词（如“骁龙8Gen3”, “16G内存”）。
- **拼写纠错**：基于编辑距离或 LLM 的上下文感知纠错（“爱凤” → “iPhone”）。

#### 2. 多路召回层
- **向量召回**：
  - 使用双塔模型生成 Query Embedding 和 Item Embedding。
  - 底层使用 FAISS/HNSWLib 进行 ANN（近似最近邻）搜索，召回 Top K。
  - **难点**：泛化能力与语义鸿沟（用户说“便宜”，召回“低价”商品）。
- **文本召回**：基于 Elasticsearch 的 BM25 算法，保证关键词的精确匹配（如特定型号词），解决向量召回对专有名词不敏感的问题。
- **个性化召回**：基于 User-Item 交互图，使用 GraphSAGE 或 Swing 算法召回“看过还看”、“买了还买”的商品。
- **多路融合**：使用 RRF (Reciprocal Rank Fusion) 或 simple Borda Count 融合各路召回的倒排列表，平衡相关性与多样性。

#### 3. 排序层
- **粗排**：从海量库（亿级）中快速筛选。通常使用双塔模型，计算 User 和 Item 的向量点积，保留相关性最高的 1000-5000 个 item。特点是计算快，特征少。
- **精排**：对粗排结果进行复杂特征计算。
  - **特征**：User 画像（历史点击、购买力）、Item 画像（价格、销量、文本特征）、交叉特征（ID 类特征统计、实时行为）。
  - **模型**：XGBoost/LRGBM（传统强基线）或 DeepFM/DIN（深度学习）。
  - **目标**：优化 pCTR (预测点击率) 和 pCVR (预测转化率)，公式 $Score = w_1 \cdot \log(pCTR) + w_2 \cdot \log(pCVR) + w_3 \cdot Price$。
- **重排**：
  - **去重与打散**：避免同一店铺商品占据首屏（Maximal Marginal Relevance 算法）。
  - **LLM 重排**：针对复杂的自然语言 Query（如“适合露营且防水的帐篷”），LLM 可以分析精排 Top N 列表的详细参数，重新打分。由于成本高，通常仅对 Top 20 或特定场景触发。

#### 4. LLM 增强与创新
- **生成式摘要**：抽取商品详情和评论，生成“用户最在意的优缺点”摘要。
- **对比问答**：用户询问“A和B哪个好”，LLM 提取关键参数（CPU、跑分、续航）生成结构化对比表。
- **交互式澄清**：当 Query 模糊时（如“买笔记本”），系统通过 LLM 生成引导性问题（“您主要用来办公还是游戏？预算多少？”），缩小范围。

### 核心指标
- **相关性指标**：NDCG@K, MRR (Mean Reciprocal Rank)。
- **业务指标**：CTR (点击率), CVR (转化率), GMV (成交总额)。
- **体验指标**：零结果率 (< 2%), 第一屏跳出率, 用户满意度 (NPS)。

### 常见考点
1. **向量检索与倒排检索的融合策略**：如何处理向量检索带来的“语义漂移”问题？如何调整 RRF 的参数 K 值？
2. **LLM 落地的性能瓶颈**：在线推理使用 LLM 重排如何保证时延（< 200ms）？通常会采用蒸馏小模型（如 BERT-Large）或仅在离线索引/特定异步流中使用 LLM。
3. **多目标排序的权衡**：当 CTR 高的便宜商品排在前面，但 GMV 贡献低时，如何调整排序公式帕累托最优解？（涉及帕累托最优 layer-wise 归一化）。
4. **数据新鲜度**：新上架商品（冷启动）没有向量 Embedding 和用户行为数据，如何召回？（利用内容特征生成 Embedding，或探索与利用机制 E&E）。

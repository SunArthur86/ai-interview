# 如何设计一个AI内容审核系统？实时检测文本、图片、视频中的有害内容。

【场景分析】
AI内容审核系统需求：实时检测有害内容（暴力/色情/仇恨/自我伤害）、多模态审核（文本+图片+视频）、误杀控制。

【多级审核架构】
1. 规则层（Rule-based，<1ms）：
   - 敏感词库：黑名单匹配 + 变体检测（谐音、拆字、Unicode）
   - 正则规则：特定模式（手机号、身份证号、银行卡）
   - 命中即拦截，零误判但低召回
2. 模型层（ML-based，10-50ms）：
   - 分类模型：Fine-tuned BERT/RoBERTa多标签分类
   - 类别：暴力、色情、仇恨言论、自我伤害、违法信息
   - 多模态：CLIP-based模型审核图片内容
   - 置信度：高于阈值拦截，灰度区转人工
3. LLM审核层（LLM-based，100-500ms）：
   - 上下文理解：判断意图而非仅字面
   - 例：「杀毒」vs「杀人」→ 上下文消歧
   - 复杂案例：隐喻、讽刺、文化敏感
4. 人工审核层：
   - 灰度案例：模型不确定的内容
   - 申诉处理：用户对审核结果不满
   - 质量抽检：定期抽检自动审核结果

【关键设计决策】
- 审核时机：预审核（发布前）vs 后审核（发布后抽检）
- 延迟 vs 准确率：实时审核需<100ms，可能牺牲准确率
- 误杀成本：误杀正常内容 → 用户体验差；漏放有害内容 → 合规风险

【持续优化】
- 误杀样本分析：False Positive → 调整阈值/规则
- 漏放样本分析：False Negative → 补充训练数据
- 对抗样本防御：恶意用户试探规则 → 快速更新
- A/B测试：不同审核策略的效果对比

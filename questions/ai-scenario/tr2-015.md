# GraphRAG 相比传统向量 RAG，在构建“全局索引”时使用了什么核心技术？它在解决“分散信息汇总”类问题时为何表现更好？

GraphRAG 的核心在于从文档中抽取实体和关系，构建知识图谱，并使用社区发现算法（如 Leiden 算法）对图谱节点进行分层聚类，生成“社区摘要”作为全局索引。在面对需要汇总全库信息的问题（如“数据集的主要主题有哪些？”）时，传统向量 RAG 仅能检索局部相关的 Top-K 文档块，难以跨文档进行归纳；而 GraphRAG 先在社区层级进行检索和预汇总，将分散在不同文档中的相关信息在图谱结构中连接起来，再由 LLM 生成最终答案。这种层级化的索引结构使得模型能够利用图谱的结构化语义，从宏观到微观逐步推理，显著提高了对全局性问题的回答质量。

### 实战案例
在处理企业年度财报分析时，传统 RAG 只能基于切片回答局部问题，难以总结“全公司三年的研发趋势”。GraphRAG 通过构建部门、项目、资金流的关系图谱，成功将散落在多份文档中的资金流向关联起来，准确汇总出宏观趋势。

### 代码示例 (Python - 模拟社区检测与索引构建)
```python
import networkx as nx
import community as community_louvain  # 或 python-louvain/leidenalg

# 假设 G 是已构建好的实体关系图 (NetworkX Graph对象)
# G.add_edges_from([("Entity_A", "Entity_B"), ("Entity_B", "Entity_C")...])

# 1. 使用社区发现算法 (Leiden/Louvain) 进行聚类
partition = community_louvain.best_partition(G)

# 2. 为每个社区生成“社区摘要”作为全局索引
community_summaries = {}
for comm_id in set(partition.values()):
    # 获取社区内所有节点
    nodes_in_comm = [n for n, c in partition.items() if c == comm_id]
    # 实战中这里会调用 LLM 对该社区的实体和关系进行总结
    community_summaries[comm_id] = f"Summary of community {comm_id} based on nodes: {nodes_in_comm}"

# 3. 检索时先检索相关社区摘要，再深入社区内的具体节点
```

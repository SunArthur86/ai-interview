# 如何设计一个AI搜索引擎（类似Perplexity）？直接给出答案而非链接列表，支持引用溯源。

【场景分析】
AI搜索引擎（如Perplexity）= 传统搜索 + LLM理解 + RAG + 引用溯源。核心价值：直接给出答案而非链接列表。

【系统架构】
1. Query理解层：
   - 意图识别：导航型/信息型/交易型
   - Query改写：同义扩展、纠错、补全
   - 子问题分解：复杂问题 → 多个子问题并行检索
2. 多源检索层：
   - Web搜索：调用搜索API（Bing/Google）获取Top结果
   - 知识库：企业内部文档/数据库
   - 实时数据：新闻API、股价API、天气API
   - 结构化数据：知识图谱查询
3. 内容提取层：
   - 网页正文提取：trafilatura/readability
   - 关键信息抽取：从多个来源提取相关段落
   - 信息去重和冲突检测
4. 答案生成层：
   - 多源信息综合：LLM整合多个来源的信息
   - 引用标注：每个事实标注来源URL和段落
   - 结构化输出：摘要 + 详情 + 引用 + 相关问题

【与传统搜索的关键差异】
- 传统搜索：返回链接列表，用户自己找答案
- AI搜索：直接返回综合答案 + 来源引用
- 多轮追问：用户可以追问深入
- 个性化：基于用户画像调整答案深度和角度

【时效性处理】
- 实时索引：新闻类Query优先检索最新内容
- 时间标注：答案中标注信息的时间（「截至2025年6月...」）
- 多版本对比：不同时间的信息可能有冲突 → LLM判断最新有效

【质量保障】
- 引用验证：答案中的每个事实都能在来源中找到
- 多数确认：关键信息需要>=2个独立来源确认
- 幻觉检测：NLI校验 + 低置信度标记
- 用户反馈：点赞/点踩 → 优化检索和生成

# 如何设计一个AI智能客服系统？要求能处理80%的常见问题，复杂问题转人工。

【场景分析】
AI客服核心目标：准确理解用户意图、快速解决问题、智能路由人工、持续学习改进。

【系统架构】
1. 意图识别层：
   - 分类器：Fine-tuned BERT / LLM zero-shot
   - 意图分类：咨询类（查订单/查物流）、操作类（退换货/修改）、投诉类、闲聊类
   - 置信度：低于阈值（0.7）→ 转人工
2. 知识检索层（RAG）：
   - 产品FAQ库 + 政策文档 + 历史工单
   - 混合检索：向量 + 关键词 + Rerank
   - 实时查询：对接CRM/ERP系统获取用户订单信息
3. 对话管理：
   - 多轮对话：追问必要信息（订单号、商品名）
   - 情绪检测：用户愤怒/焦虑 → 优先转人工
   - 上下文记忆：同一工单内保持上下文
4. 人工协作层：
   - 智能转接：自动附带对话摘要和用户画像
   - 坐席辅助：AI实时给人工座席推荐回答
   - 质检：自动评估对话质量

【关键设计决策】
- 自助率目标：80%问题AI解决，20%转人工
- 转接时机：连续2次未解决 / 用户主动要求 / 情绪异常 / 高价值客户
- 知识更新：FAQ每周更新 → 自动重建索引

【效果优化】
- Few-shot学习：历史优质对话作为示例
- 意图未覆盖检测：识别新意图 → 补充知识库
- A/B测试：不同Prompt策略的解决率对比

【监控指标】
| 指标 | 目标值 | 说明 |
| 自助解决率 | >80% | 用户未转人工即结束 |
| 首次响应时间 | <2s | AI回复延迟 |
| 转人工率 | <20% | 转接人工的比例 |
| 用户满意度 | >4.0/5 | 会话结束评分 |
| 知识命中率 | >90% | RAG检索准确率 |

# 你在项目中的角色是什么?最大的贡献是什么

**Situation：** 面试官考察个人在项目中的真实贡献和成长.
**Task：** 清晰表达个人角色和核心贡献.
**Action：** 
1. 角色定位: 后端技术负责人 / AI 工程师.
2. 核心贡献:
**架构设计：** 主导了多路检索融合架构和 Agent 编排器的设计.
**核心模块开发：** 实现了 ReAct 引擎、检索管道、熔断器、模型路由等核心模块.
**性能优化：** 主导了全链路延迟优化,将 P50 从 6.5s 降到 2.8s.
**工程体系建设：** 建立了 CI/CD 流水线、监控告警体系、评估测试框架.
3. 技术决策: 推动了从单 Agent 到多 Agent 的架构演进,解决了工具数量增长导致的准确率下
降问题.
4. 个人成长: 从"能用 LangChain 写 Demo"到"能设计和落地企业级 AI Agent 系统".
**Result：** 
系统稳定服务 20+ 企业客户.
团队从 3 人发展到 8 人.
个人从 IC 成长为技术负责人.

## 常见考点
1. **技术选型理由**：为什么最终没有使用 LangChain，而是选择自研部分组件？（提示：LangChain 过度封装导致调试困难、性能损耗、企业定制需求）
2. **性能优化细节**：P50 从 6.5s 降到 2.8s，优化点具体在哪里？（提示：并发检索、Prompt 模板缓存、流式传输、模型量化推理加速）
3. **模型路由策略**：在模型路由中，如何根据 Question 的难度动态选择使用 7B 参数模型还是 GPT-4？（提示：基于语义向量的简单分类器或 Bert 微调模型进行预判）
4. **熔断机制**：当 LLM 接口超时或限流时，Agent 的降级策略是什么？

# Token 计数和费用统计是怎么实现的

**Situation：** LLM API 按 token 计费,需要精确统计每次调用的 token 消耗,用于成本控制和计
费.
**Task：** 实现精确的 token 计数和费用统计系统.
**Action：** 
1. Token 计数方案:
输入 Token: 使用对应模型的 tokenizer 在客户端预计算.
**OpenAI：** 使用 tiktoken 库.
**Claude：** 使用 Anthropic 的 token counter API.
**本地模型：** 使用 HuggingFace tokenizer.
输出 Token: 从 API 响应的 usage 字段获取(最准确).
2. 费用计算:
3. 统计维度:
**按租户统计：** 每个客户的 token 消耗和费用.
**按功能统计：** 意图识别、RAG 生成、摘要等各模块的消耗.
**按模型统计：** 不同模型的使用量和费用占比.
**按时间统计：** 日/周/月的消耗趋势.
4. 预算告警:
每个租户设置月度 token 预算.
消耗达到 80% 时提醒,100% 时限制(或降级到小模型).
**Result：** 
cost = (input_tokens * input_price + output_tokens * output_price) /
1000
# GPT-4o: input $2.5/1M, output $10/1M
# GPT-4o-mini: input $0.15/1M, output $0.6/1M
python

Token 计数误差控制在 2% 以内.
月度成本报告自动生成,精确到每个功能模块.
通过分析发现"会话摘要"模块是 token 消耗大户(占 25%),优化后降低了 40%.

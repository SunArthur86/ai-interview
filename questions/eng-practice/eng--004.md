# 线上用户问法和离线评测数据分布不一致时怎么办

这是 RAG/QA 系统上线后最常见的问题--离线指标很好,线上效果差.

- **根本原因**
1. 离线评测集是人工构造的,问法规范、意图明确
2. 线上用户输入口语化、模糊、含错别字、混合意图
3. 长尾问题在离线集中覆盖不足

- **解决方案**
1. **线上日志回流**:收集真实用户 query → 标注 → 补充评测集
2. **Query 改写**:对线上 query 做纠错、扩展、标准化
3. **意图分类前置**:先分类意图再路由到不同处理逻辑
4. **数据增强**:用 LLM 生成多种问法变体扩充评测集
5. **分层数据集**:按 query 类型(事实型/推理型/多跳型)分别评测
6. **线上 A/B 测试**:以线上真实指标为准,不依赖离线指标
7. **持续监控**:建立 query 分布漂移检测机制

- **经验法则**:离线评测集应该至少包含 20% 的'真实线上日志'子集.

# 请详细说明 DeepSpeed ZeRO 的三个阶段及其工作原理.

- **ZeRO (Zero Redundancy Optimizer)** 通过切分训练状态消除数据并行中的冗余.

**训练状态三类：** 
1. Optimizer States (Os):Adam 的 momentum + variance,约 12 bytes/param(FP16 混合精度)
2. Gradients (G):约 2 bytes/param(FP16)
3. Parameters (P):约 2 bytes/param(FP16)

**总计：** ~16 bytes/param → 7B 模型需 ~112GB(仅训练状态)

- **ZeRO-1: 切分优化器状态 (Os)**
- 每个 GPU 只存 1/N 的优化器状态
- 显存节省:~4x(N=GPU 数)
- 通信开销:与标准 DP 相同(AllReduce 梯度)

- **ZeRO-2: 切分梯度 (Os+G)**
- 在 ZeRO-1 基础上,每个 GPU 只存 1/N 的梯度
- 显存节省:~8x
- 通信开销:Reduce-Scatter 替代 AllReduce,通信量略减

- **ZeRO-3: 切分参数 (Os+G+P)**
- 每个GPU 只存 1/N 的参数
- 显存节省:~Nx(理论上无限扩展)
- 代价:前向和反向传播时需要 All-Gather 收集参数,通信量大

- **实际选择**:
- ZeRO-1/2:通信开销小,适合中小模型
- ZeRO-3:显存极致压缩,适合超大模型(>30B)
- ZeRO-Offload:进一步将状态卸载到 CPU 内存

# 请详细说明 DeepSpeed ZeRO 的三个阶段及其工作原理.

- **ZeRO (Zero Redundancy Optimizer)** 通过切分训练状态消除数据并行中的冗余。

**训练状态三类及内存占用 (FP16 混合精度训练):** 
1. **Optimizer States (Os)**: Adam 的 momentum + variance，约 12 bytes/param (FP16 权重 + FP32 Copy + FP32 Momentum + FP32 Variance)。
2. **Gradients (G)**: 约 2 bytes/param (FP16)。
3. **Parameters (P)**: 约 2 bytes/param (FP16)。

**总计：** ~16 bytes/param。单卡 80GB 显存训练 7B 模型时：112GB (状态) + 14GB (权重) ≈ 126GB，远超单卡限制。

- **ZeRO-1: 切分优化器状态**
- **原理**: 将 Os 按维度平均切分到 N 个 GPU，每卡只存 1/N。
- **通信**: 反向传播后，梯度 `All-Reduce` 到所有卡，然后每卡只更新自己负责的那部分 Os。
- **显存节省**: 节省 ~4x 内存（去掉了 Os 占用）。
- **通信开销**: 与标准 DP (Data Parallel) 相同，仅增加了少量的 `Reduce-Scatter` 同步梯度的开销。

- **ZeRO-2: 切分梯度 + 优化器状态**
- **原理**: 在 ZeRO-1 基础上，每卡只存 1/N 的梯度。
- **通信**: 反向传播时，直接将梯度 `Reduce-Scatter` 到对应卡，不再做全量 `All-Reduce`。更新参数时，做 `All-Gather` 同步更新后的参数。
- **显存节省**: ~8x (去掉了 G + Os)。
- **通信量**: 略低于 ZeRO-1，因为梯度不需要全量同步给所有卡。

- **ZeRO-3: 切分参数 + 梯度 + 优化器状态**
- **原理**: 每个GPU 只存 1/N 的参数、梯度和优化器状态。
- **通信**: 前向传播前，`All-Gather` 获取当前 Layer 的完整参数；反向传播后，立即 `Reduce-Scatter` 梯度并释放参数。
- **显存节省**: 线性增长，理论上 ~Nx。单卡可训练万亿参数模型（受限于通信和计算效率）。
- **代价**: 通信量显著增加。由于需要按 Layer 获取参数，通信与计算重叠 的优化难度加大。

**架构对比图**:
```
标准 DP (Data Parallel):
[GPU0] Full(P,G,Os) ─┐
[GPU1] Full(P,G,Os) ─┼─ AllReduce(Gradients) ─► 每卡独立更新参数
[GPU2] Full(P,G,Os) ─┤
[GPU3] Full(P,G,Os) ─┘

ZeRO-2 (Stages 1+2):
[GPU0] P/1, G/1, Os/1 ─┐
[GPU1] P/1, G/1, Os/1 ─┼─ ReduceScatter(G) → Update Os/1 → AllGather(P)
[GPU2] P/1, G/1, Os/1 ─┤
[GPU3] P/1, G/1, Os/1 ─┘

ZeRO-3 (Stage 3):
[GPU0] P/1, G/1, Os/1 ─┐
[GPU1] P/1, G/1, Os/1 ─┼─ AllGather(仅当层计算时) → Compute → Release
[GPU2] P/1, G/1, Os/1 ─┤
[GPU3] P/1, G/1, Os/1 ─┘
```

- **实际选择策略**
- **ZeRO-1/2**: 通信开销小，适合中小模型 (<= 10B) 或网络带宽受限的环境。DeepSpeed-MoE 通常使用此阶段。
- **ZeRO-3**: 显存极致压缩，适合超大模型 (>30B)，但对 NVLink 带宽要求极高，否则会受限于通信。
- **ZeRO-Offload**: 利用 CPU 内存和 NVMe 存储进一步卸载状态，牺牲速度换取更大的模型容量。

- **## 常见考点**
  1. **通信模式区别**：ZeRO-2 和 ZeRO-3 在前向传播时的通信操作分别是什么？（ZeRO-2 已有全量参数，无需通信；ZeRO-3 需要 All-Gather）。
  2. **连续内存管理**：ZeRO-3 如何处理碎片化的显存分配？（使用 `contiguous_memory_manager` 或 `tensor_parity` 确保内存连续）。
  3. **与模型并行 (TP) 的对比**：TP 切分的是同一层的计算（需要频繁通信），ZeRO-3 切分的是模型参数存储（计算时临时聚合）。为什么大模型训练常混合使用二者？
  4. **offload 机制**：`cpu_offload` 对训练速度的具体影响有多大？通常哪些状态适合 offload？

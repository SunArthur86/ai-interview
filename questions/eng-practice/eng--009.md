# 微调后指标提升,但线上回复风格变差,你怎么排查和解决

- **典型场景**:离线 BLEU/accuracy 提升,但用户反馈回复'生硬'、'格式乱'、'不自然'.

- **排查步骤**

1. **定位变化范围**
- 对比微调前后的输出(同一 prompt)
- 用 diff 分析:是语气变了?格式变了?长度变了?
- 统计:平均回复长度、句子结构多样性

2. **检查训练数据**
- 微调数据的风格是否与目标一致?
- 是否混入了机器翻译/爬虫数据(质量低)?
- 数据分布是否偏斜(全是短回答?)

3. **检查训练超参**
- 过拟合:验证集 loss 上升?→ 减少 epoch
- 灾难遗忘:通用能力下降?→ 降低 lr 或用 LoRA
- 学习率过大:权重变化过激

4. **检查评测指标**
- 评测指标是否与用户体验一致?
- BLEU 高不代表用户满意
- 应增加'人工满意度评分'或'LLM-as-judge'

- **解决方案**
1. **混合数据**:微调数据中混入 10-20% 通用对话数据
2. **LoRA 替代 FFT**:减少对原模型能力的破坏
3. **DPO/RLHF 补充**:微调后再做偏好对齐
4. **分阶段训练**:先 SFT 任务能力 → 再 DPO 风格偏好
5. **在线 A/B**:灰度发布,实时监控用户满意度

- **经验**:风格问题比能力问题更难修复,优先用偏好数据而非 SFT 解决.

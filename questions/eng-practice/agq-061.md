# 怎么做的测试?覆盖率多少

**Situation：** AI Agent 系统的测试比传统软件更复杂:LLM 输出不确定、工具调用涉及外部服
务、端到端流程长.
**Task：** 建立多层次的测试体系,确保系统质量.
**Action：** 
1. 测试金字塔:
E2E 测试(10%):完整的用户交互流程
集成测试(30%):组件间的交互测试
单元测试(60%):单个函数/类的测试

2. 单元测试(覆盖率 85%):
工具函数、数据处理、格式解析等确定性逻辑.
Mock LLM 调用,使用固定的输入输出对测试.
**框架：** pytest + pytest-asyncio.
3. 集成测试:
**检索管道测试：** 给定 query,验证检索结果的相关性.
**工具调用测试：** 验证 Function Calling 的参数传递和结果解析.
使用 testcontainers 启动 Milvus、Redis 等依赖服务.
4. 评估测试(AI 特有):
建立评估数据集(500+ 条标注的 query-answer 对).
**自动化指标评估：** 
**检索：** Recall@10、Precision@5、NDCG@5
**生成：** BERTScore、ROUGE-L、人工评分
**端到端：** 任务完成率、用户满意度
使用 RAGAS 框架做 RAG 管道评估.
在 CI 中每次提交自动跑评估,防止回归.
5. 压力测试:
每月一次全面压测(详见架构类 Q9).
每次重大变更后做回归压测.
**Result：** 
单元测试覆盖率 85%,核心模块 95%.
评估测试在 CI 中 100% 自动化运行.
上线后的严重 bug 数量降低 70%.

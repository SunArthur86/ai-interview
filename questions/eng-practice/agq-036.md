# 怎么做的数据预处理和数据清洗

**Situation：** 企业文档质量参差不齐:有 OCR 错误、格式混乱、重复内容、过时信息等.低质量
数据直接影响检索和生成质量.
**Task：** 建立数据预处理管道,确保入库数据的质量.
**Action：** 
1. 数据清洗管道:
2. 清洗规则:
**去除噪音：** 页眉页脚、水印文字、无意义空白.
**格式标准化：** 统一编码(UTF-8)、统一标点(全角/半角).
OCR 纠错: 基于语言模型的拼写纠错(如"己术"→"技术").
**敏感信息脱敏：** 手机号、身份证号等自动打码.
3. 去重策略:
**完全重复：** MD5 hash 去重.
**近似重复：** MinHash + LSH 算法,Jaccard 相似度 > 0.85 视为重复.
**跨版本去重：** 同一文档的不同版本,只保留最新版本.
4. 质量评估打分:
文本长度(太短的 chunk 信息量不足).
语义完整性(句子是否完整).
信息密度(关键词密度、实体密度).
原始文档 → 格式转换 → 文本提取 → 清洗 → 去重 → 质量评估 → 分块 → 入库

分数低于阈值的 chunk 标记为低质量,不参与检索.
**Result：** 
数据清洗后,检索准确率提升 8%.
去重减少了 15% 的存储空间.
低质量 chunk 过滤后,LLM 生成的幻觉率降低 12%.

# 大模型部署时,瓶颈通常在算力、显存、带宽还是网络?如何排查

- **四大瓶颈及排查方法**

1. **显存 (VRAM)** - 最常见瓶颈
- 症状:OOM、无法扩大 batch
- 原因:模型权重 + KV Cache + 激活值
- 排查:`nvidia-smi` 查看 GPU 显存利用率
- 解决:量化(INT8/INT4)、PagedAttention、模型并行

2. **算力 (Compute/SM)**
- 症状:GPU 利用率高但吞吐低
- 原因:矩阵乘法计算量过大
- 排查:NVIDIA Nsight Systems 分析 kernel 占比
- 解决:Flash Attention、Tensor Core 优化、模型量化

3. **访存带宽 (Memory Bandwidth)**
- 症状:Decode 阶段 GPU 利用率低但延迟高
- 原因:逐 token 生成时反复读取权重,访存>>计算
- 排查:`nvidia-smi dmon` 看 mem utilization
- 解决:增大 batch(摊薄访存)、量化(减少读取量)

4. **网络 (Network)**
- 症状:分布式训练/推理时延迟波动大
- 原因:Tensor Parallel 跨 GPU 通信、KV Cache 传输
- 排查:`nccl-tests` 测 AllReduce 带宽
- 解决:NVLink/NVSwitch、Pipeline Parallel 减少通信

- **排查优先级**:先看显存→再看算力→再看带宽→最后看网络.

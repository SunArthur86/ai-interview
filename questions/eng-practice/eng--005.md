# 大模型推理的首 Token 延迟(TTFT)和整体吞吐量分别受什么因素影响?如何优化

- **首 Token 延迟 (Time To First Token, TTFT)**

**影响因素：** 
1. **Prefill 计算**:处理整个 prompt 的 KV Cache 计算(与 prompt 长度成正比)
2. **模型加载到 GPU**:大模型权重传输
3. **排队等待**:请求在 batch 队列中等待
4. **网络延迟**:客户端到服务端 RTT

**优化策略：** 
- 使用 Flash Attention 加速 prefill
- Prefix Caching:对共享 system prompt 缓存 KV
- 批量 prefill:多个请求的 prefill 合并
- 模型量化减少计算量

- **整体吞吐量 (Throughput, tokens/s)**

**影响因素：** 
1. **Decode 阶段**逐 token 生成,是访存瓶颈
2. **Batch Size**:越大吞吐越高但显存限制上限
3. **KV Cache 显存**:限制最大并发数

**优化策略：** 
- **连续 Batching (Continuous Batching)**:动态组合不同长度的请求
- **PagedAttention (vLLM)**:分页管理 KV Cache,减少碎片
- **Tensor Parallelism**:多 GPU 并行减少单 token 延迟
- **投机解码 (Speculative Decoding)**:小模型草稿 + 大模型校验

- **经验公式**:TTFT ∝ prompt_length × model_size / GPU_FLOPS

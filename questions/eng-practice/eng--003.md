# 全量微调(Full Fine-Tuning)与参数高效微调(LoRA/QLoRA)的区别是什么?各自适用什么场景

- **全量微调(FFT)**:更新模型所有参数.
- 优点:效果上限高,能充分学习新任务
- 缺点:显存大(需存全量梯度+优化器状态)、易灾难遗忘
- 适用:有充足 GPU 资源、任务与预训练差异大、需要最优效果

- **LoRA**:冻结原始权重,只训练低秩矩阵 A·B(rank 通常 8~64).
- 优点:训练参数减少 99%+,显存友好,可切换多个 LoRA adapter
- 缺点:表达能力有上限,极端任务效果可能不如 FFT
- 适用:资源受限、多任务管理、快速实验

- **QLoRA**:在 LoRA 基础上对原始权重做 4-bit 量化.
- 优点:显存极低(7B 模型只需 ~6GB)
- 缺点:量化可能引入精度损失
- 适用:消费级 GPU、原型验证

**选择决策树：** 
- 资源充足 + 效果优先 → FFT
- 资源受限 + 多任务 → LoRA
- 极端受限 + 单卡 → QLoRA

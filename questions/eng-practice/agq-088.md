# Prompt Engineering 有哪些关键技巧

**回答：**

**1. 结构化 Prompt 设计：**
- 明确角色定义。
- 清晰的任务描述。
- 输出格式约束。
- 必要的限制条件。
- 高质量示例。

**2. 关键技巧：**
- **Chain of Thought (CoT)：** 引导模型逐步推理 ("请一步一步分析这个问题")。
- **Few-shot Learning：** 提供 2-3 个高质量示例，模仿输入输出格式。
- **角色扮演：** "你是一个拥有 10 年经验的企业架构师"。
- **否定指令：** "不要编造信息" 比 "请确保准确" 更有效。
- **输出锚定：** 在 Prompt 末尾给出输出的开头，引导格式。

**3. 高级技巧：**
- **Self-consistency：** 多次采样后投票，提高结果可靠性。
- **ReAct Prompting：** Thought-Action-Observation 循环，结合推理与工具调用。
- **Instruction Hierarchy：** 系统指令 > 用户指令，防止 Prompt 注入。

**4. 常见陷阱：**
- Prompt 过长导致注意力分散。
- 矛盾的指令导致模型困惑。
- 缺少格式示例导致输出不可解析。

# RAG 和微调怎么选?各自优劣

**回答：**

1. **RAG (检索增强生成)：**
   - **优势：** 知识可实时更新 (改知识库即可)；可解释性好 (有来源引用)；减少幻觉；无需 GPU 训练资源。
   - **劣势：** 受检索质量影响大；上下文窗口限制；推理延迟增加；无法改变模型本身能力 (如风格)。

2. **微调：
   - **优势：** 内化知识，不依赖外部检索；可以改变行为和风格；推理无额外延迟；特定任务精度高。
   - **劣势：** 知识更新需重新训练；需高质量标注数据；需 GPU 资源；可能导致灾难性遗忘。

3. **选择策略：**
   - **知识频繁更新 / 需要引用来源：** 推荐用 RAG。
   - **特定领域术语 / 风格 / 模型能力提升：** 推荐用微调。
   - **数据量小 (< 1000 条)：** 推荐用 RAG (微调数据不足)。
   - **最佳实践：** RAG + 微调 (互补)。

4. **实际项目中的做法：**
   - 主体方案用 **RAG**：处理知识库问答、文档检索。
   - 辅以 **微调**：使用 LoRA 让模型更好地遵循指令格式、生成专业风格。
   - 效果叠加：微调后的模型 + RAG，通常比纯 RAG 效果更好。

# 如何评估Agent系统的质量?除了任务成功率还有哪些指标

- **Agent评估维度:**

| 维度 | 指标 | 说明 |
|------|------|------|
| **效果** | 任务成功率 | 最终任务是否完成 |
| **效率** | 平均步数 | 完成任务用了几步 |
| **效率** | Token消耗 | 总共花了多少Token |
| **效率** | 工具调用次数 | 调了多少次工具 |
| **鲁棒性** | 错误恢复率 | 工具失败后能否恢复 |
| **成本** | 每任务成本 | API费用 |
| **轨迹** | 步骤合理性 | 每步是否是最优选择 |
| **安全** | 安全违规率 | 是否执行了危险操作 |

- **评估方法:**

1. **端到端测试**
- 准备测试集(输入+期望输出)
- Agent自主完成任务,判断是否成功

2. **轨迹分析**
- 记录Agent每一步的Thought/Action/Observation
- 人工或LLM评估每步的合理性

3. **基准测试**
- AgentBench / SWE-Bench / GAIA / WebArena
- 标准化的Agent能力评测

4. **A/B测试**
- 不同Agent配置的线上A/B对比

# 如何系统性地评估 RAG 系统的效果?有哪些关键指标

- **RAG 评估** 需要从检索和生成两个维度分别度量.

- **Ragas 框架核心指标:**

- **1. Faithfulness(忠实度):** 生成的答案中有多少比例可以从检索到的上下文中推导出来.越高说明幻觉越少.

- **2. Answer Relevancy(答案相关性):** 生成的答案与用户查询的相关程度.越高越好.

- **3. Context Precision(上下文精确率):** 检索到的上下文中有多少是真正相关的.衡量检索质量.

- **4. Context Recall(上下文召回率):** 所有相关文档中被检索到的比例.衡量检索完整性.

- **评估方法:**
- 人工标注 Golden Set:最准但成本高
- LLM-as-Judge:用 GPT-4 评估答案质量,成本低但有偏差
- Ragas / TruLens:自动化评估框架

- **常见问题诊断:**
- Faithfulness 低 -> 检查 Prompt 是否要求引用来源
- Context Recall 低 -> 检查 Chunk 策略和 Embedding 质量
- Answer Relevancy 低 -> 检查生成 Prompt

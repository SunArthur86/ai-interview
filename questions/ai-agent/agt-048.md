# 状态管理与同步

状态管理涉及任务进度、共享事实、用户约束、工具中间结果等。同步指在多 Agent 并发读写时保持数据的一致性与可恢复性。

### 1. 原理详解

*   **全局状态共享**：维护单一 Source of Truth（如数据库行 + 版本号），各 Agent 只通过 API 更新，避免各自维护矛盾的副本。
*   **状态机管理**：将任务阶段划分为显式状态（如 `PLANNING` → `CODING` → `REVIEW` → `DONE`），拒绝非法的状态迁移，利于断点续跑和流程控制。
*   **事件驱动**：状态变更以**事件**形式发布（常与 Pub-Sub/队列结合），Agent 订阅自己关心的事件，而非轮询黑板，实现解耦。

### 2. 面试问答

**Q：多 Agent 系统为什么推荐状态机而不是纯自然语言传递一切？**

**A：** 自然语言灵活但难校验、难回放、难测试。状态机提供 **可验证迁移、清晰终止条件、可观测指标**（如卡在某个阶段多久）。自然语言可作为附件说明，不应是唯一的真相来源。

**追问应对：** 若问「状态存在哪？」——答：进程内只适合 Demo；生产环境通常使用 Redis/DB 并配合乐观锁。

### 3. 代码示例

下面使用 `enum` + 显式迁移逻辑演示小型状态机：

```python
from enum import Enum, auto
from dataclasses import dataclass

class Phase(Enum):
    INIT = auto()
    PLAN = auto()
    EXEC = auto()
    VERIFY = auto()
    DONE = auto()

# 定义合法的状态流转路径
ALLOWED = {
    Phase.INIT: {Phase.PLAN},
    Phase.PLAN: {Phase.EXEC},
    Phase.EXEC: {Phase.VERIFY},
    Phase.VERIFY: {Phase.DONE, Phase.EXEC}, # 不通过可打回重做
    Phase.DONE: set(),
}

@dataclass
class TaskState:
    phase: Phase = Phase.INIT

    def move(self, nxt: Phase) -> None:
        if nxt not in ALLOWED[self.phase]:
            raise ValueError(f"Illegal transition: {self.phase} -> {nxt}")
        self.phase = nxt
```

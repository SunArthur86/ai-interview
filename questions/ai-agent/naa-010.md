# 如何全面评估一个 Agent 系统的质量?有哪些专门的 Agent 基准

- **Agent 评估的多维度框架**

- **1. 任务完成率 (Task Success Rate)**
- 最基本的指标:任务是否正确完成
- 评测方式:人工标注 / 规则判定 / LLM-as-Judge

- **2. 效率指标**
- **步数效率**:完成任务用了多少步(越少越好)
- **Token 消耗**:完成任务用了多少 token
- **时间延迟**:首 token 延迟 + 总执行时间
- **成本效率**:每完成任务的成本($)

- **3. 过程指标**
- **工具调用准确率**:选对工具 / 传对参数
- **规划合理性**:步骤是否逻辑通顺
- **错误恢复率**:遇到错误后能否恢复

- **4. 安全指标**
- **越狱成功率**:对抗性 prompt 测试
- **信息泄露率**:是否泄露 system prompt / 敏感数据
- **权限越界率**:是否执行了未授权操作

- **专门 Agent 基准**

| 基准 | 评测维度 | 任务类型 |
|------|---------|---------|
| **AgentBench** | 综合能力 | 操作系统/数据库/游戏/网页 |
| **GAIA** | 通用 AI 助手 | 多步推理 + 工具使用 |
| **SWE-bench** | 代码修复 | GitHub PR 修复 |
| **WebArena** | 网页操作 | 真实网站交互 |
| **ToolBench** | 工具调用 | 16000+ API 调用 |
| **τ-bench** | 多轮对话 | 客服/预订场景 |

- **生产级评估最佳实践**
1. 建立 100-500 条 golden test set
2. 每次迭代跑回归测试
3. 持续从线上日志收集 bad case
4. 用 LLM-as-Judge 做批量评估(配合人工抽检)

# 为什么说 Agent = LLM + Planning + Memory + Tools?缺一块会怎样

缺 Planning：容易变成「只会接话」的聊天，长任务易跑偏或无法拆解完成。
缺 Memory：长对话会丢失线索，多会话无法延续用户偏好与任务状态。
缺 Tools：只能「空谈」，无法查询实时信息、执行代码或修改系统状态。

LLM 仍是中枢，但单靠 LLM 没有外环则不是完整 Agent。

# LATS(Language Agent Tree Search)是如何将 MCTS 引入 Agent 的?它比 ReAct 强在哪里

- **LATS (Language Agent Tree Search)** 将蒙特卡洛树搜索 (MCTS) 与 LLM Agent 结合,通过系统性探索多条路径来提升决策质量.

- **核心思想**:不是线性执行 ReAct(Thought→Action→Observation),而是构建搜索树,探索多个可能的行动分支.

- **四个阶段(类比 MCTS)**

1. **Selection(选择)**:用 UCT 公式选择最有潜力的节点
2. **Expansion(扩展)**:LLM 生成 K 个候选行动
3. **Simulation(模拟)**:执行行动,获取环境反馈
4. **Backpropagation(回传)**:用 LLM 自评更新路径价值

- **与 ReAct 的对比**

| 维度 | ReAct | LATS |
|------|-------|------|
| 探索方式 | 线性(单条路径) | 树搜索(多条路径) |
| 回溯 | ❌ 不回溯 | ✅ 可回溯到更优分支 |
| 成本 | 1x | ~5-10x(多路径探索) |
| 效果 | 基线 | 显著提升(尤其难任务) |

- **适用场景**
- 复杂推理(数学证明、代码调试)
- 决策空间大的任务
- 允许较高延迟的场景

- **局限**:计算成本高(多轮 LLM 调用),不适合实时对话.

# GraphRAG 的原理是什么?它与传统向量 RAG 有什么区别

- **GraphRAG** 是微软提出的知识图谱增强 RAG 方案,通过构建结构化知识图谱来增强检索.

- **核心原理:**
1. 文档预处理阶段:用 LLM 从文档中抽取实体和关系
2. 构建知识图谱:实体为节点,关系为边
3. 社区发现(Community Detection):将相关实体聚类
4. 生成社区摘要:为每个社区生成高层摘要
5. 查询时:先匹配社区,再在社区内精确检索

- **与传统向量 RAG 的区别:**

- **传统向量 RAG:** 文档切块 -> 向量化 -> 相似度检索.擅长局部事实查询,但无法回答全局性问题(如整个文档的主题是什么).

- **GraphRAG:** 通过知识图谱捕获实体间关系,支持多跳推理.能回答全局性问题.

- **适用场景:**
- 需要多跳推理的复杂问答
- 实体关系密集的领域(金融、法律、医学)
- 全局摘要类查询

- **代价:** 构图成本高(需要大量 LLM 调用),更新维护复杂.

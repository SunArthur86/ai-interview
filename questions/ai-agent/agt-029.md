# MCP 协议（Model Context Protocol）

### MCP 协议（Model Context Protocol）

**定义**：由 Anthropic 推动的开放标准，用于 AI 应用与外部数据源/工具之间的统一通信协议。可理解为“AI 界的 USB”。

**核心组件**：
1. **MCP Server**：暴露工具/资源的独立进程（如连接 GitHub）。
2. **MCP Client**：运行在 Host 内，负责协议转换与连接管理。
3. **Transport**：传输层，支持 stdio（本地子进程）或 HTTP/SSE（远程）。

**MCP vs Function Calling**：
*   **FC**：解决模型“如何表达”调用意图（API 层面）。
*   **MCP**：解决工具“如何暴露”与“如何连接”（系统层面）。
*   **关系**：互补。Host 可将 MCP Server 的工具映射为 FC 格式发给模型。

**企业价值**：
*   **复用**：一个 Server 供多个 Agent（IDE、桌面、Web）使用。
*   **治理**：集中管理权限、审计与版本。
*   **安全**：工具独立进程，崩溃不拖垮主应用。

**面试金句**：
*   **Q：为什么企业愿意接 MCP？**
*   **A**：降低重复建设，统一安全与观测，便于平台组与业务组分工。

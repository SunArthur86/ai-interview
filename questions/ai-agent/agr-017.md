# Rerank(重排序)在 RAG 中解决什么问题?如何选择重排序模型

- **Rerank** 是在初步检索后对候选结果进行精细排序的步骤.

- **解决的问题:**
- 向量检索(ANN)速度快但精度有限,top-k 中可能混入不相关结果
- 向量相似度高不等于与查询真正相关

- **Rerank 工作原理:**
1. 向量检索召回 top-50 或 top-100 候选
2. Rerank 模型对每个候选与查询做 cross-encoder 计算(而非 bi-encoder)
3. Cross-encoder 能捕获查询和文档的深层交互关系
4. 输出精确的相关性分数,重排后取 top-5

- **为什么 Cross-encoder 更准:**
- Bi-encoder(向量检索):查询和文档独立编码,最后算余弦相似度.快但浅.
- Cross-encoder:查询和文档拼在一起送入模型,能捕获细粒度交互.慢但准.

- **选型:**
- Cohere Rerank v3:商用效果最好
- bge-reranker-large:开源首选
- jina-reranker:轻量级方案

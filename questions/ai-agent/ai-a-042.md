# Manus(通用Agent)的设计理念是什么?它和Devin/Claude Code有什么区别

- **Manus核心:通用任务Agent**

**不限于编程,覆盖：** 数据分析、报告撰写、研究、PPT制作等.

- **架构特点:**

1. **虚拟机沙箱** - 每个任务分配独立VM
- 完整Linux环境
- 预装工具(浏览器/Office/Python等)
- 隔离安全

2. **多模态交互** - 截图+DOM+文件系统

3. **异步执行** - 后台执行长任务,完成后通知用户

- **vs Devin/Claude Code:**

| | Devin | Claude Code | Manus |
|--|-------|-------------|-------|
| 领域 | **软件工程** | **编程辅助** | **通用任务** |
| 沙箱 | Docker | 本地终端 | **完整VM** |
| 交互 | 对话+IDE | CLI/TUI | **Web+异步** |
| 产出 | 代码+PR | 代码 | **文档/PPT/数据/代码** |
| 自主性 | 高 | 中(人在环) | **高(异步执行)** |

- **Manus的启示:**
- Agent的能力上限 = 工具链的丰富度
- VM级沙箱让Agent能做任何人类在电脑上做的事
- 异步模式让Agent可以处理小时级的长任务

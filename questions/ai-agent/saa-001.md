# ReAct框架的原理是什么？

ReAct（Reasoning + Acting）让LLM交替进行推理（Thought）和行动（Action）。

循环流程：
1. Thought：LLM分析当前状态，决定下一步
2. Action：调用工具（搜索、计算、API）
3. Observation：获取工具返回结果
4. 重复直到得出最终答案

示例：
Thought: 用户问北京天气，我需要查询天气
Action: search_weather('北京')
Observation: 晴, 25°C
Thought: 已获取天气信息，可以回答了
Answer: 北京今天晴，25°C

优势：
- 推理过程可见，可调试
- 工具调用基于推理结果，更准确

局限：
- 线性执行，不擅长复杂规划
- 依赖LLM的推理能力
- 容易陷入循环

对比：Plan-and-Execute先规划全流程再执行，适合复杂任务。

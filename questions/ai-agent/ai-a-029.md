# 多Agent辩论(Debate)如何提升推理质量?有什么代价

- **Agent辩论机制:**

多个Agent针对同一问题独立给出答案,然后互相审视对方的推理过程,最终达成共识.

- **流程:**
```
问题 → Agent A给出答案+推理
→ Agent B给出答案+推理
→ Agent A审视B的推理,修正自己的
→ Agent B审视A的推理,修正自己的
→ ... N轮 ...
→ 最终答案(投票/综合)
```

- **效果:**
- 数学推理准确率提升5-15%
- 减少单个Agent的确认偏差
- 特别适合有明确正确答案的任务

- **代价:**
- **Token消耗 × Agent数量 × 轮数**
- 3个Agent × 3轮 = 9倍单Agent成本
- 延迟大幅增加

- **实践建议:**
1. 用2-3个Agent(不需要太多)
2. 2-3轮辩论足够(边际收益递减)
3. 不同Agent用不同temperature/prompt增加多样性
4. 最终用裁判Agent或投票决定答案

- **注意:** 辩论可能导致「回音壁」(互相强化错误答案),需要多样性

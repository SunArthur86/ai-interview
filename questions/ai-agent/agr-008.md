# RAG知识库索引频繁更新时,如何保证检索一致性

- **一致性挑战:**

用户提问的瞬间,文档可能正在更新 → 检索到旧/新混合数据.

- **解决方案:**

1. **双缓冲(Double Buffering)**
- 维护两个索引版本:active + standby
- 新文档写入standby → 构建完成 → 原子切换
- 检索始终读active

2. **版本号标记**
- 每个文档/chunk带version字段
- 检索时可以指定版本范围

3. **增量upsert + 软删除**
- 新文档: INSERT
- 更新: 软删除旧 + INSERT新
- 定期compact清理

4. **写后读一致性**
- 更新后立即刷新索引
- Qdrant/Milvus支持force refresh

5. **CDC(Change Data Capture)**
- 监听数据变更流
- 异步更新索引
- 最终一致性

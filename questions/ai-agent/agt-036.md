# 短期记忆

短期记忆一般指当前会话或任务周期内模型可直接访问的信息，受 Token 上限与延迟约束。

### 核心形式
1. **会话上下文**：按时间拼接完整历史。信息保真度高，但随对话变长会导致成本增加、延迟升高及模型注意力分散。
2. **滑动窗口**：仅保留最近 K 轮或 N 个 Token。成本可控，但可能丢失早期关键约束。

### Token 管理
由于计费和上下文窗口限制，需进行预算分配：System 指令 > 安全策略 > 工具定义 > 高优先级记忆 > 近期对话。超限时常采用截断、摘要或检索增强处理。

### 代码逻辑（示意）
```python
import tiktoken

class WindowedMemory:
    def __init__(self, max_tokens=3000, window_messages=20):
        self.max_tokens = max_tokens
        self.window_messages = window_messages
        self.enc = tiktoken.encoding_for_model("gpt-4o")
        self.messages = []

    def add(self, role, content):
        self.messages.append({"role": role, "content": content})
        # 1. 限制消息条数
        if len(self.messages) > self.window_messages:
            self.messages = self.messages[-self.window_messages:]
        # 2. 限制 Token 总数
        self._shrink_to_budget()

    def _count_tokens(self):
        return len(self.enc.encode("\n".join(f"{m['role']}: {m['content']}" for m in self.messages)))

    def _shrink_to_budget(self):
        while self.messages and self._count_tokens() > self.max_tokens:
            self.messages.pop(0) # 丢弃最早一条
```

# 短期记忆

短期记忆一般指当前会话或任务周期内模型可直接访问的信息，受 Token 上限与延迟约束。

### 核心形式
1.  **会话上下文**：按时间拼接完整历史。信息保真度高，但随对话变长会导致成本增加、延迟升高及模型注意力分散（"迷失中间"现象，Lost in the Middle）。
2.  **滑动窗口**：仅保留最近 K 轮或 N 个 Token。成本可控，但可能丢失早期关键约束。
3.  **摘要缓存**：将旧对话压缩为摘要，仅保留摘要和近期对话。这是滑动窗口的升级版，能保留更久远的核心意图。

### Token 管理
由于计费和上下文窗口限制，需进行预算分配：System 指令 > 安全策略 > 工具定义 > 高优先级记忆 > 近期对话。超限时常采用截断、摘要或检索增强处理。

### 实现细节与边界
*   **System Prompt 优先级**：必须始终保存在上下文最顶部，即使被截断也不能丢弃。
*   **Token 计算优化**：Message 中的 JSON 键（如 `role`, `content`）也会占用 Token，需计入预算（通常单条 Message 消耗约 5-10 tokens 的额外开销）。
*   **滑动策略**：通常保留 `System` + `Recent N`，或者在保留 `System` 的基础上，保留 `Summary` + `Recent N`。

### 架构图：Token 预算分层管理

### 代码逻辑（示意）
```python
import tiktoken

class WindowedMemory:
    def __init__(self, max_tokens=3000, window_messages=20):
        self.max_tokens = max_tokens
        self.window_messages = window_messages
        self.enc = tiktoken.encoding_for_model("gpt-4o")
        self.messages = []

    def add(self, role, content):
        self.messages.append({"role": role, "content": content})
        # 1. 限制消息条数
        if len(self.messages) > self.window_messages:
            self.messages = self.messages[-self.window_messages:]
        # 2. 限制 Token 总数
        self._shrink_to_budget()

    def _count_tokens(self):
        # 注意：真实计费应包括 role 等结构字符，这里简化计算
        return len(self.enc.encode("\n".join(f"{m['role']}: {m['content']}" for m in self.messages)))

    def _shrink_to_budget(self):
        # 保护 System Prompt (假设 messages[0] 是 system)
        while len(self.messages) > 1 and self._count_tokens() > self.max_tokens:
            # 从索引 1 开始删除，保留 System
            if self.messages[0]['role'] == 'system':
                self.messages.pop(1)
            else:
                self.messages.pop(0) # 丢弃最早一条
```

### ## 常见考点
1.  **如何处理 System Prompt 被截断？**：必须预留专门的 Token 预算给 System Prompt，或者实施硬编码保护逻辑，确保截断从索引 1 开始。
2.  **"Lost in the Middle" 现象是什么？**：指模型在处理长上下文时，对中间位置信息的关注度降低，容易遗忘。设计 Prompt 时应将关键指令放在开头或结尾。
3.  **Token 计算的精确度？**：使用 `tiktoken` 等库是估算，实际 API 的 Tokenization 可能略有差异，工程上通常预留 10%-20% 的 Buffer。
4.  **滑动窗口的 K 值如何设定？**：根据模型上下文窗口和任务性质设定。任务导向型（如编程）通常需要较长的近期历史（10-20 轮），闲聊型可较短。

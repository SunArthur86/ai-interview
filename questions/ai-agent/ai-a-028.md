# OpenAI Swarm的核心设计理念是什么?Handoff机制如何工作

- **Swarm核心:极简的多Agent编排**

- **设计哲学:**
- **无状态** - Agent本身无状态,状态在context中
- **轻量级** - 只是函数调用+context传递
- **Handoff** - Agent间通过「交接」协作

- **核心概念:**

1. **Agent** - instructions + functions + tool_name
2. **Handoff Function** - 当前Agent调用后,控制权转给另一个Agent
3. **Context Variables** - Agent间共享的状态

- **Handoff示例:**
```python
def transfer_to_billing():
return billing_agent  # 返回目标Agent

triage_agent = Agent(
name="客服",
instructions="根据用户问题分诊",
functions=[transfer_to_billing, transfer_to_tech]
)

# 用户问账单问题 → triage调用transfer_to_billing
# 控制权自动转给billing_agent
# billing_agent的回复直接返回给用户
```

- **vs AutoGen/CrewAI:**
- **更简单** - 没有GroupChat/Crew概念
- **更Pythonic** - 纯函数式
- **更灵活** - Handoff就是函数返回值
- **适合:** 客服分诊、专业路由

- **注意:** Swarm是实验性框架,生产环境需自行增强

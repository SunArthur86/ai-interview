# ReAct 框架

### 1. 概念解释
ReAct（Reasoning + Acting）是一种推理与行动交替进行的范式。它要求模型显式输出思考过程、执行动作并观察结果，形成闭环。

### 2. 核心工作流
1. **Thought**：分析当前状态，决定下一步操作。
2. **Action**：调用工具并传入参数。
3. **Observation**：获取工具返回的真实结果。
4. **循环**：根据 Observation 更新 Thought，直到任务完成或达到停止条件。

### 3. Prompt 设计关键点
- **显式前缀**：强制使用 `Thought:`、`Action:`、`Observation:` 等前缀便于解析。
- **工具清单**：清晰列出工具名称、描述和参数。
- **Few-shot**：提供 1-3 个完整轨迹示例提高格式遵从率。

### 4. 与 CoT 的区别
- **CoT**：仅在模型内部展开推理，无外部交互，依赖模型自身知识。
- **ReAct**：将推理与可执行行动绑定，利用外部反馈纠正幻觉。

# MCP(Model Context Protocol)是什么?它和 Function Calling 有什么区别

- **MCP(Model Context Protocol)** 是 Anthropic 于 2024 年提出的开放协议,标准化 LLM 与外部工具/数据的连接方式.

- **核心架构:**
- MCP Server:暴露工具和数据源(如文件系统、数据库、API)
- MCP Client:集成在 LLM 应用中,连接 Server
- 通信协议:JSON-RPC over stdio 或 SSE

- **与 Function Calling 的区别:**

- **Function Calling:** 每个 LLM 应用自己定义工具,工具定义与模型紧耦合,换模型需要改工具定义.没有标准化.

- **MCP:** 工具定义和实现独立于 LLM 应用.一个 MCP Server 可以被任何支持 MCP 的客户端使用(Claude、Cursor、Hermes 等).是工具调用的 USB-C.

- **MCP 的优势:**
1. 标准化:写一次 MCP Server,到处可用
2. 生态:社区共享 MCP Server(GitHub、Slack、数据库等)
3. 安全:Server 控制权限,Client 不直接访问资源
4. 动态发现:Client 运行时自动发现 Server 提供的工具

- **现状:** Claude Desktop、Cursor、Zed、Hermes Agent 已支持 MCP.

# Plan-and-Execute 框架

### 1. 概念解释
Plan-and-Execute 分为两个阶段：
1. **Planner（规划）**：先制定全局计划（任务分解）。
2. **Executor（执行）**：按计划逐步执行，支持重规划。

### 2. 核心流程
- **输入**：用户目标、约束条件。
- **Planning**：生成结构化步骤列表（DAG 或线性列表）。
- **Execution**：遍历步骤，调用工具或子 Agent。
- **Re-planning**：遇到失败或新信息时，局部修复或全局重规划。

### 3. 与 ReAct 的对比
| 维度 | ReAct | Plan-and-Execute |
| :--- | :--- | :--- |
| **规划方式** | 隐式、逐步贪心 | 显式、先全局后局部 |
| **灵活性** | 高，随时可改 | 中，依赖重规划机制 |
| **适用场景** | 路径不确定、探索性强 | 任务结构清晰、步骤多 |

### 4. 优缺点
- **优点**：减少短视行为，计划可审计，全局视野好。
- **缺点**：初始计划错误可能影响全局，两阶段可能增加延迟。

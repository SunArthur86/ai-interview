# 多 Agent 的「容错」具体怎么体现

体现为失败隔离 + 可替换性：例如审查 Agent 发现实现 Agent 的代码不合规，可打回重写而不污染主对话；执行 Agent 沙箱崩溃可只重启该步骤。

**工程上常配合**：重试、指数退避、断路器、降级模板。

**追问应对**：若问「会不会互相甩锅？」——答：会，所以需要明确终止条件、主席/仲裁机制、可观测日志。

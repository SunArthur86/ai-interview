# MMR(Maximal Marginal Relevance)的lambda参数是什么含义?如何调节检索结果的多样性

- **MMR 目标:** 在相关性和多样性之间做平衡,避免返回高度相似的多个结果.

- **公式:**
```
MMR = argmax [ λ × Sim(d, q) - (1-λ) × max_{d'∈S} Sim(d, d') ]

λ=1:完全按相关性排序(退化为普通检索)
λ=0:完全按多样性排序(不考虑相关性)
λ=0.5-0.7:平衡相关性和多样性(最常用)
```

- **参数选择:**
| lambda值 | 效果 | 适用 |
|---------|------|------|
| 0.3 | 极度多样 | 新闻聚合/探索 |
| 0.5 | 平衡 | **通用推荐** |
| 0.7 | 偏相关 | 精确问答 |
| 1.0 | 纯相关 | 精确检索 |

- **实际使用(LangChain):**
```python
retriever = vectorstore.as_retriever(
search_type="mmr",
search_kwargs={"k": 5, "fetch_k": 20, "lambda_mult": 0.5}
)
# fetch_k: 先取20个候选,再用MMR从中选5个
```

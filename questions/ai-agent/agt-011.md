# Reflexion 框架

### 1. 概念解释
Reflexion 框架的核心在于“反思”。它不只是尝试，而是在每次失败后生成语言化的反思文本，作为记忆存储，指导下一次尝试。

### 2. 核心机制
- **Actor**：执行任务，生成输出或动作。
- **Evaluator**：评估结果（如单元测试、打分）。
- **Reflector**：根据评估反馈生成具体的反思文本。

### 3. 工作流
1. Actor 执行任务。
2. Evaluator 给出反馈与分数。
3. Reflector 生成反思文本（例如：“应先检查单位换算”）。
4. 将反思加入上下文，Actor 重试。

### 4. 适用场景
- 可通过测试验证的任务（如代码生成、数学题）。
- 需多轮试错且易犯系统性错误的场景。

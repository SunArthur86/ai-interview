# CrewAI框架是什么？它如何实现多Agent协作？

🎯 本质：CrewAI是一个以角色(Role)为核心的多Agent协作框架，每个Agent有明确的角色、目标、背景故事和工具集，通过任务(Task)分配实现协作。

📊 CrewAI核心概念：

1. Agent（智能体）
  每个Agent定义：
  - role：角色（如"研究员"、"写手"、"审稿人"）
  - goal：目标（如"收集最新AI论文"）
  - backstory：背景故事（影响LLM的行为方式）
  - tools：可用工具列表
  - llm：使用的语言模型

2. Task（任务）
  定义具体的工作项：
  - description：任务描述
  - expected_output：预期输出格式
  - agent：负责的Agent

3. Process（流程）
  Sequential（顺序）：Agent按顺序依次执行任务
  Hierarchical（层级）：有管理Agent分配任务给子Agent

4. Crew（团队）
  将Agents和Tasks组合成一个团队
  指定执行流程

示例：
  researcher = Agent(role='研究员', goal='收集AI最新进展', tools=[search_tool])
  writer = Agent(role='技术写手', goal='撰写通俗易懂的文章')
  
  research_task = Task(description='搜索最新LLM论文', agent=researcher)
  write_task = Task(description='写一篇科普文章', agent=writer)
  
  crew = Crew(agents=[researcher, writer], tasks=[research_task, write_task])
  result = crew.kickoff()

CrewAI vs LangGraph：
| 特性 | CrewAI | LangGraph |
|------|--------|-----------|
| 设计哲学 | 角色驱动 | 状态图驱动 |
| 上手难度 | 简单 | 中等 |
| 灵活性 | 中等 | 高 |
| 多Agent | 天然支持 | 需自己设计 |
| 适用 | 团队协作场景 | 复杂工作流 |

CrewAI适合需要角色分工的场景——像组建一个虚拟团队，每个成员有明确职责。

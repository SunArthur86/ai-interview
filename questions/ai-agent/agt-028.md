# Tool Use / Tool Calling

### Tool Use / Tool Calling

**定义**：Tool 是 Agent 的“能力单元”，封装了名称、描述、参数 Schema 和执行函数。

**最佳实践**：
1. **描述**：是模型选择工具的关键。应包含功能、何时用、何时不用的示例。
2. **参数设计**：少而精，优先枚举，避免自由文本，日期统一 ISO 格式。
3. **返回值**：优先 JSON，超大结果需摘要或分页，错误信息结构化。
4. **错误处理**：区分可重试（429、5xx）与不可重试（4xx）错误，敏感操作需人工确认。

**面试金句**：
*   **Q：Tool 与普通函数的区别？**
*   **A**：Tool 多了面向模型的元数据（描述、Schema）和统一执行接口（日志、超时、权限）。
*   **Q：工具返回 10MB 日志怎么办？**
*   **A**：不应直接返回。应摘要、写入对象存储返回链接、或提供 grep 工具缩小范围。

### 代码示例
```python
from langchain_core.tools import tool

@tool
def search_product(query: str, category: str = "") -> str:
    """在电商站内搜索商品。用户要找商品、比价时用；不要用于闲聊。
    Args:
        query: 搜索关键词
        category: 可选类目
    """
    return f"Results for {query} in {category}" 
```

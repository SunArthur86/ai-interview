# RRF(Reciprocal Rank Fusion)为什么比加权分数融合更鲁棒

- **RRF原理:**
```
RRF_score(d) = Σ 1 / (k + rank_i(d))

k通常取60
rank_i(d): 文档d在第i个检索系统中的排名
```

- **为什么比加权分数更鲁棒:**

1. **不需要分数归一化**
- BM25分数和向量相似度分数量级完全不同
- 加权平均需要复杂的归一化
- RRF只看排名,不关心分数绝对值

2. **对异常值不敏感**
- 一个检索系统的极端高分不会主导结果
- 排名是相对的,天然归一化

3. **简单可调**
- 只有一个超参数k
- k=60是一个经验值,几乎不需要调

- **对比示例:**
```
**文档A：** BM25排名=1, 向量排名=5
**文档B：** BM25排名=3, 向量排名=1

RRF(A) = 1/61 + 1/65 = 0.0311
RRF(B) = 1/63 + 1/61 = 0.0320
→ B更优(两个系统都排在前面)
```

# 基于效用的 Agent 和基于目标的有什么区别

目标型关注「是否达成」；效用型在多个冲突目标间做权衡（成本、时延、风险、用户偏好），选综合最优而非单点达成。

### 深度解析与数学原理

**1. 目标型 Agent**
- **核心逻辑**：基于二元状态。设定一个明确的 Goal $G$，状态空间 $S$ 中存在目标状态 $S_{goal}$。
- **判据**：$Success = (CurrentState == S_{goal})$。
- **适用场景**：逻辑解谜、迷宫寻路、代码生成（通过测试用例即成功）。
- **局限**：无法处理「有多条路径，但路径优劣不同」的情况，也无法处理「无法完全达成，但需要部分妥协」的情况。

**2. 效用型 Agent**
- **核心逻辑**：基于最大化效用函数 $U(s, a)$。Agent 不追求绝对达成，而是追求行动序列带来的期望收益最大化。
- **权衡维度**：
  - $Cost$：Token 消耗、API 调用费用、计算时间。
  - $Risk$：操作失败概率、数据安全风险。
  - $Preference$：用户个性化偏好（如「结果越快越好，哪怕准确率略降」）。
- **决策公式**：$Action^* = \operatorname{argmax}_{a} \sum_{s'} P(s'|s,a) \cdot U(s')$。

**3. 决策对比示意图**
```text
场景：预定去机场的出租车，有两个方案

方案 A：豪华专车，10分钟到，费用 $100
方案 B：经济拼车，30分钟到，费用 $20

[ Goal-Based Agent ]
User Goal: "预定一辆车"
Result: A 或 B 均可，可能随机选一个（或者选第一个满足条件的）。
Failure Mode: 可能选了 $100 的车，导致用户不满，因为 Goal 只定义了「预定」。

[ Utility-Based Agent ]
Utility Function: U = 0.6 * (1/CostNorm) + 0.4 * (1/TimeNorm)
Result: 根据 Utility 分数计算，方案 B 可能得分更高（若用户对价格敏感），
         Agent 选择 B。
Success Mode: 自动在冲突目标中找到帕累托最优解。
```

**4. 实际应用**
- 在企业级 Agent 中，Utility Function 往往通过 `Weights` 配置。例如：对于金融类任务，将「准确性」权重设为 0.9，「速度」设为 0.1；对于闲聊机器人，则反之。

## 常见考点
1.  **如何定义具体的效用函数？**（考点：线性加权、层次分析法 AHP，或通过强化学习从人类反馈中学习 Reward Model）。
2.  **目标型和效用型能否结合？**（考点：可以，先用 Goal-Based 过滤不可行解，再用 Utility-Based 对可行解排序）。
3.  **在多智能体协作中，效用冲突怎么解决？**（考点：博弈论概念，如纳什均衡，或者引入中心仲裁者）。

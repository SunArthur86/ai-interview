# 安全性

### 安全性

**核心风险**：Prompt 注入、越权访问、敏感数据泄露、资源耗尽。

**防护措施**：
1. **权限控制**：模型无身份，必须在服务端绑定用户身份，执行前鉴权（行级/列级）。
2. **输入验证**：清洗参数，防止 SQL 注入、路径穿越（`../../etc/passwd`），使用白名单。
3. **敏感操作确认**：删除、转账等操作需人工确认（HITL）或二次 Token。
4. **频率限制**：按用户/工具维度限流，防止刷量与成本失控。
5. **脱敏与审计**：敏感信息不进入上下文，全量记录调用日志。

**面试金句**：
*   **Q：如何防止模型通过工具泄露敏感数据？**
*   **A**：最小权限原则、结果脱敏（掩码）、禁止把密钥放进返回值、敏感字段仅在服务端处理。

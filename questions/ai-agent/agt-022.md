# 检索策略

### 检索策略

**6.1-6.3 检索模式**
- **向量检索**：擅长语义理解，弱于专有名词匹配。
- **关键词检索 (BM25)**：擅长精确匹配（如型号、ID）。
- **混合检索**：融合两者分数，兼顾语义与字面。

**6.4 多路检索与融合 (RRF)**
RRF (Reciprocal Rank Fusion) 不依赖分数尺度，仅根据排名融合，公式：$1/(k+rank)$。

**面试 Q11：为什么混合检索比单路向量更有效？**
A：向量捕获语义，但可能漏掉专有名词；BM25 补精确匹配。二者融合提高鲁棒性，尤其在技术文档与电商场景。

**6.5 查询改写**
将口语/模糊问题转化为利于检索的形式，或生成多路查询。

**6.6 HyDE**
先生成假设性答案，再用假设答案去检索，拉近 Query 与 Doc 的向量距离。

**面试 Q12：HyDE 适合什么场景？**
A：适合表述差距大、短查询难对齐长文档；不适合强事实约束且易胡编的领域。

### 代码示例：RRF
```python
def rrf_fuse(ranked_lists, k=60):
    scores = {}
    for ranks in ranked_lists:
        for rank, doc_id in enumerate(ranks, start=1):
            scores[doc_id] = scores.get(doc_id, 0) + 1.0 / (k + rank)
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

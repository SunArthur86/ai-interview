# Agent服务的限流和弹性设计怎么做?LLM API限流如何处理

- **限流层级:**

1. **用户级限流**
- 每用户/分钟N次请求
- 每用户/天M Token配额
- Token Bucket / Leaky Bucket算法

2. **系统级限流**
- 总并发Agent数限制
- 全局Token/分钟限制
- 队列排队等待

- **LLM API限流处理:**

```python
# 指数退避重试
async def call_llm_with_retry(prompt, max_retries=5):
for i in range(max_retries):
**try：** 
return await llm.generate(prompt)
except RateLimitError:
wait = 2 ** i + random.uniform(0, 1)
await asyncio.sleep(wait)
raise LLMUnavailableError()

# 多供应商故障转移
providers = ["openai", "anthropic", "zhipu", "deepseek"]
for provider in providers:
**try：** 
return await call(provider, prompt)
except (RateLimit, Timeout):
continue
```

- **弹性设计:**
- **断路器** - 连续失败后熔断,避免雪崩
- **降级** - 大模型限流时临时切小模型
- **排队** - 超过容量时排队等待
- **超时** - 单步超时+全局超时

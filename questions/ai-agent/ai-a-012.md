# 当Agent有几十上百个工具时,如何高效选择正确的工具

- **工具过载问题:** LLM上下文有限,工具太多导致选择困难和幻觉.

- **解决方案(分层策略):**

1. **工具检索(Tool RAG)**
- 将工具描述做embedding
- 根据用户意图检索Top-K相关工具
- 只将相关工具传给LLM

2. **工具分类路由**
- 先用LLM/分类器选择工具类别
- 再在类别内选择具体工具

3. **层级工具**
- 粗粒度工具作为入口
- 调用后动态暴露细粒度工具

4. **工具使用历史**
- 记录哪些工具组合在什么场景下被使用
- 用历史数据辅助选择

5. **MCP动态发现**
- MCP Server按需暴露工具
- Agent先列出可用工具,再选择

- **实践建议:**
- <15个工具:直接放系统prompt
- 15-50个工具:Tool RAG
- 50+个工具:分类路由 + Tool RAG

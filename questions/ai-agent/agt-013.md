# LangChain Agent 实现

### 1. 概念解释
LangChain 是一个编排框架，其中 Agent 指由 LLM 决定工具调用的循环体，而 **AgentExecutor** 负责驱动这个循环直到任务结束。

### 2. 核心组件
- **Tools**：封装外部能力，包含 name、description 和 args_schema。
- **Agent**：推理策略，决定如何组合工具与 Prompt（如 ReAct 模式）。
- **AgentExecutor**：运行时引擎，处理循环、错误、中间步骤记录。

### 3. AgentExecutor 工作流
1. 接收用户输入。
2. 调用 LLM 决定下一步（生成回答或工具调用）。
3. 若需调用工具，执行工具并获取 Observation。
4. 将结果回填消息历史，重复步骤 2。
5. 满足停止条件后返回最终答案。

### 4. 自定义 Agent 步骤
1. 定义 Tools 与清晰的 Description。
2. 选择 Prompt 模板与输出解析器。
3. 组装 Agent 与 AgentExecutor。
4. 配置 `max_iterations` 与 `handle_parsing_errors`。

# Agent如何通过模型路由降低成本?什么场景用大模型,什么场景用小模型

- **模型路由策略:**

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 规划/决策 | **大模型**(Sonnet/Opus) | 需要推理能力 |
| 简单分类 | **小模型**(Flash/Haiku) | 低成本足够 |
| 代码生成 | 大模型 | 准确性要求高 |
| 文本摘要 | **中模型** | 平衡性价比 |
| 工具参数提取 | **小模型** | 结构化任务 |
| 错误诊断 | 大模型 | 需要推理 |
| 格式转换 | **小模型** | 机械任务 |

- **路由实现:**

```python
class ModelRouter:
def select_model(self, task_type, complexity):
if task_type in ["plan", "reason", "debug"]:
return "claude-sonnet"
elif complexity > 0.7:
return "claude-sonnet"
elif task_type in ["extract", "format", "classify"]:
return "claude-haiku"
**else：** 
return "glm-4-flash"  # 最便宜
```

- **成本对比示例:**
- 全部用Sonnet: $100/1000任务
- 混合路由: $30/1000任务(70%省70%)
- 质量损失: <5%

- **关键:** 定期评估路由准确性,确保简单任务确实用小模型

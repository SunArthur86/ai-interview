# 多模态RAG（Multimodal RAG）是什么？如何处理图文混合检索？

🎯 本质：多模态RAG在传统文本RAG基础上增加了图像、表格、音频等多模态数据的检索和生成能力。

📊 多模态RAG架构：

1. 文档处理
  PDF/Word中的文本 -> 文本切分 + 向量化
  图片 -> CLIP/BLIP生成图像Embedding
  表格 -> 转为Markdown或结构化文本
  混合内容 -> 按页/按节切分

2. 多模态Embedding
  文本编码：text-embedding-ada-002 / BGE
  图像编码：CLIP Image Encoder
  统一空间：CLIP将文本和图像映射到同一向量空间
  优势：可以用文本查询检索图片，反之亦然

3. 多模态检索
  方案A: 统一向量空间检索（CLIP）
    text_query -> CLIP -> embedding -> 搜索图文混合向量库
  方案B: 多路检索+融合
    文本检索器 + 图像检索器分别检索
    用RRF或加权融合结果
  方案C: 多模态LLM直接理解（GPT-4V/Claude）
    将检索到的图片直接送入VLM

4. 多模态生成
  纯文本答案：LLM基于检索到的文本+图片描述生成
  图文答案：VLM(GPT-4V/Qwen-VL)同时理解文字和图片
  表格答案：LLM基于结构化数据生成

挑战：
1. 跨模态对齐：文本"猫"和猫的图片在向量空间要对齐
2. 图表理解：复杂图表(折线图/柱状图)需要专门处理
3. 分块策略：图文混排文档如何合理切分
4. 评估困难：多模态答案的质量难以自动评估

工具：LangChain MultiVectorRetriever、LlamaIndex MultiModalIndex

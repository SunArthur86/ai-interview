# 情景记忆与语义记忆

情景记忆与语义记忆源于认知心理学，在 Agent 中对应不同的存储与检索策略。

1.  **情景记忆**：关于具体事件（时间、地点、参与者）。例如「昨天下午用户要求改 PDF」。
    *   **实现**：带时间戳的对话片段、工具日志。
    *   **特点**：高度个人化，随时间衰减。
2.  **语义记忆**：关于一般性知识、规则。例如「公司退货政策是 7 天」。
    *   **实现**：知识库条目、FAQ、图谱三元组。
    *   **特点**：共享、稳定，更新频率低。

### 为什么要区分？
*   **检索特征**：情景检索偏时间邻近；语义检索偏概念相似。
*   **治理策略**：情景隐私级别高且易过期；语义需长期维护且多租户共享。

### 实现策略
*   **情景记忆**：通常存储在向量数据库中，检索时强依赖时间衰减权重，防止过时信息干扰当前任务。
*   **语义记忆**：可以是向量库，也可以是知识图谱。对于规则类信息，KG 的结构化查询更精准；对于通用知识，向量库更灵活。

### 代码模型
```python
@dataclass
class EpisodicRecord:
    ts: float
    summary: str  # 具体发生了什么
    embedding: list[float] # 向量化后的内容
    user_id: str

@dataclass
class SemanticFact:
    key: str     # 如 'billing.policy'
    value: str   # 通用规则
    source: str  # 来源文档
    embedding: list[float] # 用于语义关联检索
```

### ## 常见考点
1.  **两种记忆的隔离性？**：必须物理或逻辑隔离。情景记忆按用户隔离，隐私敏感；语义记忆通常是全局共享的。
2.  **情景记忆何时归档？**：当短期记忆溢出或会话结束时，将对话历史通过 LLM 转化为 Episode 记录存入长期记忆。
3.  **如何利用知识图谱增强语义记忆？**：通过实体链接将文本中的实体（如“iPhone 15”）映射到图谱节点，利用图结构进行推理（如查找关联配件），而不仅仅是文本相似度匹配。

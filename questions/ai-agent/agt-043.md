# 为什么需要多智能体

多智能体系统指多个相对独立的 Agent 在协作协议下共同完成复杂任务。

### 为什么需要多智能体？
**单 Agent 的瓶颈**：
1.  **注意力漂移**：长提示词下，模型易忽略中间的约束。
2.  **推理链断裂**：复杂任务中，单次调用难以进行长程规划与回溯。
3.  **能力局限**：单一 System Prompt 难以同时兼顾严谨规划、创意发散与代码执行。

**多 Agent 的优势**：
1.  **专业化**：每个 Agent 专注于特定领域（如代码审查），Prompt 更短更稳。
2.  **并行化**：独立子任务可并行执行，缩短墙钟时间。
3.  **容错性**：某一 Agent 失败可隔离重试，不致全盘崩溃。

### 协作模式与架构
```text
      [ 用户请求 ]
           │
           ▼
    ┌──────────────┐
    │  管理者 Agent │
    │  (拆解任务)   │
    └──────┬───────┘
           │
     ┌─────┴─────┬────────────┐
     ▼           ▼            ▼
[ 研究员 ]  [ 程序员 ]    [ 审核员 ]
(并行/串行)  (执行代码)    (质量把关)
     │           │            │
     └─────┬─────┴────────────┘
           │
           ▼
    [ 结果汇总/迭代 ]
```

### 代码逻辑（示意）
```python
roles = [
    ("analyst", "你只输出需求要点。"),
    ("coder", "你只输出代码。"),
    ("reviewer", "你只审查代码安全。")
]
for name, hint in roles:
    agent = SimpleAgent(name, hint)
    result = agent.run(task)
```

## 常见考点
1.  **通信机制**：多 Agent 之间如何通信？（如：自然语言消息传递、共享的黑板机制/消息队列、或直接函数调用）。
2.  **冲突解决**：当两个 Agent 的意见不一致时（如程序员和审核员吵架），如何解决？（通常引入裁判 Agent 或由 Manager Agent 依据最终目标进行裁决）。
3.  **成本控制**：多 Agent 会成倍增加 Token 消耗，如何优化？（例如：Agent 间使用更紧凑的协议格式而非自然语言，或设置最大迭代轮数）。
4.  **动态规划**：Agent 的数量和角色是静态配置的还是动态生成的？（高级系统会根据任务难度动态生成新的子 Agent）。

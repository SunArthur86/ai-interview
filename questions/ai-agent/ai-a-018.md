# Agent的短期记忆(工作记忆)如何设计?上下文窗口满了怎么办

- **Agent记忆层级:**

```
即时记忆 = 当前prompt上下文窗口(如128K tokens)
↓ 溢出
短期记忆 = 最近N轮对话摘要
↓ 归档
长期记忆 = 向量数据库中的历史交互
```

- **上下文满了的处理策略:**

1. **滑动窗口** - 只保留最近K轮对话
- 简单但丢失早期上下文

2. **摘要压缩** - 定期总结旧对话
- 每N轮将前N-1轮总结为一段摘要
- LangChain的ConversationSummaryMemory

3. **向量检索** - 将历史存入向量库
- 当前问题embedding检索相关历史
- 只注入相关的历史片段

4. **实体记忆** - 提取并记住关键实体
- 记住用户提到的人名/项目名/偏好
- LangChain的EntityMemory

5. **分层压缩**
- 最近3轮:完整保留
- 3-10轮前:摘要
- 10轮前:只存向量索引

- **最佳实践:** 滑动窗口 + 摘要压缩 + 向量检索的组合

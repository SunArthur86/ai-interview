# AutoGen / CrewAI 等多 Agent 框架

### 1. 概念解释
多 Agent 框架（如 AutoGen, CrewAI）将任务分配给多个不同“角色”的 Agent。它们通过对话或消息传递协作完成任务。

### 2. 核心设计
- **角色定义**：每个 Agent 有特定的 System Prompt、工具集和目标。
- **协作模式**：
  - **对话驱动**：像群聊一样互相发送消息。
  - **流程驱动**：像流水线一样按顺序传递工作。

### 3. 优势与挑战
- **优势**：分工明确（如写代码 vs 审查代码），利用角色互补提高质量；支持并行探索。
- **挑战**：协调成本高，可能出现“互相附和”或对话死循环。

### 4. 防止“互相附和”的策略
- 引入独立的**批评者** 角色。
- 使用外部工具验证（如单元测试、检索）。
- 设定明确的停止条件和异议处理流程。

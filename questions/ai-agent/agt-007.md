# Agent 的挑战与局限

### 1. 幻觉
**问题**：模型编造事实或工具调用参数。
**对策**：
- 检索增强：使用 RAG 进行事实 grounding。
- 约束解码：限制模型生成的词表或格式。
- 验证器：使用外部工具或规则校验输出。

### 2. 安全风险
**问题**：Prompt 注入、工具滥用、敏感数据外泄。
**对策**：
- **分层信任域**：隔离不同权限的运行环境。
- **输出过滤**：检测并拦截敏感内容。
- **秘密保护**：禁止将密钥等敏感信息直接输入模型上下文。

### 3. 成本控制
**问题**：长链路调用与大上下文导致 Token 消耗过高。
**对策**：
- **摘要**：压缩中间历史信息。
- **小模型路由**：使用轻量模型处理简单任务。
- **缓存与批处理**：复用工具调用结果，合并请求。

### 4. 可解释性
**问题**：决策过程黑盒，难以排错。
**对策**：
- 记录完整轨迹。
- 提供引用来源。
- 保存关键决策日志与回放机制。

### 5. 评估困难
**问题**：传统 NLP 指标（如 BLEU）不适用任务完成度评估。
**对策**：
- **过程指标**：工具选择是否正确、步骤是否冗余。
- **结果指标**：任务是否最终成功完成。
- **人工抽检**：结合 A/B 测试与人工评审。

# 生产Agent如何实现可观测性?需要追踪哪些关键指标

- **Agent可观测性三层:**

- **1. 追踪(Tracing) - 每步执行记录**
```
**Trace：** 
├─ Step 1: Thought(分析) → Action(search) → Result(3 docs)
├─ Step 2: Thought(阅读) → Action(read_file) → Result(content)
├─ Step 3: Thought(总结) → Action(finish)

**每步记录：** 时间戳/Token数/延迟/模型/温度
```

- **2. 指标(Metrics) - 聚合统计**
- 平均任务完成率
- 平均步数/Token/成本
- P50/P99延迟
- 工具调用成功率
- 模型分布(哪个模型被调用最多)

- **3. 日志(Logs) - 调试信息**
- 错误堆栈
- 完整prompt和response
- 工具输入输出

- **工具选择:**

| 工具 | 特点 | 适用 |
|------|------|------|
| **LangSmith** | LangChain生态原生 | LangChain/LangGraph |
| **Phoenix(Arize)** | 开源,支持多框架 | 通用 |
| **W&B Tracing** | ML生态整合 | 研究/实验 |
| **OpenTelemetry** | 标准化 | 生产系统 |
| **自建** | 完全可控 | 特殊需求 |

- **关键:** 追踪每个LLM调用的完整input/output,不只是最终结果

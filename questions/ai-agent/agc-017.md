# 多 Agent 协作和单 Agent 多工具怎么选

单 Agent 多工具：实现简单、延迟低，适合任务边界清晰。多 Agent：角色分工、并行探索、对抗审查（如「批评者 Agent」）；但带来协调成本与一致性问题。选型看任务分解结构、组织边界、延迟与成本。

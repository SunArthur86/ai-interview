# 重排序（Reranking）

### 重排序

**7.1 为什么需要重排**
向量检索 (Bi-Encoder) 为追求速度独立编码，交互不足；Cross-Encoder 深度交互精度高但慢，用于精排。

**7.2 对比**
- **Bi-Encoder**: 快，可 ANN，精度略低。
- **Cross-Encoder**: 慢，只能小批量，精度高。

**7.4 MMR (最大边际相关性)**
在相关性与多样性间权衡，避免返回的 Top-K 全是重复内容。

**面试 Q13：重排放在检索后哪一步？**
A：一般在召回 Top-K (几十到几百) → Cross-Encoder 精排取 Top-N (3–10) → 再生成，平衡延迟与效果。

# Query Rewrite(查询重写)有哪些策略?什么场景需要用

- **Query Rewrite** 是在检索前对用户查询进行改写增强的技术.

- **常见策略:**

- **1. 查询扩展(Query Expansion):** 将一个查询扩展为多个相关查询.如 Transformer 原理 -> Transformer 自注意力机制 + Transformer 位置编码 + Transformer 训练方法.

- **2. 查询分解(Query Decomposition):** 将复杂问题拆分为子问题.如比较 A 和 B 的优劣 -> A 的优势是什么 + B 的优势是什么 + 两者对比.

- **3. 回指消解(Coreference Resolution):** 多轮对话中的代词消解.如它 -> 指代上一轮提到的具体名词.

- **4. HyDE:** 生成假设文档作为查询.

- **适用场景:**
- 用户查询模糊或过短
- 多轮对话场景
- 复杂推理类问题

- **代价:** 每次检索多一次 LLM 调用,增加延迟.需要在效果和延迟间权衡.

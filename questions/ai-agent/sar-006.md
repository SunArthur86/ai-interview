# RAG系统评估有哪些指标？如何系统地评估RAG质量？

🎯 本质：RAG评估需要从检索质量和生成质量两个维度，使用RAGAS/Triumf等框架系统评估。

📊 RAGAS评估框架（四大核心指标）：

1. Faithfulness（忠实度）
  生成答案是否完全基于检索到的文档
  检测幻觉：答案中是否有文档不支持的内容
  评估方法：将答案分解为claims，逐个验证是否被context支持

2. Answer Relevancy（答案相关性）
  生成的答案是否直接回答了用户问题
  检测偏题/啰嗦：答案是否包含了不相关的内容
  评估方法：用LLM从答案反推可能的问题，计算与原问题的相似度

3. Context Precision（上下文精确度）
  检索到的文档中有多少是真正相关的
  排在前面的文档是否更有用
  评估方法：检查相关文档在检索结果中的排名位置

4. Context Recall（上下文召回率）
  是否所有需要的信息都被检索到了
  检测遗漏：是否有应该检索到但没检索到的信息
  评估方法：对比ground truth答案和检索文档的覆盖度

其他评估维度：
| 维度 | 指标 | 说明 |
|------|------|------|
| 检索 | Hit Rate | 正确文档在Top-K中的比例 |
| 检索 | MRR | 正确文档的平均倒数排名 |
| 检索 | NDCG | 考虑排序位置的增益 |
| 生成 | BLEU/ROUGE | 与参考答案的n-gram重叠 |
| 生成 | 人工评估 | 准确性/完整性/流畅性 |
| 端到端 | Latency | P50/P99响应延迟 |
| 端到端 | 成本 | 每次查询的token消耗 |

评估工具：
- RAGAS: 最流行的RAG评估框架
- TruLens: RAG+LLM应用追踪和评估
- DeepEval: 单元测试风格的LLM评估
- LlamaIndex Evaluator: 内置评估器

# RAG系统评估有哪些指标？如何系统地评估RAG质量？

RAG系统的评估通常分为三个核心维度：检索质量、生成质量和端到端质量。

### 1. 检索阶段评估 (Retrieval Evaluation)
评估检索到的上下文是否相关、准确。

*   **Context Precision (上下文精度)**：检索结果中相关文档的排序情况。相关文档是否排在前面？计算公式通常参考平均精度。
*   **Context Recall (上下文召回)**：标准答案中引用的信息，有多少被包含在检索到的上下文中。即检索是否有遗漏。
*   **MRR (Mean Reciprocal Rank)**：第一个相关文档出现在排序第几位。倒数排名越高越好。
*   **NDCG (Normalized Discounted Cumulative Gain)**：考虑排序位置和相关性等级的指标，关注高相关性文档是否排在前列。

### 2. 生成阶段评估 (Generation Evaluation)
评估LLM基于上下文生成的答案质量。

*   **Faithfulness (忠实度)**：答案是否严格依据检索到的上下文。**这是RAG评估的核心，用于检测幻觉。**
*   **Answer Relevance (答案相关性)**：生成的答案是否解决了用户的问题。

### 3. 端到端评估 (End-to-End Evaluation)
*   **RAGAS**：最流行的开源框架。流程如下：
    ```text
    ┌──────────┐     ┌──────────────┐     ┌──────────────┐
    │ Question │ ──> │ LLM Generate │ ──> │  LLM Verify  │
    └──────────┘     │   Answer     │     │  (Is it      │
                     └──────┬───────┘     │  Faithful?)  │
                            │             └──────┬───────┘
                            v                    │
                     ┌──────────────┐            v
                     │   Context    │     ┌──────────────┐
                     │   (Retrieved)│     │  Score (0-1) │
                     └──────────────┘     └──────────────┘
    ```
*   **TruLens (RAG Triad)**：由Context Relevance、Faithfulness、Answer Relevance三个指标组成三角评估。
*   **ARES**：基于对抗训练的评估方法，无需人工标注，通过构造噪声数据来评估。

### 4. 系统性能评估
*   **Latency (延迟)**：首字生成时间 (TTFT) 和总生成时间。
*   **Throughput (吞吐量)**：单位时间内处理的请求数。

## 常见考点
1. **如何在没有标准答案（Ground Truth）的情况下评估？**
   使用 LLM-as-a-Judge 模式，利用强模型（如GPT-4）根据Prompt生成的答案与检索Context的一致性来打分，不依赖人工标注的标准答案。
2. **RAGAS 中 Context Recall 和 Faithfulness 的区别？**
   Recall 关注检索阶段是否“找全了”（相对标准答案），Faithfulness 关注生成阶段是否“瞎编了”（相对检索Context）。
3. **如何解决评估中的分数波动问题？**
   设置多个样本或使用更强的一致性Prompt，必要时引入人工抽检（Human-in-the-loop）。

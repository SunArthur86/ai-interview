# Claude Code的Agent架构有什么特点?它如何处理文件系统和代码理解

- **Claude Code核心设计:**

1. **CLI优先** - 不是IDE插件,是终端原生
2. **文件系统操作** - 读/写/搜索/grep/patch
3. **代码执行** - 可以运行测试/构建/lint
4. **上下文管理** - 自动压缩+选择性加载

- **工具集:**
```
Read(file)      - 读取文件
Write(file)     - 写入文件
Search(pattern) - 内容搜索
Bash(cmd)       - 执行命令
Patch(file)     - 精确编辑
git operations  - 版本控制
```

- **关键设计:**

1. **AGENTS.md / CLAUDE.md** - 项目级配置文件
- 告诉Agent项目的编码规范/架构/测试方法
- 类似于给新人的README

2. **增量式代码理解**
- 不一次性读取整个代码库
- 按需搜索→读取相关文件→理解局部

3. **Diff-based修改**
- 用patch而非全文重写
- 改动可追溯、可审查

4. **自我验证**
- 改完后自动运行测试
- 测试失败自动修复

- **与Devin区别:** Claude Code更轻量,运行在用户本地环境,不依赖云端沙箱

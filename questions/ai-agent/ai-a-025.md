# Agent如何做多步推理规划?Forward Planning vs Backward Planning的区别

- **Forward Planning(前向规划):**
- 从当前状态出发,向目标推进
- 「我现在有什么 → 下一步做什么 → 最终达到目标」
- 适合:步骤较少、目标明确的任务

- **Backward Planning(反向规划):**
- 从目标倒推需要的步骤
- 「要达到X → 需要先完成Y → 需要先完成Z」
- 适合:目标复杂但路径需要推理的任务

- **对比:**

| | Forward | Backward |
|--|---------|----------|
| 起点 | 当前状态 | 目标状态 |
| 方向 | 向前推 | 向后推 |
| 适用 | 简单线性 | **复杂多步** |
| 效率 | 快 | 需更多推理 |
| 示例 | 「写完代码→测试→部署」 | 「要部署→需要测试→需要代码」 |

- **混合策略:**
1. 先Backward:分解目标到子目标
2. 再Forward:按依赖顺序执行
3. 执行中动态调整

- **LLM规划prompt设计:**
```
目标：{goal}
当前状态：{state}
可用工具：{tools}

请制定详细计划：
1. 倒推实现目标需要哪些步骤
2. 标注每个步骤的依赖关系
3. 标注每步使用的工具
```

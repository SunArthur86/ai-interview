# LangGraph 状态机

### 1. 概念解释
LangGraph 将 Agent 工作流建模为**图（Graph）**结构。节点代表处理逻辑（如模型调用、工具执行），边代表状态流转。相比 LangChain 的固定循环，它更灵活。

### 2. 核心要素
- **State**：共享状态，通常用 TypedDict 定义，由各节点增量更新。
- **Node**：函数，接收 state 并返回 state 更新。
- **Edge**：连接节点，包含普通边和条件边。

### 3. 优势
- **显式编排**：适合复杂流程（如条件分支、回退、并行）。
- **人机协同**：轻松插入“人工节点”等待审核。
- **可控性**：结构化的轨迹，易于调试与观测。

### 4. 与 LangChain Agent 对比
| 维度 | LangChain Agent | LangGraph |
| :--- | :--- | :--- |
| **控制流** | 隐式循环 | 显式图结构 |
| **适用场景** | 通用工具调用 | 复杂业务流程、多阶段流水线 |

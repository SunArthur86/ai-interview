# FAISS中IndexFlatIP和IndexHNSW有什么区别?如何选择

- **FAISS索引类型对比:**

| | IndexFlatIP | IndexHNSW |
|--|------------|-----------|
| 原理 | 暴力搜索(计算所有点) | 图结构近似搜索 |
| 精度 | **100%精确** | 近似(95%+ recall) |
| 速度 | 慢(O(n)) | **快(O(log n))** |
| 内存 | 低(只存向量) | 高(存图结构) |
| 构建时间 | 无需构建 | 需要构建 |
| 适合规模 | **<10万** | **>10万** |

- **选择建议:**
```
数据量 < 10万 → IndexFlatIP(暴力搜索,精度最高)
数据量 10万-100万 → IndexHNSW(图搜索,速度快)
数据量 > 100万 → IVF-PQ(聚类+量化,内存友好)
```

- **IndexHNSW关键参数:**
- **M**:每个节点的连接数(16-48,越大越精确越耗内存)
- **efConstruction**:构建搜索宽度(200-500)
- **efSearch**:查询搜索宽度(50-200,运行时可调)
- efSearch调大 → 精度提升但速度下降

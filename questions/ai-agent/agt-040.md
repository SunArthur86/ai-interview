# 记忆检索策略

检索策略决定在生成下一轮回复前，从记忆池中召回哪些信息。单一策略往往不足，需混合使用。

### 常见策略
1.  **基于时间**：优先召回最近对话。适合排障、联调等强时效任务。
2.  **基于相关性**：通过向量相似度召回。适合用户换说法的开放域问答。
3.  **基于重要性**：优先召回高权重信息（如用户核心目标）。避免早期硬约束被淹没。

### Generative Agents 多因子融合
斯坦福 Generative Agents 论文提出综合打分公式：
$$ Score = w_{rel} \cdot \widehat{rel} + w_{rec} \cdot \widehat{recency} + w_{imp} \cdot \widehat{importance} $$
其中：
*   $\widehat{rel}$: 相关性（向量余弦）
*   $\widehat{recency}$: 近期性（时间衰减函数，如 $e^{-\alpha \Delta t}$）
*   $\widehat{importance}$: 重要性（通常由 LLM 生成时打分 1-10）

### 实现难点
*   **去重**：同一事实的不同表述需语义对齐，避免冗余占用上下文。
*   **限长**：按得分排序后进行 Token 装箱，或分层注入（摘要+细节）。

### 代码逻辑（示意）
```python
def score_memories(memories, query_vec, w_rel=0.5, w_rec=0.3, w_imp=0.2):
    now = time.time()
    for m in memories:
        rel = cosine_sim(query_vec, m.vec)
        # 使用指数衰减模拟时间遗忘，alpha控制衰减速度
        recency = math.exp(-0.1 * (now - m.last_access_ts))
        # importance 通常在写入时由 LLM 预估 (1-10)，需归一化处理
        norm_imp = m.importance / 10.0 
        m.score = w_rel*rel + w_rec*recency + w_imp*norm_imp
    return sorted(memories, key=lambda x: x.score, reverse=True)[:10]
```

### ## 常见考点
1.  **如何平衡不同权重的效果？**：$w$ 值是超参数。对话类应用通常 $w_{rec}$ 较高；知识问答类 $w_{rel}$ 较高；Agent 任务执行中 $w_{imp}$ 需较高以保证核心目标不被遗忘。
2.  **Token 装箱问题**：简单的截断可能导致一条很重要的长消息被切掉。需实现“装箱算法”，即尝试放入整个 Message，如果放不下才跳过，以保证信息的完整性。
3.  **如何处理检索失败？**：当召回分数均低于阈值时，应明确告知 LLM “未找到相关记忆”，而不是强行注入噪音信息，这能有效减少幻觉。
4.  **元数据过滤在检索前还是检索后？**：应该尽可能在检索前（Pre-filter）进行。例如先筛选 `user_id=123`，再在子集中计算向量相似度，既能加速又能保证准确性。

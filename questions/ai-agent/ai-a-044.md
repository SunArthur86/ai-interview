# OpenAI Codex CLI的架构有什么特点?它的前缀缓存和上下文压缩如何工作

- **Codex CLI核心架构:**

- **1. 无状态请求设计**
- 每次API调用包含完整上下文(不依赖服务端会话)
- 好处:可水平扩展、无会话恢复成本

- **2. 前缀缓存(Prefix Caching)**
- 系统prompt + AGENTS.md + 工具定义 = 固定前缀
- 前缀部分的KV Cache可复用
- **节省50-80%的Token计算成本**
- OpenAI/Anthropic/Zhipu都支持

- **3. 上下文压缩(Context Compaction)**
- 超过阈值时自动触发
- 保留最近6条消息 + 旧消息摘要
- 摘要由LLM自动生成

- **4. 工具沙箱**
- 命令执行有超时+权限限制
- 敏感操作需要审批(3级:auto/suggest/always-ask)

- **5. AGENTS.md级联加载**
```
/AGENTS.md          ← 全局规则
/project/AGENTS.md  ← 项目规则
/project/src/AGENTS.md ← 模块规则
```
- 按目录层级合并,深层覆盖浅层

- **与Claude Code的区别:**
- Codex:无状态 + 前缀缓存优化(成本更低)
- Claude Code:有状态会话 + 更强的推理能力

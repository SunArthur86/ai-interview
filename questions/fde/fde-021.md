# 客户的 AI 应用 P95 延迟 8 秒,用户抱怨太慢,你会怎么优化到 3 秒以内

- **AI 应用延迟优化方案(8s -> 3s)**

- **诊断:延迟分解**
用 OpenTelemetry 追踪每个环节耗时:
- Embedding 查询向量化:100ms
- 向量检索:200ms
- Rerank:500ms
- LLM 生成(最大头):6000ms
- 网络传输:200ms

- **优化措施(按收益排序)**

1. **流式输出**(体感延迟降低 50%)
- 首 token 延迟 < 1s 即可
- 用户看到流式输出不会觉得慢

2. **减小上下文**(LLM 延迟降低 30-40%)
- 检索结果从 Top-10 减到 Top-5
- 去掉冗余的 system prompt
- 上下文 token 数直接线性影响生成时间

3. **模型降级**(LLM 延迟降低 50%)
- 简单问题用 GLM-4-Flash(2s 内完成)
- 复杂问题才用 GLM-4-Plus
- 模型路由实现自动选择

4. **语义缓存**(缓存命中直接返回)
- 高频问题直接返回缓存
- 命中率 20-30% 时整体延迟降低 20%

5. **Rerank 异步化**
- 先返回 Top-3 初步结果
- Rerank 在后台异步执行
- 用户不需要等待 Rerank 完成

- **预期效果**
- 流式首 token < 1.5s
- 完整生成 P95 < 3s
- 缓存命中 < 200ms

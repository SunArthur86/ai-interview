# 如何把一个 AI 应用快速部署到客户的服务器上

- **AI 应用客户部署 Checklist**

- **1. 容器化(最重要)**
```dockerfile
FROM python:3.11-slim
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

- **2. docker-compose 一键启动**
```yaml
version: '3'
services:
  app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - API_KEY=${API_KEY}
    depends_on:
      - redis
  redis:
    image: redis:7-alpine
```

- **3. 部署步骤**
```bash
# 客户服务器上
git clone <repo>
cd <project>
cp .env.example .env  # 填入 API Key
docker-compose up -d
# 验证
curl localhost:8000/health
```

- **4. 常见坑**
- 客户服务器是 ARM 架构 → Docker buildx 跨平台构建
- 客户没有公网 → 用内网 IP + 反向代理
- 客户防火墙限制 → 只开放必要端口
- GPU 驱动版本不匹配 → 提前确认 CUDA 版本

- **FDE 经验**:永远准备一个 `install.sh` 脚本,客户只需一行命令就能装好.

- **## 常见考点**
1. **环境隔离**：面试官常问 `.env` 文件管理。答案：`.env` 必须加入 `.gitignore`，提供 `.env.example` 模板，敏感信息通过环境变量注入容器，不硬编码在代码中。
2. **健康检查**：Dockerfile 中应包含 `HEALTHCHECK` 指令（如 `curl -f http://localhost:8000/health || exit 1`），配合 K8s 或 Docker 的自动重启策略。
3. **GPU 穿透/利用**：如果本地部署大模型，需熟悉 `nvidia-docker` 运行时（`runtime: nvidia`）以及如何通过 `docker-compose.yml` 指定 `deploy: resources: reservations: devices` 来调用 GPU。

# 如何设计一个支持多模型的 AI 网关?让客户可以根据场景切换不同的 LLM

- **多模型 AI 网关架构**

**请求流：** 用户请求 -> API Gateway(认证/限流) -> 模型路由器(场景选择) -> 模型适配层(统一接口) -> 实际调用 -> 统一响应 -> 缓存层

- **核心组件**

1. **模型路由器**:根据 query 复杂度/预算/延迟要求选择模型
- 简单FAQ -> GLM-4-Flash(便宜快速)
- 代码问题 -> GPT-4(代码能力强)
- 长文档 -> Claude 3(长上下文)
- 默认 -> GLM-4-Plus(性价比)

2. **统一适配层**:不同模型 API 格式不同,统一为一致接口
3. **降级链**:主模型(GPT-4) -> 故障 -> 备用(Claude) -> 故障 -> 兜底(GLM-4)
4. **成本控制**:Token 预算管理 + 语义缓存 + 模型降级

- **客户价值**:一个 API 地址,背后自动选最优模型,降低成本同时保证效果.

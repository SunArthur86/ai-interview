# 客户的数据不能出公司网络,你怎么帮他部署 AI 方案

- **数据不出网的 AI 部署方案**

**架构对比图：**
```text
【方案一：全本地化部署 (私有化)】
┌─────────────────────────────────────┐
│  客户内网环境 │
│  ┌───────────┐  ┌───────────┐       │
│  │ 业务应用   │─>│ 向量数据库 │       │
│  └─────┬─────┘  └───────────┘       │
│        │                            │
│        ▼                            │
│  ┌───────────┐  ┌───────────┐       │
│  │推理服务    │  │ 嵌入模型  │       │
│  │(vLLM/TGI) │<─│ (BGE/M3)  │       │
│  └─────┬─────┘  └───────────┘       │
│        │                            │

【方案二：混合模式 (隐私计算/脱敏)】
```

- **方案详解**

**方案 1: 全本地化部署**
*   **栈**: 开源模型 (Llama 3 8B/70B 或 Qwen-72B) + vLLM/TGI 推理加速 + 本地向量库 (Milvus/PGVector) + 应用层 (FastAPI/LangChain)。
*   **优势**: 数据完全不出网，合规性最高。
*   **劣势**: 需要 GPU 服务器集群，运维成本高，模型效果通常弱于 SOTA 闭源模型。

**方案 2: 混合模式**
*   **机制**:
    *   **敏感数据**: 存储和处理在本地向量库。
    *   **非敏感推理**: LLM 推理走 API。
    *   *关键细节*: 构造 Prompt 时，不将原文传给云端，而是只传检索到的元数据或经过高度脱敏的特征；或者在本地运行小模型进行脱敏重写后再发给云端。
*   **优势**: 利用云端模型强能力，减少数据暴露。

**方案 3: 专用云 (VPC / 私有化云)**
*   **机制**: 云厂商提供专用实例 (如火山引擎 VPC、AWS Outposts)。
*   **数据隔离**: 数据在客户专属的云空间，物理或逻辑隔离，符合部分监管要求。

- **合规要点与技术实现**
1.  **数据脱敏**:
    *   API 调用前自动替换 PII (姓名/手机号/身份证) 为 `<PERSON>`, `<PHONE>` 等占位符。
    *   可使用微软 Presidio 或专门的 NER 模型进行预处理。
2.  **审计日志**:
    *   记录所有 AI 调用的输入/输出。注意：审计日志本身也是敏感数据，需加密存储并严格控制访问权限。
3.  **权限控制 (RBAC)**:
    *   不同角色访问不同知识库。例如：HR 只能访问 HR 文档的向量库空间，技术部访问技术文档。
4.  **国密算法**:
    *   政府/金融客户可能要求传输和存储使用 SM2/SM4 加密。需确保推理框架和通信库支持国密。
5.  **等保三级**:
    *   需要过等保测评的系统有额外要求：包括网络隔离、数据备份恢复、防入侵策略等。

## 常见考点
1.  **本地部署的 GPU 资源如何评估？**
    *   *追问*: 如果我要跑一个 70B 的模型，需要什么配置？
    *   *回答*: 70B 模型量化到 4-bit 至少需要约 40GB 显存（INT4），建议 2-3 张 A800/A10 (24G/48G) 做张量并行；同时要考虑并发量，QPS 越高需要的 GPU 越多。
2.  **混合模式下的“数据出境”法律界定？**
    *   *追问*: 如果我把文档的 Hash 发给云端算 Embedding，算不算数据出网？
    *   *回答*: 严格来说，Hash 也可能被还原或通过侧信道攻击泄露信息。最合规的做法是 Embedding 模型也部署在本地，只将向量检索后的结果（通常是文本片段）经过严格脱敏后发给云端 LLM。
3.  **如何更新本地部署的模型知识？**
    *   *追问*: 不重新训练模型，如何让本地 LLM 知道公司新业务？
    *   *回答*: 使用 RAG (检索增强生成)，将新文档存入本地向量库；或者使用 PEFT (LoRA 微调) 但成本较高。生产环境推荐 RAG。

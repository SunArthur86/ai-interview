# 客户的数据不能出公司网络,你怎么帮他部署 AI 方案

- **数据不出网的 AI 部署方案**

- **方案 1: 全本地化部署**
- 开源模型(Llama 3 8B/70B)+ vLLM 推理服务
- 本地向量库(Milvus)+ 应用层(FastAPI)
- 优势:数据完全不出网
- 劣势:需要 GPU 服务器,运维成本高

- **方案 2: 混合模式**
- 敏感数据本地处理(向量检索在本地)
- 非敏感的 LLM 推理走 API(只传 prompt 不传数据原文)

- **方案 3: 专用云**
- 云厂商提供专用实例(如火山引擎 VPC)
- 数据在客户专属的云空间

- **合规要点**
1. **数据脱敏**:API 调用前自动替换 PII(姓名/手机号/身份证)
2. **审计日志**:记录所有 AI 调用的输入/输出
3. **权限控制**:RBAC,不同角色访问不同知识库
4. **国密算法**:政府/金融客户可能要求 SM2/SM4 加密
5. **等保三级**:需要过等保测评的系统有额外要求

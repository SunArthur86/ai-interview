# 客户问'我们需要微调模型吗',你怎么帮他判断

- **要不要微调模型--FDE 判断框架**

- **先试这些(不需要微调)**
1. Prompt Engineering -> 80% 的问题可以解决
2. RAG(知识库检索)-> 解决'模型不知道客户数据'的问题
3. Few-shot Learning -> 给几个示例就好
4. 工具调用 -> 让模型调 API 获取信息

- **什么时候才需要微调?**
1. 输出格式固定且严格(如特定 JSON schema)
2. 需要特定领域术语/行话
3. Prompt 怎么调都效果不够
4. 需要降低延迟(小模型微调后接近大模型效果)
5. 成本压力(微调小模型替代 GPT-4 API)

- **微调方式选择**
- LoRA: 500-5000条数据,单卡,效果好
- QLoRA: 500-5000条数据,消费级GPU,较好
- 全量微调: 10000+条数据,多卡,最好

- **FDE 建议客户的标准话术**:先用 Prompt + RAG 做一版,如果效果不够再考虑微调.微调不是银弹,80% 的场景不需要.

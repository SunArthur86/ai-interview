# FDE 如何在一周内交付一个让客户买单的 AI PoC(概念验证)

- **一周交付 PoC 的 FDE 方法论**

- **Day 1(周一):需求锁定**
- 上午:客户深度访谈(1-2小时),问 5 个核心问题
- 下午:梳理需求文档(1 页),发客户确认
- 晚上:确定技术方案和测试数据集

- **Day 2-3(周二-周三):核心开发**
- 搭建 MVP pipeline,用客户真实数据
- 聚焦核心场景(不要做边缘 case)
- 技术栈:Python + Streamlit + GLM-4 API
- 目标:核心功能跑通

- **Day 4(周四):优化 + 测试**
- 调优 Prompt(对 10-20 个真实问题优化)
- 界面美化(虽然只是 PoC,但客户看脸)
- 准备演示脚本(5 个最亮眼的场景)

- **Day 5(周五):演示 + 复盘**
- 现场演示(30分钟展示 + 30分钟讨论)
- 拿到客户反馈:满意?还需要什么?
- 讨论:从 PoC 到生产的路径和预算

- **关键原则**
1. **用客户真实数据**,不要用通用 demo 数据
2. **解决最痛的 1 个问题**,不要贪多
3. **演示 > 文档**,让客户亲手操作
4. **准备好'下一步'**,演示成功后立刻谈生产化方案

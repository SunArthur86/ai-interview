# 客户的知识库有 10 万篇文档,每周新增 500 篇,如何设计高效的数据更新管道

- **大规模知识库增量更新管道**

- **全量初始化(一次性)**
1. 批量处理 10 万篇文档
2. 分布式处理(10 个 worker 并行)
3. 分块 + Embedding + 入向量库
4. 预计时间:10 万篇 x 2s/篇 / 10 worker = ~5.5 小时

- **增量更新管道**

1. **变更检测**
- 监控文档目录/SharePoint/Confluence
- 文件 hash 比对(新增/修改/删除)
- Cron job 每 1 小时扫描一次

2. **处理流水线**
```
新增文档 -> 文档解析 -> 分块 -> Embedding -> 入库
修改文档 -> 删除旧分块 -> 重新分块 -> 重新Embedding -> 入库
删除文档 -> 从向量库删除对应分块
```

3. **去重与合并**
- 同一文档的不同版本只保留最新
- 跨文档去重(MinHash 近似检测)

4. **质量检查**
- Embedding 成功率 > 99%
- 分块大小分布检查(异常值告警)
- 随机抽检 10 篇,人工验证

5. **版本管理**
- 每次更新记录版本号
- 支持回滚到上一个版本
- 向量库快照备份

- **性能优化**
- Embedding 用批量 API(一次 100 条)
- 向量库 upsert 用批量操作
- 增量更新在低峰期执行(凌晨 2 点)

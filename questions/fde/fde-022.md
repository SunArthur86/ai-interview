# 如何为客户建立 AI 系统的自动化评估流水线

- **AI 自动化评估流水线设计**

- **评估维度**
1. **Faithfulness (忠实度)**: 回答是否基于检索到的上下文，没有幻觉。**原理**：通过 LLM 将答案拆解为原子事实，逐条核对检索上下文的支持度。
2. **Answer Relevancy (相关性)**: 回答是否切题。**原理**：基于生成的答案反向生成问题，计算反向生成问题与原始问题的余弦相似度。
3. **Context Precision (检索精度)**: 检索结果是否包含正确答案。**原理**：计算检索到的文档中，相关文档的排序靠前程度（如 NDCG 指标）。
4. **Context Recall (检索召回)**: 所有相关信息是否都被检索到。**原理**：计算标准答案中的声明能在检索到的上下文中找到的比例。

- **实现方案**

1. **黄金测试集构建**
- 从客户真实问题中选 200-500 条，需覆盖高频场景和长尾 Corner Case。
- 人工标注标准答案和评分，确保数据质量的信度（Kappa 系数 > 0.8）。
- 每次迭代回归测试，建立版本基线。

2. **自动化评估工具**
- **RAGAS**: 开源 RAG 评估框架，集成上述指标计算逻辑。
- **LLM-as-Judge**: 用 GPT-4/Claude 3 Opus 评估输出质量，需设计精细的 Prompt 模板，采用 CoT（思维链）减少评分波动。
- **人工抽检**: 每天随机抽 20 条人工复核，作为自动化指标校准的 Ground Truth。

3. **CI/CD 集成**
- 每次 Prompt/配置变更 -> 自动触发评估流水线。
- 评分下降超阈值（如 > 5%）-> 阻止上线并报警。

4. **线上监控**
- 用户点赞/点踩率（显式反馈）。
- 转人工率（模型兜底阈值触发的频率）。
- 用户追问率（追问多说明没答好或用户困惑）。

- **评估报告模板**
- **指标总览**: Faithfulness 92%, Relevancy 88%, Context Precision 85%。
- **环比变化**: 较上周 +2% (Context Recall 提升，归因于 Chunk Size 调整)。
- **Bad Case 分析**: Top 5 错误类型（如：多跳推理失败、时效性问题、格式错误）。
- **优化建议**: 数据补充/模型调整/Prompt 修改/参数调优（如 temperature, top_k）。

- **## 常见考点**
1. **评估指标本身的可靠性**：如何解决 LLM-as-Judge 的偏好和随机性？（答案：使用更强的模型如 GPT-4，采用多数投票法，定期与人工标注对齐）。
2. **黄金数据集维护**：当系统业务逻辑变更时，如何维护测试集的有效性？（答案：建立数据过期机制，定期注入新样本，剔除不再适用的旧问题）。
3. **评估成本控制**：全量评估成本过高怎么办？（答案：采用分层采样，高频全量跑轻量级指标，低频跑深度语义指标；或使用较小参数量的模型做初步筛选）。
4. **线上负反馈处理**：如何自动化处理线上用户点踩的数据？（答案：自动识别并归入困难样本集，用于后续的 RLHF 微调或 Prompt 优化）。

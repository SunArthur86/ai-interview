# FDE 和传统软件工程师(SWE)有什么核心区别

- **核心区别**

| 维度 | SWE(软件工程师) | FDE(前沿部署工程师) |
|------|-------------------|----------------------|
| 工作地点 | 公司办公室 | 客户现场 |
| 需求来源 | 产品经理写 PRD | 客户口头描述,自己提炼 |
| 代码质量 | 追求优雅、可维护 | 先跑通再说,快速迭代 |
| 技术栈 | 专精某一端(前/后端) | 全栈 + AI + DevOps |
| 成功标准 | 代码上线、Bug 少 | 客户满意、签合同 |
| 反馈周期 | 季度/月度 | 天级别 |

- **FDE 的独特挑战**
- 没有详细需求文档,要学会从客户对话中挖掘真实需求
- 客户环境千差万别,需要快速适配
- 既要写代码又要做方案汇报
- 多客户并行,时间管理要求高

- **一句话理解**:SWE 是在工厂里造产品的,FDE 是拿着产品到客户现场安装调试的.

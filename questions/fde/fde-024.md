# 项目交付后,如何把 AI 系统的运维知识转移给客户团队

- **AI 系统知识转移 SOP**

- **交付物清单**
1. **技术文档**
- 架构图(系统全貌)
- API 文档(接口规格 + 示例)
- 运维手册(常见问题 + 排查步骤)
- 数据字典(知识库结构 + 字段说明)

2. **培训计划(3 天)**
- Day 1: 系统架构 + 核心概念讲解
- Day 2: 日常运维实操(监控/日志/重启/更新)
- Day 3: 故障排查演练(模拟常见问题)

3. **视频录制**
- 录制关键操作视频(部署/Prompt修改/数据更新)
- 客户随时可以回看

- **重点培训内容**

1. **Prompt 维护**:如何修改 System Prompt 而不破坏效果
2. **知识库更新**:如何添加新文档、删除旧文档
3. **监控看板**:哪些指标要看、什么值是异常
4. **故障处理**:服务挂了怎么重启、API 超时怎么排查

- **持续支持**
- 上线后 2 周每日跟进
- 上线后 1 月每周复盘
- 建立专属沟通群(飞书/钉钉)
- 约定 SLA:紧急问题 2 小时响应

- **避免的坑**
- 不要只给文档,一定要实操培训
- 客户运维人员技术水平参差不齐,文档要'小白也能看懂'
- 交接不是一次性的事,需要 1-2 个月的过渡期

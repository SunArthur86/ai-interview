# FDE 需要具备哪些核心技能?非 CS 背景的人能做 FDE 吗

- **FDE 核心技能树**

- **1. 工程能力(必须)**
- Python(最核心)+ 至少一门后端语言(Java/Go)
- 全栈能力:前端 React/Vue + 后端 FastAPI/Flask
- 数据库:SQL + 向量数据库(Milvus/Pinecone)
- DevOps:Docker + Linux + CI/CD

- **2. AI 能力(必须)**
- Prompt Engineering
- RAG 系统搭建(Embedding + 检索 + 生成)
- Agent 开发(LangChain/LlamaIndex)
- 模型 Fine-tuning 基础(LoRA/QLoRA)
- API 集成(OpenAI/GLM/Claude)

- **3. 软技能(同样重要)**
- 客户沟通:能听懂'业务语言',翻译成'技术方案'
- 快速学习:客户行业知识要在 1 周内入门
- 项目管理:多客户并行管理
- 方案宣讲:能用客户听得懂的话讲技术

- **非 CS 背景能做 FDE 吗?**
- 门槛确实高,不是学几个月 Python 就行
- 但如果有强领域知识(金融/医疗/法律)+ 编程基础 → 很有竞争力
- 因为 FDE 最难的不是技术,是理解客户业务

- **## 常见考点**
1. **LangChain vs LlamaIndex**：面试官常问框架选择。答案：LangChain 生态全、适合做 Agent 和复杂链；LlamaIndex 专注数据索引和 RAG，检索性能更好，通常是搭配使用。
2. **向量数据库原理**：需理解 HNSW（Hierarchical Navigable Small World）索引原理，它是一种基于图的索引算法，牺牲内存换取检索速度（召回率与性能的平衡）。
3. **非 CS 背景的转化路径**：面试官可能质疑你的工程落地能力。需强调熟悉 Python 异步编程、并发处理以及 Docker 容器化部署，这是交付的硬底线。

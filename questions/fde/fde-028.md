# 作为 FDE,你在客户现场发现的通用需求,如何有效反馈给产品团队

- **FDE 反馈闭环最佳实践**

- **FDE 的双向价值**
- 向客户:交付定制化解决方案
- 向产品:反馈通用需求,驱动产品演进

- **反馈渠道(分层)**

1. **紧急反馈**(立即处理)
- 线上 Bug / 安全漏洞
- 直接发工单 + @产品负责人

2. **周期性反馈**(每周)
- 每周写 FDE 週报(1 页)
- 内容:本周客户反馈 Top 5 + 需求优先级排序
- 发产品团队周会讨论

3. **季度需求评审**(每季度)
- 汇总多个客户的共性需求
- 写 PRD 草稿(附带客户案例和数据支撑)
- 在季度规划会上正式提案

- **反馈模板(FDE Insight Report)**
- 客户名称(脱敏)
- 场景描述:客户在什么场景下遇到什么问题
- 当前 workaround:目前怎么绕过
- 建议方案:产品应该怎么改进
- 影响范围:X 个客户有同样需求
- 商业价值:不做会丢多少订单 / 做了能赚多少

- **推动产品改进的技巧**
1. 用数据说话('5 个客户都提到了这个需求')
2. 关联商业价值('不做可能影响续约')
3. 提供 MVP 方案('加一个配置项就能解决 80% 的场景')
4. 自己先写原型验证可行性

- **## 常见考点**
1. **需求优先级判定**：如何区分“客户的伪需求”和“产品的真机会”？（考察商业敏锐度）
2. **处理被拒需求**：如果产品团队拒绝了你的提案，你该如何处理？（考察沟通与推动力）
3. **通用性定义**：多少个客户反馈才算“通用需求”？是看数量还是看客户体量？（考察判断力）
4. **数据量化**：如何在没有直接数据支持的情况下，估算一个需求的商业价值？（考察分析能力）
